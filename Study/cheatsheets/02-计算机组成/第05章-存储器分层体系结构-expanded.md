# 计算机组成 · 存储器分层体系结构 — 复习资料

> 来源：2-3-计算机组成计算机类讲义.pdf 物理页 [65, 79]
> 说明：⭐=高频考点，⚠=易错点；> Caveat: 为陷阱/技巧；末尾 Worked Examples 为代表性题目（无书内答案的标 [PROPOSED — verify]）。

## 本章概览

本章解决计算机系统中存储器的**容量、速度、价格**三者矛盾问题。核心思路是采用**存储器层次结构**（Cache—主存—辅存），通过程序局部性原理使整体表现接近最快层、容量接近最大层、价格接近最便宜层。内容涵盖：存储器基本概念与分类 → 半导体 RAM/ROM 原理 → 存储器与 CPU 连接及容量扩展 → Cache 高速缓冲（映射/替换/写策略）→ 虚拟存储器。Cache 与存储器层次、存储器扩展与编址计算、DRAM 与 SRAM 是本章三大高频考点。

## 存储器概述

**字地址与字节地址**：字地址是存放一个机器字的存储单元对应的地址；字节地址是存放一个字节的单元对应的地址。若可编址的最小单位是字存储单元，则称为**按字寻址**的计算机；若最小单位是字节，则称为**按字节寻址**的计算机。*(讲义 p65)*

**存储器的特性指标** ⭐：*(讲义 p65)*

| 指标 | 定义 |
|---|---|
| 存储容量 | 存储器中可以容纳的存储单元总数 |
| 存取时间 $t_a$ | 又称存储器访问时间 |
| 存储周期 $t_n$ | 连续启动两次读操作所需间隔的最小时间 |
| 传输率 | 数据传入或传出存储单位的速率 |
| 带宽 | 单位时间内存储器可读写的字节数（或二进制的位数） |
| 价格 | 一般用每位价格来表示 |

> Caveat (专项练习-单选12): 存储容量计算公式为：容量 = 末地址 − 首地址 + 1。*(讲义答案 p76)*

**存储器的分类**：*(讲义 p65)*

| 分类依据 | 类别 |
|---|---|
| 按存取方式 | 顺序存取、直接存取、随机存取、相联存取 |
| 按存储介质 | 半导体存储器、磁表面存储器、光存储器 |
| 按读写功能 | ROM、RAM |
| 按信息的可保存性 | 非永久记忆存储器、永久记忆存储器 |
| 按在计算机系统中的作用 | 主存储器、辅助存储器、Cache 等 |

> Caveat (专项练习-单选19): 磁带属于顺序存取存储器；磁盘、光盘、U 盘属于直接存取。*(讲义答案 p76)*

**存储器的层次结构** ⭐：在存储系统中存在如下关系：存取时间越短，每位价格越高；容量越大，每位价格越低；容量越大，存取时间越长。解决容量/速度/价格矛盾的方法是采用**存储器层次结构（Memory Hierarchy）**。*(讲义 p65)*

在通用存储层次中，从上到下有如下关系：*(讲义 p66)*

- 每位价格降低
- 容量增大
- 存取时间增大
- 处理器访问存储器的频度降低

微机系统中存储器层次结构主要是**缓存—主存**和**主存—辅存**两个层次。*(讲义 p66)*

> Caveat (专项练习-多选2): 三级存储系统由 Cache、主存和辅存组成。*(讲义答案 p77)*

## 半导体随机存储器

**半导体存储位元**：存储位元用于保存一位二进制信息，需具备以下 3 个条件：①呈现两种稳态（或半稳定），分别代表 1 和 0；②能够写入（至少一次）来设置状态；③能够读出状态。*(讲义 p66)*

常用半导体存储位元对比：*(讲义 p66)*

| 类型 | 速度 | 集成度 | 功耗 | 价格 | 主要用途 |
|---|---|---|---|---|---|
| 双极型 TTL | 高 | 低 | 大 | 高 | 小容量高速存储器 |
| MOS（金属氧化物场效应晶体管） | 较低 | 高 | 小 | 低 | 大容量存储器 |

> 计算机中 MOS 器件组成的存储器最为常用。*(讲义 p66)*

**SRAM 与 DRAM** ⭐⚠：RAM 分为静态 RAM（SRAM）和动态 RAM（DRAM）。*(讲义 p66)*

| 特性 | SRAM（静态随机存储器） | DRAM（动态随机存储器） |
|---|---|---|
| 速度 | 快 | 比 SRAM 慢 |
| 集成度 | 低 | 高 |
| 容量 | 小 | 大 |
| 是否需要刷新 | 不需要（"静态"含义） | 需要定时刷新（"动态"含义） |
| 主要用途 | 高速缓冲存储器（Cache） | 个人计算机主存（使用最多） |

> Caveat (第五章单选24-25): DRAM 读出是破坏性的，读后需立即重写；刷新是按行进行的，不是按位或按列。*(题目第五章单选24-25)*

> Caveat (专项练习-判断7): 动态 RAM 靠电容储存电荷的原理存储信息，一般在 2ms 时间内必须刷新一次，刷新与行地址有关。*(讲义答案 p79)*

> Caveat (专项练习-多选4): SRAM 记忆单元由 6 个 MOS 管组成，利用双稳态触发器存储信息；DRAM 利用栅极电容存储信息，需读后再生并定时刷新。半导体存储器大多用随机存取方式，但相联存储器按内容访问，不按地址访问，因此并非所有半导体存储器都采用随机存取方式。*(讲义答案 p77)*

**存储位元的组织形式**：存储器芯片中存储位元的组织形式有三种：*(讲义 p67)*

| 组织形式 | 说明 |
|---|---|
| 一维地址译码方式（单译码方式） | [原文未述] |
| 二维地址译码的位选方式 | [原文未述] |
| 二维地址译码的字选方式 | **常用形式** |

采用地址线的复用和方阵型行列结构，每增加一个专用的地址引脚，便使得行地址和列地址的指示范围加倍，因此存储器芯片的容量以 **4 的倍数**增长。*(讲义 p67)*

**DRAM 刷新** ⚠：由于 DRAM 存储单元被访问是随机的，有可能某些存储单元长期得不到访问，无读出也就无重写，其原信息必然消失，必须采用定时刷新的方法。*(讲义 p67)*

常用刷新方式有三种：*(讲义 p67)*

| 刷新方式 | 特点 |
|---|---|
| 集中式 | [原文未述] |
| 分散式 | [原文未述] |
| 异步式 | [原文未述] |

实际应用中常使用 DRAM 控制器与 DRAM 配合使用。DRAM 控制器是 CPU 与 DRAM 芯片之间的接口电路，负责完成刷新、刷新/访存裁决等操作。借助 DRAM 控制器，可把 DRAM 看作 SRAM 一样使用。*(讲义 p67)*

## 半导体只读存储器

**计算机为什么配置 ROM**：ROM 主要用于存放系统程序、常用功能的库例程和功能表。由于计算机加电后必须运行程序，不能只用易失性的 RAM 构成主存储器，需要有 ROM 存储引导程序。*(讲义 p67)*

**ROM 的分类**：*(讲义 p68)*

| 类型 | 全称 |
|---|---|
| MROM | 半导体掩模只读存储器（Masked Read Only Memory） |
| PROM | 一次性编程只读存储器（Programmable ROM） |
| EPROM | 可擦写可编程只读存储器（Erasable PROM） |
| EEPROM | 电可擦写可编程只读存储器（Electrically Erasable PROM） |
| FLASH Memory | 快闪存储器 |

> Caveat (专项练习-多选6): ROM 是只读的，不能存，只能取。ROM 和 RAM 采用的都是随机存取方式。*(讲义答案 p77)*

## 存储器与 CPU 连接

**存储器与 CPU 的连接** ⭐：存储器同 CPU 连接时，要完成**地址线、数据线和控制线**的连接。CPU 对存储器进行读/写操作时：首先由地址总线给出地址信号，然后发出读/写控制信号，最后在数据总线上进行信息交流。*(讲义 p67-68)*

存储芯片的引脚包括：地址、数据、片选信号、读写信号等。*(讲义 p68)*

**存储容量扩展方法** ⭐⚠：常用方法有位扩展法、字扩展法、字位同时扩展法。*(讲义 p68)*

| 扩展方法 | 说明 | 关键点 |
|---|---|---|
| 位扩展法 | 扩展数据位宽 | 并联数据线，地址线和控制线共用 |
| 字扩展法 | 扩展存储字数 | 需使用译码器（如 74138 3-8 译码器、74139 2-4 译码器） |
| 字位同时扩展法 | 同时扩展字数和位宽 | 先位扩展再字扩展（或反之） |

> Caveat (专项练习-单选13): 字扩展看地址线位数（决定组数），位扩展看数据线位数（决定每组片数）。*(讲义答案 p76)*

**标准解题套路**（for 存储器芯片数量计算）*(讲义 p68, 答案 p76)*
1. 确定总容量（如 $64\text{KB} = 64\text{K} \times 8$ 位）和单片容量（如 $16\text{K} \times 1$ 位）。
2. 计算字扩展组数 = 总字数 / 单片字数（如 $64\text{K} / 16\text{K} = 4$ 组）。
3. 计算每组位扩展片数 = 总位数 / 单片位数（如 $8 / 1 = 8$ 片）。
4. 总片数 = 组数 × 每组片数（如 $4 \times 8 = 32$ 片）。

**标准解题套路**（for 按字/按字节编址的寻址范围计算）*(讲义 p65, 答案 p75-76)*
1. 确定总存储容量（字节）。
2. 确定编址单位：
   - 按字节编址：编址单位 = 1 字节
   - 按字编址：编址单位 = 字长 / 8 字节（如 32 位字长 → 4 字节）
   - 按半字编址：编址单位 = 字长 / 16 字节（如 32 位字长 → 2 字节）
3. 寻址范围 = 总容量 / 编址单位。

> Caveat (专项练习-单选4): 32 位字长，4MB 按字编址：一个字 = 4 字节，$4\text{MB} / 4\text{B} = 1\text{M}$。*(讲义答案 p75)*

> Caveat (专项练习-单选5): 32 位字长，16MB 按半字寻址：半字 = 2 字节，$16\text{MB} / 2\text{B} = 8\text{M} = 2^{23}$。*(讲义答案 p75)*

> Caveat (第一章单选2): 存储单位换算中 $1\text{KB} = 1024\text{B}$，而非 $1000\text{B}$；$1\text{MB} = 1024\text{KB}$。*(题目第一章单选2)*

**加速 CPU 和存储器之间有效传输的途径**：由于 CPU 重点提高处理速度，主存重点提高容量，速度差异越来越大，可采取以下措施：*(讲义 p68)*

1. 主存储器采用更高速的技术来缩短读出时间，或加长存储器的字长
2. 采用并行操作的双端口存储器
3. 在每个存储器周期中存取几个字（多体交叉存储器）
4. 在 CPU 和主存储器之间插入高速缓冲存储器（Cache）

**主存地址在各模块中的安排**：一个由若干模块组成的主存储器是线性编址的，地址安排有两种方式：*(讲义 p68)*

| 组织方式 | 特点 | 优点 | 缺点 |
|---|---|---|---|
| 顺序方式 | 某模块存取时其他模块不工作 | 故障隔离；扩容方便 | 各模块串行工作，带宽受限 |
| 交叉方式 | 连续地址分布在相邻不同模块内，同一模块内地址不连续 | 对连续字成块传送可实现多模块流水式并行存取，大大提高带宽 | [原文未述] |

交叉方式使用场合为成批数据读取。*(讲义 p68)*

> Caveat (专项练习-判断8): 双端口存储器属于空间并行技术（物理上具有两套读写机构），多模块交叉存储器属于时间并行技术（一套读写机构分时使用）。*(讲义答案 p79)*

## 高速缓冲存储器（Cache）

**程序的局部性原理** ⭐：多级存储体系能够解决存储器容量、速度、价格之间矛盾的关键是**程序执行的局部性原理**。程序在执行时所呈现的局部性规律，即在一段较短时间内，程序的执行仅限于某个部分，相应地，它所访问的存储器空间也局限在某个空间。*(讲义 p68-69)*

| 局部性类型 | 含义 | 主要原因 |
|---|---|---|
| 时间局部性 | 某条指令/数据被访问后，不久后很可能再次被访问 | 程序中有大量循环操作 |
| 空间局部性 | 访问某个内存单元后，不久后其附近的内存单元也要被访问 | 程序的顺序执行 |

**Cache 的原理** ⭐：高速缓冲存储器是为了解决 CPU 和主存之间速度不匹配而采取的重要技术。基本原理：*(讲义 p69)*

- Cache 的主要目的是为了解决存储器**速度**问题
- Cache 容量远小于主存，必须考虑两者间的**地址变换**问题
- Cache 中使用的所有算法均由**硬件**实现
- Cache 技术实现的基本目标是提高**命中率**
- Cache 和主存之间以**块**为单位交换信息
- Cache 与主存**同时**进行读写

> Caveat (专项练习-单选15): Cache 的存在是为了提高内存的访问速度（提高 CPU 效率），而非扩大主存容量。*(讲义答案 p76)*

**设计 Cache 需要考虑的因素**：*(讲义 p69)*

1. Cache 容量的大小
2. 主存地址映射到 Cache 的方法
3. 当新的数据块装入 Cache 时，替换掉原数据块的策略
4. CPU 执行写操作时写的策略
5. 数据块大小的选择
6. Cache 数目的选择

**三种映射方式** ⭐⚠：映射功能的选择决定了 Cache 的结构，通常采用三种技术：*(讲义 p69)*

| 映射方式 | 优点 | 缺点 |
|---|---|---|
| 全相联 | 灵活，不易产生冲突 | 比较电路难于实现，且效率低，速度慢 |
| 直接映射 | 硬件简单，成本低 | 容易产生冲突，不能有效利用 Cache 空间 |
| 组相联 | 以上两种方式的结合（组间直接映射，组内全相联映射） | [原文未述] |

> Caveat (专项练习-判断9): 在 Cache 组相联存储器中，设 $c$ 为组字段长度，$r$ 为块字段长度，当 $c=0$ 时，则成为全相联地址映像（不分组，所有块都是一个组）。*(讲义答案 p79)*

**替换策略** ⭐：当使用全相联和组相联方式时，新数据块装入 Cache 需使用替换策略：*(讲义 p69)*

| 替换策略 | 原理 | 特点/缺点 |
|---|---|---|
| FIFO（先进先出） | 替换在 Cache 中停留时间最长的块 | 用循环或环形缓冲技术很容易实现 |
| LFU（最不经常使用） | 计数周期限定在两次替换之间的时间间隔 | 不能严格反映近期访问情况；新调入行计数值小，容易被替换出去 |
| LRU（近期最少使用） | 替换近期最少使用的块 | 目前使用较多，能有效提高命中率 |
| 随机替换 | 随机选择替换块 | 硬件容易实现并且速度快，实际模拟显示性能不错 |

> Caveat (专项练习-单选16): LRU 是理论上最好的替换算法，因为其符合程序的局部性原理。*(讲义答案 p76)*

**Cache 的写操作** ⚠：对 Cache 块内写入的信息，必须与被映象的主存块内的信息完全一致。*(讲义 p69-70)*

| 写策略 | 别名 | 优点 | 缺点 |
|---|---|---|---|
| 写直达法 | 通过式写（Write Through） | 最简单；数据不会出现不一致 | 对写操作没有高速缓存的作用 |
| 写回法 | Write Back | 速度快 | 存在数据不一致隐患 |

**Cache 块大小与命中率的关系**：关系复杂，取决于特定程序的局部性特征，目前还没有找到确定的最优值。较大的块能减少装入 Cache 的块数（少量块导致装入的数据很快被改写）；当块变得较大时，每个附加的字离所要的字更远，被使用的可能性更小。*(讲义 p70)*

> Caveat (专项练习-多选5): 字块大可充分利用空间局部性，但：①失效损失变大（不命中时需花更多时间从主存读块）；②字块太大则 Cache 项数变少，命中的可能性变小。字块不能太大。*(讲义答案 p77)*

**Cache 级数与统一/分立**：*(讲义 p70)*

| 设计选择 | 方案 A | 方案 B |
|---|---|---|
| 级数 | 单级 Cache | 两级（或以上）Cache — 研究表明可提高性能 |
| 组织方式 | 统一 Cache — 可充分利用 Cache 空间，便于管理 | 分立 Cache — 取消指令和数据的访问冲突，便于实现流水线 |

## 虚拟存储器

**虚拟存储器的概念** ⭐：虚拟存储器只是一个容量非常大的存储器的**逻辑模型**，不是任何实际的物理存储器。它借助于磁盘等辅助存储器来扩大主存容量，使之能被更大或更多的程序所使用。虚拟存储器不仅是解决存储容量和存取速度矛盾的一种方法，也是管理存储设备的有效方法。*(讲义 p70)*

- **虚拟地址（虚地址）**：程序的逻辑地址
- **物理地址（实地址）**：由 CPU 地址引脚送出，用于访问主存的地址

**虚拟存储器与 Cache 的区别** ⚠：*(讲义 p70-71)*

| 比较维度 | Cache | 虚拟存储器 |
|---|---|---|
| 实现方式 | 用硬件实现 | 用软件、硬件相结合组成 |
| 对操作系统的透明性 | 对操作系统透明 | [原文未述] |
| 对未命中的敏感度 | 较低 | 对未命中更加敏感 |

主存—外存层次的基本信息传送单位可采用几种不同方案：*(讲义 p71)*

| 传送单位 | 说明 |
|---|---|
| 段 | 利用程序的模块化性质，按照程序的逻辑结构划分成的多个相对独立部分 |
| 页 | 定长的页 |
| 段页 | [原文未述] |

> Caveat (专项练习-判断10): 虚拟存储器由 DRAM 和硬盘及软件构成，不能说"虚拟存储器由硬盘构成"。*(讲义答案 p79)*

> Caveat (专项练习-单选17): 虚拟存储器的设计来自程序的局部性原理。*(讲义答案 p76)*

| 带宽 | 单位时间内存储器可读写的字节数（或位数） |
| 寻址范围 | $= \text{总容量} / \text{编址单位}$ |
| 芯片总片数 | $= \text{字扩展组数} \times \text{每组位扩展片数}$ |
| 字扩展组数 | $= \text{总字数} / \text{单片字数}$ |
| 位扩展片数 | $= \text{总位数} / \text{单片位数}$ |
| 地址范围容量 | $= \text{末地址} - \text{首地址} + 1$ |
| DRAM 刷新周期 | 一般 2ms 内必须刷新一次 |
| 地址线复用容量增长 | 每增加一个专用地址引脚，容量以 4 的倍数增长 |

## 高频考点与易错提醒

1. **存储器层次结构** ⭐：三级存储系统 = Cache + 主存 + 辅存；从上到下价格降、容量增、速度降、频度降。
2. **SRAM vs DRAM** ⭐⚠：SRAM 不需刷新、用作 Cache；DRAM 需刷新、用作主存；DRAM 读出为破坏性，刷新按**行**进行。
3. **存储器扩展与编址计算** ⭐⚠：位扩展看数据线、字扩展看地址线；按字/半字/字节编址时编址单位不同；十六进制地址范围求容量 = 末地址 − 首地址 + 1。
4. **Cache 映射方式** ⭐⚠：全相联（灵活但慢）、直接映射（快但易冲突）、组相联（折中）；组相联 $c=0$ 时退化为全相联。
5. **Cache 替换算法** ⭐：LRU 最好（符合局部性原理），FIFO 最简单，随机替换性能也不错。
6. **Cache 写策略** ⚠：写直达数据一致但无高速缓存效果，写回快但有数据不一致隐患。
7. **局部性原理** ⭐：时间局部性（循环操作）、空间局部性（顺序执行）；是 Cache 和虚拟存储器的共同理论基础。
8. **虚拟存储器** ⭐⚠：非物理存储器，由 DRAM + 硬盘 + 软件构成；与 Cache 区别在于实现方式（软硬件结合 vs 纯硬件）和对未命中的敏感度。
9. **双端口 vs 多模块交叉** ⚠：双端口 = 空间并行，多模块交叉 = 时间并行。
10. **存储单位换算** ⚠：$1\text{KB} = 1024\text{B}$，非 $1000\text{B}$。

## 一句话记忆要点

- **层次结构**：从上到下，价格降、容量增、速度降、访问频度降。
- **SRAM vs DRAM**：SRAM 快不刷新做 Cache，DRAM 慢要刷新做主存，刷新按行不按列。
- **ROM 分类**：MROM → PROM → EPROM → EEPROM → FLASH，可擦写性递增。
- **存储扩展**：位扩展并联数据线，字扩展用译码器扩地址，总片数 = 组数 × 每组片数。
- **编址计算**：按字编址除以字长字节数，按字节编址除以 1，容量 = 末地址 − 首地址 + 1。
- **Cache 映射**：全相联灵活但慢，直接映射快但冲突，组相联折中，$c=0$ 退化为全相联。
- **替换算法**：LRU 最好（符合局部性），FIFO 最简单，随机也不错。
- **写策略**：写直达一致但慢，写回快但有隐患。
- **虚拟存储器**：逻辑模型非物理实体，软硬件结合实现，对未命中更敏感。

## Worked Examples

### 例 1：内存编址范围计算 *(来自 exam-signals 候选 4)*

*Tags: 按字节编址, 地址范围, 十六进制运算*

**问题**：内存按字节编址，地址从 A4000H 到 CBFFFH，共有多少个字节？*(题目第五章单选12)*

**解答**：

**方法一：十六进制减法***(讲义答案 p76)*

1. 明确存储容量计算公式：容量 = 末地址 − 首地址 + 1。
2. 十六进制减法：$\text{CBFFFH} - \text{A4000H} = \text{27FFFH}$。
3. 加 1 得到字节数：$\text{27FFFH} + 1 = \text{28000H}$。
4. 转换为十进制：$\text{28000H} = 2 \times 16^4 + 8 \times 16^3 = 131072 + 32768 = 163840$ Bytes $= 160$ KB。

**方法二：二进制分析（书内答案方法）***(讲义答案 p76)*

1. 将地址转换为二进制：
   - $\text{A4000H} = 1010\ 0100\ 0000\ 0000\ 0000\text{B}$
   - $\text{CBFFFH} = 1100\ 1011\ 1111\ 1111\ 1111\text{B}$
2. 发现低 14 位作为一个从全 0 到全 1 的地址范围，这样的地址范围共有 10 组。
3. 每组 $2^{14} = 16\text{K}$，所以一共 $10 \times 16\text{K} = 160\text{K}$。

**答案**：C（160K）
