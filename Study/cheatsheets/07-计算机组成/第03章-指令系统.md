## 一、本章概览

本章介绍指令系统——计算机硬件与软件之间的接口。内容涵盖四大板块：（1）指令系统的发展与性能要求，包括基本概念、设计要求、计算机语言发展史；（2）机器指令的设计要素，包括指令构成、指令格式、指令字长、指令集设计考虑因素；（3）指令和操作数的寻址方式，包括指令寻址与操作数寻址的分类、有效地址与形式地址的概念、十种常用寻址方式；（4）RISC技术，包括其定义、主要特点及相对于CISC的优点。指令系统的格式与功能直接影响到机器的硬件结构、系统软件和适用范围，是计算机组成原理的核心章节。

## 二、知识点详解

### 第一节 指令系统的发展与性能要求

#### 知识点1 基本概念

- **指令**：指令就是要计算机执行某种操作的命令。
- **指令系统**：一台计算机中所有机器指令的集合，称为这台计算机的指令系统。
  - 指令系统是表征一台计算机性能的重要因素，它的格式与功能不仅直接影响到机器的硬件结构，而且也直接影响到系统软件，影响到机器的适用范围。
- **系列机**：是指基本指令系统相同、基本体系结构相同的一系列计算机。推出的机种指令系统一定包含所有旧机种的全部指令。

#### 知识点2 指令系统应满足的要求

- **完备性**：汇编语言编写各种程序时，指令系统直接提供的指令足够使用，而不必用软件来实现。
- **有效性**：指利用该指令系统所编写的程序能够高效率地运行。
- **规整性**，包含三个方面：
  - **对称性**：在指令系统中所有的寄存器和存储器单元都可同等对待，所有的指令都可使用各种寻址方式；
  - **匀齐性**：一种操作性质的指令可以支持各种数据类型；
  - **指令格式和数据格式的一致性**：指令长度和数据长度有一定的关系，以方便处理和存取。
- **兼容性**：各机种上基本软件可以通用。

#### 知识点3 计算机语言发展史 ⚠(练习9,10)

- 计算机语言发展的历史：机器语言→汇编语言→高级语言。
- 机器语言、汇编语言依赖于特定计算机的硬件结构和指令系统，因此被称为与硬件相关的语言，也被称为**低级语言**。
- 高级语言更贴近于数学语言和自然语言，与计算机的硬件结构及指令系统无关，在编写程序方面比汇编语言优越。

### 第二节 机器指令的设计要素

#### 知识点1 机器指令的构成

一条机器指令需要提供的信息主要有：

- **操作码**（Operation Code）
- **源操作数地址**（Source Operand Reference）
- **目的操作数地址**（Result Operand Reference）
- **下一条指令的地址**（Next Instruction Reference）

大多数CPU的下一指令地址为隐含的（由程序计数器得到）。源操作数和结果操作数存放位置包括：

- 主存或虚存
- CPU寄存器
- I/O设备

#### 知识点2 机器指令格式 ⭐(练习17,19)

- **机器指令格式**：机器指令用二进制代码表示的结构形式，通常由**操作码字段**和**地址码字段**组成。
  - 操作码字段表征指令的操作特性与功能；
  - 地址码字段通常指定参与操作的操作数的地址。
- 计算机指令最主要的元素是**操作码**，它指明即将完成的操作、源操作数和目的操作数的引用方式，以及通常隐式指明下一个指令的引用方式。操作码采用编码形式表示，CPU中有专门的电路来解释每个操作码。
- 常用的指令格式中地址码有：**三地址、二地址、单地址和零地址**几种形式。
- 从操作数的物理位置来说，又可归结为3种类型：
  - **存储器—存储器（SS）型指令**
  - **寄存器—寄存器（RR）型指令**
  - **寄存器—存储器（RS）型指令**

#### 知识点3 指令字长 ⚠(练习20,21,22)

- **指令字长**：一个指令字中包含二进制代码的位数，称为指令字长。常见指令字长有：单字长、半字长、双字长。
- **使用多字长指令的目的**：提供足够的地址位来解决访问内存任何单元的寻址问题。
- **使用多字长指令的缺点**：必须两次或多次访问内存以取出一整条指令。
- **等长指令字结构**：在一个指令系统中，如果各种指令字长是相等的，称为等长指令字结构。
- **变长指令字结构**：如果各种指令字长随指令功能而异，比如有的指令是单字长指令，有的指令是双字长指令，就称为变长指令字结构。

#### 知识点4 指令集设计考虑哪些因素

在指令集设计的最根本出发点中，最重要的方面包括有：

- **操作指令表**（Operation Repertoire）：应提供多少和什么样的操作，操作的复杂程度；
- **数据类型**（Data Types）：所支持的数据类型；
- **指令格式**（Instruction Format）：指令的（位）长度、地址数目、各个字段的大小等；
- **寄存器**（Registers）：能被指令访问的CPU寄存器数目以及它们的用途；
- **寻址方式**（Addressing Mode）：指定操作数地址的产生方式。

### 第三节 指令和操作数的寻址方式

#### 知识点1 寻址方式 ⭐⚠(练习1,2,3,4,5,7,8,14,16,18)

- **寻址方式**：形成操作数或指令地址的方式，称为寻址方式。
- 寻址方式主要分为两类：
  - **指令的寻址方式**：较为简单，种类少；
  - **操作数的寻址方式**：较复杂，种类多，很灵活。
- **寻址方式的作用**：增强指令的功能，缩短指令的长度，提高指令的执行速度，增加操作数存放的位置的灵活性等。

**指令的寻址方式：**

- **顺序寻址方式**：通过程序计数器；
- **跳跃寻址方式**：通过跳转指令。

**操作数寻址的核心概念：**

- **有效地址（EA）**：操作数的真正存储位置。
- **形式地址（A）**：也称偏移量，它是指令字结构中给定的地址量。
- **寻址过程**：就是把操作数的形式地址（A），变换为操作数的有效地址（EA）的过程。

**常用的寻址方式有（十种）：**

1. 隐含寻址
2. 立即寻址
3. 直接寻址
4. 间接寻址
5. 寄存器寻址
6. 寄存器间接寻址
7. 相对寻址
8. 基址寻址
9. 变址寻址
10. 堆栈寻址

> **注**：讲义知识点正文仅列出上述十种寻址方式的名称，未逐一展开定义与有效地址公式。各寻址方式的具体含义及 EA 计算方法在本章专项练习及答案解析中涉及，详见第五节高频考点提醒。

### 第四节 RISC技术

#### 知识点1 RISC的特点 ⭐(练习11,12)

- **RISC**：精简指令系统计算机的英文缩写，即 Reduced Instruction Set Computer。
- **CISC**：复杂指令系统计算机（Complex Instruction Set Computer），与RISC对应。

**RISC的主要特点（7点）：**

1. 选取使用频率较高的一些简单指令以及一些很有用但又不复杂的指令，让复杂指令的功能由频度高的简单指令的组合来实现。
2. 指令长度固定，指令格式种类少，寻址方式种类少。
3. 只有取数/存数（LOAD/STORE）指令访问存储器，其余指令的操作都在寄存器内完成。
4. 采用流水线技术，大部分指令在一个时钟周期内完成。采用超标量和超流水线技术，可使每条指令的平均执行时间小于一个时钟周期。
5. 控制器采用组合逻辑控制，不用微程序控制。
6. CPU中有多个通用寄存器。
7. 采用优化的编译程序。

**与CISC机相比，RISC机的主要优点（4点）：**

1. 充分利用VLSI芯片的面积；
2. 提高计算机运算速度；
3. 便于设计，可降低成本，提高可靠性；
4. 有效支持高级语言程序。

## 五、高频考点与易错提醒

> 以下考点信号均来自本章专项练习及答案解析，知识点正文未展开的内容标注「(练习N，反推)」。

1. ⚠ **寻址方式的目的**是高频易错点：主要目的是缩短指令长度、扩大寻址空间、提高编程灵活性 (练习1，反推)。
2. ⚠ **单地址指令中的隐含操作数**是高频考点：单地址指令只能显式存放一个操作数地址，另一个常采用隐含寻址方式 (练习2，反推)。
3. ⚠ **寄存器寻址 vs 寄存器间接寻址**是高频易错点：寄存器寻址操作数在寄存器中；寄存器间接寻址操作数在主存单元中，寄存器中存的是主存地址 (练习3、4，反推)。
4. ⚠ **变址寻址 EA 公式**是高频考点：$EA = (X) + D$，其中 X 为变址寄存器内容，D 为形式地址 (练习5，反推)。
5. ⚠ **基址寻址中基址寄存器**是易错点：基址寄存器内容由操作系统确定，在程序执行过程中不可变 (练习14，反推)。
6. ⚠ **变址寻址 vs 基址寻址**是高频易混点：变址寻址中变址寄存器提供修改量、指令提供基准量；基址寻址中基值寄存器提供基准量、指令提供位移量 (练习9，反推)。
7. ⚠ **先变址再间址的 EA**是高频考点：$EA = ((X) + D)$，先变址得到地址，再对该地址间接寻址 (练习16，反推)。
8. ⚠ **寻址速度对比**是高频考点：立即寻址最快（操作数在指令中，不需访存）、间接寻址最慢（需二次访存）、立即寻址不需要访存 (练习6，反推)。
9. ⚠ **隐含寻址**是高频考点：指令格式中不明确给出操作数地址而是隐含指定；累加器和寄存器均可作为隐含地址 (练习8，反推)。
10. ⚠ **扩展操作码**是高频考点：一种指令优化技术，让操作码的长度随地址数的减少而增加，不同地址数的指令可以具有不同的操作码长度，目的是在不扩大指令长度的情况下增加指令数量 (练习17、19，反推)。
11. ⚠ **PC 自动增量与编址方式的关系**是高频易错点：按字编址时取单字长指令后 PC 加 1；按字节编址时取单字长（16位=2字节）指令后 PC 加 2，取双字长指令后 PC 加 4 (练习20、21、22，反推)。
12. ⚠ **堆栈指针 SP**是易错点：SP 指向栈顶元素，不是表示堆栈内数据个数；出栈操作顺序与进栈相反 (练习10、15，反推)。
13. ⚠ **汇编语言源程序长度**是易错点：汇编语言源程序通常比高级语言源程序更长（不是更短），因为汇编指令功能更简单，表达同样程序需更多条指令 (练习9，反推)。
14. ⚠ **RISC 指令功能**是易错点：RISC 指令功能尽可能简单（不是"尽可能强"） (练习11，反推)。
15. ⚠ **RISC 指令集**是易错点：RISC 可以不设置乘除法指令和浮点运算指令 (练习12，反推)。
16. ⚠ **跳跃寻址的功能**是高频考点：可实现程序的条件转移或无条件转移 (练习8，反推)。
17. ⚠ **程序控制类指令**是考点：功能是改变程序执行的顺序 (练习6，反推)。
18. ⚠ **位操作类指令**是考点：可对 CPU 内部通用寄存器或主存某一单元任一位进行状态检测或强置 (练习13，反推)。
19. ⚠ **运算型指令 vs 转移性指令的寻址**是考点：前者取操作数，后者决定程序转移地址 (练习7，反推)。
20. ⚠ **取指令操作**是易错点：取指令是控制器固有的自动功能，不需要在指令控制下完成 (练习5[多选]、判断4，反推)。

## 六、一句话记忆要点

1. **指令系统** = 所有机器指令的集合，是软硬件接口；系列机保证指令系统向下兼容。
2. **指令系统四要求**：完备、有效、规整（对称/匀齐/一致）、兼容。
3. **机器指令四要素**：操作码 + 源操作数地址 + 目的操作数地址 + 下一条指令地址（通常隐含由PC给出）。
4. **指令格式** = 操作码字段 + 地址码字段；地址码分三/二/单/零地址；按物理位置分 SS/RR/RS 型。
5. **多字长指令**：扩大寻址范围但需多次访存取指令。
6. **寻址方式** = 形成操作数或指令地址的方式；EA 是真正存储位置，A 是形式地址（偏移量），寻址 = A → EA。
7. **十种操作数寻址**：隐含、立即、直接、间接、寄存器、寄存器间接、相对、基址、变址、堆栈。
8. **RISC 七特点**：简单指令、固定长度、仅LOAD/STORE访存、流水线、组合逻辑控制、多通用寄存器、优化编译。
9. **RISC 四优点**：充分利用VLSI面积、提高速度、便于设计降成本提可靠性、支持高级语言。
10. **编址与PC**：按字编址PC加1，按字节编址PC加字节数（单字长加2，双字长加4）。
