## 一、程序的顺序执行与并发执行

**程序顺序执行**：一个应用程序由若干程序段组成，每个程序段完成特定功能，按某种先后次序顺序执行，仅当前一程序段执行完后才运行后一程序段。用结点代表各程序段操作，I 代表输入、C 代表计算、P 代表打印，前趋关系为 $I_i \to C_i \to P_i$。

**顺序执行的三特征**：

| 特征 | 含义 |
|------|------|
| 顺序性 | 处理机严格地顺序执行，每一操作必须在下一个操作开始之前结束 |
| 封闭性 | 程序在封闭的环境运行，独占全机资源，资源状态（除初始状态外）只有本程序才能改变，执行结果不受外界因素影响 |
| 可再现性 | 只要程序执行时的环境和初始条件相同，不论不停顿执行还是"停停走走"执行，都可获得相同的结果 |

**程序并发执行**：多道程序环境下，多个程序并发执行时产生三个特征。

| 特征 | 含义 |
|------|------|
| 间断性 | 并发程序间共享资源、相互合作形成相互制约关系，导致"执行-暂停-执行"的活动规律 |
| 失去封闭性 | 资源为多个程序共享，资源状态由这些程序共同改变，任一程序运行时环境必然受其它程序影响 |
| 不可再现性 | 失去封闭性导致失去可再现性 |

## 二、进程的概念与特征 ⭐

**进程概念**：在多道程序环境下，允许多个程序并发执行，此时它们失去封闭性并具有间断性及不可再现性。为此引入进程概念，以描述和控制程序的并发执行，实现操作系统的并发性和共享性（最基本的两个特性）。

**进程实体（进程映像）**：由程序段、相关数据段和 PCB 三部分构成。
- 创建进程实质上是创建进程映像中的 PCB；撤销进程实质是撤销进程的 PCB。
- 进程映像是静态的，进程则是动态的。
- **PCB 是进程存在的唯一标志！** ⚠

**进程的典型定义**：
1. 进程是程序的一次执行过程。
2. 进程是一个程序及其数据在处理机上顺序执行时所发生的活动。
3. 进程是具有独立功能的程序在一个数据集合上运行的过程，是系统进行资源分配和调度的一个独立单位。

**进程的五大特征**：

| 特征 | 说明 |
|------|------|
| 动态性 | 进程最基本的特征。由创建而产生，由调度而执行，由撤销而消亡，有一定生命期。程序则是一组有序指令的集合，是静态的。 |
| 并发性 | 多个进程实体同存于内存中，且能在一段时间内同时运行。引入进程的目的就是使其能并发执行。程序（没有建立 PCB）不能参与并发执行。 |
| 独立性 | 进程实体是一个能独立运行、独立获得资源和独立接受调度的基本单位。 |
| 异步性 | 进程按各自独立的、不可预知的速度向前推进。需配置进程同步机制保证结果可再现。 |

> Caveat (本章单选4)：进程与程序的根本区别是动态和静态特点。动态性是进程最重要的特性，以此来区分文件形式的静态程序。

> Caveat (本章多选21)：进程是一个独立的运行单位，也是操作系统进行资源分配和调度的基本单位，它包括 PCB、程序和数据以及执行栈区。

## 三、进程的状态及转换 ⭐

**三种基本状态**：

| 状态 | 定义 |
|------|------|
| 就绪状态 | 进程已分配到除 CPU 以外的所有必要资源，只要再获得 CPU 便可立即执行。多个就绪进程按策略（如优先级）排成就绪队列。 |
| 执行状态 | 进程已获得 CPU，其程序正在执行。单处理机系统中只有一个进程处于执行状态；多处理机系统中则有多个。 |
| 阻塞状态（等待/封锁状态） | 正在执行的进程因某事件（如 I/O 请求、申请缓冲区失败等）暂时无法继续执行，引起进程调度，受阻进程处于暂停状态。通常也排成阻塞队列，较大系统中根据阻塞原因设置多个阻塞队列。 |

**三态转换**：

| 转换 | 触发条件 |
|------|----------|
| 就绪 → 执行 | 调度程序为就绪进程分配处理机 |
| 执行 → 阻塞 | 正在运行的进程需要等待某事件（如 I/O 请求）而无法运行，让出处理机 |
| 阻塞 → 就绪 | 进程所等待的事件发生了（如 I/O 完成） |
| 执行 → 就绪 | 时间片用完被暂停；或可抢先式调度中优先级高的进程到来，低优先级进程被强制撤下处理机 |

> Caveat (本章单选16)：在进程三态转换中，不能直接由阻塞态到运行态。

> Caveat (本章单选18)：I/O 操作完成前进程为阻塞态；完成后变为就绪态（不是运行态）。

> Caveat (本章判断题34)：进程不止三种基本状态，实际系统中还存在创建、终止等状态。

**五状态模型**：在三种基本状态基础上增加创建状态和终止状态。

**挂起操作**：引入挂起原语 Suspend 和激活原语 Active 后，产生以下状态转换。

| 转换 | 说明 |
|------|------|
| 活动就绪 → 静止就绪 | Readya 用 Suspend 挂起后变为 Readys，不再被调度执行 |
| 活动阻塞 → 静止阻塞 | Blockeda 用 Suspend 挂起后变为 Blockeds，期待事件出现后变为静止就绪 |
| 静止就绪 → 活动就绪 | Readys 用 Active 激活后变为 Readya |
| 静止阻塞 → 活动阻塞 | Blockeds 用 Active 激活后变为 Blockeda |

**引起挂起的原因**：
1. 终端用户的需要（研究执行情况或修改程序）
2. 父进程请求（考查修改子进程或协调子进程活动）
3. 负荷调节的需要（实时系统负荷较重时挂起不重要进程）
4. 操作系统的需要（检查资源使用情况或记账）

**引入挂起后的状态转换**：
- NULL → 创建
- 创建 → 活动就绪
- 创建 → 静止就绪
- 执行 → 终止

## 四、进程控制

**原语（原子操作）**：所谓原语即原子操作，要么全做，要么不做，一般通过屏蔽中断来完成。

> Caveat (本章判断题32)：原语在执行过程中不允许被中断。

**六种原语操作**：

| 原语 | 功能 |
|------|------|
| create | 进程创建 |
| halt | 进程撤销 |
| block | 进程阻塞 |
| wakeup | 进程唤醒 |
| suspend | 进程挂起 |
| active | 进程激活 |

**引起创建进程的四类事件**：
1. 用户登录（分时系统中用户登录成功后建立进程）
2. 作业调度（多道批处理系统中调度到作业时创建进程）
3. 提供服务（系统为用户请求创建服务进程，如打印进程）
4. 应用请求（用户进程自己创建新进程）

> Caveat (本章单选12)：用户登录、高级调度发生时、操作系统响应用户请求、用户打开浏览器程序都会导致创建新进程。

**进程创建过程**（create 原语按以下步骤）：
1. 申请空白 PCB，为新进程申请唯一的数字标识符，从 PCB 集合中索取一个空白 PCB
2. 为新进程分配运行所需资源（内存、文件、I/O 设备、CPU 时间等，从操作系统或父进程获得）
3. 初始化进程控制块 PCB：
   - 初始化标识信息（系统分配的标识符和父进程标识符）
   - 初始化处理机状态信息（程序计数器指向程序入口地址，栈指针指向栈顶）
   - 初始化处理机控制信息（状态设为就绪或静止就绪，优先级通常设为最低）
4. 如果进程就绪队列能够接纳新进程，将新进程插入就绪队列

> Caveat (本章单选11)：为进程分配 CPU 不是由进程创建原语完成的，而是由进程调度实现的。

**引起进程终止的事件**：
1. 正常结束（任务已完成）
2. 异常结束（存储区越界、保护错、非法指令、特权指令错、运行超时、算术运算错、I/O 故障等）
3. 外界干预（操作员或 OS 干预、父进程请求、父进程终止）

**引起进程阻塞和唤醒的事件**：
1. 向系统请求共享资源失败
2. 等待某种操作的完成
3. 新数据尚未到达
4. 等待新任务的到达

**进程阻塞过程**：正在执行的进程发生上述事件后，调用阻塞原语 block 将自己阻塞。**阻塞是进程自身的一种主动行为**。立即停止执行，将 PCB 中现行状态由"执行"改为"阻塞"，并将 PCB 插入阻塞队列。

**进程唤醒过程**：被阻塞进程所期待的事件发生时，由有关进程调用唤醒原语 wakeup。将阻塞进程从阻塞队列移出，PCB 中现行状态由阻塞改为就绪，将 PCB 插入就绪队列。

**进程挂起与激活**：出现挂起事件时系统利用挂起原语 suspend() 将指定进程挂起；出现激活事件时系统利用激活原语 active() 将指定进程激活。

## 五、进程组织

**进程实体的三部分**：进程控制块（PCB）、程序段、数据段，其中最核心的是 PCB。

**PCB 的作用**：
1. 作为独立运行基本单位的标志（PCB 是进程存在的唯一标志）
2. 能实现间断性运行方式（保存和恢复 CPU 现场信息）
3. 提供进程管理所需要的信息（程序和数据地址指针、资源清单等）
4. 提供进程调度所需要的信息（进程状态、优先级、等待时间、已执行时间等）
5. 实现与其它进程的同步与通信（信号量、通信队列指针等）

**PCB 的四方面信息**：

| 信息类别 | 内容 |
|----------|------|
| 进程标识符 | 内部标识符（系统使用的数字序号）；外部标识符（创建者提供，含父/子进程标识、用户标识） |
| 处理机状态 | 处理机各种寄存器内容（中断时保存于 PCB 中，供恢复执行用） |
| 进程调度信息 | 进程状态及有关进程调度的信息 |
| 进程控制信息 | [原文未详述] |

> Caveat (本章多选29)：保存在 PCB 结构中的是进程标识符、进程当前状态以及代码段指针。

**程序段**：能被进程调度程序调度到 CPU 执行的程序代码段。程序可被多个进程共享，即多个进程可以运行同一个程序。

**数据段**：进程对应的程序加工处理的原始数据，或程序执行时产生的中间或最终结果。

**PCB 的三种组织方式**：

| 方式 | 说明 | 特点 |
|------|------|------|
| 线性方式 | 所有 PCB 组织在一张线性表中，首址存于内存专用区域 | 实现简单、开销小，但查找需扫描整张表，适合进程数目不多的系统 |
| 链接方式 | 相同状态进程的 PCB 通过链接字链接成队列（就绪队列、阻塞队列、空白队列等） | 就绪队列可按优先级排列，阻塞队列可按阻塞原因排成多个 |
| 索引方式 | 根据进程状态建立几张索引表（就绪索引表、阻塞索引表等），表目记录 PCB 地址 | 各索引表首地址记录在内存专用单元中 |

## 六、进程通信 ⭐

**进程通信**：进程之间的信息交换。P、V 操作称为低级通信。高级通信是指进程之间以较高效率传送大量数据的通信方式。高级通信方式可分为三大类：共享内存、消息传递以及管道机制。

**1. 共享存储器**：通信进程之间存在一块可直接访问的共享空间，通过写/读操作实现信息交换。用户进程空间一般独立，需通过特殊系统调用实现共享；进程内的线程是自然共享进程空间的。

**2. 消息传递**：进程间数据交换以格式化的消息为单位。通过发送消息和接收消息两个原语进行数据交换。

| 通信方式 | 说明 |
|----------|------|
| 直接通信方式 | 发送进程直接把消息发送给接收进程，挂在接收进程的消息缓冲队列上，接收进程从消息缓冲队列中取得消息 |
| 间接通信方式（信箱通信） | 发送进程把消息发送到中间实体（信箱），接收进程从信箱取得消息。广泛应用于计算机网络中，相应的通信系统称为电子邮件系统 |

> Caveat (本章单选19)：信箱通信属于消息传递中的间接通信方式，借助于收发双方进程之外的共享数据结构作为通信中转。

**3. 管道通信**：消息传递的一种特殊方式。"管道"是指用于连接一个读进程和一个写进程以实现通信的共享文件（pipe 文件）。写进程以字符流形式将大量数据送入管道；读进程从管道中接收数据。

管道机制必须提供三方面的协调能力：
1. 互斥
2. 同步
3. 确定对方的存在

**4. 客户机-服务器系统**：网络环境中的主流通信实现机制，主要实现方法分为三类：套接字、远程过程调用和远程方法调用。

> Caveat (本章多选22)：进程之间通信主要有共享内存、消息传递、系统管道等方式。

## 七、线程 ⭐

**线程的引入目的**：减少程序在并发执行时所付出的时空开销，使 OS 具有更好的并发性。

**进程的两个基本属性**：
1. 进程是一个可拥有资源的独立单位（需拥有磁盘/内存地址空间、I/O 设备、已打开文件、信号量等）
2. 进程是一个可独立调度和分派的基本单位（每个进程有唯一 PCB，系统根据 PCB 感知进程存在并进行调度）

**线程概念**：将进程的两个属性分开——资源拥有单位仍为进程，调度和执行的基本单位变为线程，"轻装上阵"，拥有资源的基本单位不施以频繁切换。

**线程的三个状态**：

| 状态 | 说明 |
|------|------|
| 执行状态 | 线程已获得处理机正在运行 |
| 就绪状态 | 线程已具备各种执行条件，只须再获得 CPU 便可立即执行 |
| 阻塞状态 | 线程执行中因某事件受阻而处于暂停状态（如执行从键盘读入数据的系统调用时被阻塞） |

**线程控制块 TCB**：系统为每个线程配置 TCB，包含以下内容。

| 内容 | 说明 |
|------|------|
| 线程标识符 | [原文未详述] |
| 寄存器状态 | 包括程序计数器 PC 和堆栈指针中的内容 |
| 堆栈 | 保存局部变量和返回地址 |
| 线程运行状态 | 描述线程处于何种运行状态 |
| 优先级 | 描述线程执行的优先程度 |
| 线程专有存储器 | 保存线程自己的局部变量拷贝 |
| 信号屏蔽 | 对某些信号加以屏蔽 |

**线程的三种实现方式**：

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 内核支持线程 KST | 在内核空间实现，创建/阻塞/撤消/切换都在内核空间完成，内核为每个线程设置 TCB | ①多处理器可同时调度同进程多线程并行执行；②一个线程阻塞可调度同进程其它线程；③数据结构和堆栈小，切换快开销小；④内核本身可多线程 | 同进程内线程切换需用户态转核心态，模式切换开销较大 |
| 用户级线程 ULT | 在用户空间实现，创建/撤消/同步/通信无需内核支持，与内核无关 | ①切换不需转内核空间；②调度算法可进程专用；③与 OS 平台无关 | ①系统调用阻塞问题：一个线程阻塞则进程内所有线程被阻塞；②不能利用多处理机多重处理，内核每次仅分配一个 CPU 给进程 |
| 组合方式 ULT/KST | 内核支持多个 KST 建立/调度/管理，同时允许用户应用程序建立/调度/管理 ULT | 兼具两者优点 | [原文未述] |

> Caveat (本章判断题36)：对用户级线程来说，进程中的某个用户级线程被阻塞，则整个进程都被阻塞，即进程中的其他用户级线程也被阻塞。

> Caveat (本章判断题31)：同一进程或不同进程内的线程都可以并发执行。

> Caveat (本章单选2)：线程包含 CPU 现场，可以独立执行程序；线程没有自己独立的地址空间，它共享其所属进程的空间；进程可以创建多个线程。

**进程与线程的比较**：

| 比较项 | 进程 | 线程 |
|--------|------|------|
| 调度基本单位 | 传统 OS 中既是资源分配也是调度单位 | 引入线程后，线程是调度和执行的基本单位。同一进程中线程切换不引起进程切换；跨进程线程切换会引起进程切换 |
| 并发性 | 进程间可并发 | 同进程内多线程亦可并发，使 OS 具有更好并发性 |
| 拥有资源 | 拥有系统资源 | 不拥有系统资源（仅有必不可少资源），可访问隶属进程的资源 |
| 独立性 | 高 | 相对进程而言独立性低 |
| 系统开销 | 切换代价远高于线程 | 切换代价低；同进程多线程具有相同地址空间，同步和通信更容易 |
| 支持多处理机 | [原文未详述] | 多线程多处理器 |

> Caveat (本章多选24)：资源分配的基本单位是进程，CPU 调度的基本单位是线程。

> Caveat (本章多选23)：在同一进程中线程切换不会引起进程切换；从一个进程的线程切换到另一进程的线程时才会引起进程切换。

> Caveat (本章单选8)：线程模型中，操作系统分配 CPU 以外的资源以进程为单位。

## 八、信号量与死锁（考点补充）

> 以下内容来自本章专项练习的考点信号，知识点正文未展开，但考试反复涉及。

**信号量**：信号量小于 0 时，其绝对值表示等待使用该资源的进程个数。

> Caveat (本章单选3)：V 操作的作用是归还资源，将资源数加一，然后判断资源数是否小于等于 0，若小于等于 0 说明有进程阻塞在当前资源上，唤醒一个进程。

> Caveat (本章单选20)：信号量当前值为 -4，表示系统中有 4 个进程在等待。

> Caveat (本章判断题33)：对计数信号量 S 执行 P 操作后，当 S.value < 0 时才表示有进程进入了信号量的等待队列中。

> Caveat (本章判断题40)：若用信号量作为同步工具，多个 P 和 V 顺序不当，也会产生死锁。

**死锁的必要条件**：
1. 互斥控制
2. 请求和保持
3. 不可剥夺控制
4. 循环等待

**死锁预防**：资源静态分配法破坏了"请求和保持"条件来预防死锁发生。

> Caveat (题目P30-T18)：死锁四个必要条件中，"互斥"条件通常由设备物理特性决定（如打印机），无法通过操作系统策略破坏，而其他三个条件（请求和保持、不剥夺、环路等待）均可破坏。

> Caveat (本章单选10)：CPU 不是竞争使用的，进程经分配调度后独占 CPU，不会因竞争 CPU 产生死锁。

**死锁最小资源数公式**：
$$R_{\min} = n \times (m - 1) + 1$$
其中 $n$ 为进程数，$m$ 为每个进程最多需要的资源数。

> Caveat (题目P28-T8)：死锁最小资源数计算时，注意"每个进程最多需要 M 台"与"每个进程已占有 M-1 台"的等价转换，此时系统处于临界死锁状态，再多分配 1 个资源即可打破死锁。

**死锁解除**：解决死锁只有撤销进程、剥夺资源。

## 九、处理机调度相关考点

> 以下内容来自本章专项练习的考点信号。

**时间片轮转法（RR）**：
- 就绪队列中诸进程轮流在 CPU 上运行，每次运行一个时间片
- 当时间片结束时，运行进程自动让出 CPU，进入**就绪队列**（不是等待队列）⚠
- 时间片长度选择过长，退化为先进先出算法（FCFS）
- 采用可变长度的时间片可以改善调度性能

> Caveat (本章单选1)：时间片结束时进程进入就绪队列，不是等待队列。

> Caveat (题目P30-T20)：时间片轮转法（RR）计算平均周转时间时，必须严格按照时间片大小交替执行，注意进程完成后从就绪队列移除，切勿按整体运行时间简单相加。

**进程调度概念**：
- 进程获得处理器运行是通过调度实现的
- 优先级分静态和动态两种，动态优先级根据运行情况随时调整
- 单处理器系统中任何时刻只有一个进程处于运行态（但死锁时可能全部阻塞，CPU 空闲）
- 进程申请处理器得不到满足时处于**就绪态**（不是阻塞态）⚠

**抢占式调度的原则**：
1. 优先权原则
2. 短进程优先原则
3. 时间片原则

**特权态与指令**：
- 处理机的所有指令可以在系统态（核心态）中执行
- 引入用户态和核心态是为了避免用户程序错误使用特权指令，保护操作系统不被用户程序破坏

> Caveat (题目P5-T20)：用户态与核心态指令判断：修改内存单元的值在用户自己的地址空间内是合法的非特权指令；而屏蔽中断、设置时钟、停机、I/O 操作均为核心态特权指令。

## 十、一句话记忆要点

- **顺序执行**三特征：顺序性、封闭性、可再现性；**并发执行**三特征：间断性、失去封闭性、不可再现性。
- **进程 = 程序段 + 数据段 + PCB**，PCB 是进程存在的唯一标志，进程是动态的、程序是静态的。
- **三态转换**：就绪↔执行（调度/时间片用完），执行→阻塞（等 I/O），阻塞→就绪（I/O 完成）；**阻塞不能直接到执行**。
- **原语** = 原子操作，不可中断，不可分割。
- **进程创建**四步：申请 PCB → 分配资源 → 初始化 PCB → 插入就绪队列；**不分配 CPU**。
- **PCB 组织**三方式：线性（简单慢）、链接（按状态/优先级排队）、索引（按状态建表）。
- **进程通信**三方式：共享存储器、消息传递（直接/间接信箱）、管道（互斥+同步+确定对方存在）。
- **线程**是调度和执行的基本单位，进程是资源分配的基本单位；用户级线程阻塞→整个进程阻塞。
- **死锁最小资源数**：$R_{\min} = n \times (m-1) + 1$。
- **信号量 < 0** 时，绝对值 = 等待进程数；**V 操作**后值 ≤ 0 则唤醒一个进程。

## Worked Examples

### 例 1：死锁临界资源计算 *(来自 exam-signals 候选 1)*

*Tags: 死锁预防, 资源分配*

**问题**：某系统有 8 台打印机，K 个进程竞争，每个进程最多需要 3 台。求可能发生死锁的 K 的最小值。

**解答**：`[PROPOSED — verify]`
1. 明确死锁发生的临界条件：每个进程都已获得部分资源但均未达到最大需求，且此时系统已无剩余资源。
2. 设每个进程最多需要 $m$ 台，当前已获得 $m-1$ 台。若系统总资源 $R$ 满足 $R = K \times (m-1)$，则系统无空闲资源，发生死锁。
3. 代入公式：$8 = K \times (3-1)$，解得 $K = 4$。
4. 验证：当 $K=3$ 时，最坏情况每个进程分得 2 台，共占 6 台，系统还剩 2 台，足够分给一个进程使其运行完毕并释放资源，不会死锁。
5. 结论：发生死锁的最小 $K$ 值为 4。

### 例 2：时间片轮转法平均周转时间计算 *(来自 exam-signals 候选 2)*

*Tags: 时间片轮转, 平均周转时间*

**问题**：4 个进程 A、B、C、D 同时到达，运行时间分别为 11、7、2、4，时间片为 1，求平均周转时间。

**解答**：`[PROPOSED — verify]`
1. 按时间片轮转原则，队列初始顺序为 A, B, C, D。
2. 模拟执行过程：A(1), B(1), C(1), D(1) → 剩余 A(10), B(6), C(1), D(3)。
3. 继续轮转：C 在第 7 个时间单位完成。D 在第 14 个时间单位完成。B 在第 20 个时间单位完成。A 在第 24 个时间单位完成。
4. 计算各进程周转时间（完成时间 - 到达时间 0）：C=7, D=14, B=20, A=24。
5. 求平均：$\frac{7 + 14 + 20 + 24}{4} = \frac{65}{4} = 16.25$。
