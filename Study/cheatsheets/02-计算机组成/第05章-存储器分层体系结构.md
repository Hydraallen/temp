## 一、本章概览

本章解决计算机系统中存储器的核心矛盾——**容量、速度、价格三者不可兼得**。围绕这一矛盾，本章依次讲解：存储器的基本特性与分类、半导体 RAM/ROM 的工作原理、存储器与 CPU 的连接及容量扩展、并行存储技术（双端口/多模块交叉）、高速缓冲存储器（Cache）的映射与替换策略、虚拟存储器的基本概念。全章构成"存储器层次结构"的完整知识链：从存储位元→芯片→主存→Cache→虚拟存储器，层层递进，是计算机组成原理的核心章节。

## 二、知识点详解

### 第一节 存储器概述

#### 知识点1 存储器的特性 ⭐(练习1,2,3)

- **字地址**：存放一个机器字的存储单元，通常称为字存储单元，相应的单元地址。
- **字节地址**：存放一个字节的单元，称为字节存储单元，相应的地址称为字节地址。
- **按字寻址**：计算机中可编址的最小单位是字存储单元。
- **按字节寻址**：计算机中可编址的最小单位是字节。

**存储器的特性：**

| 特性 | 定义 |
|---|---|
| 存储容量 | 在一个存储器中可以容纳的存储单元总数 |
| 存取时间 $t_a$ | 又称存储器访问时间 |
| 存储周期 $t_n$ | 连续启动两次读操作所需间隔的最小时间 |
| 传输率 | 数据传入或传出存储单位的速率 |
| 带宽 | 单位时间内存储器可读写的字节数（或二进制的位数） |
| 价格 | 一般用每位价格来表示 |

#### 知识点2 存储器的分类

存储器的分类方法：

| 分类依据 | 类别 |
|---|---|
| 按存取方式 | 顺序存取、直接存取、随机存取、相联存取 |
| 按存储介质 | 半导体存储器、磁表面存储器、光存储器 |
| 按读写功能 | ROM、RAM |
| 按信息的可保存性 | 非永久记忆存储器、永久记忆存储器 |
| 按在计算机系统中的作用 | 主存储器、辅助存储器、Cache 等 |

#### 知识点3 存储器的层次结构 ⭐(练习1,2,3)

存储器是计算机的核心部件之一。其性能直接关系到整个计算机系统性能的高低。

**存储系统中存在如下关系：**
- 存取时间越短，每位价格就越高。
- 容量越大，每位价格就越低。
- 容量越大，存取时间就越长。

计算机系统既需要大容量、低每位价格，又需要高速度。解决这个难题的方法是采用**存储器层次结构（Memory Hierarchy）**，而不只是依赖单一的存储部件或技术。

**在通用存储层次中，从上到下有如下关系：**
- 每位价格降低
- 容量增大
- 存取时间增大
- 处理器访问存储器的频度降低

在微机系统中，存储器的层次结构主要是**缓存—主存**和**主存—辅存**这两个存储层次。

### 第二节 半导体随机存储器

#### 知识点1 半导体存储位元

存储器的构成单元是存储位元，用于保存一位二进制的信息。

**存储位元的性质（3个条件）：**
- 呈现两种稳态（或半稳定），分别代表二进制的 1 和 0
- 能够写入（至少一次）来设置状态
- 能够读出状态

**常用的半导体存储位元：**

| 类型 | 特点 | 用途 |
|---|---|---|
| 双极型半导体 TTL（Transistor-Transistor Logic） | 速度高，驱动能力强，但集成度低，功耗大，价格高 | 多用于小容量高速存储器 |
| 金属氧化物场效应晶体管 MOS（Metal Oxide Semiconductor） | 集成度高，功耗小，工艺简单，成本低，但速度较低 | 主要用于大容量存储器 |

> 在计算机中 MOS 器件组成的存储器是最为常用的。

#### 知识点2 半导体存储器的分类 ⭐(练习11) ⚠(练习4多选)

计算机中使用的半导体存储器主要有两类：半导体 RAM（Random Access Memory）和半导体 ROM（Read Only Memory）。RAM 具有易失性，而 ROM 是非易失性的。

**RAM 分为：**

| 类型 | 特点 |
|---|---|
| 静态 RAM（Static RAM, SRAM） | 速度快，集成度低，容量小，使用方便；不需要刷新操作；造价较高，通常用作高速缓冲存储器（Cache） |
| 动态 RAM（Dynamic RAM, DRAM） | 速度比 SRAM 慢，容量大，需要定时刷新操作；目前在个人计算机中使用最多；必须定时刷新，否则存储的数据会丢失 |

**存储位元的组织形式：**
- 一维地址译码方式（单译码方式）
- 二维地址译码的位选方式
- 二维地址译码的字选方式

> 常用的形式为**二维地址译码的字选方式**。

**地址线复用与方阵型行列结构：** 采用地址线的复用和方阵型行列结构，每增加一个专用的地址引脚，便使得行地址和列地址的指示范围加倍，因此存储器芯片的容量以 **4 的倍数**增长。

**DRAM 刷新：** 由于 DRAM 存储单元被访问是随机的，有可能某些存储单元长期得不到访问，无读出也就无重写，其原信息必然消失，必须采用定时刷新的方法。

**常用的刷新方式有三种：**
- 集中式
- 分散式
- 异步式

> 在实际应用中，经常使用 DRAM 控制器与 DRAM 配合使用。DRAM 控制器是 CPU 与 DRAM 芯片之间的接口电路，负责完成刷新、刷新/访存裁决等操作。借助 DRAM 控制器，可把 DRAM 看作 SRAM 一样使用。

### 第三节 半导体只读存储器

#### 知识点1 计算机为什么配置ROM

ROM 主要用于存放系统程序、常用功能的库例程和功能表，当程序或数据需要永久保存在主存中，不需要从辅助存储器中调入时，存放在 ROM 中。

**计算机中配置 ROM 的原因：** 由于目前绝大部分计算机的工作方式为指令流驱动，因此要求计算机加电后必须要运行程序，因此不能只用易失性的 RAM 构成主存储器，需要有 ROM 存储引导程序。

#### 知识点2 ROM的分类 ⭐(练习3多选,6多选)

常用 ROM 芯片有如下几种：

| 类型 | 全称 |
|---|---|
| MROM | Masked Read Only Memory（半导体掩模只读存储器） |
| PROM | Programmable ROM（一次性编程只读存储器） |
| EPROM | Erasable PROM（可擦写可编程只读存储器） |
| EEPROM | Electrically Erasable PROM（电可擦写可编程只读存储器） |
| FLASH Memory | 快闪存储器 |

### 第四节 存储器与CPU连接

#### 知识点1 存储器与CPU如何连接

存储器和 CPU 的连接，必须按照芯片提供的引脚特征进行连接。CPU 对存储器进行读/写操作：
1. 首先由**地址总线**给出地址信号
2. 然后发出**读操作或写操作的控制信号**
3. 最后在**数据总线**上进行信息交流

因此，存储器同 CPU 连接时，要完成**地址线、数据线和控制线**的连接。

一般的存储芯片的引脚包括：地址、数据、片选信号、读写信号等。

#### 知识点2 存储容量扩展方法 ⭐(练习13,14) ⚠(练习13,14)

常用的存储容量扩展方法：

| 方法 | 说明 |
|---|---|
| 位扩展法 | 扩展数据位宽 |
| 字扩展法 | 扩展地址范围（需使用译码器） |
| 字位同时扩展法 | 同时扩展字和位 |

在字扩展中需要使用译码器，常用的有 **3-8 译码器 74138** 和 **2-4 译码器 74139**。

> 存储芯片与 CPU 芯片相连时，特别要注意片与片之间的地址线、数据线和控制线（如片选等）的连接。

#### 知识点3 加速CPU和存储器之间的有效传输

CPU 重点提高处理速度，主存重点提高容量，速度差异越来越大，使主存存取速度成为系统瓶颈。可采取的加速途径：

- 主存储器采用更高速的技术来缩短存储器的读出时间，或加长存储器的字长
- 采用并行操作的**双端口存储器**
- 在每个存储器周期中存取几个字（**多体交叉存储器**）
- 在 CPU 和主存储器之间插入一个**高速缓冲存储器（Cache）**，以缩短读出时间

#### 知识点4 主存地址在各模块中的安排 ⚠(练习8判断)

一个由若干个模块组成的主存储器是线性编址的。地址在各模块中的安排有两种方式：

| 方式 | 特点 | 优点 | 缺点 |
|---|---|---|---|
| **顺序组织方式** | 某个模块进行存取时，其他模块不工作 | 某模块故障时其他模块可照常工作；增添模块扩充容量方便 | 各模块串行工作，存储器带宽受限 |
| **交叉组织方式** | 连续地址分布在相邻的不同模块内，同一模块内地址不连续 | 对连续字的成块传送可实现多模块流水式并行存取，大大提高带宽 | — |

> 交叉组织方式使用场合为**成批数据读取**。

### 第五节 高速缓冲存储器

#### 知识点1 程序的局部性原理 ⭐(练习17)

多级存储体系能够解决存储器容量、速度、价格之间矛盾的关键：**程序执行的局部性原理**。

程序局部性原理：程序在执行时所呈现的局部性规律，即在一段较短时间内，程序的执行仅限于某个部分，相应地，它所访问的存储器空间也局限在某个空间。

| 表现 | 含义 | 主要原因 |
|---|---|---|
| **时间局部性** | 如果某条指令被执行，则不久以后该指令很可能再次被执行；如果某条数据结构被访问，则不久以后该数据结构很可能再次被访问 | 程序中有大量的循环操作 |
| **空间局部性** | 一旦程序访问了某个内存单元，不久以后，其附近的内存单元也要被访问，即程序在一段时间内所访问的存储器空间可能集中在一定的范围之内 | 最常见情况就是程序的顺序执行 |

#### 知识点2 Cache的原理 ⭐(练习15)

高速缓冲存储器是为了解决 CPU 和主存之间速度不匹配而采取的一项重要技术。

**Cache 的基本原理：**
- Cache 的主要目的是为了解决存储器速度问题
- Cache 的容量要远小于主存，因此必须考虑两者间的地址变换问题
- Cache 中使用的所有算法均由**硬件**实现
- Cache 技术实现的基本目标是**提高命中率**
- Cache 和主存之间以**块**为单位交换信息
- Cache 与主存**同时**进行读写

#### 知识点3 设计Cache需要考虑的因素

在 Cache 系统的设计中，需要考虑的基本要素有：
- Cache 容量的大小
- 主存地址映射到 Cache 的方法
- 当新的数据块装入 Cache 时，替换掉原数据块的策略
- CPU 执行写操作时写的策略
- 数据块大小的选择
- Cache 数目的选择

#### 知识点4 三种映射 ⭐(练习16) ⚠(练习9判断)

映射功能的选择决定了 Cache 的结构，通常采用三种技术：

| 映射方式 | 特点 | 优点 | 缺点 |
|---|---|---|---|
| **全相联** | 主存任意块可映射到 Cache 任意位置 | 灵活，不易产生冲突 | 比较电路难于实现，且效率低，速度慢 |
| **直接映射** | 主存块只能映射到 Cache 固定位置 | 硬件简单，成本低 | 容易产生冲突，不能有效利用 Cache 空间 |
| **组相联** | 组间采用直接映射，组内采用全相联映射 | 以上两种方式的结合 | — |

> 当使用全相联和组相联方式时，当新的一块数据装入 Cache 时，原存储的一块数据就有可能被替换掉，会使用到替换策略。

**常用的替换策略：**

| 替换算法 | 说明 |
|---|---|
| 先进先出（FIFO） | 替换掉那些在 Cache 中停留时间最长的块。用循环或环形缓冲技术很容易实现 |
| 最不经常使用（LFU） | 将计数周期限定在对这些特定行两次替换之间的时间间隔内，不能严格反映近期访问情况；没有考虑对新调入行的处理，新调入行的计数值小，容易被替换出去 |
| 近期最少使用（LRU） | 目前使用较多的一种策略，能够有效地提高命中率 |
| 随机替换 | 硬件容易实现并且速度快，实际模拟显示其性能还是不错的 |

#### 知识点5 Cache的写操作 ⚠(练习5多选)

当程序运行过程中需要对某个单元进行写操作时，会出现如何使 Cache 与主存内容保持一致的问题。目前采用的方法主要有以下 2 种：

| 写策略 | 别名 | 优点 | 缺点 |
|---|---|---|---|
| **写直达法** | 通过式写（Write Through） | 数据不会出现不一致 | 对写操作没有高速缓存的作用 |
| **写回法** | Write Back | 速度快 | 存在数据不一致隐患 |

**Cache 块大小与命中率的关系：**
- 关系复杂，取决于特定程序的局部性特征，目前还没有找到确定的最优值
- 较大的块能减少装入 Cache 的块数（少量块导致装入的数据很快被改写）
- 当块变得较大时，每个附加的字离所要的字更远，被使用的可能性就更小

**Cache 数目的设计问题：**

| 问题 | 说明 |
|---|---|
| 单级与两级 Cache | 使用两级或两级以上 Cache 确实可以提高性能 |
| 统一和分立 Cache | 统一 Cache：可充分利用 Cache 空间，便于管理；分立 Cache：取消了指令和数据的访问冲突，便于实现流水线 |

### 第六节 虚拟存储器

#### 知识点1 虚拟存储器的概念 ⭐(练习17) ⚠(练习10判断)

虚拟存储器只是一个容量非常大的存储器的**逻辑模型**，不是任何实际的物理存储器。它借助于磁盘等辅助存储器来扩大主存容量，使之能被更大或更多的程序所使用。

虚拟存储器不仅是解决存储容量和存取速度矛盾的一种方法，而且也是管理存储设备的有效方法。

- **虚拟地址（虚地址）**：程序的逻辑地址
- **虚拟地址空间**：程序的逻辑地址空间
- **物理地址（实地址）**：由 CPU 地址引脚送出，用于访问主存的地址

**主存—外存层次的基本信息传送单位：**

| 方案 | 说明 |
|---|---|
| 段 | 利用程序的模块化性质，按照程序的逻辑结构划分成的多个相对独立的部分 |
| 页 | 定长的页 |

#### 知识点2 虚拟存储器和Cache的区别 ⚠(练习10判断)

| 对比项 | Cache | 虚拟存储器 |
|---|---|---|
| 实现方式 | 用硬件实现 | 用软件、硬件相结合组成 |
| 对操作系统的透明性 | 对操作系统透明 | — |
| 对未命中的敏感度 | 较低 | 对未命中更加敏感 |

## 五、高频考点与易错提醒

1. ⭐ **存储系统三级层次**（Cache、主存、辅存）是高频考点。(练习1, 判断3)
2. ⭐ **存取速度排序**：半导体内存 > 硬盘 > 软盘 > 磁带/光盘，半导体（电子）速度远快于机械结构。(练习2,18)
3. ⭐ **易失性判断**：RAM 断电丢失，ROM 断电不丢失；主存储器是易失存储器。(练习3, 判断6)
4. ⚠ **寻址范围计算**：需注意按字编址/按半字编址/按字节编址的区别——字长决定每单元位数，总容量÷每单元位数=寻址范围。(练习4,5，反推)
5. ⚠ **存储芯片容量计算**：$2^{\text{地址线根数}} \times \text{数据线位数}$ = 芯片容量。(练习6,9,10，反推)
6. ⚠ **32位处理器最大地址空间**：$2^{32} = 4\text{GB}$（按字节编址）。(练习7，反推)
7. ⚠ **地址范围字节计算**：末地址 − 首地址 + 1 = 总字节数，注意十六进制运算。(练习12，反推)
8. ⚠ **存储器扩展（芯片数计算）**：字扩展数 = 总字数/芯片字数；位扩展数 = 总位数/芯片位数；总芯片数 = 字扩展数 × 位扩展数。(练习13,14，反推)
9. ⭐ **DRAM 需要刷新**，SRAM 不需要刷新；DRAM 速度不如 SRAM。(练习11, 判断7)
10. ⭐ **Cache 目的**：解决 CPU 和主存速度不匹配，提高内存访问速度（进而提高 CPU 效率）。(练习15)
11. ⭐ **LRU 替换算法**是理论上最好的 Cache 替换策略，符合程序局部性原理。(练习16)
12. ⭐ **虚拟存储器理论基础**：程序的局部性原理。(练习17)
13. ⚠ **顺序存取存储器**：磁带属于顺序存取；磁盘/光盘/U盘属于直接存取。(练习19)
14. ⚠ **磁盘定位**：依靠磁臂移动定位磁道，依靠磁盘旋转定位扇区。(练习20)
15. ⚠ **主存容量单位**：基本单位是字节（Byte），不是字长；与地址总线位数相关，与 CPU 时钟频率无直接关系。(多选1)
16. ⚠ **半导体存储器并非全部随机存取**：相联存储器按内容访问，不按地址访问。(多选4)
17. ⚠ **Cache 字块大小**：字块大可利用空间局部性增加命中机会，但太大使失效损失变大且 Cache 项数变少使命中可能性变小。(多选5)
18. ⚠ **组相联映射特例**：$c=0$ 时退化为全相联映射。(判断9)
19. ⚠ **虚拟存储器构成**：由 DRAM 和硬盘及软件共同构成，不能简单说"由硬盘构成"。(判断10)
20. ⚠ **双端口存储器是空间并行，多模块交叉存储器是时间并行**。(判断8)

## 六、一句话记忆要点

1. **存储器特性**：容量、存取时间 $t_a$、存储周期 $t_n$、传输率、带宽、价格——周期≥存取时间。
2. **层次结构**：从上到下——价格降、容量增、存取时间增、访问频度降；微机中两个层次：缓存—主存、主存—辅存。
3. **SRAM vs DRAM**：SRAM 快、小、贵、不刷新、做 Cache；DRAM 慢、大、需刷新、做主存。
4. **ROM 配置原因**：加电必须运行程序，不能只用易失性 RAM，需 ROM 存引导程序。
5. **存储器扩展**：位扩展加数据线、字扩展加地址线（需译码器 74138/74139）、字位同时扩展两者皆需。
6. **顺序 vs 交叉组织**：顺序串行容错好但带宽受限；交叉并行带宽高适合成批读取。
7. **局部性原理**：时间局部性（循环→近期重访）、空间局部性（顺序执行→邻近访问）——是 Cache 和虚拟存储器的理论基础。
8. **Cache 原理**：硬件实现、以块为单位交换、目标是提高命中率、与主存同时读写。
9. **三种映射**：全相联灵活但慢、直接映射简单但冲突多、组相联折中（组间直接、组内全相联）。
10. **替换算法**：LRU 最好（符合局部性原理），FIFO 最简单，随机替换硬件易实现。
11. **Cache 写策略**：写直达（一致但无缓存效果）、写回（快但有不一致隐患）。
12. **虚拟存储器 vs Cache**：Cache 纯硬件、对 OS 透明；虚存软硬件结合、对未命中更敏感；虚存传送单位为段/页/段页。
