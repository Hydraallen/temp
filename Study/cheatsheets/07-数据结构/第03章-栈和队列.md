## 一、本章概览

本章介绍两种**操作受限的线性表**——栈与队列。栈限制在表的一端（栈顶）进行插入和删除，遵循**后进先出（LIFO）**原则；队列限制在一端（队尾）插入、另一端（队头）删除，遵循**先进先出（FIFO）**原则。每种结构又分别介绍了**顺序存储**和**链式存储**两种实现方式，并引出顺序队列中的"假上溢"问题及其解决方案——循环队列。本章是后续树、图等复杂数据结构的基础，也是函数调用、递归、括号匹配等实际应用的核心支撑。

## 二、知识点详解

### 1. 栈的定义 ⭐⚠ (练习2,3,6,9,10,11,14,17,20,判断2,8,9)

- **定义**：栈是限制在表的一端进行插入、删除运算的线性表。允许插入、删除的这一端被称为**栈顶**，另一个固定端称为**栈底**。当表中没有元素时称为**空栈**。
- **要点**：
  1. **线性结构**：栈元素具有线性（即前驱后继）关系。
  2. **限制操作**：限制了线性表插入和删除的位置。
  3. **单向延伸性**：栈底是固定的，栈顶随着插入和删除操作的进行而变化。
  4. **插入**也称为进栈、压栈、入栈；**删除**也称为出栈、弹栈。
  5. 栈是一种**后进先出（LIFO — Last In First Out）**的数据结构。
- **重要注意**：栈只对线性表插入和删除的**位置**进行了限制，并没有限定插入和删除操作进行的**时间**。由于出栈可随时进行，只要某个元素位于栈顶就可以出栈，所以元素最终的出栈次序可能是多样的。

---

### 2. 顺序栈 ⭐ (练习4,5,判断1,4,6)

- **定义**：用顺序存储方式实现的栈称为顺序栈。类似于顺序表的定义，栈中的数据元素用一个预设足够大的一维数组来实现：`datatype data[MAXSIZE]`，栈底位置可以设置在数组的任一个端点，而栈顶是随着插入和删除而变化的，因此需要一个栈顶指针来标示，设变量 `int top` 作为栈顶指针使用，指明当前栈顶的位置。
- **类型描述**：

```c
#define MAXSIZE 1024 //设定栈的最大长度为 1024，可根据实际情况进行修改
typedef struct
{
    datatype data[MAXSIZE];
    int top;
}SeqStack
```

- **指针定义**：定义一个指向顺序栈的指针：`SeqStack *s;`

---

### 3. 链栈 ⭐ (练习1,4,判断1,5)

- **定义**：用链式存储结构实现的栈称为链栈。通常链栈用单链表表示，因此其结点结构与单链表的结构相同。
- **类型描述**：

```c
typedef struct snode
{
    datatype data;
    struct snode *next;
}StackNode, * LinkStack;
```

- **指针定义**：定义 top 为栈顶指针变量：`LinkStack top;`

---

### 4. 上溢 (练习19)

- **定义**：栈满时再做进栈运算产生的空间溢出。

---

### 5. 下溢

- **定义**：栈空时再做退栈运算产生的无元素可退。

---

### 6. 队列的定义 ⭐ (练习9,10,12,15,17,18,判断3)

- **定义**：队列是仅允许在表的一端进行插入运算，而在表的另一端进行删除运算的受限的线性表。允许插入的一端称为**队尾**，允许删除的一端称为**队头**。
- **要点**：
  1. **线性结构**：相邻元素具有前驱后继关系。
  2. **限制操作**：限制了线性表插入和删除的位置。
  3. **插入**也称为入队、进队；**删除**也称为出队。
  4. 队列是一种**先进先出（FIFO — First In First Out）**的数据结构。

---

### 7. 顺序队列 (练习18,判断1,4)

- **定义**：顺序存储的队称为顺序队列。因为队的队头和队尾都是活动的，因此，除了队列的数据区外还有队头、队尾两个指针。
- **类型定义**：

```c
define  MAXSIZE  1024          //队列的最大容量
typedef  struct
{datatype  data[MAXSIZE];      //队员的存储空间
  int rear,front;              //队头队尾指针
}SeQueue;
```

> [?] 注：原文 `define` 疑为 `#define` 的 OCR 识别遗漏，但按原文保真保留。

- **指针定义**：定义一个指向队的指针变量：`SeQueue *q;`
- **申请存储空间**：`q= (SeQueue *)malloc(sizeof(SeQueue));`
- **队列的数据区**为：`q->data[0]～q->data[MAXSIZE-1]`
- **队头指针**：`q->front`
- **队尾指针**：`q->rear`

---

### 8. 链队列 ⭐⚠ (练习7,8,13,判断1,5,7)

- **定义**：链式存储的队称为链队列。和链栈类似，用单链表来实现链队列，根据队的 FIFO 原则，为了操作上的方便，分别需要一个头指针和尾指针。其实这就是一个带头尾指针的单链表，只要对它的操作按照队列的规则进行，它就是一个链队。
- **结构说明**：头指针 `front` 和尾指针 `rear` 是两个独立的指针变量，共同标示这个链队。从结构性上考虑，将二者封装在一个结构中，用指向头尾指针结点的变量 `q` 标示这个链队。
- **链队示意图**（图3-2）：
  - a) 非空队：`q → [front|rear] → [////] → [a₁] → [a₂] → ... → [aₙ]∧`
  - b) 空队：`q → [front|rear] → [////]∧`
  - c) 链队中只有一个元素结点：`q → [front|rear] → [////] → [a₁]∧`
- **类型描述**：

```c
typedef struct node
{
    datatype  data;
    struct  node *next;
} QNode;          //链队结点的类型
typedef struct
{
    QNode *front,*rear;
}LQueue;          //将头尾指针封装在一起的链队
```

- **指针定义**：定义指向链队的指针：`LQueue *q;`

---

### 9. 假上溢 ⚠ (练习19)

- **定义**：在顺序队列中由于队的头尾位置只增不减，导致队尾位置已到队列空间的上界不能再做入队操作，而随着出队操作的进行，队首位置增加，使队列空间尚有空闲单元，这种队列已满的假象被称为**假上溢**。

---

### 10. 循环队列 (练习19)

- **定义**：将顺序队列的向量空间视为一个首尾相接的圆环进行运算的队列。
- **作用**：克服了假上溢现象，充分利用了向量空间。

## 四、重点对比与易混辨析

### 栈 vs 队列

| 对比项 | 栈 | 队列 |
|--------|-----|------|
| 操作受限位置 | 仅在表的一端（栈顶）插入和删除 | 一端（队尾）插入，另一端（队头）删除 |
| 操作原则 | LIFO（后进先出） | FIFO（先进先出） |
| 固定端 | 栈底固定 | [原文未述] |
| 线性结构 | 是 | 是 |
| 顺序存储 | 顺序栈 | 顺序队列 |
| 链式存储 | 链栈 | 链队列 |

### 顺序栈 vs 链栈

| 对比项 | 顺序栈 | 链栈 |
|--------|--------|------|
| 存储方式 | 一维数组 `data[MAXSIZE]` | 单链表 |
| 栈顶指针 | `int top`（标示数组位置） | `LinkStack top`（指针） |
| 结点结构 | data 数组 + top 封装在 struct | `struct snode` 含 data 和 next |
| 栈底位置 | 可设在数组任一端点 | [原文未述] |

### 顺序队列 vs 链队列

| 对比项 | 顺序队列 | 链队列 |
|--------|----------|--------|
| 存储方式 | 一维数组 `data[MAXSIZE]` | 单链表 |
| 指针数量 | 队头 `front` + 队尾 `rear` | 队头 `front` + 队尾 `rear`（封装在一个结构中） |
| 指针封装 | `front`、`rear` 在同一 struct 内 | `QNode *front, *rear` 封装在 `LQueue` 内 |
| 特有问题 | 假上溢 | [原文未述] |
| 解决方案 | 循环队列 | [原文未述] |

### 上溢 vs 下溢 vs 假上溢

| 概念 | 发生场景 | 原因 |
|------|----------|------|
| 上溢 | 栈满时再做进栈运算 | 栈空间已满 |
| 下溢 | 栈空时再做退栈运算 | 栈中无元素可退 |
| 假上溢 | 顺序队列队尾到上界但前方有空闲 | 头尾位置只增不减 |

## 五、高频考点与易错提醒

1. ⚠ **栈的出栈序列判断**是高频易错点：栈只限制插入/删除位置，不限制操作时间，因此出栈次序多样。判断某序列是否可能时，需逐元素模拟入栈/出栈过程。(练习3,6,判断2,9)

2. ⚠ **链栈插入操作步骤**：`p->next=top; top=p;`——先链接新结点到旧栈顶，再修改栈顶指针。顺序不能反。(练习1)

3. ⚠ **链队列入队操作步骤**：`r->next=p; r=p;`——先将新结点链接到队尾，再更新队尾指针。(练习7)

4. ⚠ **链队列出队时指针修改**：当队列中只有一个元素时，删除后头、尾指针都要修改；多于一个元素时只修改头指针。(练习8)

5. ⚠ **共享栈满条件**：两栈共享空间 `S[1,m]`，栈1底在 `S[1]`，栈2底在 `S[m]`，栈满条件为 `top[1]+1=top[2]`。(练习5，反推)

6. ⚠ **后缀表达式转换**：需掌握中缀表达式转后缀表达式的方法。(练习4，反推)

7. ⭐ **栈的典型应用**：函数调用/递归、括号匹配（含HTML标签匹配）、撤销（undo）操作。(练习9,11,14,17)

8. ⭐ **队列的典型应用**：打印机任务调度、售票柜台服务、多道程序设计。(练习12,15)

9. ⚠ **顺序存储需考虑溢出，链式存储一般不需要**：顺序存储分配固定连续空间，需考虑溢出；链式存储动态分配空间，一般不需要考虑溢出（除非内存不够）。(判断4,5)

10. ⚠ **共享栈栈底设置**：两个栈共享连续内存空间时，栈底应分别设在空间两端，以提高利用率。(判断6)

11. ⚠ **循环单链表队列可不设队头指针**：队头可由队尾指针指向的最后一个结点的 next 域得到。(判断7)

12. ⚠ **堆栈适合处理顺序与输入顺序相反的问题**，队列适合处理顺序与输入顺序相同的问题。(判断8)

13. ⚠ **循环队列的目的**是克服假溢出（假上溢）问题，充分利用向量空间。(练习19)

## 六、一句话记忆要点

- **栈**：一端进出，后进先出（LIFO），栈底固定栈顶动。
- **队列**：一端进一端出，先进先出（FIFO），队头出队尾进。
- **顺序栈**：数组 + `top` 指针，栈底可设数组任一端。
- **链栈**：单链表头作栈顶，插入删除均在链头。
- **上溢/下溢**：满栈再压→上溢，空栈再弹→下溢。
- **假上溢**：顺序队列头尾只增不减，队尾到顶但前方有空——假满现象。
- **循环队列**：向量空间首尾相接成环，克服假上溢，充分利用空间。
- **链队列**：单链表 + 头尾指针封装一体，队尾设在链表尾最方便入队。
