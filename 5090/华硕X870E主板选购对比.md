## 5. 三个真正的分水岭

在 SATA 数量、USB4、第一条 PCIe 5.0 x16 全系相同的前提下，**只有下面 3 项真正区分档次**：

### 分水岭 1：第二条 x16 是否 CPU 直连 PCIe 5.0
- **是真 x8/x8 的**：**保留的两款（Dark Hero、ProArt）均属此类**——第二条 x16 走 CPU 直连、可 x8+x8 拆分。
- **对比（已移除）**：Strix 全系的第二条 x16 只是**物理 x16、电气 PCIe 4.0 x4，且走芯片组**，不能 x8/x8，正因如此（叠加无 10G）本轮已被移除。
- 影响对象：**双计算卡 / 双 5090**、PCIe 5.0 扩展卡（高速采集卡、网卡、M.2 扩展卡）用户——这类第二张卡才真吃带宽
- **不影响**：RTX 3060 之类**副卡**（额外显示输出/编码/轻量推理），它不吃带宽，插芯片组 4.0 x4 就够（详见 [6.5](#65-多卡场景未来加第二张卡的通道与拆分逻辑)）

### 分水岭 2：CPU 直连 PCIe 5.0 M.2 的数量
- **2 条**：**保留的两款（Dark Hero、ProArt）均为 2 条**（原有 3 条的 Hero / Strix X870E-E 已随本轮移除）。
- 因此这一项在两款间不再构成差异；下方关于「第 3 条 CPU M.2 从显卡 16 条通道里借」的原理仍适用于理解通道分配（见下节）。

### 分水岭 3：万兆（10G）有线网口
- **有**：**保留的两款均有**——DARK HERO（10G+5G）、ProArt（10G+2.5G），这正是本轮筛选保留它们的原因。
- **没有（已移除）**：原 Strix 全系与 Crosshair HERO 均无 10G（最高 5G），已被移除。
- 现实提醒：家里没有 10G 交换机 / NAS 的话，这一项其实用不满；本轮按用户硬性要求以 10G 为门槛。

## 6. 9950X + RTX 5090 专项建议（重点）

### 6.1 平台事实

**AMD Ryzen 9 9950X**
- Zen 5 架构，16 核 32 线程
- TDP **170W**，PPT ≈ 230W
- **无 3D V-Cache**
- AM5 接口，PCIe 5.0
- 总计 **28 条可用 PCIe 通道** = 16（显卡）+ 4（CPU 直连 M.2）+ 4（芯片组链路）+ 4（额外 CPU M.2 / 其他）

### 6.2 五条关键推论

#### ① 供电相数：ProArt 的 16+2+2 / 80A 方案余量充足，别为相数付钱

9950X 满载 PPT 约 230W（170W TDP，比 9900X 高约 40%）。以 **ProArt 的 16 相 × 80A ≈ 1280A** 理论供电能力计算，喂 230W PPT 的 9950X 依然绰绰有余，实际负载占用率仍低、温度余量充足——这类 80A 级供电稳喂 9950X 没有问题。

Dark Hero 的 **20~24 相 110A（2000A+）** 供电堆料，是给 **液氮/极限超频、长时间 AVX-512 全核满载**这类极端场景准备的。对一颗默认 170W TDP 的 9950X，日常与全核生产力负载下这些多出来的相数依旧是过剩的，**不会带来任何频率、温度或稳定性上的可感知收益**，只有拉到极限超频才用得上。

> 结论：供电对 9950X 而言，ProArt 的 80A 档已完全够用；Dark Hero 更强的供电与散热用料属于「用料更好」的加分项，而非 9950X 必需——但它叠加上「第二网口更快（5G vs 2.5G）」，本次仍略倾向 Dark Hero（见 6.3）。

#### ② 两款保留板都带第二条 CPU 直连 x16，但单卡都用不上，且风冷双卡装不下——这不是本次的付费点

单卡状态下，RTX 5090 占用第一条 CPU 直连 PCIe 5.0 x16 即可满血运行。**当前**这台机器不需要第二条 CPU 直连 x16。

保留的两款（**Dark Hero、ProArt**）**恰好都带**第二条 CPU 直连 PCIe 5.0 x16、可 x8+x8 拆分——这原本是双卡刚需。但本次显卡是 **TUF 5090（3.6 槽 / 348mm 风冷）**，两张这种卡在标准 ATX 上**物理装不下**（见 ④ 与 [6.5](#65-多卡场景未来加第二张卡的通道与拆分逻辑)），所以这条双 x16 能力**对本单卡配置价值为 0**，两款在这一项上打平、不构成谁更值的理由。

> 结论：既然两款都有、单卡又都用不上，选板就不看这条，而看**第二网口速率、供电 / 散热用料、M.2 布局、外观与价格**这些真有差异的项（见 6.3）。

#### ③ 通道原理：CPU 直连 M.2 别插到抢显卡通道（本次两款均只有 2 条 CPU 5.0 M.2，天然不踩坑）

Ryzen 9000 只有 28 条通道。在某些「3 条 CPU 直连 5.0 M.2」的板上，**插满第 3 条 CPU M.2 会触发 x8+x4+x4 拆分、显卡从 x16 掉到 x8**——但**保留的两款都只有 2 条 CPU 直连 5.0 M.2**（Dark Hero M.2_1/2、ProArt M.2_1/2），这两条走独立通道、不动显卡，**天然不会把显卡拖到 x8**。第 3 块及以后的 SSD 都走芯片组 PCIe 4.0 x4，对游戏 / 日常体感差异极小。

对 RTX 5090（实测数据，见来源）：
- PCIe 5.0 x8 的带宽（约 32GB/s 单向）**等于 PCIe 4.0 x16**，游戏实测平均损失仅约 **1%**（TechPowerUp / GamersNexus 的 5090 PCIe scaling 测试）；作为对比 PCIe 3.0 x16 平均掉约 **4%**
- **但生产力/内容创作不同**：DaVinci Resolve 等场景下 5.0 x8 ≈ 4.0 x16 几乎无损，然而在**带宽更受限**的配置（更低代次或更少通道，如降到 4.0 x8 甚至 4.0 x4）下，个别 3D / 视频工作流可损失高达 **约 25%**（Puget Systems / Tom's Hardware）。这正是「副卡 vs 计算卡」要区别对待的根源

> 提示：各板 lane sharing 规则不同，**装机前仍建议查该型号说明书的 lane sharing 表**，确认 SSD + 显卡 + 扩展卡组合不会互相降速。

#### ④ TUF 5090 是 3.6 槽 / 348mm 的超厚超长卡 → 优先看 M.2 散热布局和免工具卸载

本次选定的**华硕 TUF Gaming RTX 5090 OC 实测占 3.6 槽厚、348mm 长（348 × 146 × 72 mm，ASUS 官网 techspec）**，是市面上最厚最长的 5090 之一，比"3 槽卡"整整厚了半槽以上，实际后果只会更严重：
- **会完全压住第一条 M.2 散热片**（位于显卡正下方），高负载时 SSD 温度上升明显；146mm 的高度 + 348mm 的长度还可能盖到第二条 M.2 位
- **会挡住第二条 PCIe 槽，甚至连第三条也够不着**（3.6 槽的散热鳍片会向下侵占到第 4 槽位置；对单卡无所谓，但影响声卡/采集卡）
- 拆装显卡时，PCIe 卡扣被这么厚的散热鳍片死死挡住，非常难按——所以 **Q-Release / Q-Release Slim 免工具卸载对这张卡尤其重要**

选板要点：
1. 第一条 M.2 是否有**独立厚散热马甲**，最好带背面散热（Dark Hero 的 3D VC 均热板是这一档最好的）
2. 是否有 **Q-Release / Q-Release Slim 免工具显卡卸载**——对 5090 这种重卡是刚需体验
3. 优先把主力 SSD 装在**不被显卡覆盖**的 M.2 位

#### ⑤ 电源与供电接口

- TUF Gaming RTX 5090 OC TGP **575W**（默认功率墙与公版一致，OC 版只提频不提功率墙；可手动上调至 600W），+ 9950X 满载约 230W（PPT）+ 其余
- **单卡整机建议 1000W 以上电源**——与 **ASUS 官方对该卡的建议电源（1000W）一致**（推荐 1000–1200W ATX 3.1 / 原生 12V-2×6 接口）；这张卡为 **单个 16-pin（12V-2×6）供电**，接线时务必插到底避免接触电阻过大
- **若未来上双 5090**：两卡实测整机可拉到约 **1.5kW** 负载（hostkey 双 5090 实测），电源需 **1600W 级起步、留头建议 2000W**，且需两路独立 12V-2×6 供电——这是把双 5090 从「想法」变成「现实」的第一道硬门槛
- 主板端需确认 **CPU 供电为双 8Pin（EPS 8+8）**——保留的两款（Dark Hero / ProArt）均满足

### 6.3 选购建议：Dark Hero vs ProArt（本次单卡配置）

保留的两款都过了全部硬性条件，且都属「带第二条 CPU 直连 x16」的高阶档；**本次是单张 TUF 5090 风冷、双 x16 用不上（风冷双卡装不下）**，所以取舍落在两款之间。

#### 硬性筛选条件（两款均满足）

1. **最大内存容量 ≥ 192GB**（用户硬性要求）。两款均为 **4×DIMM / 最大 256GB**（即 4×64GB，ASUS 官方规格页明确标称），256GB > 192GB。
2. **CPU 供电为双 8Pin（EPS 8+8）**。两款均满足。
3. **有线网口必须有 10G**（本轮硬性要求）。两款均配 10G（Dark Hero 10G+5G、ProArt 10G+2.5G）——这正是只保留它们的原因。
4. **主板预算 ≤ ¥8000**。两款均在预算内（Dark Hero ¥4,899、ProArt ¥4,699）。
5. **机箱非背插（不接受 BTF / 背插板）**。两款均为常规正面走线板。

> 关于 ≥192GB 如何达成：4-DIMM 板最大 256GB 即 **4×64GB**，天然覆盖；想正好 192GB 也可用 **4×48GB**。具体单条模组兼容型号以官方 QVL 为准。

#### 结论：单卡配置下，Dark Hero 更合适

> **推荐：ROG Crosshair X870E DARK HERO（¥4,899）**
> **备选：ProArt X870E-Creator（¥4,699）** —— 更适合工作站外观 / 22110 M.2 / 8K 输出，或将来改薄卡·水冷卡做真双卡

**为什么单卡下倾向 Dark Hero：**
- **ProArt 最大卖点「双 CPU 直连 PCIe 5.0 x16」在本配置用不上**：本次是单张 **TUF 5090（3.6 槽 / 348mm 风冷）**，两张这种卡在标准 ATX 上**物理装不下**（第一张就吃到第 4 槽位、第二条 x16 被散热鳍片贴死，见 6.2 ④ / 6.5），双 x16 无从发挥。两款都有这条能力、都用不上，等于打平。
- **第二网口 Dark Hero 更快**：主网口两款同为 10G，但**第二口 Dark Hero 是 5G、ProArt 只有 2.5G**。
- **供电与散热用料 Dark Hero 更强**：Dark Hero 20(110A)+2+2 的 VRM、3D VC 均热板 M.2 散热、无风扇设计，压 5090 正下方那条 M.2 更从容；ProArt 是 16(80A)+2+2（对 9950X 也够，但用料一档）。
- 单卡状态 5090 独占第一条 CPU 直连 x16 满血；两款各有 2 条 CPU 直连 M.2、走独立通道不抢显卡。

**什么时候反而选 ProArt：**
- 想要**工作站外观**、需要 **22110 长盘位**（ProArt M.2_4 支持 22110）、或看重 **8K DP Alt 输出**（DP Alt 最高 8K@60Hz，全系最强）。
- **确有把握将来改用薄卡 / 水冷卡（2~2.5 槽）做真双卡** —— 那时 ProArt 的双 CPU 直连 x16 才真正有意义；但用当前这张 3.6 槽 TUF 风冷卡凑双卡并不现实。

**⚠️ 关于「未来加第二张卡」的现实（两款同理）：**
- **加第二张 5090 当计算卡**：带宽上必须走 CPU 直连 x8/x8（两款都支持；若掉到芯片组 4.0 x4 会成瓶颈，部分 3D / 视频工作流损失约 **25%**，见 6.2 ③ 与 6.5）。但**更硬的死结是物理**：TUF 3.6 槽风冷卡两张在标准 ATX 根本叠不下——想双 5090，现实里必须换**水冷 / 涡轮 / 2 槽薄卡**，或上 E-ATX / 开放式机箱。
- **加一张 RTX 3060 类副卡**：副卡不吃带宽，插芯片组 4.0 x4 槽即可，两款都够用。
- 双卡整机功耗实测约 **1.5kW**，需 **1600W+ 电源**（留头 2000W）、两路独立 12V-2×6——把双卡从想法变现实的硬门槛。

> **已移出型号一句话汇总**：原 X870E 全系 13 款，经硬性条件（内存 <192GB / 网口无 10G / 超预算 / 需背插机箱）累计移除 11 款——包括原 ROG Strix 全线（X870E-E / E NEO / A NEO / H，均无 10G）与 Crosshair HERO（无 10G），仅 Dark Hero、ProArt 两款满足全部条件、进入上面的对比。移除清单见文首「本文档筛选说明」。

### 6.4 一句额外的话：CPU 搭配

如果这台机器的**主要用途是游戏**，9950X 配 RTX 5090 偏生产力——9950X 同样无 3D V-Cache，**纯游戏帧数仍明显低于同为 16 核但带 3D 缓存的 9950X3D（以及 8 核的 9800X3D）**，在 1080p/1440p 高刷场景下差距尤为明显，而 5090 恰恰是最容易被 CPU 拖后腿的卡。

9950X 的优势在于 16 核 32 线程带来的强生产力/多线程性能，配 5090 属于**游戏 + 生产力兼顾、偏生产力**的搭配（多核渲染、编译、虚拟机）。如果你的实际使用确实是这种混合场景，那这个搭配是合理的；如果纯游戏，建议在下单前重新考虑 CPU。

### 6.5 多卡场景：未来加第二张卡的通道与拆分逻辑

这一节解释「为什么副卡随便插、计算卡必须走 CPU 直连」，让你理解 6.3 里「加副卡」与「加第二张 5090」两种加卡场景背后的原理。

#### 通道从哪来：CPU 直连 vs 芯片组（PCH）

Ryzen 9000（Zen 5）对外一共 **28 条可用 PCIe 通道**：其中 **16 条**是给显卡的 PCIe 5.0，另外几条给 CPU 直连 M.2 和芯片组链路。主板上第二条 x16 槽的电气规格，取决于它接在哪：

- **接 CPU（本次两款的第二条 x16 都是这类：Dark Hero、ProArt）**：那 16 条显卡通道可**对半拆成 x8 + x8**（BIOS 里叫 bifurcation）。两张卡各拿 PCIe **5.0 x8**，每条约 32GB/s——等于满血 4.0 x16，带宽充足。
- **接芯片组（ProArt 另有 1 条物理 x16、电气 PCIe 4.0 x4 的 PCH 槽；Dark Hero 无此额外槽，两条 x16 都在 CPU 上）**：这类 PCH 槽物理是 x16 长度，但**电气只有 PCIe 4.0 x4**，而且要和其他 PCH 设备（部分 M.2、USB）**共享那条上行 4.0 x4 链路**。带宽只有 CPU 直连 x8 的约 1/4，且可能被别的设备挤占。（原 Strix 全系把「第二条 x16」也做成这种 PCH x4，因而不能双卡满速，已随本轮移除。）

> 一句话：**「x8+x8」是把显卡的通道分一半给第二张卡（CPU 直连，快）；芯片组 x4 是从一条细管子里再分（慢、且要抢）。**

#### 加 RTX 3060 之类副卡 → 芯片组 4.0 x4 完全够

副卡的用途是额外显示输出 / 视频编码 / 轻量推理，本身**不吃 PCIe 带宽**。RTX 3060 虽是 x16 卡，但其算力早在到达 PCIe 瓶颈前就先饱和了——即便降到 4.0 x4，游戏/日常几乎无可感损失（3060 本身是瓶颈，不是通道）。所以副卡：**ProArt 直接插那条 PCH 4.0 x4 槽最省心；Dark Hero 没有独立 PCH 槽，副卡插第二条 CPU x8 槽即可（会让显卡走 x8，游戏约 -1%，可接受）**。两款都能容纳副卡，不需要为此加钱。

#### 加第二张 5090 做生产力 → 必须 CPU 直连，且门槛很高

**带宽层面**：第二张 5090 是真正在跑计算的卡，PCIe 带宽会实打实影响渲染/AI 数据吞吐。若它只有芯片组 4.0 x4，等于把一张 575W 的卡喂在一根细管子上，部分 3D/视频工作流可损失约 25%（见 ②③ 的 Puget/Tom's 数据）。因此**只有 CPU 直连 x8/x8 的板才适合双 5090——保留的两款（Dark Hero / ProArt）都满足**。

**但在掏钱买板之前，先确认能不能过这三道现实门槛**：

1. **无 NVLink，不能游戏叠加**：40/50 系消费卡早已取消 SLI/NVLink，两张 5090 **无法合并渲染同一帧游戏**。第二张卡只对**能吃多 GPU 的计算任务**有意义——渲染农场、并行 AI 推理/训练、多卡显存池化（2×32GB=64GB，可跑单卡放不下的大模型）。想靠它提升游戏帧数是无效的。
2. **功耗与电源**：双卡实测整机可达约 **1.5kW**，需 **1600W+ 电源**（留头 2000W）、两路独立 12V-2×6。
3. **物理空间与散热**：本次选定的 **TUF 5090 是 3.6 槽 / 348mm** 超厚超长卡。两张在标准 ATX 上，第二条 x16 通常在第一张卡正下方 3~4 槽处——第一张的散热鳍片已侵占到第 4 槽，**第二条槽被直接盖住，两张 TUF 风冷卡叠不下**，现实里要么用**涡轮/水冷/2 槽版**更薄的专业卡，要么上 EATX/开放式机箱。**TUF 风冷版做双卡并不合适。**
4. **CPU 供数**：9950X 是 16 核 32 线程，在多卡计算供数上比 9900X 更从容，不再是明显短板。不过在超大规模并行下，消费级平台仍可能吃紧——实测有消费级平台带不动双 5090、换到 HEDT/服务器平台（如 EPYC）才稳的案例。追求双卡满血生产力，长远看 CPU/平台才是瓶颈，而不是主板贵不贵。

> 结论：双 5090 技术上可行、且在**显存受限的 AI/渲染**上收益明显，但它是一套**围绕电源、机箱、散热、平台**的系统级工程，不是「买块贵主板就成」。没有明确计划时，为它多花的主板钱是「保险费」，不是「性能」。
