## 一、指令系统基本概念

**指令**：指令就是要计算机执行某种操作的命令。*(讲义 p036)*

**指令系统**：一台计算机中所有机器指令的集合，称为这台计算机的指令系统。指令系统是表征一台计算机性能的重要因素，它的格式与功能不仅直接影响到机器的硬件结构，而且也直接影响到系统软件，影响到机器的适用范围。*(讲义 p036)*

**系列机**：是指基本指令系统相同、基本体系结构相同的一系列计算机。推出的机种指令系统一定包含所有旧机种的全部指令。*(讲义 p036)*

### 指令系统应满足的要求 ⭐

| 要求 | 含义 |
|---|---|
| 完备性 | 汇编语言编写各种程序时，指令系统直接提供的指令足够使用，而不必用软件来实现 |
| 有效性 | 利用该指令系统所编写的程序能够高效率地运行 |
| 规整性—对称性 | 所有寄存器和存储器单元都可同等对待，所有指令都可使用各种寻址方式 |
| 规整性—匀齐性 | 一种操作性质的指令可以支持各种数据类型 |
| 规整性—一致性 | 指令长度和数据长度有一定的关系，以方便处理和存取 |
| 兼容性 | 各机种上基本软件可以通用 |

*(讲义 p036)*

### 计算机语言发展史

计算机语言发展历史：机器语言 → 汇编语言 → 高级语言。*(讲义 p036)*

- 机器语言、汇编语言依赖于特定计算机的硬件结构和指令系统，因此被称为与硬件相关的语言，也被称为低级语言。*(讲义 p036)*
- 高级语言更贴近于数学语言和自然语言，与计算机的硬件结构及指令系统无关，在编写程序方面比汇编语言优越。*(讲义 p036)*

> Caveat (题目9): 汇编语言源程序通常比高级语言源程序**更长**，因为汇编指令功能比高级语言更简单，表达同样程序往往需要更多条指令。

> Caveat (题目多选10): 汇编语言需要汇编后才能执行，不能直接在机器上执行。

## 二、机器指令格式与设计要素

### 机器指令的构成

一条机器指令需要提供的信息主要有：*(讲义 p036)*

- **操作码**（Operation Code）
- **源操作数地址**（Source Operand Reference）
- **目的操作数地址**（Result Operand Reference）
- **下一条指令的地址**（Next Instruction Reference）

大多数CPU的下一条指令地址为隐含的（由程序计数器 PC 得到）。源操作数和结果操作数的存放位置包括：主存或虚存、CPU 寄存器、I/O 设备。*(讲义 p036-037)*

### 机器指令格式 ⭐

机器指令格式是机器指令用二进制代码表示的结构形式，通常由**操作码字段**和**地址码字段**组成。*(讲义 p037)*

- 操作码字段：表征指令的操作特性与功能。*(讲义 p037)*
- 地址码字段：通常指定参与操作的操作数的地址。*(讲义 p037)*

计算机指令最主要的元素是操作码，它指明即将完成的操作、源操作数和目的操作数的引用方式，以及通常隐式指明下一个指令的引用方式。操作码采用编码形式表示，CPU 中有专门的电路来解释每个操作码。*(讲义 p037)*

**常用地址码格式**：*(讲义 p037)*

| 格式 | 说明 |
|---|---|
| 三地址 | [原文未述具体细节] |
| 二地址 | [原文未述具体细节] |
| 单地址 | [原文未述具体细节] |
| 零地址 | [原文未述具体细节] |

**按操作数物理位置分类**：*(讲义 p037)*

| 类型 | 全称 | 说明 |
|---|---|---|
| SS 型 | 存储器—存储器 | 两个操作数均在主存中 |
| RR 型 | 寄存器—寄存器 | 两个操作数均在寄存器中 |
| RS 型 | 寄存器—存储器 | 一个操作数在寄存器，一个在主存 |

> Caveat (题目多选1): 一地址格式指令中，可能有一个操作数，也可能有两个操作数（另一个是隐含的）；执行一条一地址指令需 **2 次访问主存**（取指令 1 次 + 取操作数 1 次）。

### 指令字长 ⭐⚠

**指令字长**：一个指令字中包含二进制代码的位数。常见指令字长有：单字长、半字长、双字长。*(讲义 p037)*

- 使用多字长指令的目的：提供足够的地址位来解决访问内存任何单元的寻址问题。*(讲义 p037)*
- 使用多字长指令的缺点：必须两次或多次访问内存以取出一整条指令。*(讲义 p037)*

| 结构 | 定义 |
|---|---|
| 等长指令字结构 | 在一个指令系统中，各种指令字长是相等的 |
| 变长指令字结构 | 各种指令字长随指令功能而异（如有的单字长、有的双字长） |

*(讲义 p037)*

### 指令集设计考虑因素

在指令集设计的最根本出发点中，最重要的方面包括：*(讲义 p037)*

| 方面 | 内容 |
|---|---|
| 操作指令表（Operation Repertoire） | 应提供多少和什么样的操作，操作的复杂程度 |
| 数据类型（Data Types） | 所支持的数据类型 |
| 指令格式（Instruction Format） | 指令的（位）长度、地址数目、各个字段的大小 |
| 寄存器（Registers） | 能被指令访问的 CPU 寄存器数目以及它们的用途 |
| 寻址方式（Addressing Mode） | 指定操作数地址的产生方式 |

### 扩展操作码 ⭐⚠

> Caveat (题目17): 扩展操作码**不是**简单地扩大操作码字段的位数，而是一种**指令优化技术**——让操作码的长度随地址数的减少而增加，不同地址数的指令可以具有不同的操作码长度。即在使用地址数减少时，让操作码占用一定的地址码位置，从而在不扩大指令长度的情况下表示更多的指令。

> Caveat (题目19): 扩展操作码的目的是在同样指令长度的情况下**增加指令数量**。

## 三、寻址方式 ⭐⭐⭐

**寻址方式**：形成操作数或指令地址的方式，称为寻址方式。*(讲义 p037)*

寻址方式主要分为两类：*(讲义 p037)*

| 类别 | 特点 |
|---|---|
| 指令的寻址方式 | 较简单，种类少 |
| 操作数的寻址方式 | 较复杂，种类多，很灵活 |

**寻址方式的作用**：增强指令的功能，缩短指令的长度，提高指令的执行速度，增加操作数存放的位置的灵活性等。*(讲义 p037)*

> Caveat (题目1): 指令系统采用不同寻址方式的主要目的是**缩短指令长度，扩大寻址空间，提高编程灵活性**。

> Caveat (题目多选3): 寻址方式的主要目的**不是**提高访问速度、简化指令译码电路增加内存容量、或增加内存容量。

### 指令的寻址方式

| 方式 | 实现机制 |
|---|---|
| 顺序寻址方式 | 通过程序计数器（PC） |
| 跳跃寻址方式 | 通过跳转指令 |

*(讲义 p037-038)*

> Caveat (题目8): 跳跃寻址可实现程序的**条件转移或无条件转移**。

### 操作数的寻址方式

**有效地址（EA）**：操作数的真正存储位置。*(讲义 p038)*

**形式地址（A）**：也称偏移量，它是指令字结构中给定的地址量。*(讲义 p038)*

寻址过程就是把操作数的形式地址（A），变换为操作数的有效地址（EA）的过程。*(讲义 p038)*

讲义列出十种常用寻址方式：隐含寻址、立即寻址、直接寻址、间接寻址、寄存器寻址、寄存器间接寻址、相对寻址、基址寻址、变址寻址、堆栈寻址。*(讲义 p038)*

以下表格中 EA 公式与特征综合自讲义定义及本章答案解析：*(讲义 p038, 题目3-5, 题目多选6-9)*

| 寻址方式 | 有效地址 EA | 操作数位置 | 访存次数 | 关键特征 |
|---|---|---|---|---|
| 隐含寻址 | [原文未述] | [原文未述] | [原文未述] | 指令中不明确给出操作数地址，隐含指定；累加器/寄存器可作为隐含地址 |
| 立即寻址 | 操作数即地址字段值 | 指令内 | 0（不访存） | 地址字段指出的不是操作数的地址，而是操作数本身 |
| 直接寻址 | $EA = A$ | 主存 | 1 | 直接访问内存地址 |
| 间接寻址 | $EA = (A)$ | 主存 | $\geq 2$ | 先从指令取地址，再用该地址访存，速度最慢 |
| 寄存器寻址 | $EA = R_i$ | 寄存器 | 0 | 操作数在寄存器中 |
| 寄存器间接寻址 | $EA = (R_i)$ | 主存 | 1 | 寄存器中存储的是主存地址，该地址对应的操作数才是真正操作数 |
| 相对寻址 | $EA = (PC) + A$ | 主存 | 1 | A 为偏移量（补码表示） |
| 基址寻址 | $EA = (BR) + A$ | 主存 | 1 | 基址寄存器内容由 OS 确定，执行中不可变；A 为位移量 |
| 变址寻址 | $EA = (IX) + A$ | 主存 | 1 | 变址寄存器提供修改量，A 为基准量 |
| 堆栈寻址 | [原文未述] | 堆栈 | [原文未述] | SP 指向栈顶元素 |

*(讲义 p038, 题目3-5, 题目多选6-9)*

### 寻址方式速度对比 ⭐⚠

| 寻址方式 | 速度 | 访存次数 | 说明 |
|---|---|---|---|
| 立即寻址 | **最快** | 0 | 操作数就在指令中，不需要寻址 |
| 寄存器寻址 | 快 | 0 | 操作数在寄存器中 |
| 直接寻址 | 中 | 1 | 直接访问内存地址 |
| 寄存器间接寻址 | 中 | 1 | 寄存器存主存地址，需访存取操作数 |
| 间接寻址 | **最慢** | $\geq 2$ | 先取地址再访存 |

*(题目多选6)*

> Caveat (题目多选6): 取得操作数速度最快的是**立即寻址**，最慢的是**间接寻址**，不需要访存的寻址方式是**立即寻址**。

### 变址寻址 vs 基址寻址 ⭐⚠

| 对比项 | 基址寻址 | 变址寻址 |
|---|---|---|
| 寄存器提供 | 基准量 | 修改量 |
| 指令提供（形式地址 A） | 位移量 | 基准量 |
| 寄存器内容由谁确定 | 操作系统 | 用户 |
| 执行过程中可变性 | **不可变** | 可变 |
| EA 计算 | $EA = (BR) + A$ | $EA = (IX) + A$ |

*(题目14, 题目多选9)*

> Caveat (题目14): 基址寻址中，基址寄存器内容由**操作系统**确定，在程序执行过程中**不可变**。

> Caveat (题目16): 先变址再间址的寻址方式，有效地址 $EA = ((X) + D)$，即先变址得到 $(X)+D$，再以此作为地址进行间接寻址。

### 堆栈寻址 ⚠

> Caveat (题目10): 堆栈指针 SP 指向的是堆栈的**栈顶元素**，**不是**堆栈内所存储的数据的个数。

> Caveat (题目15): 进栈操作顺序为 $(A) \to M_{sp}$，$(SP) - 1 \to SP$；出栈操作顺序与之**相反**：$(SP) + 1 \to SP$，$(M_{sp}) \to A$。

### 标准解题套路：PC 自动增量计算 ⭐⚠

**题型**：给出机器字长、编址方式、指令长度，求取指令后 PC 的值。*(题目20-22)*

1. 确定编址方式：按字编址还是按字节编址。*(题目20-22)*
2. 确定指令长度：单字长、双字长等，换算为字节数。*(题目20-22)*
3. 计算PC增量：*(题目20-22)*
   - 按字编址：PC 增量 = 指令字长数（单字长加 1，双字长加 2）
   - 按字节编址：PC 增量 = 指令字节数（16 位单字长 = 2 字节 → 加 2；32 位双字长 = 4 字节 → 加 4）
4. 新 PC 值 = 原 PC 值 + PC 增量。*(题目20-22)*

> Caveat (题目20-22): 机器字长 16 位，按字编址取单字长指令 PC 加 **1**；按字节编址取单字长指令 PC 加 **2**；按字节编址取双字长指令 PC 加 **4**。

### 标准解题套路：相对寻址偏移量计算 ⭐⚠

**题型**：给出当前 PC 值、目标地址、指令长度，求偏移量。*(题目18)*

1. 确定指令长度（字节数）。*(题目18)*
2. 计算取指后 PC 值：取指后 $PC = \text{原 } PC + \text{指令字节数}$。*(题目18)*
3. 计算偏移量：$\text{偏移量} = \text{目标地址} - \text{取指后 } PC$。*(题目18)*
4. 偏移量用补码表示。*(题目18)*

> Caveat (题目18): 注意 PC 在取指后已自增，偏移量是相对于**取指后的 PC 值**而非原 PC 值。

## 四、RISC 与 CISC ⭐

**RISC**（Reduced Instruction Set Computer）：精简指令系统计算机。*(讲义 p038)*

**CISC**（Complex Instruction Set Computer）：复杂指令系统计算机。*(讲义 p038)*

### RISC 的主要特点

1. 选取使用频率较高的一些简单指令以及一些很有用但又不复杂的指令，让复杂指令的功能由频度高的简单指令的组合来实现。*(讲义 p038)*
2. 指令长度固定，指令格式种类少，寻址方式种类少。*(讲义 p038)*
3. 只有取数/存数（LOAD/STORE）指令访问存储器，其余指令的操作都在寄存器内完成。*(讲义 p038)*
4. 采用流水线技术，大部分指令在一个时钟周期内完成。采用超标量和超流水线技术，可使每条指令的平均执行时间小于一个时钟周期。*(讲义 p038)*
5. 控制器采用组合逻辑控制，不用微程序控制。*(讲义 p038)*
6. CPU 中有多个通用寄存器。*(讲义 p038)*
7. 采用优化的编译程序。*(讲义 p038)*

### RISC 的优点（与 CISC 相比）

| 优点 | 说明 |
|---|---|
| 充分利用 VLSI 芯片面积 | [原文未述细节] |
| 提高计算机运算速度 | [原文未述细节] |
| 便于设计，可降低成本，提高可靠性 | [原文未述细节] |
| 有效支持高级语言程序 | [原文未述细节] |

*(讲义 p038)*

### RISC vs CISC 对比 ⭐

| 对比项 | RISC | CISC |
|---|---|---|
| 指令长度 | 固定 | 可变（多种长度） |
| 指令格式种类 | 少 | 多 |
| 寻址方式种类 | 少 | 丰富 |
| 访存指令 | 只有 LOAD/STORE | 各种指令都能访存 |
| 控制器方式 | 组合逻辑控制 | 微程序控制 |
| 通用寄存器 | 多个 | 较少 |
| 指令执行时间 | 大部分一个时钟周期 | 多数需多个时钟周期 |
| 编译优化 | 采用优化编译程序 | [原文未述] |

*(讲义 p038, 题目多选7)*

> Caveat (题目11): RISC 的思想是寻址方式种类尽量**少**，指令功能尽可能**简单**——而非"指令功能尽可能强"。

> Caveat (题目12): RISC 的主要目标是**减少指令数**；RISC 可以不设置乘除法指令和浮点运算指令。

> Caveat (题目判断9): RISC 机使用了大量的通用寄存器，以优化编译和执行速度。

> Caveat (题目判断10): RISC 机指令条数少，寻址方式少，结构相对简单，便于设计、成本低、可靠性高。

## 五、高频考点与易错提醒

### 高频考点

1. **寻址方式**（考频最高 ×14）：有效地址计算、速度对比、先变址再间址逻辑、相对寻址偏移量计算。*(题目1-8,16,18, 多选3,6,8-9)*
2. **PC 自动增量计算**（×3）：编址方式（按字/按字节）与指令长度共同决定。*(题目20-22)*
3. **扩展操作码**（×2）：原理与目的。*(题目17,19)*
4. **RISC 特点**（×7）：与 CISC 的对比。*(题目11-12, 多选4,7, 判断9-10)*
5. **指令格式**（×8）：地址码类型、SS/RR/RS 型、一地址指令访存次数。*(题目多选1,17,19)*

### 易错提醒

- ⚠ 寻址方式速度：立即最快（不访存），间接最慢（多次访存），直接居中（一次访存）。*(题目多选6)*
- ⚠ PC 增量：按字编址加指令字数，按字节编址加指令字节数。*(题目20-22)*
- ⚠ 基址寄存器由 OS 确定且不可变，变址寄存器由用户使用且可变。*(题目14, 多选9)*
- ⚠ 堆栈指针 SP 指向栈顶元素，不是数据个数。*(题目10)*
- ⚠ 扩展操作码不是简单扩大操作码位数，而是地址数减少时操作码扩展到地址码位置。*(题目17)*
- ⚠ 先变址再间址：$EA = ((X)+D)$，不是 $(X)+(D)$。*(题目16)*
- ⚠ 汇编语言源程序通常比高级语言**更长**。*(题目9)*
- ⚠ RISC 指令功能尽可能**简单**，不是尽可能强。*(题目11)*

## 六、一句话记忆要点

- **指令系统**：所有机器指令的集合，决定硬件结构与软件适用范围。*(讲义 p036)*
- **寻址方式**：形式地址 A → 有效地址 EA 的变换过程；立即最快不访存，间接最慢多次访存。*(讲义 p038)*
- **变址 vs 基址**：变址寄存器给修改量（用户可变），基址寄存器给基准量（OS 不可变）。*(题目14, 多选9)*
- **PC 增量**：按字编址加字数，按字节编址加字节数。*(题目20-22)*
- **扩展操作码**：地址少了操作码来占，不增长度多指令。*(题目17,19)*
- **RISC**：指令少而简、长度固定、只有 LOAD/STORE 访存、组合逻辑控制、多寄存器、流水线。*(讲义 p038)*

### 例 1：相对寻址转移指令偏移量计算 *(来自 exam-signals 候选 3)*

*Tags: 相对寻址, PC 自增, 补码偏移量*

**问题**：设相对寻址的转移指令占两个字节，第一字节为操作码，第二字节为相对偏移量（补码表示）。CPU 每当从存储器取出一个字节，PC 自动加 1。设当前 PC 的内容为 2000H，如果需要转移到 2008H 地址，求该转移指令的第二字节的内容。*(题目18)*

**解答**：

1. 确定指令长度：转移指令占 **2 字节**，CPU 每取一字节 PC 加 1，因此取指结束后 PC 自增了 2。*(题目18)*
2. 计算取指后 PC 值：*(题目18)*
$$PC_{\text{取指后}} = 2000\text{H} + 2\text{H} = 2002\text{H}$$
3. 计算目标地址与取指后 PC 的差值（即偏移量）：*(题目18)*
$$\text{偏移量} = 2008\text{H} - 2002\text{H} = 06\text{H}$$
4. 偏移量用补码表示，$06\text{H}$ 为正数，其补码即为其本身。*(题目18)*

**答案**：第二字节的内容为 **06H**（选项 B）。*(题目18)*
