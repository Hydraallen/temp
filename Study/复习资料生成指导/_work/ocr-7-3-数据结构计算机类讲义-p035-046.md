【知识点 3-1】栈的定义
栈是限制在表的一端进行插入、删除运算的线性表。允许插入、删除的这一端被称为栈顶，另一个固定端称为栈底。当表中没有元素时称为空栈。
理解栈的定义有以下要点：
1) 线性结构：栈元素具有线性（即前驱后继）关系
2) 限制操作：限制了线性表插入和删除的位置
3) 单向延伸性：栈底是固定的，栈顶随着插入和删除操作的进行而变化
4) 插入也称为进栈、压栈、入栈，删除也称为出栈、弹栈
5) 栈是一种后进先出（LIFO — Last In First Out）的数据结构。
需要注意的是，栈只对线性表杀入和删除的位置进行了限制，并没有限定插入和删除操作进行的时间，由于出栈可随时进行，只要某个元素位于栈顶就可以出栈，所以，元素最终的出栈次序可能是多样的。
【知识点 3-2】顺序栈
用顺序存储方式实现的栈称为顺序栈。类似于顺序表的定义，栈中的数据元素用一个预设足够大的一维数组来实现：datatype data[MAXSIZE]，栈底位置可以设置在数组的任一个端点，而栈顶是随着插入和删除而变化的，因此需要一个栈顶指针来标示，设变量int top作为栈顶指针使用，指明当前栈顶的位置，同样将 data 和 top 封装在一个结构中，顺序栈的类型描述如下：
#define MAXSIZE 1024 //设定栈的最大长度为 1024，可根据实际情况进行修改
typedef struct
{
    datatype data[MAXSIZE];
    int top;
}SeqStack
定义一个指向顺序栈的指针：SeqStack *s;
【知识点 3-3】链栈
用链式存储结构实现的栈称为链栈。通常链栈用单链表表示，因此其结点结构与单链表的结构相同，链栈的类型描述如下：：
typedef struct snode
{
    datatype data;
    struct snode *next;
}StackNode, * LinkStack;
定义 top 为栈顶指针变量：LinkStack top;
【知识点 3-4】上溢
栈满时再做进栈运算产生的空间溢出。
【知识点 3-5】下溢
栈空时再做退栈运算产生的无元素可退。

队列是仅允许在表的一端进行插入运算，而在表的另一端进行删除运算的受限的线性表。允许插入的一端称为队尾，允许删除的一端称为队头。

理解队列的定义有以下要点:
1) 线性结构：相邻元素具有前驱后继关系
2) 限制操作：限制了线性表插入和删除的位置
3) 插入也称为入队、进队，删除也称为出队
4) 队列是一种先进先出（FIFO — First In First Out）的数据结构。

【知识点 3-7】顺序队列

顺序存储的队称为顺序队列。因为队的队头和队尾都是活动的，因此，除了队列的数据区外还有队头、队尾两个指针。顺序队的类型定义如下：
define  MAXSIZE  1024          //队列的最大容量
typedef  struct
{datatype  data[MAXSIZE];      //队员的存储空间
  int rear,front;              //队头队尾指针
}SeQueue;
定义一个指向队的指针变量：
  SeQueue  *q;
申请一个顺序队的存储空间：
  q= (SeQueue *)malloc(sizeof(SeQueue));
队列的数据区为：
  q->data[0]～q->data[MAXSIZE-1]
队头指针：q->front
队尾指针：q->rear

【知识点 3-8】链队列

链式存储的队称为链队列。和链栈类似，用单链表来实现链队列，根据队的FIFO原则，为了操作上的方便，我们分别需要一个头指针和尾指针。如图3-1所示，其实这就是一个带上尾指针的单链表，只要我们对它的操作按照队列的规则进行，它就是一个链队。

【知识点3-9】假上溢
在顺序队列中由于队的头尾位置只增不减，导致队尾位置已到队列空间的上界不能再做入队操作，而随着出队操作的进行，队首位置增加，使队列空间尚有空闲单元，这种队列已满的假象被称为假上溢。

【知识点3-10】循环队列
将顺序队列的向量空间视为一个首尾相接的圆环进行运算的队列；克服了假上溢现象，充分利用了向量空间。
