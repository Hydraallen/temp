## 二、知识点详解

### 概述与数值表示基础

**计算机数值表示的考虑因素**：选择计算机数值表示方式时应全面考虑四方面因素。
1. 要表示数的类型（小数、整数、实数或复数，或它们的组合）
2. 可能遇到的数值范围（范围越大，对存储和处理能力要求越高）
3. 数值精确度（精确度越高，需要更多二进制位）
4. 数据存储和处理所需要的硬件代价（影响计算机整体造价）

**数值表示特点**：计算机中常用数据表示格式有定点格式和浮点格式两种。
- 定点格式：容许的数值范围有限，但处理硬件比较简单
- 浮点格式：容许的数值范围很大，但处理硬件比较复杂

数值在计算机中的表示具有如下特点：
- 二进制表示
- 数据的编码化
- 正负号的数字化
- 小数点位置的约定
- 数据有模

### 定点数编码：原码与补码

**数值编码基本概念**：
- **真值**：书写表示的数值，如 +3、−5 等，由人识别
- **机器数（机器码）**：数值在计算机中的编码表示，为二进制数串形式，供机器使用
- **定点数表示法**：将机器中所有数据的小数点位置固定不变的数据表示法；通常为了处理简单，将定点数表示成纯小数或纯整数

**原码** ⭐：采用一位符号位的最简单、最直观的表示法，又称为符号——幅值表示法。

定点整数原码定义：
$$\begin{cases} x & x \geq 0 \\ 2^n - x = 2^n + |x| & x < 0 \end{cases}$$


定点小数原码定义：
$$\begin{cases} x & x \geq 0 \\ 1 - x = 1 + |x| & x < 0 \end{cases}$$


原码性质：
- 最高位表示符号，0 位为正，1 为负
- 0 在原码表示中不唯一，有 +0、−0 之分
- n 位原码总共有 $2^n$ 种编码，共可表示 $2^n - 1$ 个数（因为 0 用了两个编码）
- 负数的原码大于正数的原码
- 实质是表示数值的绝对值；由真值转换为原码：将 + 写成 0，− 写成 1，数值位不变

原码优点：表示直观；易于实现乘、除法运算

原码缺点：加减运算时既要考虑数的符号又要考虑幅值，处理较为复杂；0 有两种表示

> Caveat (判断3): 在用原码表示的机器数中，零的编码不是唯一的；只有补码中零的表示才唯一。

**补码** ⭐⚠：补码表示法是计算机中应用最广泛的一种数据编码形式。

有模运算概念：有模运算是指在一定数值范围内进行的运算（如钟表）。计算机中采用有限的二进制位表示数据，因此所有运算都是有模运算。对于有模运算，减一个数等于加上该数对模的补数，补码的概念即来源于此。正数用本身代表，负数用其补数代表。

定点整数补码定义：
$$\begin{cases} x & x \geq 0 \\ 2^{n+1} + x = 2^{n+1} - |x| & x < 0 \end{cases}$$


定点小数补码定义：
$$\begin{cases} x & x \geq 0 \\ 2 + x = 2 - |x| & x < 0 \end{cases}$$


求负数补码的方法（两种等价方法）：
1. 符号位置 1，其余各位 0 变 1、1 变 0，然后在最末位上加 1
2. 从低位向高位找到第一个 1，这个 1 和右边各位的 0 保持不变，左边的各高位按位取反

补码性质：
- 最高位表示符号，0 位为正，1 为负
- 0 的表示唯一，即编码为全 0 的情况
- 补码表示法比原码表示法多表示一个数据
- 负数补码值大于正数补码值
- 补码算术右移时，要将符号位复制
- 补码算术左移时，末位补 0 即可
- 定点整数补码位数扩展时，要将符号位向左复制；定点小数补码进行位数扩展时，只需要在原机器码后补 0 即可

> Caveat (单选16): 8 位补码的表示范围是 $-128 \sim +127$，负数比正数多表示一个，不要错记为 $-127 \sim +127$。

> Caveat (单选12): $[X]_{补} = X_0 X_1 X_2 \cdots X_n$（n 为整数），它的模是 $2^{n+1}$。

> Caveat (单选10): 用 $n+1$ 位字长（其中 1 位符号位）表示定点整数时（原码），所能表示的数值范围是 $0 \leq |N| \leq 2^n - 1$。

> Caveat (多选3): 补码特点：整数用补码存储、0 有唯一表示、采用补码可简化 CPU 实现；但补码与真值的对应关系并不直观，需要计算。

**十进制负数求补码标准解题套路**（for 补码表示题）
1. 写出该数绝对值的二进制形式
2. 添加符号位（负数符号位为 1）得到原码
3. 将原码转换为补码：符号位不变，数值位按位取反，末位加 1（或用"找第一个 1"法）
4. 如需十六进制，每 4 位一组转换

**原码与补码对比**：

| 特性 | 原码 | 补码 |
|---|---|---|
| 符号位 | 最高位，0 正 1 负 | 最高位，0 正 1 负 |
| 零的表示 | 不唯一（+0、−0） | 唯一（全 0） |
| n 位可表示数个数 | $2^n - 1$ | $2^n$（多一个） |
| 加减运算 | 复杂（需考虑符号和幅值） | 简单（符号位与数值位一同运算） |
| 乘除运算 | 易于实现 | [原文未述] |
| 应用广泛度 | [原文未述] | 最广泛 |


### 浮点数表示

**浮点数的二进制表示** ⭐：浮点数表示法思想来源于十进制科学计数法，目的是增大相同位数二进制数所表示数的范围，但并不能增加表示数的个数，即降低了数的表示精度。

$$\pm S \times B^{E}$$


存储在二进制字的三个字段中：
- 符号：正或负
- 有效数 $S$（Significant，也称尾数）：一般采用补码形式的定点小数
- 指数 $E$（Exponent）：一般采用移码形式
- 基值 $B$：隐含的，不需存储（对所有数都相同）

**规格化表示**：为了提高数据表示精度，当尾数值不为 0 时，其绝对值应 $\geq 0.5$，即尾数域最高有效位应为 1，否则要修改阶码同时左右移小数点。

**机器零**：当浮点数尾数为 0（不论阶码为何值），或阶码值比能表示的最小值还小（不管尾数为何值），计算机都把该浮点数看成零值，称为机器零。

**IEEE 754 标准** ⭐：为便于程序从一类处理器移植到另一类处理器、促进研究更为复杂的数值运算程序而定义，已获广泛认可，用于当代所有各类处理器和算术协处理器。定义了 32 位单精度和 64 位双精度两种格式。

**IEEE 754 单精度浮点数真值求解标准解题套路**（for 浮点数转换题）
1. 十六进制转二进制（32 位）
2. 提取各段：第 1 位符号位 $S$；后 8 位阶码 $E$；后 23 位尾数 $M$
3. 计算阶码真值：$e = E - 127$
4. 计算尾数真值：$1.M$（补隐藏的 1）
5. 组合结果：$(-1)^S \times 1.M \times 2^e$

### 文字与多媒体信息表示

**编码**：
- 西文：七单位 ASCII 码（美国国家信息交换标准字符码），包括 10 个十进制数码、26 个英文字母和一定数量的专用符号
- 汉字表示需从三个方面解决：

| 汉字编码类型 | 说明 |
|---|---|
| 汉字输入编码 | 为直接使用西文标准键盘输入汉字而设计的输入编码方法 |
| 汉字内码 | 用于汉字信息的存储、交换、检索等操作的机内代码 |
| 汉字字模码 | 用点阵表示的汉字字形代码，是汉字的输出形式 |


> Caveat (判断7): 在微型计算机的汉字系统中，一个汉字的内码占 2 个字节。

**语音和图像的计算机内表示**：

语音表示步骤：采样 → 量化 → 编码

图像表示：将视觉形象转换为由点阵构成的用二进制表示的数字化图像，转化过程包含抽样和量化两个步骤。

图形与图像的区别：
- 位图图像基本元素是像素点
- 图形基本元素是图元，使用图形指令描述图元，一般能经数学公式计算得出

### 校验码

**校验码** ⭐：

| 校验码类型 | 全称/提出者 | 功能 | 特点 |
|---|---|---|---|
| 奇偶校验码 | — | 检错码 | 通过增加冗余位使"1"的个数保持奇数或偶数；简单但漏检率太高，只能检测出一位错 |
| 循环冗余码（CRC） | Cyclic Redundancy Code，也称多项式码 | 检错码 | 在计算机网络和数据通信中应用最为广泛；漏检率低且便于实现 |
| 海明码 | R. Hamming, 1950 年首次提出 | 纠错码 | 可以纠正一位差错的编码 |


> Caveat (判断2): 奇偶校验可以检错但不能纠错。

> Caveat (判断4): 8421 BCD 码用二进制求和时，当和超过 9 时，需要做加 **6** 调整修正（非加 8）。

> Caveat (判断9): 常用的 BCD 码为 8421BCD 码，其中的 8 表示 $D_3$ 位的权重（$2^3 = 8$）。

### 定点加减法运算

**补码加减法** ⭐：计算机中广泛采用补码进行定点加减运算。优势：符号同数值一同运算，一次运算同时获得结果的符号和数值；使用补码可将减法运算变为加法运算，运算器里只需要一个加法器。

补码加减法运算公式：
$$[X+Y]_{补}=[X]_{补}+[Y]_{补}$$


$$[-X]_{补}=-[X]_{补}$$


$$[X-Y]_{补}=[X]_{补}+[-Y]_{补}=[X]_{补}-[Y]_{补}$$


由 $[X]_{补}$ 求 $[-X]_{补}$ 的方法：将连符号在内依次按位取反，末位加 1。

补码加减运算规则：
1. 参加运算的操作数用补码表示
2. 补码的符号位与数值位同时进行加运算
3. 若作加，两数补码直接相加；若作减，将减数补码连同符号位一起按位取反，末位加 1，然后再与被减数相加
4. 运算结果即为和/差的补码

> Caveat (单选9): 计算机系统中采用补码运算的目的是为了简化计算机的设计（精简加减法器的硬件复杂度），而非提高运算速度或精度。

**溢出** ⭐⚠：运算结果超过了机器数能表示的范围。
- **上溢**：两个正数相加，结果大于机器所能表示的最大正数
- **下溢**：两个负数相加，结果小于机器所能表示的最小负数

溢出检测方法：

| 检测方法 | 别名 | 原理 | 溢出判定 |
|---|---|---|---|
| 双符号位法 | 变形补码 / 模 4 补码 | 正数两个符号位均为 0，负数均为 1 | 结果符号位出现"01"或"10"表示溢出 |
| 单符号位法 | — | 比较符号位进位与最高有效位进位 | $V = C_f \oplus C_0$ 为 1 表示溢出 |


$$V = C_{f} \oplus C_{0}$$


其中 $C_f$ 为符号位产生的进位，$C_0$ 为最高有效位产生的进位。

> Caveat (单选20): 在定点运算器中，无论采用双符号位还是单符号位，必须有溢出判断电路，一般用异或门来实现。

### 定点乘除法运算

**定点乘法** ⭐：计算机中乘、除法的实现方式有三种：软件实现（指令系统中无乘除法指令）；在加/减法器基础上增加左移、右移位及其他逻辑线路实现（指令系统中设置乘除法指令）；设置专用高速阵列乘除运算器（指令系统中设置乘除法指令）。

| 乘法方法 | 原理 | 特点 |
|---|---|---|
| 原码一位乘 | 将被乘数左移一位相加变为部分积与被乘数相加后右移一位；k 次"累加与右移" | 每步只求一位乘数对应的新部分积，与原部分积累加后右移一次，重复 k 次 |
| 补码一位乘（Booth 算法） | 使用补码直接进行乘法运算 | 由 Booth 夫妇首先提出 |
| 阵列乘法 | 资源重复设置，依靠器件数量换取运算速度 | 器件数量大但结构十分规整，易于大规模集成电路实现 |


**定点除法** ⭐：除法是乘法的逆运算，乘法可通过累加、右移实现，因此除法可通过左移、减来实现。

定点小数除法要求：除数 $\neq 0$；$|被除数| < |除数|$。

| 除法方法 | 原理 | 特点/缺点 |
|---|---|---|
| 恢复余数法 | 不论是否够减都先减除数；余数为正（符号位 0）上商 1，左移继续；余数为负（符号位 1）上商 0，加回除数恢复余数，左移继续 | 实际操作步骤不定，不便于控制；恢复余数需多作一次加法，延长执行时间；计算机中一般不采用 |
| 加减交替法 | 余数为正时商 1，左移减除数；余数为负时商 0，左移加除数 | 克服了恢复余数法的缺点 |


加减交替法做除法的步骤：
1. 判断除数是否为 0
2. 判断被除数是否大于除数
3. 根据每一步余数的符号，判断下次运算是做加法还是减法
4. 最后的运算结果，余数要进行右移

除法运算一般：被除数最大可取 2n 位，除数为 n 位，商也取 n 位。

### 逻辑运算

**逻辑运算**：逻辑数是指不带符号的二进制数。利用逻辑运算可进行两个数的比较，或从某个数中选取某几位操作。

计算机中的逻辑运算主要指四种基本运算：

| 运算 | 名称 |
|---|---|
| 逻辑非 | NOT |
| 逻辑加 | OR |
| 逻辑乘 | AND |
| 逻辑异 | XOR |


### ALU 与运算器组成

**ALU** ⭐：算术/逻辑单元（ALU）是计算机实际完成数据算术和逻辑运算的部分，是运算器的核心部件。

ALU 与 CPU 其余部分的互连关系：
- 数据存在寄存器中提交给 ALU，运算结果也存于寄存器
- 寄存器是 CPU 内的临时存储位置，通过信号路径连接到 ALU
- ALU 将设置标志作为运算结果（如溢出标志等），标志值也存于 CPU 内的寄存器中
- 控制器提供控制 ALU 操作和数据移入移出 ALU 的信号

**ALU 组成需要考虑的两个问题**：
1. ALU 多功能的实现：不将输入 $A_1, B_1$ 和下一位进位数 $C_1$ 直接进行全加，而是将 $A_1$ 和 $B_1$ 先组合成由控制参数 $S_0, S_1, S_2, S_3$ 控制的组合函数 $X$ 和 $Y$，然后再将 $X, Y$ 和下一位进位数通过全加器进行全加
2. ALU 实现快速运算的方法：
   - 行波 ALU 存在问题：串行进位，运算时间很长
   - 先行进位：低位进位可以直接传送到最高进位上去，因而可以实现高速运算

**运算器的构成**：运算器是进行数据处理的器件，核心部分是 ALU，还包括存放数据的寄存器、传送数据的内部总线 IB 以及进行数据选择的多路选择器。

运算器逻辑组织的差异体现在以下方面：
- 数据宽度：运算器一次可以并行处理的二进制位数
- 表示方式：支持定点或浮点表示法
- 运算方法：具体运算的实现方法
- 内总线结构：单总线、双总线或三总线
- 指令功能：支持的运算类型

**内部总线**：指 CPU 内部连接各寄存器及运算部件之间的总线。运算器内部结构有单总线、双总线和三总线三种形式。总线形式不同，运算器的内部结构也就不同，使得运算的并行性也不同。

> 注：讲义仅提及三种总线形式不同使得运算并行性不同，未详述各自具体特点。

### 浮点算术运算

**浮点加减法运算的 4 个基本阶段** ⭐⚠：
1. 0 操作数检查
2. 比较阶码并完成对阶
3. 尾数加减
4. 规格化结果

**对阶方法** ⚠：对阶时，总是使小阶向大阶看齐，即小阶的尾数向右移位（相当于小数点左移），每右移一位，其阶码加 1，直到两数的阶码相等为止，右移的位数等于阶差。

> Caveat (判断8、13): 浮点加减法运算时，对阶采用**小阶向大阶看齐**的原则，小阶尾数右移，切勿大阶向小阶看齐。

**尾数加减时的溢出判定** ⚠：浮点运算中，尾数溢出不能采用定点运算中的处理方法，因为可以通过改变阶码的值使尾数不再溢出。只有当阶码上溢（或下溢出，根据机器定义的不同）时，整个浮点数才发生溢出。

**补码数的规格化**：
- 尾数运算结果的绝对值必须 $\geq 1/2$
- 补码规格化形式：$0.1XXXXX$ 和 $1.0XXXXX$ 两种
- IEEE 754 形式：尾数要求为 $1.M$ 形式

**规格化方法**：

| 方法 | 操作 | 适用场景 | 次数限制 |
|---|---|---|---|
| 左规 | 尾数左移一位，阶码减 1 | 尾数运算结果为非规格化时 | 可能要进行多次 |
| 右规 | 尾数右移一位，阶码加 1 | 尾数运算结果溢出（双符号位为 01 或 10）时 | 只需进行一次 |


> Caveat (多选5): 对阶操作不会引起阶码上溢或下溢；右规和尾数舍入都可能引起阶码上溢；左规时可能引起阶码下溢；尾数溢出时结果不一定溢出。

**浮点乘法运算基本过程** ⭐：
1. 测试 0
2. 被乘数的指数加乘数的指数
3. 尾数相乘
4. 规格化和舍入处理

两规格化浮点数相乘，不需要右规，但可能需要左规 1 次。

**浮点除法运算基本过程** ⭐：
1. 测试 0
2. 被除数的指数减除数的指数
3. 尾数相除
4. 规格化和舍入处理

浮点除法中，不再要求 $|被除数| < |除数|$。由于被除数可能大于除数，商可能需要右规处理（右规一次即可）；也可能需要左规处理（如商的绝对值小于 1/2，需左规一次）。

**舍入方法** ⭐：

| 舍入方法 | 原理 | 特点/缺点 |
|---|---|---|
| 截断法 | 将欲保留的末位数据右面的数据统统截掉，无论是 0 还是 1 | 简单，但容易产生积累误差效应 |
| 末位恒置 1 法 | 将欲保留的末位数据恒置 1，无论右面是 0 还是 1，也无论现在末位的值是 0 还是 1 | 简单，但容易产生积累误差效应 |
| 0 舍 1 入法 | 一种比较合理的方法 | 当尾数为 $0.11\cdots1$ 时会造成尾数溢出，需调整阶码，运算步骤不规整 |
| 查表法 | [原文未述具体规则] | [原文未述] |


IEEE 标准列出的 4 种可供选择的方法：

| IEEE 舍入方法 | 原理 |
|---|---|
| 就近舍入（Round To Nearest） | 结果被舍入成最近的可表示的数 |
| 朝 +∞ 舍入（Round Toward +∞） | 结果向正无穷大方向取舍 |
| 朝 −∞ 舍入（Round Toward −∞） | 结果向负无穷大方向取舍 |
| 朝 0 舍入（Round Toward 0） | 结果朝 0 取舍 |

| 补码减法 | $[X-Y]_{补}=[X]_{补}+[-Y]_{补}$ | — |
| 求 $[-X]_{补}$ | 连符号在内依次按位取反，末位加 1 | — |

## 四、重点对比与易混辨析

**上溢 vs 下溢**：

| 对比项 | 上溢 | 下溢 |
|---|---|---|
| 定义 | 两正数相加，结果大于最大正数 | 两负数相加，结果小于最小负数 |
| 浮点运算中 | 右规和尾数舍入可能引起阶码上溢 | 左规可能引起阶码下溢 |


**浮点运算 vs 定点运算溢出判定**：

| 对比项 | 定点运算 | 浮点运算 |
|---|---|---|
| 尾数溢出处理 | 即为运算溢出 | 不一定溢出，可通过改变阶码使尾数不再溢出 |
| 整体溢出判定 | 运算结果超出表示范围即溢出 | 仅当阶码上溢（或下溢）时整个浮点数才溢出 |


**浮点乘法 vs 浮点除法规格化**：

| 对比项 | 浮点乘法 | 浮点除法 |
|---|---|---|
| 指数运算 | 被乘数指数 + 乘数指数 | 被除数指数 − 除数指数 |
| 尾数运算 | 尾数相乘 | 尾数相除 |
| 规格化需求 | 不需要右规，可能需要左规 1 次 | 可能右规 1 次，也可能左规 1 次 |
| 被除数/被乘数限制 | [原文未述] | 不再要求 $\|被除数\| < \|除数\|$ |


## 五、高频考点与易错提醒

1. **补码特性与范围** ⭐：补码零唯一、比原码多表示一个数、8 位补码范围 $-128 \sim +127$（非 $-127 \sim +127$）
2. **IEEE 754 浮点数转换** ⭐：十六进制→二进制→提取符号/阶码/尾数→阶码减 127→补隐藏 1
3. **浮点加减对阶原则** ⚠：小阶向大阶看齐，小阶尾数右移
4. **溢出检测** ⭐：双符号位法（01/10 溢出）和单符号位法（$V = C_f \oplus C_0$），用异或门实现
5. **奇偶校验** ⚠：只能检错不能纠错，只能检测一位错
6. **8421 BCD 码修正** ⚠：求和超过 9 时需加 **6** 修正（非加 8）
7. **补码求负数方法** ⭐：符号位置 1，其余取反末位加 1；或从低位向高位找第一个 1，左边取反右边不变
8. **模的概念** ⭐：$[X]_{补} = X_0 X_1 \cdots X_n$ 的模是 $2^{n+1}$
9. **浮点运算溢出** ⚠：尾数溢出不等于结果溢出，阶码溢出才是真正的溢出
10. **补码引入目的** ⭐：简化计算机设计（精简加减法器硬件复杂度），非提高速度或精度
11. **进制转换易错** ⚠：二进制 01100100 转十六进制是 **64H**（非 46H）

## 六、一句话记忆要点

- **原码**：符号幅值，零不唯一，n 位表示 $2^n-1$ 个数，乘除方便加减难。
- **补码**：有模运算的补数，零唯一，多表示一个数，加减统一为加法。
- **求负数补码**：取反末位加 1，或找第一个 1 左边取反右边不变。
- **溢出检测**：双符号位看 01/10，单符号位用 $C_f \oplus C_0$ 异或门。
- **浮点数**：$\pm S \times B^E$，尾数补码、阶码移码、基值隐含。
- **IEEE 754**：1 位符号 + 8 位阶码 + 23 位尾数，阶码减 127，尾数补隐藏 1。
- **对阶**：小阶向大阶看齐，小阶尾数右移。
- **规格化**：左规（尾数左移阶码减 1，可多次），右规（尾数右移阶码加 1，仅一次）。
- **校验码**：奇偶检 1 位错不纠错， CRC 漏检低应用广，海明码纠 1 位错。
- **浮点溢出**：尾数溢出不等于结果溢出，阶码溢出才是真溢出。

## Worked Examples

### 例 1：IEEE 754 单精度浮点数转换 *(来自 exam-signals 候选 2)*
*Tags: IEEE754, 阶码, 尾数*
**问题**：采用 IEEE 754 单精度浮点数格式表示一个数为 45100000H，求该数的值。

**解答**：
1. 十六进制转二进制：$45100000H = 0100\ 0101\ 0001\ 0000\ 0000\ 0000\ 0000\ 0000$
2. 提取各段：符号位 $S = 0$；阶码 $E = 10001010B = 138D$；尾数 $M = 001\ 0000\ \cdots\ 0$
3. 计算阶码真值：$e = E - 127 = 138 - 127 = 11$
4. 计算尾数真值：$1.M = 1.001B = 1 + 1 \times 2^{-3} = 1.125$
5. 组合结果：$(-1)^S \times 1.M \times 2^e = +1.125 \times 2^{11}$

答案为 B：$(+1.125)_{10} \times 2^{11}$。
