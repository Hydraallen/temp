6.1.1 域名系统的概念
Internet 上的主机名：用英文字母和数字组成。如：mail. tsinghua. edu. cn。
域名是一个逻辑概念，用于标识主机，并不反映主机所在的物理地点。
域名系统 DNS (Domain Name System)：负责将主机名转换为 IP 地址。Internet 的域名系统
被设计成一个联机分布式数据库系统，并采用客户／服务器模式。
域名服务器程序：在专设的结点上运行。运行该程序的计算机称为域名服务器。

6.2 文件传输协议 FTP、TFTP
6.2.1 文件传输协议 FTP
基于 TCP。使用端口号 20（数据连接）和 21（控制连接）。传送文件类型分为两种：ASOI（文本）文件和二进制（非文本）文件。
FTP 服务器进程由两部分组成：一个主进程，负责接受请求；若干个从属进程，负责处理单个请求。工作时需要建立两个连接：控制连接和数据连接。
匿名 FTP:使用默认的帐号(anonymous)、口令即可登录访问的 FTP 服务。
特点：必须先获得文件的副本，才能修改文件，然后将该副本传回到服务器。
6.2.2 简单文件传送协议 TFTP
TFTP 是一个很小且易于实现的文件传送协议。
TFTP 使用客户服务器方式和使用 UDP 数据报，因此 TFTP 需要有自己的差错改正措施。
TFTP 只支持文件传输而不支持交互。
TFTP 没有一个庞大的命令集，没有列目录的功能，也不能对用户进行身份鉴别 TFTP 的主要特点是：
每次传送的数据 PDU 中有 512 字节的数据，但最后一次可不足 512 字节。
数据 PDU 也称为文件块(block)，每个块按序编号，从 1 开始。
支持 ASCII 码或二进制传送。
可对文件进行读或写。
使用很简单的首部。

6.3 远程终端协议 Teelnet

远程登录是为用户提供的以终端方式与Internet上的主机建立在线连接的一种服务。在这种连接建立之后，用户的计算机就可以作为远程主机的一台终端来使用该主机提供的各种资源(用户能获得哪些服务资源取决于该主机所拥有资源的丰富程度和用户拥有的权限)。

实现远程登录的工具程序很多，最常见的就是Telnet程序(与Telnet协议同名)，它在Unix系统和Windows、DOS系统中都可以找到。

为了使用Telnet，需要在用户的本地计算机上运行一个名为telnet的程序。这个程序可以通过Internet把用户的计算机与指定的远程主机连接起来。一旦连接成功，telnet就成为用户主机与另外一个计算机的中介物，用户通过键盘输入的任何信息都被发送远程主机；在远程主机屏幕上显示的信息也被传回到用户的屏幕上，其效果就好像本地的键盘和屏幕与远程主机直接相连一样。

在Internet术语中，用户所在的计算机被称为本地(Local)计算机或本地主机(Host)；而用户所连接的另一台计算机则称为远程(Remote)计算机或远程主机，且不管这个远程主机距离有多远，无论是在同一房间或是在世界的另一端都无所谓。

6.4 电子邮件 SMTP、POP3 和 IMAP

6.4.1 电子邮件概况

电子邮件是 Internet 最早开发的而且是使用最广泛的应用之一。早期的电子邮件系统仅由文件传输协议组成，存在不少问题。1982 年制订了 ARPANET 的“简单电子邮件协议”SMTP 和“电子邮件报文格式”MAIL。此后，CCITT 制订了“报文处理系统”MHS 的标准 X. 400。为传送多媒体数据，1993 年又制订了“通用因特网扩充”MIME 标准。目前，因特网的电子邮件系统已成为事实上的标准，而 X. 400 则由于过于复杂而基本上被淘汰。

6.4.2 电子邮件系统组成与服务

一般由两个子系统组成：

用户代理 UA：用户与系统的接口，本地程序，提供收发邮件的界面。

邮件服务器：运行在后台的服务器软件，负责消息的管理、传送与提交。

6.4.3 电子邮件系统的基本功能

撰写：给用户提供创建、编辑消息的环境。

传输：自动地将消息传送给接收者，一般为基于连接方式。

报告：向发送者反馈消息的传送情况。

呈现：显示来信的内容。

处理：使收信人可根据情况按不同方式对来信进行处理。

6.4.4 SMTP 协议与 POP3 协议

在电子邮件系统中，最重要的协议就是 SMTP 协议与 POP3 协议。

发送邮件使用的是 SMTP 协议，而接收邮件则使用 POP3 协议。

■ SMTP 协议

在 Internet 电子邮件系统中，使用简单邮件传送协议 SMTP 进行邮件传送。

SMTP 规定在两个通信的 SMTP 客户进程和服务器进程之间如何交换信息。

SMTP 使用客户 / 服务器模式，在 TCP 连接上传送 ASCII 码邮件。

负责发送邮件的 SMTP 进程是 SMTP 客户，而负责接收邮件的 SMTP 进程是 SMTP 服务器。

发送前要先建立 TCP 连接，SMTP 客户与目的主机的 SMTP 服务器（使用周知端口 25）建立 TCP 连接。在这个连接上，可以发送多个电子邮件（如果有的话）。当电子邮件都发送完之后，TCP 连接就被释放。

■ POP3 协议

邮局协议版本 3 (POP3)很简单，但它的功能有限。客户 POP3 软件安装在收信人的计算机上；服务器 POP3 软件安装在邮件服务器上。

POP 客户与 POP 服务器（使用 TCP 端口 110）建立 TCP 连接。POP 客户然后发送用户名和口令，访问邮箱。用户可以列出清单，并逐个读取邮件报文。

6.5 万维网 WWW

6.5.1 什么是 www

www 是 World Wide Web 的简称。类似的简称有 Web、W3、3W 等，译为万维网。www 使用超文本(Hypertext)组织、查找和表示信息，利用了从一个站点到另一个站点的链接。这样就彻底摆脱了以前查询工具只能按特定路径一步步地查找信息的限制。另外它还具有连接已有信息系统(Gopher、FTP、News)的能力。www 具有友好的用户查询接口，是目前广为流行的最受欢迎的、最先进的信息服务工具。

超文本文件使用超文本标记语言(HTML)格式写成，这种语言是欧洲粒子物理实验室(CERN)提出的 www 描述性语言。www 文本不仅含有文本和图象，还含有作为热链的词、词组、句子、图象、图标。这些热链通过颜色和字体的改变与普通文本区别开来，它含有指向其他 Internet 信息 URL 地址(Unifoiy Resource Locator)。将鼠标移到热链上，点击之，Web 就根据热链所指向的 URL 地址跳到不同站点、不同文件。链接同样可以指向声音、视频等多媒体超文本，与多媒体一起形成超媒体(Hypermedia)。www 实现了文本与声音、图象的同时传输，大大扩展了 Internet 的传输信息范围。

6.5.2 WWW 的历史

1989 年，位于瑞士的欧洲粒子物理实验室实施了超媒体项目。它最初的目标是让分散在世界各地的物理学家能共同分享最新研究成果和进行科研合作，它只是作为高能物理界通告科研成果和交流思想的一种手段，最初方案是由 Tim Bemer Lee 负责实施的。

1990 年 11 月，NeXT 发布了最初的 World Wide Web 模型。同年就诞生了行模式和 NeXT Step 浏览器，它们能访问超文本和 Internet 新闻组。一年后，CERN 向外界发布了 W3。此后，它在高能物理界之外也蓬勃发展起来，最先在欧洲，然后到美国。

1993 年 NCSA 开发出了 www 的图形化浏览器 Mosaic。这样 www 的应用就变得方便直观了。自此以后，www 以每月百分之二十几的速度增长，到 1994 年底，全世界已有超过一万台的 www 服务器，商业 www 服务器也出现了。

1994 年，由部分设计 Mosaic 的人员组成的 Netscape Communicator 公司推出了功能强大的商业 Web 图形浏览器 Netscape Navigator。它克服了 Mosaic 的缺点，提供了一个更加方便、直观的、易于使用的图形界面。

1996 年和 1997 年，微软公司相继发布了其著名的 www 浏览器 Internet Explore 3.0 和 4. O。尤其是 IE4.O，由于实现了 Windows 的紧密集成，使其操作更加简便，界面更加友好。

目前，Mozilla Firefox、Google Chrome、Internet Explore、傲游、Safari 等被称为当今最受欢迎的 Web 浏览器。

6.5.4 URL 地址

URL 地址是 www 中用来寻找资源地址的办法，URL 是 Uniform Resource Locator 的缩写，译为统一资源定位器或普遍资源定位器。URL 地址的思想是为了使所有的信息资源都能得到有效利用，从而将分散的孤立的信息点连接起来，实现资源的统一寻址。URL 大致由三部分组成：协议、主机名、目录或文件名，如图 6.1 所示。

协议即访问页的方式，也就是浏览器用于取得超文本文件的协议或程序。如果浏览器利用 HTTP 方式来访问文件，那么协议部分就是 http;如果用 FTP，则是 ftp;用 Gopher，就是 gopher;以此类推。协议必须与系统安装的信息服务器匹配以便能正常工作。例如，在一台没有安装 FTP 服务器的机器上，用户就无法使用 FTP 的 URL。

主机名是 Internet 上用于标识信息保存的地方，例如 www.netcom.com，ftp.apple.com 或 www.aol.com。同一个主机名因使用不同的协议而有不同的 URL，如：

6.6.1  DHCP概念
