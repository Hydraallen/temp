## 一、本章概览

本章解决计算机系统中各功能部件（CPU、存储器、I/O 设备）之间如何互连、如何通信、以及主机与外设之间如何进行数据传送的问题。全章分为三大块：**总线技术**（第一至四节，涵盖互连结构、总线定义/特性/性能指标/分类、总线连接方式、总线设计要素）、**I/O 接口基础**（第五节，涵盖接口功能、类型、端口寻址）、**I/O 控制方式**（第六至九节，从程序查询→程序中断→DMA→通道，逐步降低 CPU 介入程度，提升数据传送效率）。本章是计算机组成原理中篇幅最大、考点最密集的章节之一，中断机制与 DMA 机制是重中之重。

## 二、知识点详解

### 第一节 计算机系统互连结构

#### 1. 互联结构

- **定义**：互连结构（Interconnection Structure）是连接各种模块的通路的集合。这一结构的设计取决于模块之间所必须交换的信息。
- **要点**：
  - 冯·诺依曼型计算机由五大部件组成，部件之间必然要通信，必须有使部件连接在一起的通路。
  - 组成计算机各模块（CPU、存储器、I/O）之间传送的信号，互连结构必须支持传送的信号类型。
  - 目前最普遍的互连结构是**总线和各种多总线结构**（包括层次式总线）。
  - 在现代计算机系统中，互连结构已成为决定计算机性能的重要因素之一。

### 第二节 总线的基本概念

#### 2. 总线的定义及分类 ⭐（单选1, 单选2, 单选7, 单选8, 单选9, 判断1）

- **定义**：总线（Bus）是连接两个或多个设备的公共通信通路。总线是构成计算机系统的互联机构，是多个系统功能部件之间进行数据传送的公共通路。总线的关键特征是**共享传输媒质**。
- **分类（单处理器系统）**：
  - **内部总线**：CPU 内部连接各寄存器及运算部件之间的总线。
  - **系统总线**：CPU 同计算机系统的其他具有高速传输功能的部件。
  - **I/O 总线**：中、低速 I/O 设备之间互相连接的总线。
- 多处理器系统中，还要有**通信总线**。
- **总线标准**：可视为系统与各模块、模块与模块之间的一个互连的标准界面。使用总线标准，可使不同厂家生产的相同功能部件互换使用。目前常用的总线标准有 ISA、EISA、PCI、STD 等。

#### 3. 总线的特性 ⭐（多选2）

- **机械特性**：总线在机械连接方式上的一些性能，保证机械上的可靠连接。
- **电气特性**：总线的每一根传输线上信号的传递方向和有效的电平范围，确保电气上正确连接。
- **功能特性**：总线中每根传输线的功能。
- **时间特性**：总线中的任一条线在什么时间内有效。

#### 4. 总线的性能指标

- **总线宽度**：数据总线的根数。
- **标准传输率**：在总线上每秒能传输的最大字节数。
- **时钟同步/异步**：总线上的数据与时钟同步工作的称同步总线，与时钟不同步工作的称异步总线。
- **总线复用**：地址总线和数据总线共用一组物理线路，叫做总线的多路复用。
- **信号线数**：地址总线、数据总线和控制总线三种总线数的总和。
- **总线控制方式**：包括并发工作、自动配置、仲裁方式、逻辑方式、计数方式等。
- **其他指标**。

#### 5. 总线的内部结构

- **早期总线**：实际上是处理器芯片引脚的延伸，是处理器与 I/O 设备适配器的通道。
  - 不足之处：
    - CPU 是总线上惟一的主控者。即使后来增加了具有简单仲裁逻辑的 DMA 控制器以支持 DMA 传送，仍不能满足多 CPU 环境的要求。
    - 总线信号是 CPU 引脚信号的延伸，故总线结构紧密与 CPU 相关，通用性较差。
- **当代流行总线**：一些标准总线，追求与结构、CPU、技术无关的开发标准，并满足多个主控者（包括 CPU）环境需求。在这种结构中，CPU 和它私有的 cache 一起作为一个模块与总线相连。

#### 6. 总线的分类（按功能） ⭐（多选1, 判断1）

系统总线按功能可分为：
- **数据总线**：数据线提供系统模块间传送数据的路径。
- **地址总线**：地址线用于指定数据总线上数据的来源和去向。
- **控制总线**：控制线用来控制对数据地址线的访问和使用。
- 除此之外，还有为模块提供电源的**电源线**。

> 总线标准界面对它两端的模块都是透明的，即界面的任一方只需根据总线标准的要求完成自身一面接口的功能要求，而无需了解对方接口与总线的连接要求。按总线标准设计的接口可视为通用接口。

### 第三节 总线连接方式

#### 7. 单总线结构 ⭐（单选4）

- **结构**：CPU、主存储器，以及所有 I/O 设备均通过一组总线连接。
- **优点**：结构简单，总线控制也较简单，系统易于扩展。
- **缺点**：只能支持信息传送量相对较小的计算机系统。

#### 8. 双总线结构

- **带存储总线的双总线结构**：在 CPU 和主存之间增设一组总线（存储总线），属于局部总线。
- **带 IOP 的双总线**：IOP 一方面通过 I/O 总线与众多外部设备相连，另一方面又与连接 CPU 和 MM 的系统总线相连。

#### 9. 中间层结构

- **背景**：多媒体技术与网络技术的发展要求高速度大容量的数据传输，传统总线力不从心。
- **方案**：构造与系统其余部分紧密集成的高速总线，仅要求一个在处理器总线和高速总线之间建立连接（中间层结构）。
- **结构要点**：
  - 连接处理器和高速缓存控制器的局部总线。
  - 高速缓存控制器连接到支持主存储器的系统总线上。
  - 高速缓存控制器集成到连接高速总线的桥或缓冲设备中。
  - **总线桥接器（简称桥）**：由总线控制器、数据缓冲区及总线加速器组成，总线控制器在总线争用时起仲裁作用。
- **优点**：高速总线使高需求设备与处理器更紧密集成，同时又独立于处理器。处理器和高速总线速度及信号线定义的不同都可以容忍。处理器结构的变化不影响高速总线，反之亦然。

### 第四节 总线设计要素

#### 10. 总线仲裁 ⭐⚠（单选5, 判断4, 判断5）

仲裁方法分为**集中式**和**分布式**两类。

- **集中式**：一个称为总线控制器或仲裁器的硬件设备负责分配总线时间。可以是独立的模块，也可以是 CPU 的一部分。
  - **链式查询方式**：
    - 特点：离中央仲裁器最近的设备具有最高优先权，离总线控制器越远，优先权越低。
    - 优点：只用很少几根线就能按一定优先次序实现总线控制，链式结构很容易扩充设备。
    - 缺点：对询问链的电路故障很敏感，优先级固定。
  - **计数器定时查询方式**：可方便地改变优先级。
  - **独立请求方式**：
    - 优点：响应时间快，即确定优先响应的设备所花费的时间少。对优先次序的控制相当灵活。
- **分布式**：没有中央控制器，每个模块中包含访问控制逻辑，多个仲裁器竞争使用总线。

> ⚠ 判断4指出"定时查询方式响应速度最快"为错误，独立请求方式才是最快的。(判断4)

#### 11. 总线定时 ⭐⚠（单选6, 判断6）

定时指事件出现在总线上的时序关系，分为同步时序和异步时序两种。

- **同步**：事件在总线上的时刻由总线时钟信号来确定。传输频率较高，适用于各功能模块速度相差不多的情况。
- **异步**：应答方式或互锁机制。总线周期长度可变，适用于快速、慢速设备连接到同一总线。

> ⚠ 异步通信中，不互锁方式不需等待对方应答，传输速度最快但可靠性较低；全互锁方式可靠性高但速度最慢。(单选6)

#### 12. 总线数据传输模式

- **读、写**：总线传输的最基本形式，所有总线系统都必须支持。
- **读—修改—写**：主要用于共享存储器的保护，如进程管理中的 PV 原语、管程等。
- **写后读**：主要用于数据校验。
- **成块传输**：现代总线具有的特征，可实现与高速设备的信息传输。

#### 13. 总线宽度

- **数据总线的宽度**：数据总线越宽，一次能传送的位数就越多。
- **地址总线的宽度**：地址总线越宽，可以访问的单元就越多。

#### 14. 总线复用

总线的信号线可归为**专用**和**复用**两类。

- **专用**：始终只负责一项功能，或始终分配给计算机部件的一个物理子集。
  - 优点：总线冲突减少，具有高吞吐量。
  - 缺点：增加了系统的规模和成本。
- **复用**：
  - 优点：使用的总线数量少，节省空间和成本。
  - 缺点：控制电路略显复杂，潜伏着性能降低的危险（共享总线的特定事件不能同时发生）。

### 第五节 输入输出系统概述

#### 15. 为什么需要 I/O 接口

一个 I/O 接口不是简单地把设备和系统总线连接起来的机械连接器，而是包含了在外设与总线之间执行通信功能的逻辑。

主机和外设信息交换必须通过接口的原因：
- 各种外设使用不同的操作方法，将控制一定范围的设备的必要逻辑放入处理器内是不切实际的。
- 外设的数据传送速度一般比存储器或处理器慢得多，使用高速的系统总线直接与外设通信是不切实际的。
- 外设经常使用与处理器不同的数据格式和字长度。

#### 16. I/O 接口的功能 ⭐（单选3, 多选7）

- 通过系统总线或中央交换器与处理器和存储器连接。
- 通过专用数据线与一个或多个外设连接。

具体功能包括：
- 控制和定时
- 处理器通信
- 设备通信
- 数据缓冲
- 检错

**接口传送的信息**：数据信息、控制信息和状态信息。大多数计算机把外部设备的状态信息视为输入数据，把控制信息看成输出数据，在接口中分设各自相应的寄存器，赋以不同的端口地址，各种信息分时地使用数据总线传送到各自的寄存器中去。

**接口（Interface）与端口（Port）的区别**：
- **端口**：接口电路中可以被 CPU 直接访问的寄存器。
- **接口**：若干个端口加上相应的控制逻辑电路才组成接口。
- 通常一个接口包含：**数据端口**（存放数据信息）、**命令端口**（存放控制命令）、**状态端口**（存放状态信息）。
- CPU 通过输入指令从端口读取信息，通过输出指令把信息写入端口。有的端口只能写或只能读，有的端口既可以读又可以写。

#### 17. 接口的类型

- 按数据传送方式：串行接口和并行接口。
- 按主机访问 I/O 设备的控制方式：程序查询式接口、中断接口、DMA 接口、通道、I/O 处理机等。
- 按功能选择的灵活性：可编程接口和不可编程接口。
- 按通用性：通用接口和专用接口。
- 按输入输出的信号：数字接口和模拟接口。

#### 18. I/O 端口寻址方式 ⭐⚠（单选4, 多选3）

I/O 端口寻址方式有两种：

| | 存储器映射方式（统一编址） | I/O 映射方式（独立编址） |
|---|---|---|
| **编址方式** | 端口地址与存储器地址统一编址 | I/O 端口地址与存储器地址分别独立编址 |
| **优点** | 访存指令同时也可进行外设端口的访问，指令格式统一 | 主存和外设都有自己独立的地址空间 |
| **缺点** | 外设端口要占用一部分主存的地址空间 | 访问外设和主存所使用的指令不同 |

> ⚠ 单总线结构中，外设和主存都在一条总线上，地址线宽度相同，可以统一编址。(单选4)

### 第六节 程序查询方式及其接口

#### 19. 程序直接控制方式

由 CPU 执行一段输入输出程序来实现内存与外设之间数据传送的方式。分为两种：
- **无条件传送**：I/O 端口总是准备好接收主机的输出数据，或总是准备好向主机输入数据，只适用于一些简单设备。
- **程序查询方式**：可适用于大多数的低速外设。

#### 20. 程序查询的基本思想

CPU 在程序中查询外设的工作状态，来保证数据传送的正确进行。

**工作过程**：
```
1. 预置传送参数
2. 向外设接口发出命令字
3. 从外设接口取回状态字
4. 查询外设标志
5. 传送数据
6. 判断传送是否结束
```

- 接口中至少有两个寄存器：**数据缓冲寄存器**（数据端口）和**设备状态寄存器**（状态端口）。
- 状态寄存器中最重要的标志是"外设准备就绪"标志。
- 对多个外设进行信息交换时，可采用**轮询**的方式。

### 第七节 程序中断方式及其接口

#### 21. 程序查询方式的缺点

- 在查询过程中，CPU 长期处于踏步等待状态，使系统效率大大降低。
- CPU 在一段时间内只能和一台外设交换信息，其他设备不能同时工作。
- 不能发现和处理预先无法估计的错误和异常情况。

#### 22. 程序中断方式的基本思想 ⭐⚠（单选10, 单选15, 多选4, 多选6, 多选9, 多选10, 判断8, 判断9）

- **核心思想**：实现主动权的变化。程序查询方式中 CPU 主动查询外设状态；中断方式中，外设主动向 CPU 报告自己的状态。
- 中断方式在一定程度上实现了 CPU 和外设的并行工作，使 CPU 的效率得到充分发挥。
- **中断系统**是计算机实现中断功能的软、硬件总称。一般在 CPU 中配置中断机构，在外设接口中配置中断控制器，在软件上设计相应的中断服务程序。

**中断的基本类型**：
- 自愿中断和强迫中断
- 内中断和外中断
- 向量中断和非向量中断
- 单重中断和多重中断

**内中断（内部异常）**：与当前执行指令有关，中断信号来源于 CPU 内部。是一个算法指令，由软中断指令启动。例子：地址非法、校验错、页面失效、存取访问控制错、算术操作溢出、数据格式非法、除数为零、非法指令、用户程序执行特权指令。

**内中断进一步分类**（图 6-5）：
- **陷阱、陷入（trap）**：由当前[为之?疑为"指令"]的异常，如系统调用。
- **故障（fault）**：由错误条件引起的，可能被故障处理程序修复，如缺页中断。
- **终止（abort）**：不可恢复的致命错误造成的结果，终止处理程序不再将控制返回给引发终止的应用程序，如[权?疑为"机器"]故障。

**外中断**：信号来源为 CPU 外部，包括 I/O 中断请求、人工干预。分为**可屏蔽中断**（通过 INTR 信号线）和**不可屏蔽中断**（通过 NMI 信号线）。

> ⚠ "除数为零"属于内部中断（异常），不是外部中断或软件中断。(单选10)
> ⚠ 内部异常中，故障返回原指令重新执行，陷阱返回下一条指令，终止不返回。(多选9)
> ⚠ 不可屏蔽中断即使在关中断（IF=0）情况下也会被响应，优先级最高。(多选10)

**中断源**：引起计算机中断的事件。可采用具有存储功能的触发器来记录中断源，称为**中断请求触发器（INTR）**。

**中断请求信号传送到 CPU 的方式**：
- 独立请求线
- 公共请求线
- 二维结构

**确定中断优先级的原则**：对提出中断请求后需要立刻处理否则会造成严重后果的中断源规定最高优先级；对可以延迟响应和处理的中断源规定较低的优先级。

**中断判优方法**：
- **软件判优法**：灵活性强，但速度慢。
- **硬件判优电路**：优先级别固定，但速度快，是目前常用的方法。

**CPU 响应中断必须满足的条件**：
- CPU 接收到中断请求信号
- CPU 允许中断
- 一条指令执行完毕

#### 23. 中断隐指令 ⭐（多选5）

CPU 响应中断之后，经过某些操作，转去执行中断服务程序。这些操作由硬件直接实现，称为**中断隐指令**。

**中断隐指令完成的操作**：
- 保存断点
- 暂不允许中断（关中断）
- 引出中断服务程序

> ⚠ 中断隐指令完成的是关中断、保护断点、引出中断服务程序；保存通用寄存器内容是在进入中断服务程序后进行的操作，不属于中断隐指令。(多选5)

#### 24. 向量中断 ⭐⚠（单选14）

- **硬件向量中断**：当 CPU 响应某一中断请求时，硬件能自动形成并找出与该中断源对应的中断服务程序的入口地址。
- **向量地址的两种情况**：
  - 向量地址是中断服务程序的入口地址。
  - 向量地址是中断向量表的指针。

> ⚠ 中断向量地址是"中断服务程序入口地址的地址"，而不是中断服务程序入口地址本身。(单选14)

**单重中断与多重中断流程对比**：

```
单重中断流程：
取指令 → 执行指令 → [中断?]
  No → 取指令
  Yes → 关中断 → 断点→EPC → 中断周期（中断识别，地址→PC）
        → 保护现场 → 设备中断服务子程序
        → 恢复现场 → 开中断 → 中断返回（EPC→PC）

多重中断流程：
取指令 → 执行指令 → [中断?]
  No → 取指令
  Yes → 关中断 → 断点→EPC → 中断周期（中断识别，地址→PC）
        → 保护现场（含新嵌套）→ 开中断
        → 设备中断服务子程序
        → 关中断，恢复现场 → 开中断（中断返回 EPC→PC）
```

**中断现场**：发生中断时 CPU 的主要状态，最重要的是断点，还有通用寄存器的状态。现场的保护和恢复方法有纯软件和软、硬件相结合两种。

> ⚠ 现场保护是指保护中断服务子程序中要用的各种寄存器（含标志寄存器）中的内容，不是保护断点地址。(单选12)

**中断嵌套**：层次可以多层，越在里层的中断请求越急迫，优先级越高，优先得到 CPU 服务。

**多重中断注意事项**：
- 若系统中有 $n$ 级中断，CPU 中就要有 $n$ 个中断请求触发器与之对应，称为**中断请求寄存器**。
- 多级中断系统中每一级可以是一个或多个中断源。多级中断之间可以实现中断嵌套，但**同一级内的中断源的中断不能嵌套**。
- 多级中断系统同样使用**堆栈**保存断点信息。

> ⚠ 保存现场信息最有效的方法是采用堆栈。(单选11)

**多重中断中开/关中断的时机**：

| 时机 | 操作 |
|---|---|
| 应**开中断**的情况 | ① 已响应中断请求转向中断服务程序，在保护完中断现场之后；② 在中断服务程序执行完毕，即将返回被中断的程序之前 |
| 应**关中断**的情况 | ① 当响应某一级中断请求，不再允许被其他中断请求打断时；② 在中断服务程序的保护和恢复现场之前 |

**中断屏蔽**：中断源发出中断请求后，并不一定能真正送到 CPU，可以用程序方式有选择地封锁部分中断。实现方式是使用**中断屏蔽寄存器**，与中断请求寄存器相对应。在多级中断系统中，中断屏蔽寄存器的内容是一个很重要的程序现场。

**中断全过程五个阶段**：
1. 中断请求
2. 中断判优
3. 中断响应
4. 中断处理
5. 中断返回

**中断控制机构至少应包括**：
- 中断请求电路
- 中断优先级电路
- 向量地址形成部件

### 第八节 DMA 方式及其接口

#### 25. 直接存储器访问 DMA ⭐⚠（单选16, 单选20, 多选8, 判断10）

- **定义**：直接存储器访问 DMA（Direct Memory Access）是一种完全由硬件执行 I/O 交换的工作方式。在外设和内存之间开辟一条"直接数据通道"，在不需要 CPU 干预也不需要软件介入的情况下进行高速数据传送。

#### 26. DMA 方式的特点

- 它使内存与 CPU 的固定联系脱钩。内存既可被 CPU 访问，又可被外设访问。
- 在数据块传送时，内存地址的确定、传送数据的计数等都用硬件电路直接实现。
- 内存中要开辟专用缓冲区，及时供给和接收外设的数据。
- DMA 传送速度快，CPU 和外设并行工作，提高了系统的效率。
- DMA 在传送开始前要通过程序进行预处理，结束后要通过中断方式进行后处理。

**DMA 方式的主要优点**：速度快。由于 CPU 不参加传送操作，省去了 CPU 取指令、取数、送数等操作。数据传送过程中没有保存现场、恢复现场之类的工作。

**DMA 方式的主要应用**：
- 磁盘数据块的读写
- 通信设备的数据交换
- 大批量数据采集系统
- DRAM 的刷新

**DMA 控制器的功能**：
- 接受外设发出的 DMA 请求，并向 CPU 发出总线请求。
- 确定传送数据的内存单元地址及传送长度，并自动修改内存地址计数值和传送长度计数值。
- 规定数据在内存与外设之间的传送方向，发出读写或其他控制信号。
- 向 CPU 报告 DMA 操作的结束（通过中断方式）。

**DMA 控制器的基本组成**：
- 内存地址计数器
- 传送长度计数器
- 数据缓冲寄存器
- DMA 请求触发器
- 控制/状态逻辑
- 中断机构

**DMA 控制器的引线**：
- 地址总线
- 数据总线
- 控制数据传送方式的信号线
- DMA 控制器与外设之间的联络信号线
- DMA 控制器与 CPU 之间的联络信号线

**主要信号**：DREQ、DACK、HRQ、HLDA。

**DMA 控制器与 CPU 共享内存的三种方法**：

| 方法 | 方式描述 | 优点 | 缺点 |
|---|---|---|---|
| **停止 CPU 访问内存** | DMA 控制器要求 CPU 放弃总线使用权，接管后开始数据传送，完毕后交还 | 控制简单，适用于高速设备成组数据传送 | DMA 传送过程中 CPU 处于空闲状态，无法执行其他任务 |
| **周期挪用** | I/O 设备发出 DMA 请求时挪用一个或多个内存周期进行数据传送，CPU 暂停访存但可继续执行其他任务 | 实现了 I/O 传送，较好地发挥了内存和 CPU 的效率。适用于 I/O 周期大于内存存储周期的情况 | 需要额外的逻辑来处理周期挪用，可能增加系统复杂性 |
| **DMA 与 CPU 交替访问内存** | 一个 CPU 周期内分为两部分，一部分供 DMA 控制器访存，另一部分供 CPU 访存 | 可发挥 CPU 最高效率，确保 DMA 传送顺利进行。适用于 CPU 工作周期比内存存取周期长得多的情况 | 需要更复杂的硬件逻辑，可能增加成本和复杂性 |

**DMA 传送过程的三个阶段**：
1. **DMA 传送的预处理**（由 CPU 完成）
2. **数据传送**（由 DMA 控制器完成）
3. **传送后的结束处理**（由 CPU 完成）

> 前后两个阶段由 CPU 完成，只有数据传送由 DMA 控制器完成。

#### 27. DMA 和中断的区别 ⭐⚠（多选8, 判断10）

| 比较项 | 中断方式 | DMA 方式 |
|---|---|---|
| **本质** | 程序切换，需要保护和恢复现场 | 除了开始和结尾，不占用 CPU 的任何资源 |
| **响应时机** | 每条指令执行完毕时 | 每个机器周期结束时 |
| **CPU 干预** | 传送过程需要 CPU 干预 | 传送过程不需要 CPU 干预，数据传送速率非常高 |
| **优先级** | 较低 | DMA 请求优先级高于中断请求 |
| **适用场景** | 具有对异常事件的处理能力 | 仅限于完成传送信息块的 I/O 操作，适合高速外设成组数据传送 |

> ⚠ 中断 I/O 方式请求的是 CPU 处理，DMA 方式请求的是总线使用权。(多选8)
> ⚠ 中断 I/O 方式下数据传送通过软件完成，DMA 方式下数据传送由硬件完成。(多选8)
> ⚠ 中断 I/O 方式只适合中、低速外部设备，而非所有设备。(多选8)

### 第九节 通道方式及其接口

#### 28. DMA 方式的不足

- DMA 方式只能进行简单的数据传输，在某些场合下不够。
- DMA 传送方式在预处理和后处理阶段仍需要 CPU 的大量介入，不适合连接大量的外围设备。

#### 29. 通道控制方式与 DMA 方式的区别

- 通过执行**通道程序**，通道具有更强的独立处理数据输入输出的功能。
- 一个通道可以同时控制许多台同类或不同类的设备。

**具有通道的计算机 4 级连接**：主机 → 通道 → 设备控制器 → 外部设备。

#### 30. 通道的功能

- 接受 CPU 的 I/O 指令，按指令要求与指定的外设进行联系。
- 执行通道程序，进行设备控制。
- 实施内存和外设间的数据传送。
- 从外设获得设备的状态信息，形成并保存通道本身的状态信息，根据要求将这些状态信息送到内存的指定单元，供 CPU 使用。
- 将外设的中断请求和通道本身的中断请求按次序及时报告 CPU。

**设备控制器的功能**：
- 从通道接收控制信号，控制外部设备完成所要求的操作。
- 向通道反馈外部设备的状态。
- 将外部设备的各种不同信号转换为通道能识别的标准信号。

#### 31. 通道的分类

按照输入输出信息的传送方式，通道可分为三种类型：

| 类型 | 数据传送基本单位 | 特点 | 适用场景 |
|---|---|---|---|
| **字节多路通道** | 字节 | 各个子通道分时使用通道 | 连接大量低速设备 |
| **选择通道** | [原文未明确，按上下文为数据块] | 又称高速通道，物理上可连接多个设备，但这些设备不能同时工作 | 连接高速外围设备 |
| **数组多路通道** | 数据块 | 物理上可连接多个设备，多个设备可同时工作，但只能有一个设备进行传输型操作，其他设备进行控制型操作 | [原文未述] |

**采用通道的计算机系统中，输入输出操作分两个层次**：
- 主 CPU 执行 I/O 指令生成输入输出程序。
- 通道执行所生成的程序，完成输入输出。

| **独立请求方式** | [原文未明确固定/可变] | 响应时间快，优先次序控制灵活 | [原文未述] |

### 2. 同步定时 vs 异步定时

| | 同步定时 | 异步定时 |
|---|---|---|
| **时序确定方式** | 总线时钟信号 | 应答方式或互锁机制 |
| **总线周期** | 固定 | 可变 |
| **传输频率** | 较高 | [原文未述] |
| **适用场景** | 各功能模块速度相差不多 | 快速、慢速设备连接到同一总线 |

### 3. I/O 端口寻址方式对比

| | 统一编址（存储器映射） | 独立编址（I/O 映射） |
|---|---|---|
| **编址** | 端口与存储器统一编址 | 端口与存储器分别独立编址 |
| **优点** | 访存指令可访问外设端口，指令格式统一 | 主存和外设有独立地址空间 |
| **缺点** | 外设端口占用主存地址空间 | 访问外设和主存使用不同指令 |

### 4. 四种 I/O 控制方式对比（CPU 介入程度由大到小）

| 方式 | CPU 介入程度 | 数据传送方式 | 特点 |
|---|---|---|---|
| **程序查询** | 最大（全程参与） | 软件 | CPU 踏步等待，效率低 |
| **程序中断** | 中等（每数据中断一次） | 软件（程序切换） | CPU 与外设并行工作，需保护/恢复现场 |
| **DMA** | 较小（仅预处理和后处理） | 硬件 | 数据传送不需 CPU 干预，速率高，适合高速外设 |
| **通道** | 最小（仅发 I/O 指令） | 硬件+通道程序 | 通道执行通道程序，可同时控制多台设备 |

### 5. 内中断 vs 外中断

| | 内中断（内部异常） | 外中断 |
|---|---|---|
| **信号来源** | CPU 内部 | CPU 外部 |
| **与指令关系** | 与当前执行指令有关 | 与当前指令执行无关 |
| **子类型** | 陷阱、故障、终止 | I/O 中断请求、人工干预 |
| **屏蔽性** | 不可屏蔽 | 分为可屏蔽中断（INTR）和不可屏蔽中断（NMI） |
| **典型例子** | 除数为零、溢出、缺页、非法指令 | I/O 设备中断、定时器中断 |

### 6. DMA 共享内存三种方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| **停止 CPU 访问内存** | 控制简单 | CPU 空闲 | 高速设备成组传送 |
| **周期挪用** | 较好发挥内存和 CPU 效率 | 需额外逻辑 | I/O 周期 > 内存存储周期 |
| **交替访问** | 发挥 CPU 最高效率 | 硬件复杂 | CPU 工作周期 >> 内存存取周期 |

## 五、高频考点与易错提醒

- ⚠ **总线基本概念**是高频考点：总线的关键特征是"共享传输媒质"；使用总线结构的主要优点是减少信息传输线的条数。(单选1, 单选2)
- ⚠ **总线分类**：CPU 向外引出的信号包括地址线、数据线和控制线（不是"系统总线"这一概念）。(单选1)
- ⚠ **总线标准识别**：ISA、EISA、PCI、PCI-Express 是总线标准；CRT、RISC、SCSI、RAM、MIPS 不是总线标准。(单选7)
- ⚠ **PCI 总线属于系统总线**，不是通讯总线或片级总线。(单选9)
- ⚠ **总线仲裁**：独立请求方式响应速度最快，不是计数器定时查询。(判断4)
- ⚠ **总线仲裁**：某一时刻只能有一个主设备控制总线。(判断5)
- ⚠ **异步通信**：不互锁方式速度最快但可靠性较低；全互锁可靠性高但速度最慢。(单选6)
- ⚠ **统一编址与独立编址**的区别是高频易错点：单总线结构中外设可以和主存统一编址。(单选4)
- ⚠ **"除数为零"属于内部中断（异常）**，不是外部中断或软件中断。(单选10)
- ⚠ **中断向量地址**是"中断服务程序入口地址的地址"，不是入口地址本身。(单选14)
- ⚠ **现场保护**是指保护中断服务子程序中要用的各种寄存器（含标志寄存器）中的内容，不是保护断点地址。(单选12)
- ⚠ **中断标志**的符号是 IF（Interrupt Flag）。(单选13)
- ⚠ **多级中断保存现场**最有效的方法是采用堆栈。(单选11)
- ⚠ **中断隐指令**完成关中断、保护断点、引出中断服务程序；保存通用寄存器内容不属于中断隐指令。(多选5)
- ⚠ **多重中断**：在保护现场时关中断，执行中断处理程序时开中断。(多选6)
- ⚠ **内部异常返回地址**：故障返回原指令重新执行，陷阱返回下一条指令，终止不返回。并非所有内部异常都返回原指令。(多选9)
- ⚠ **不可屏蔽中断（NMI）**：即使在关中断状态下也会被响应，优先级最高。(多选10)
- ⚠ **DMA 特点**：数据传输过程中完全不需要 CPU 的介入（但传输启动和结束仍需 CPU）。(单选16)
- ⚠ **中断 vs DMA**：中断请求的是 CPU 处理，DMA 请求的是总线使用权；中断通过软件传送数据，DMA 通过硬件传送数据；中断适合中低速设备，DMA 适合高速设备。(多选8)
- ⚠ **CPU 对外设控制方式按介入程度从小到大**：通道方式 < DMA 方式 < 中断方式 < 程序方式。(判断10)
- ⚠ **USB 接口**通过使用集线器（HUB），理论上可连接 127 台外设。(判断7)（此知识点源自练习答案，正文未述）

## 六、一句话记忆要点

1. **互连结构**：连接各模块的通路集合，最普遍的是总线结构。
2. **总线定义**：共享传输媒质的公共通信通路，分内部/系统/I/O 三类。
3. **总线特性**：机械、电气、功能、时间四大特性。
4. **总线性能指标**：宽度、传输率、同步/异步、复用、信号线数。
5. **总线按功能分**：数据总线、地址总线、控制总线（加电源线）。
6. **单总线**：简单易扩展但容量小；**双总线**：增设存储总线或 IOP；**中间层**：桥接器连接高速总线与处理器总线。
7. **总线仲裁**：集中式（链式查询—简单但固定优先级；计数器定时—可变优先级；独立请求—最快最灵活）vs 分布式。
8. **总线定时**：同步（时钟、频率高、速度相近）vs 异步（握手、周期可变、快慢混合）。
9. **总线传输模式**：读写、读—修改—写（共享保护）、写后读（校验）、成块传输。
10. **I/O 接口**：不是简单连接器，含通信逻辑；功能包括控制定时、通信、缓冲、检错。
11. **接口 vs 端口**：端口是接口中 CPU 可直接访问的寄存器；接口 = 多个端口 + 控制逻辑。
12. **I/O 端口寻址**：统一编址（访存指令访问端口，但占主存空间）vs 独立编址（独立空间，但需不同指令）。
13. **程序查询**：CPU 主动查状态，踏步等待效率低。
14. **中断方式**：外设主动报告状态，CPU 与外设并行工作；响应条件=有请求+允许中断+指令执行完毕。
15. **中断隐指令**（硬件完成）：保存断点→关中断→引出中断服务程序。
16. **向量中断**：硬件自动形成中断服务程序入口地址；向量地址=入口地址的地址。
17. **多重中断**：用堆栈保存现场；保护/恢复现场时关中断，执行服务程序时开中断；同级不嵌套。
18. **DMA**：纯硬件传送，不需 CPU 干预；三阶段=预处理（CPU）+数据传送（DMA控制器）+后处理（CPU）。
19. **DMA vs 中断**：DMA 请求优先级更高，响应在机器周期结束（非指令结束），数据传送由硬件完成。
20. **DMA 共享内存**：停止CPU访存（简单但CPU空闲）、周期挪用（效率好但需额外逻辑）、交替访问（最高效但硬件复杂）。
21. **通道方式**：执行通道程序，一个通道控制多台设备；4级连接=主机→通道→设备控制器→外设。
22. **通道分类**：字节多路（低速多设备）、选择通道（高速单工作）、数组多路（高速多设备可同时工作但单传输）。
