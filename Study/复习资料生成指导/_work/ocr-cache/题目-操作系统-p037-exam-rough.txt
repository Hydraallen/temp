21. B
解析：A：时间片轮转调度算法。所有作业轮流获得CPU，对CPU繁忙型和I/O繁忙型相对公平，但频繁切换对I/O型有利（可及时响应I/O），并非“不利于I/O型”。B：先来先服务调度算法（FCFS）。CPU繁忙型作业长期占用CPU，I/O繁忙型作业需等待，导致I/O设备空闲，系统吞吐量下降。因此FCFS有利于CPU繁忙型，不利于I/O繁忙型。C：短作业优先算法。短作业（通常也是I/O型）优先，有利于I/O繁忙型，不利于长CPU繁忙型。D：优先权调度算法。取决于优先权设定，若优先权高则有利，无固定偏向。
考点：不同调度算法对CPU繁忙型和I/O繁忙型作业的影响
22. B
解析：单道式运行：作业依次执行，每次只运行一个作业。4个作业同时到达，执行时间均为2小时。周转时间 = 完成时刻 - 到达时刻（同时到达，到达时刻均为0）。作业1：0时刻开始，2时刻完成 → 周转时间 = 2h 作业2：2时刻开始，4时刻完成 → 周转时间 = 4h 作业3：4时刻开始，6时刻完成 → 周转时间 = 6h 作业4：6时刻开始，8时刻完成 → 周转时间 = 8h 平均周转时间 = (2 + 4 + 6 + 8) / 4 = 20 / 4 = 5h
考点：周转时间的计算（单道批处理系统）
23. D
解析：A：先来先服务。这是一种调度算法，不是优先级类型。B：动态优先级。优先级在运行过程中可以改变（如随等待时间或执行时间调整）。C：短作业。这是一种调度算法，且作业长度事先未知时无法确定静态优先级。D：静态优先级。创建进程时确定，运行期间保持不变。
考点：静态优先级与动态优先级的区别
24. B
解析：短作业优先（SJF）：执行时间短的先运行。顺序为：J1（T1）→ J2（T2）→ J3（T3）。同时到达，到达时间=0。J1 周转时间 = T1 J2 周转时间 = T1 + T2 J3 周转时间 = T1 + T2 + T3 平均周转时间 = (T1 + (T1+T2) + (T1+T2+T3)) / 3 = (3T1 + 2T2 + T3) / 3
考点：短作业优先调度算法的周转时间计算
