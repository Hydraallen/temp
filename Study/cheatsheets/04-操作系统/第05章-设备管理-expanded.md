# 操作系统 · 设备管理 — 复习资料

> 来源：4-3-操作系统-讲义.pdf 物理页 [94, 116]
> 说明：⭐=高频考点，⚠=易错点；> Caveat: 为陷阱/技巧；末尾 Worked Examples 为代表性题目（无书内答案的标 [PROPOSED — verify]）。

## 一、本章概览

设备管理是操作系统的四大资源管理功能之一，核心目标是屏蔽 I/O 设备的物理细节、缓和 CPU 与 I/O 设备间速度不匹配的矛盾，并为用户程序提供统一、方便的设备使用接口。本章涵盖七大块：I/O 设备分类与设备控制器、I/O 子系统层次结构、I/O 控制方式（程序查询 / 中断驱动 / DMA / 通道）、I/O 核心子系统（调度 / 高速缓存 / 缓冲区）、设备分配与回收、SPOOLing 技术、磁盘管理（访问时间 / 调度算法 / RAID）。其中 SPOOLing 原理与虚拟设备、I/O 控制方式对比、磁盘调度算法计算、设备独立性是高频考点。*(讲义 p94)*

| 字符设备 | 无结构类型，传输基本单位为字符 | 不可寻址，传输速率低 | 交互式终端、打印机 | 常采用中断驱动方式 |

**按设备的共享属性分** ⭐*(讲义 p94)*

| 类型 | 定义 | 例子 | 特点 |
|---|---|---|---|
| 独占设备 | 一段时间内只允许一个进程访问的设备，即临界资源 | 打印机 | 多个并发进程应互斥访问 |
| 共享设备 | 一段时间内允许多个进程同时访问的设备 | 磁盘 | 每一时刻仍只允许一个进程访问；是文件系统和数据库系统的物质基础 |
| 虚拟设备 | 通过虚拟技术将一台独占设备变换为若干台逻辑设备，供若干个进程同时使用 | [原文未述] | [原文未述] |

> Caveat (第五章-单选-13): 共享设备是指一段时间内可被多个进程同时访问的设备，只有磁盘满足。打印机在同一时间段被多个进程访问会打印错乱；磁带机旋转到读写位置需较长时间，不适合分时共享。

### 2. 设备控制器 ⭐

**定义**：设备控制器位于 CPU 与设备之间，既要与 CPU 通信，又要与设备通信，还具有按照 CPU 发来的命令去控制设备工作的功能。*(讲义 p95)*

**基本功能**（6 项）*(讲义 p94)*

1. 接收和识别命令
2. 数据交换
3. 标识和报告设备的状态
4. 地址识别
5. 数据缓冲区
6. 差错控制

**组成**（3 部分）*(讲义 p95)*

| 组成部分 | 说明 | 关键要素 |
|---|---|---|
| 设备控制器与处理机的接口 | 实现 CPU 与设备控制器之间的通信 | 三类信号线：数据线、地址线、控制线。数据线连接两类寄存器：①数据寄存器（存放输入/输出数据）；②控制/状态寄存器（存放控制信息或设备状态信息） |
| 设备控制器与设备的接口 | 一个控制器可连接一个或多个设备 | 每个接口中有数据、控制和状态三种类型信号；I/O 逻辑根据地址信号选择设备接口 |
| I/O 逻辑 | 实现对设备的控制 | CPU 发送启动命令的同时通过地址线发送地址，I/O 逻辑对地址译码，根据命令对所选设备进行控制 |

> Caveat (第五章-单选-2): 在设备控制器中用于实现设备控制功能的是 I/O 逻辑，而非接口。接口用来传输信号，I/O 逻辑用来实现对设备的控制。

## 三、I/O 子系统层次结构 ⭐

整个 I/O 系统可视为具有 4 个层次的系统结构（不含硬件层），各层次及其功能如下：*(讲义 p96)*

| 层次（自上而下） | 功能 |
|---|---|
| 用户层 I/O 软件 | 产生 I/O 请求，格式化 I/O，Spooling |
| 设备独立性软件 | 映射、保护、分块、缓冲、分配 |
| 设备驱动程序 | 设置设备寄存器；检查状态 |
| 中断处理程序 | 执行 I/O 操作 |
| 硬件 | [最底层，非软件层] |

**各层详解**：*(讲义 p96)*

- **用户层 I/O 软件**：实现与用户交互的接口，用户可直接调用与 I/O 操作有关的库函数对设备进行操作。大部分 I/O 软件在操作系统内部，但仍有一小部分在用户层（库函数及完全运行于内核之外的程序），必须通过一组系统调用来获取操作系统服务。*(讲义 p96)*

- **设备独立性软件** ⭐：设备独立性也称设备无关性，使得应用程序独立于具体使用的物理设备。为此引入了**逻辑设备**和**物理设备**两个概念。在应用程序中，使用**逻辑设备名**来请求设备。该层用于实现用户程序与设备驱动器的统一接口、设备命名、设备的保护以及设备的分配与释放等，同时为设备管理和数据传送提供必要的存储空间。*(讲义 p96)*

> Caveat (第五章-单选-5): 程序员利用系统调用打开 I/O 设备时，通常使用的设备标识是逻辑设备名。用户程序对 I/O 设备的请求采用逻辑设备名，程序实际执行时使用物理设备名。

> Caveat (第五章-单选-7): 将系统调用参数翻译成设备操作命令的工作由设备无关的操作系统软件（设备独立性软件）完成，而非设备驱动程序。设备驱动程序负责执行操作系统发出的 I/O 命令。

> Caveat (第五章-多选-30): 具备设备独立性的系统中，更换物理设备只需更换驱动程序，无须修改应用程序。

- **设备驱动程序**：与硬件直接相关，用于具体实现系统对设备发出的操作指令，驱动 I/O 设备工作。*(讲义 p96)*

> Caveat (第五章-多选-28): 设备驱动程序的功能包括：接收 I/O 命令和参数并检查合法性、查询设备状态、发出 I/O 命令启动设备。数据的分析和缓冲不是设备驱动程序的功能，由进程或操作系统完成。

- **中断处理程序**：用于保存被中断进程的 CPU 环境，转入相应的中断处理程序进行处理，处理完毕再恢复被中断进程的现场后返回。*(讲义 p96)*

## 四、I/O 控制方式 ⭐

对设备的控制，早期使用轮询的可编程 I/O 方式（程序查询方式），后来发展为中断驱动方式、DMA 方式、通道方式。*(讲义 p96)*

### 1. 程序查询方式

处理机向控制器发出 I/O 指令启动输入设备时，同时把状态寄存器中的忙/闲标志 busy 置为 1，然后不断循环测试 busy（称为轮询）。当 busy=1 时表示设备尚未输完，继续测试；当 busy=0 时表明数据已送入控制器的数据寄存器，处理机取出数据送入内存。*(讲义 p96)*

### 2. 中断驱动方式

允许 I/O 设备主动打断 CPU 的运行并请求服务，从而"解放"CPU。CPU 发出读命令后可继续做其他工作，在每个指令周期的末尾检查中断。*(讲义 p96)*

- **I/O 控制器角度**：接收读命令 → 从设备读数据 → 数据进入数据寄存器 → 发中断信号 → 等待 CPU 取数据 → 将数据放数据总线传到 CPU 寄存器。*(讲义 p97)*
- **CPU 角度**：发读命令 → 保存上下文转去执行其他程序 → 指令周期末尾检查中断 → 有中断则保存上下文执行中断处理程序 → 从 I/O 控制器读数据传到寄存器并存入主存 → 恢复上下文继续运行。*(讲义 p97)*

中断驱动方式比程序直接控制方式有效，但由于数据中每个字在存储器与 I/O 控制器之间的传输都必须经过 CPU，仍会消耗较多 CPU 时间。*(讲义 p97)*

### 3. DMA 方式 ⭐

DMA（直接存储器存取）方式的基本思想是在 I/O 设备和内存之间开辟直接的数据交换通路，彻底"解放"CPU。*(讲义 p97)*

**特点**：*(讲义 p97)*

1. 基本单位是数据块。
2. 所传送的数据，从设备直接送入内存，或者相反。
3. 仅在传送一个或多个数据块的开始和结束时，才需 CPU 干预，整块数据的传送是在 DMA 控制器的控制下完成的。

**DMA 控制器的组成**（4 类寄存器） ⭐*(讲义 p98)*

| 寄存器 | 全称 | 功能 |
|---|---|---|
| CR | 命令/状态寄存器 | 接收从 CPU 发来的 I/O 命令或有关控制信息，或设备的状态 |
| MAR | 内存地址寄存器 | 输入时存放设备→内存的目标地址；输出时存放内存→设备的源地址 |
| DR | 数据寄存器 | 暂存从设备到内存或从内存到设备的数据 |
| DC | 数据计数器 | 存放本次要传送的字（节）数 |

**DMA 与中断驱动方式的主要区别** ⭐*(讲义 p98)*

| 对比维度 | 中断驱动方式 | DMA 方式 |
|---|---|---|
| 中断时机 | 每个数据需要传输时中断 CPU | 一批数据全部传送结束时才中断 CPU |
| 数据传送控制者 | 中断处理时由 CPU 控制完成 | DMA 控制器控制完成 |
| 数据路径 | 设备 → CPU 寄存器 → 内存 | 设备 ↔ 内存（直接通路） |

> Caveat (第五章-多选-25): DMA 是一个控制器，不是处理机。DMA 需要 CPU 指出所取数据的地址和长度（存放在 DMA 控制器的寄存器中）。CPU 不是给出 DMA 可以解释的程序，而是发出一条指令并设置寄存器值来启动 DMA。

> Caveat (第五章-单选-19): DMA 适合磁盘等高速块设备与内存间的信息交换，而打印机、键盘、鼠标等低速/字符设备只能以串行方式与存储器沟通。

### 4. 通道方式

I/O 通道是指专门负责输入/输出的处理机。I/O 通道方式是 DMA 方式的发展，可进一步减少 CPU 的干预，把对一个数据块的读（写）为单位的干预减少为对一组数据块的读（写）及有关控制和管理为单位的干预。同时可实现 CPU、通道和 I/O 设备三者的并行操作。*(讲义 p98)*

**通道与一般处理机的区别**：通道指令的类型单一，没有自己的内存，通道所执行的通道程序放在主机的内存中，即通道与 CPU 共享内存。*(讲义 p98)*

**通道与 DMA 方式的区别** ⭐*(讲义 p98)*

| 对比维度 | DMA 方式 | 通道方式 |
|---|---|---|
| 数据块大小/内存位置控制 | 由 CPU 控制 | 由通道控制 |
| 设备控制能力 | 每个 DMA 控制器对应一台设备 | 一个通道可控制多台设备 |

> Caveat (第五章-单选-6): 通道是一种特殊的处理机，属于硬件机制。SPOOLing 技术、缓冲池、内存覆盖都是在内存基础上通过软件实现的。

> Caveat (第五章-判断-37): 通道是通过通道程序来对 I/O 设备进行控制的。

**通道类型**（来自专项练习答案）：通道可分为字节多路通道、数组选择通道、数组多路通道三种。*(第五章-多选-23)*

### 四种 I/O 控制方式总对比 ⭐

| 方式 | 数据传输单位 | CPU 干预程度 | 数据路径 | 主要特点 |
|---|---|---|---|---|
| 程序查询 | 字/字符 | 全程轮询（busy 标志） | 设备→CPU→内存 | CPU 效率最低 |
| 中断驱动 | 字/字符 | 每个字中断一次 | 设备→CPU 寄存器→内存 | CPU 可做其他工作，但频繁中断 |
| DMA | 数据块 | 仅块开始和结束时干预 | 设备↔内存（直接） | DMA 控制器控制，CPU 基本解放 |
| 通道 | 一组数据块 | 通道程序执行期间基本不干预 | 设备↔内存（通道控制） | 专用处理机，可控制多台设备 |

## 五、I/O 核心子系统

### 1. I/O 调度

I/O 调度就是确定一个好的顺序来执行 I/O 请求。操作系统通过为每个设备维护一个请求队列来实现调度。当一个应用程序执行阻塞 I/O 系统调用时，该请求加到相应设备的队列上。I/O 调度会重新安排队列顺序，以改善系统总体效率和应用程序的平均响应时间。*(讲义 p98)*

### 2. 高速缓存

操作系统中使用磁盘高速缓存技术来提高磁盘的 I/O 速度。磁盘高速缓存技术不同于通常意义下的介于 CPU 与内存之间的小容量高速存储器，而是指**利用内存中的存储空间来暂存从磁盘中读出的一系列盘块中的信息**。因此，磁盘高速缓存**逻辑上属于磁盘，物理上则是驻留在内存中的盘块**。*(讲义 p98)*

> Caveat (第五章-单选-14): 在系统内存中设置磁盘高速缓冲区的主要目的是减少磁盘 I/O 次数（匹配磁盘和内存速度差异）。

**引入缓冲区的原因** ⭐*(讲义 p99)*

1. 缓和 CPU 与 I/O 设备间速度不匹配的矛盾。
2. 减少对 CPU 的中断频率，放宽对 CPU 中断响应时间的限制。
3. 解决数据粒度不匹配的问题。
4. 提高 CPU 和 I/O 设备之间的并行性。

> Caveat (第五章-单选-15): CPU 输出数据速度远超打印机速度时，应采用缓冲技术来缓和速度不匹配的矛盾。

> Caveat (第五章-判断-36): 引入缓冲技术的主要目的是提高 CPU 与设备之间的并行程度。

### 3. 缓冲区

**单缓冲** ⭐*(讲义 p99)*

在设备和处理机之间设置一个缓冲区。设备和处理机交换数据时，先把被交换数据写入缓冲区，然后需要数据的设备或处理机从缓冲区取走数据。

设 $T$ 为从磁盘把一块数据输入缓冲区的时间，$M$ 为操作系统将缓冲区中数据传送到用户区的时间，$C$ 为 CPU 对这一块数据处理的时间。*(讲义 p99)*

- 当 $T > C$ 时，整个过程用时 $M + T$。*(讲义 p99)*
- 当 $T < C$ 时，整个过程用时 $M + C$。*(讲义 p99)*

$$\text{单缓冲处理每块数据的用时} = \max(C, T) + M$$

*(讲义 p99)*

**双缓冲** ⭐*(讲义 p99)*

I/O 设备输入数据时先装填缓冲区 1，缓冲区 1 填满后才开始装填缓冲区 2，与此同时处理机可从缓冲区 1 取数据放入用户进程处理。缓冲区 1 数据处理完后，若缓冲区 2 已填满，处理机又从缓冲区 2 取数据处理，而 I/O 设备又可以装填缓冲区 1。注意：必须等缓冲区 2 充满才能让处理机从缓冲区 2 取出数据。*(讲义 p99)*

研究双缓冲处理一块数据的用时，规定初始状态：工作区是空的，其中一个缓冲区是满的，另一个是空的。*(讲义 p99)*

> Caveat (第五章-单选-10): 单缓冲同一时刻只能实现单向传输（但可在一段时间内发送、另一段时间内接收）；双缓冲同一时刻可以实现双向传输。

**缓冲池***(讲义 p99)*

讲义中仅给出标题，未展开描述。据专项练习答案：缓冲池是系统的共用资源，可供多个进程共享，既能用于输入又能用于输出。一般包含三种类型的缓冲：①空闲缓冲区；②装满输入数据的缓冲区；③装满输出数据的缓冲区。为管理方便，可将相同类型的缓冲区链成一个队列。*(第五章-单选-9)*

> Caveat (第五章-单选-9): 为了使多个并发进程能有效地进行输入和输出，最好采用缓冲池结构的缓冲技术（循环缓冲、单缓冲、双缓冲属于专用缓冲）。

> Caveat (第五章-单选-11): 缓冲技术的缓冲池在主存中。

> Caveat (第五章-多选-26): 缓冲区是一个存储区域，可由专门的硬件寄存器组成，也可利用内存来实现；可用于输入设备和输出设备；也可用于块设备和字符设备。

## 六、设备分配与回收

### 1. 设备分配的总原则 ⭐

充分发挥设备的使用效率，尽可能地让设备忙碌，又要避免由于不合理的分配方法造成进程死锁。*(讲义 p100)*

### 2. 设备的三种使用方式

| 使用方式 | 说明 | 例子 |
|---|---|---|
| 独占式使用设备 | 申请设备时若空闲则独占，不允许其他进程申请，直到释放 | 打印机 |
| 分时式共享使用设备 | 各进程的每次 I/O 操作请求通过分时交替进行 | 磁盘 |
| 以 SPOOLing 方式使用外部设备 | 对 I/O 操作进行批处理，实质是以空间换时间 | [原文未述] |

*(讲义 p100)*

> Caveat (第五章-单选-4): SPOOLing 技术是一种以空间换时间的技术；请求分页系统中的页面调度算法是以时间换空间的技术。

### 3. 设备分配的数据结构

讲义正文此处内容缺失，据专项练习答案：进行设备分配时所需的数据结构有设备控制表、控制器控制表、通道控制表、系统设备表等。*(第五章-多选-21)*

> Caveat (第五章-单选-3): 设备映射表（DMT）的作用是建立逻辑设备与物理设备的对应关系。

### 4. 设备分配策略

**分配方式** ⭐*(讲义 p101)*

| 分配方式 | 适用设备 | 时机 | 特点 |
|---|---|---|---|
| 静态分配 | 独占设备 | 作业开始执行前，一次性分配全部设备 | 不会出现死锁，但设备使用效率低 |
| 动态分配 | 共享设备（独占设备也可） | 进程执行过程中按需分配，用完立即释放 | 有利于提高设备利用率，但可能造成死锁 |

**分配算法**：常用的动态设备分配算法有先请求先分配、优先级高者优先等。*(讲义 p101)*

> Caveat (第五章-判断-34): 时间片分配是进程调度的常用算法，不是设备分配算法。设备分配常用算法主要有先来先服务和最高优先级优先。

> Caveat (第五章-多选-27): 独占设备往往采用静态分配方式；共享设备往往采用动态分配方式（每个时刻只有一个作业使用）。

### 5. 设备分配的安全性 ⚠

| 安全性方式 | 机制 | 优点 | 缺点 |
|---|---|---|---|
| 安全分配方式 | 进程发出 I/O 请求后便阻塞，直到 I/O 完成才被唤醒 | 设备分配安全 | CPU 和 I/O 设备串行工作（对同一进程而言） |
| 不安全分配方式 | 进程发出 I/O 请求后继续运行，可发多个 I/O 请求 | 一个进程可同时操作多个设备，迅速推进 | 有可能产生死锁 |

*(讲义 p102)*

## 七、SPOOLing 技术 ⭐

### 1. 相关概念

为了缓和 CPU 的高速性与 I/O 设备低速性间的矛盾，引入了脱机输入、脱机输出技术。该技术利用专门的外围控制器，先将低速 I/O 设备上的数据传送到高速磁盘上，或者相反。*(讲义 p102)*

外围操作与 CPU 对数据的处理同时进行，在**联机**情况下实现的同时外围操作称为 SPOOLing，或称为**假脱机操作**。*(讲义 p102)*

> Caveat (第五章-判断-38): SPOOLing 系统是假脱机、真联机技术，不是脱机 I/O 系统。

### 2. SPOOLing 的组成 ⭐

| 组成部分 | 位置 | 功能 |
|---|---|---|
| 输入井和输出井 | 磁盘上 | 输入井模拟脱机输入时的磁盘，收容 I/O 设备输入的数据；输出井模拟脱机输出时的磁盘，收容用户程序的输出数据。数据以文件形式组织（井文件），链接成输入/输出队列 |
| 输入缓冲区和输出缓冲区 | 内存中 | 缓和 CPU 和磁盘之间速度不匹配的矛盾。输入缓冲区暂存输入设备→输入井的数据；输出缓冲区暂存输出井→输出设备的数据 |
| 输入进程和输出进程 | [原文未述] | 输入进程（预输入进程）模拟脱机输入时的外围控制器；输出进程（缓输出进程）模拟脱机输出时的外围控制器 |
| 井管理程序 | [原文未述] | 控制作业与磁盘井之间信息的交换 |

*(讲义 p102)*

> Caveat (第五章-多选-24): SPOOLing 系统的主要组成部分是输入井和输出井、输入缓冲区和输出缓冲区、输入进程和输出进程。

> Caveat (第五章-判断-35): 实现 SPOOLing 系统时，必须在磁盘上开辟出称为输入井和输出井的专门区域。

> Caveat (第五章-多选-29): SPOOLing 技术需要外存支持，建立在多道程序设计基础上，可让多个作业共享一台独占式设备。设备与输入/输出井之间的数据传送由操作系统实现，不是由用户作业控制。

### 3. SPOOLing 的特点 ⭐

1. **提高了 I/O 的速度**：对数据的 I/O 操作从对低速 I/O 设备演变为对磁盘缓冲区中数据的存取。*(讲义 p103)*
2. **将独占设备改造为共享设备**：实际上并没有为任何进程分配设备，而只是在磁盘缓冲区中为进程分配一个空闲块和建立一张 I/O 请求表。*(讲义 p103)*
3. **实现了虚拟设备功能**：宏观上虽然是多时使用一台独占设备，而每个进程都认为自己独占了一个设备（逻辑设备）。*(讲义 p103)*

> Caveat (第五章-单选-12): SPOOLing 技术实现将一台物理 I/O 设备虚拟为多台逻辑 I/O 设备，即实现设备的虚拟分配。

> Caveat (第五章-单选-16): 在 SPOOLing 系统中，用户进程实际分配到的是外存区（虚拟设备），而非物理设备。

> Caveat (第五章-判断-39): 虚拟设备是指把一个物理设备变换成多个对应的逻辑设备。

> Caveat (第五章-判断-40): 虚拟设备是通过 SPOOLing 技术实现的，不是通过缓冲技术实现的。

### 4. 假脱机打印机系统

打印机属于独占设备，利用假脱机技术可将它改造为可供多个用户共享的打印设备。假脱机打印系统主要有以下三部分：*(讲义 p103)*

1. **磁盘缓冲区**：在磁盘上开辟的存储空间，暂存用户程序的输出数据，可设置空盘块队列、满盘块队列等。*(讲义 p103)*
2. **打印缓冲区**：设置在内存中，暂存从磁盘缓冲区送来的数据，以后再传送给打印设备。*(讲义 p103)*
3. **假脱机管理进程和假脱机打印进程**：管理进程为每个要求打印的用户数据建立假脱机文件并放入队列；打印进程依次对队列中的文件进行打印。*(讲义 p103)*

## 八、磁盘管理

### 1. 磁盘访问时间 ⭐

一次磁盘服务的总时间为寻道时间、旋转延迟时间和传输时间三者之和。*(讲义 p103)*

**（1）寻道时间 $T_s$***(讲义 p103)*

把磁臂（磁头）移动到指定磁道所经历的时间，是启动磁臂的时间 $s$ 与磁头移动 $n$ 条磁道所花费的时间之和：

$$T_s = m \times n + s$$

其中，$m$ 是常数，与磁盘驱动器的速度有关（一般磁盘 $m=0.2$，高速磁盘 $m \le 0.1$）；$s$ 为磁臂启动时间，约为 2ms。一般温盘寻道时间大体上为 5～30ms。*(讲义 p103)*

**（2）旋转延迟时间 $T_r$***(讲义 p103)*

指定扇区移动到磁头下面所经历的时间。如硬盘旋转速度为 15000r/min，每转需时 4ms，平均旋转延迟时间 $T_r$ 为 2ms；软盘旋转速度为 300r/min 或 600r/min，平均 $T_r$ 为 50～100ms。*(讲义 p103)*

**（3）传输时间 $T_t$***(讲义 p103)*

把数据从磁盘读出或向磁盘写入数据所经历的时间，与每次所读/写的字节数 $b$ 和旋转速度有关：

$$T_t = \frac{b}{rN}$$

其中，$r$ 为磁盘每秒钟的转数；$N$ 为一条磁道上的字节数。*(讲义 p103)*

当一次读/写的字节数相当于半条磁道上的字节数时，$T_r$ 与 $T_t$ 相同 [原文模糊，OCR 原文为"$T_t$ 与 $T_t$ 相同"，疑为 $T_r$ 与 $T_t$]。*(讲义 p103)*

**总访问时间** ⭐*(讲义 p103)*

$$T_a = T_s + \frac{1}{2r} + \frac{b}{rN}$$

其中 $\frac{1}{2r}$ 为平均旋转延迟时间。*(讲义 p103)*

> Caveat (第五章-单选-1): 磁盘调度的目的是缩短寻道时间。调度算法直接决定寻道时间，从而决定总的存取时间。

> Caveat (第五章-判断-32): 旋转延迟的大小与文件的物理结构有关（连续分配、链接分配和索引分配影响物理地址是否连续，进而影响旋转延迟）。

### 2. 磁盘调度算法 ⭐

**（1）FCFS（先来先服务）***(讲义 p104)*

根据进程请求访问磁盘的先后次序进行调度。优点是公平、简单，不会出现某一进程的请求长期得不到满足的情况；缺点是平均寻道时间可能较长。

**（2）SSTF（最短寻道时间优先）** ⭐⚠*(讲义 p104)*

要求访问的磁道与当前磁头所在磁道距离最近，以使每次的寻道时间最短。不能保证平均寻道时间最短，并且可能导致某个进程发生"饥饿"现象。

> Caveat (第五章-单选-18): SSTF 容易使磁头一直在某个磁盘区域来回移动，远离该区域的进程请求可能长期得不到满足，即引起饥饿。

**（3）SCAN（电梯调度算法）** ⭐*(讲义 p104)*

不仅考虑将要访问的磁道与当前磁道间的距离，更优先考虑磁头当前的移动方向。磁头从磁盘的一端出发向另一端移动，遇到所需磁道就进行服务，一旦当前方向上没有请求了，磁头移动方向就反过来继续服务。避免了"饥饿"现象。磁头移动规律类似电梯运行，故常称为电梯调度算法。

**（4）CSCAN（循环扫描）***(讲义 p104)*

规定磁头单向移动。当磁头移到最外的磁道并访问后，立即返回到最里的欲访问磁道，即将最小磁道号紧接着最大磁道号构成循环，进行循环扫描。

**四种磁盘调度算法对比** ⭐

| 算法 | 原理 | 优点 | 缺点 | 是否会饥饿 |
|---|---|---|---|---|
| FCFS | 按请求先后次序服务 | 公平、简单 | 平均寻道时间可能较长 | 否 |
| SSTF | 选距当前磁头最近的磁道 | 每次寻道时间短 | 不能保证平均最短；可能导致饥饿 | 是 |
| SCAN | 优先考虑磁头移动方向，到底反向 | 避免饥饿 | [原文未述] | 否 |
| CSCAN | 单向移动到底，立即返回另一端循环扫描 | [原文未述] | [原文未述] | 否 |

*(讲义 p104)*

> Caveat (第五章-单选-17): 电梯调度、FCFS、CLOCK 算法都可以是磁盘调度算法，而银行家算法是死锁避免算法，存在根本不同。

> Caveat (第五章-单选-20): 时间片轮转调度算法和优先级高者优先算法是进程调度算法；LRU 是页面淘汰算法；最短寻找时间优先算法是磁盘调度算法。

### 3. 磁盘阵列（RAID）

| 级别 | 冗余/校验方式 | 特点 | 利用率/成本 |
|---|---|---|---|
| RAID0 | 无冗余 | 并行交叉存取，缩短 I/O 请求排队时间 | 成本低，无容错能力 |
| RAID1 | 磁盘镜像 | 读请求拥有最短寻道时间 | 磁盘容量利用率仅 50% |
| RAID2 | 海明校验码 | 数据字/字节方式交叉存放，并行存取 | 成本较高（多个冗余盘） |
| RAID3 | 奇偶校验盘 | RAID2 的简化版，并行传输 | 如 7 盘时利用率 6/7 |
| RAID4 | 奇偶校验盘（独立存取） | 数据条带交叉存放，访问请求可并行满足 | [原文未述] |
| RAID5 | 分散校验块 | 校验块分散到所有数据盘块，独立传送，各磁盘可独立读/写 | [原文未述] |
| RAID6 | 双重冗余 | 可容忍两个数据磁盘错误，设置专用异步校验盘 | 价格昂贵，改进有限 |
| RAID7 | 同步传送 | 所有 I/O 传送同步进行，每盘带高速缓存，可分别控制 | 目前最高档次的磁盘阵列 |

*(讲义 p104-p105)*

## 九、公式 / 算法 / 定理速查表

| 名称 | 公式 | 说明 |
|---|---|---|
| 寻道时间 | $T_s = m \times n + s$ | $m$：常数；$n$：移动磁道数；$s$：启动时间 |

## 十、高频考点与易错提醒

1. **磁盘调度目的 = 缩短寻道时间**（不是旋转延迟/传输时间） ⭐*(第五章-单选-1)*
2. **SSTF 可能引起饥饿**；FCFS、SCAN、CSCAN 不会 ⚠*(第五章-单选-18)*
3. **SPOOLing 是以空间换时间**；请求分页页面调度是以时间换空间 ⚠*(第五章-单选-4)*
4. **SPOOLing 是假脱机、真联机**，不是脱机 I/O 系统 ⚠*(第五章-判断-38)*
5. **虚拟设备通过 SPOOLing 技术实现**，不是通过缓冲技术 ⚠*(第五章-判断-40)*
6. **用户程序通过系统调用使用外部设备**，不是通过 I/O 指令直接使用 ⚠*(第五章-判断-31)*
7. **DMA 是控制器不是处理机**；需要 CPU 指出数据地址和长度 ⚠*(第五章-多选-25)*
8. **DMA 适合磁盘**等高速块设备，不适合键盘/鼠标/打印机 ⭐*(第五章-单选-19)*
9. **通道是硬件机制**（特殊处理机）；SPOOLing、缓冲池、内存覆盖是软件实现 ⭐*(第五章-单选-6)*
10. **系统调用用逻辑设备名**；设备无关软件负责翻译成设备操作命令 ⭐*(第五章-单选-5, 第五章-单选-7)*
11. **缓冲池在主存中**；缓冲区可用内存实现，可用于输入/输出、块设备/字符设备 ⚠*(第五章-单选-11, 第五章-多选-26)*
12. **双缓冲可实现双向传输**，单缓冲只能单向 ⭐*(第五章-单选-10)*
13. **磁盘是共享设备**但每个时刻最多一个作业访问 ⭐*(第五章-判断-33)*
14. **独占设备→静态分配；共享设备→动态分配** ⚠*(第五章-多选-27)*
15. **设备驱动程序不做数据分析和缓冲**（由进程或 OS 完成） ⚠*(第五章-多选-28)*
16. **更换物理设备不需修改应用程序**（设备独立性） ⚠*(第五章-多选-30)*
17. **旋转延迟与文件物理结构有关** ⚠*(第五章-判断-32)*

## 十一、一句话记忆要点

- **I/O 控制方式**：程序查询（CPU 全程轮询）→ 中断驱动（每字中断）→ DMA（块传输，直接内存）→ 通道（一组块，专用处理机），CPU 干预递减。*(讲义 p96-p98)*
- **DMA 四寄存器**：CR（命令/状态）、MAR（内存地址）、DR（数据）、DC（数据计数器）。*(讲义 p98)*
- **单缓冲用时**：$\max(C,T)+M$，记住取较大值再加传送时间。*(讲义 p99)*
- **SPOOLing**：假脱机、真联机；输入井/输出井在磁盘，输入缓冲区/输出缓冲区在内存；以空间换时间；将独占设备改造为共享设备，实现虚拟设备。*(讲义 p102-p103)*
- **磁盘访问时间**：$T_a = T_s + \frac{1}{2r} + \frac{b}{rN}$，调度算法只影响 $T_s$（寻道时间）。*(讲义 p103)*
- **磁盘调度**：FCFS 公平但慢；SSTF 最短但会饥饿；SCAN 电梯双向；CSCAN 单向循环。*(讲义 p104)*
- **设备独立性**：逻辑设备名请求 → 设备无关软件翻译 → 驱动程序执行；换设备不改程序。*(讲义 p96)*

## Worked Examples

### 例 1：SSTF 磁盘调度移动距离计算

*Tags: SSTF, 磁盘调度算法*

**问题**：某个磁盘系统采用最短寻道时间优先（SSTF）磁盘调度算法，请求柱面读写的磁盘请求队列如下：27、136、58、100、72、40，当前磁头位置是 80 柱面。求磁盘总的移动距离。*(第五章-单选-8)*

**解答**：

1. SSTF 要求每次选择距当前磁头最近的柱面服务。*(讲义 p104)*
2. 当前磁头在 80，逐一选择最近柱面：
   - 80 → 72：移动 $|80-72|=8$
   - 72 → 58：移动 $|72-58|=14$
   - 58 → 40：移动 $|58-40|=18$
   - 40 → 27：移动 $|40-27|=13$
   - 27 → 100：移动 $|27-100|=73$（此时 27 最近的是 100，因为 136 距 27 为 109）
   - 100 → 136：移动 $|100-136|=36$
3. 总移动距离 $= 8 + 14 + 18 + 13 + 73 + 36 = 162$。
4. 答案为 **D. 162**。*(第五章-单选-8)*

### 例 2：SCAN 磁盘调度服务顺序

*Tags: SCAN, 电梯调度算法*

**问题**：磁盘请求队列为 98、183、37、122、14、124、65、67，磁头最初在 53 道，磁头正向 0 道方向移动。采用 SCAN 算法，写出服务顺序。*(讲义 p104)*

**解答**：

1. SCAN 算法优先考虑磁头当前的移动方向，在当前方向上依次服务，直到该方向上没有请求再反向。*(讲义 p104)*

   - 依次服务 65、67、98、122、124、183 道。
