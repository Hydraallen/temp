# 操作系统 · 内存管理 — 复习资料

> 来源：4-3-操作系统-讲义.pdf 物理页 [70, 93]
> 说明：⭐=高频考点，⚠=易错点；> Caveat: 为陷阱/技巧；末尾 Worked Examples 为代表性题目（无书内答案的标 [PROPOSED — verify]）。

## 一、本章概览
本章解决计算机系统中至关重要的“内存资源分配与管理”问题。内容涵盖从底层多层次存储器体系架构，到程序的编译链接与装入机制，再到各类连续与非连续的内存分配方案（重点为分页与分段管理）。在此之上，引出基于局部性原理的虚拟存储器技术，深入探讨请求分页系统的实现机制（缺页中断、页表机制）及各类页面置换算法，最后剖析内存分配策略及“抖动”现象的预防。本章是操作系统的核心高频考区。

## 二、存储器概述与程序装入

**多层次存储系统**：计算机存储层次至少应具有三级（CPU寄存器、主存、辅存），高档计算机可细分为6层。层次越高（越靠近CPU），访问速度越快，价格越高，容量越小。
- 寄存器、高速缓存、主存、磁盘缓存属于OS存储管理范畴（掉电易失）；磁盘、可移动介质属于设备管理范畴。

**程序的装入步骤**：用户程序运行需先装入内存，通常经过：编译（形成目标模块） → 链接（形成装入模块） → 装入（装入内存）。

**程序的装入方式对比** ⭐

| 装入方式 | 适用环境 | 地址变换时机/特点 | 特性 |
| :--- | :--- | :--- | :--- |
| 绝对装入 | 单道程序环境 | 编译时给出绝对地址，无需修改 | 逻辑地址与物理地址完全相同 |
| 可重定位装入(静态重定位) | 多道程序环境 | 装入时一次完成地址变换 | 作业运行期间不能在内存中移动，不能再申请内存 |
| 动态运行时装入(动态重定位) | 多道程序环境 | 真正执行时才进行地址变换 | 需重定位寄存器支持；可分配到不连续区域；可部分装入；便于共享 |

> Caveat (题目21): 动态重定位系统中，系统仅设置一个重定位寄存器（在进程切换时保存和恢复），而非为每个进程分配一个物理的重定位寄存器。

**程序的链接方式**：
1. 静态链接：运行前链接成完整可执行程序，不再拆开。
2. 装入时动态链接：装入内存时边装入边链接。
3. 运行时动态链接：执行中需要目标模块时才链接。优点是便于修改、更新和共享。

**覆盖与交换技术**（扩充内存方法）：
- **覆盖**：用于同一程序/进程中。将用户空间分固定区和覆盖区，常驻部分放固定区，按调用关系分段放覆盖区，在需要前调入。对用户不透明（需程序员给出结构）。
- **交换（对换）**：用于不同进程/作业之间。把等待状态的程序从内存移至辅存（换出），把准备好竞争CPU的程序移入内存（换入）。现代OS中仍广泛使用。

## 三、内存分配管理 ⭐

### 1. 连续分配管理方式

**单一连续分配**：内存分系统区和用户区，无外部碎片，有内部碎片。只能用于单用户、单任务OS。
**固定分区分配**：最简单的多道程序存储管理。划分为固定大小分区，允许并发。分“大小相等”和“大小不等”。会产生内部碎片，无外部碎片。

**动态分区分配（可变分区）** ⭐：根据进程实际需要动态分配。无内部碎片，有外部碎片。

**动态分区分配算法**：

| 算法名称 | 空闲分区链接/排序原则 | 分配查找方式 | 特点 |
| :--- | :--- | :--- | :--- |
| 首次适应算法(First-fit) | 按地址递增 | 找满足要求的第一个 | 简单，无需对空闲区链排序 |
| 最佳适应算法(Best-fit) | 按容量递增 | 找满足要求的第一个 | 产生许多难以利用的小碎片 |
| 最坏适应算法(Worst-fit) | 按容量递减 | 找满足要求的第一个 | 对大进程不利 |
| 邻近适应算法(Next-fit) | 按地址递增(循环) | 从上次查找结束位置继续 | |
> Caveat (题目13): 首次适应算法不需要对空闲区链进行排序，只需按地址递增的顺序链接即可。

**动态可重定位分区分配（紧凑）** ⚠：将小分区合并为大分区以装入程序。紧凑会变更程序存放位置，需挂起进程，且需要重定位寄存器支持。

### 2. 非连续分配管理方式

**基本分页管理方式** ⭐
- **页面与物理块**：进程逻辑空间分页，内存物理空间分块（页和块等大）。最后一页常装不满形成“页内碎片”（内部碎片）。
- **页面大小**：适中（通常 $1\text{KB} - 8\text{KB}$），且必须为 $2$ 的幂。太小导致页表过长；太大导致页内碎片大。
- **地址结构**：32位地址分为页号(P)和页内偏移量(W)。0~11位为页内地址（每页 $4\text{KB}$），12~31位为页号（最多 $2^{20}$ 页）。由硬件直接实现。
- **页表**：记录页面在内存中对应的物理块号，存放在内存中。注意区分：页表项第二部分是**物理块号**，地址第二部分是**页内偏移**。

**基本分页地址变换机构** ⭐
- 进程未执行时，页表始址和长度存放在PCB中；被调度时，装入**页表寄存器(PTR)**。
- 基本地址变换机构访存：CPU每存取一个数据，需访问**2次**内存（第1次查页表形成物理地址，第2次获取数据）。
> Caveat (题目3): 仅有基本地址变换机构的分页系统中，单次存取需访问2次内存。

**具有快表(TLB)的地址变换机构** ⭐
- 快表(联想寄存器)存放当前访问的页表项，容量通常 $16\sim 512$ 表项。命中率可达 $90\%$ 以上。
> Caveat (题目18/题目44-T18): 访存次数计算规则：若TLB命中，仅需1次访存（直接获取数据）；若未命中，需访问多级页表，最后还要加1次访存获取数据。不可漏掉最后取数据的1次。

**基本分段管理方式**
- **目的**：考虑用户和程序员需要，满足方便编程、信息保护和共享、动态链接及增长。
- **分段**：按逻辑信息划分（如主程序段MAIN、子程序段X）。段长不等，逻辑地址为**二维特性**（段号 + 段内地址）。例如段号16位，段内地址16位，最多 $64\text{K}$ 个段，段最大 $64\text{KB}$。
- **段表**：记录段号、段长、本段在主存的始址。
- **共享与保护**：通过两个作业段表项指向同一个物理副本实现共享。不能修改的代码称为**纯代码或可重入代码**，可以共享。

**基本分页与基本分段区别** ⭐

| 对比维度 | 分页存储管理 | 分段存储管理 |
| :--- | :--- | :--- |
| 目的与单位 | 信息的物理单位（系统行为），提高内存利用率 | 信息的逻辑单位（用户行为），满足用户需要 |
| 大小 | 页大小固定，由系统决定 | 段长度不固定，决定于用户程序 |
| 地址空间 | 一维（单一线性地址） | 二维（需给出段名和段内地址） |
| 共享 | 难以实现 | 易于实现（如纯代码段共享） |

## 四、虚拟存储与请求分页 ⭐

**局部性原理** ⭐：虚拟内存的实现基础。
1. **时间局部性**：某指令被执行或数据被访问后，不久可能再次执行/访问（典型原因：大量循环操作）。
2. **空间局部性**：访问某存储单元后，不久其附近的单元也将被访问（典型原因：程序顺序执行）。

**虚拟存储器定义与特征** ⭐
- 定义：具有请求调入和置换功能，从逻辑上扩充内存容量的存储器系统。实际不存在，对用户透明。
- 特征：多次性（分多次调入）、对换性（换进换出）、虚拟性（逻辑扩充）。
> Caveat (题目19): 虚拟存储器的最大实际容量受限于 $\min(\text{计算机地址结构}, \text{内存容量} + \text{外存容量})$。并不是内外存简单相加。

**请求分页管理方式** ⭐
- **页表机制**：在基本分页基础上，页表项增加4个字段。完整结构：`页号 | 物理块号 | 状态位P | 访问字段A | 修改位M | 外存地址`。
  - *状态位 P*：指示是否在内存。决定是否缺页。
  - *访问字段 A*：记录访问次数或未访问时间，供置换参考。
  - *修改位 M*：调入后是否被修改过（换出时决定是否写回外存）。
  - *外存地址*：通常是物理块号，供调入参考。
  > Caveat (题目40): 决定是否发生缺页故障的是状态位（有效位），而非修改位。

**缺页中断机制** ⚠
当访问页不在内存时产生缺页中断。它与一般中断的区别：
1. 缺页中断属于**内部中断**，在指令执行期间产生和处理（非指令执行完后）。
2. 一条指令在执行期间可能产生**多次缺页中断**。

## 五、页面置换算法与抖动 ⭐

**页面置换算法**：内存无空闲块时，选择换出页面的算法。不适当的算法会导致“抖动”。

| 算法 | 原理 / 策略 | 优缺点与特性 |
| :--- | :--- | :--- |
| OPT (最佳置换) | 淘汰以后永不使用或最长时间不再被访问的页 | 缺页率最低，用于评价其他算法，**无法实现** |
| FIFO (先进先出) | 淘汰在内存中驻留时间最久的页 | 实现简单，性能差。会出现 **Belady异常** |
| LRU (最近最久未使用) | 淘汰最近最长时间未访问过的页 | 性能好，需寄存器和栈硬件支持。不会出现 Belady 异常 |
| CLOCK (简单时钟/NRU) | 循环扫描缓冲区，淘汰使用位(访问位)为0的页 | 开销小，性能接近LRU |

- **Belady 异常** ⚠：进程分配的物理块数增大，缺页次数不减反增的异常现象。仅 **FIFO算法** 会出现，LRU和OPT绝不会出现。

**改进型 CLOCK 算法** ⭐：在使用位($u$)基础上增加修改位($m$)，每帧有4种情况，优先淘汰 $u=0, m=0$ 的页面。

**页面分配策略与抖动** ⭐
- **抖动**：进程频繁发生缺页中断，大部分时间用于页面置换而非执行指令。主要原因是频繁访问的页面数高于可用物理页帧数。
- **预防方法**：
  1. 采取局部置换策略（限制抖动影响范围）。
  2. 把工作集算法融入处理机调度（调入新作业前确保驻留页足够）。
  3. 利用“$L=S$”准则（缺页平均时间 $L$ 接近缺页服务时间 $S$ 时，利用率最大）。
  4. 暂停部分进程（优先级最低的进程调出到磁盘）。
- **工作集**：某段时间间隔 $\Delta$（窗口尺寸）里，进程实际访问页面的集合。它是窗口尺寸 $\Delta$ 的非降函数。

## 记号约定
- $u.size$ = 请求的分区大小
- $m.size$ = 表中空闲分区大小
- $\Delta$ = 工作集窗口尺寸
- $L$ = 缺页之间的平均时间
- $S$ = 平均缺页服务时间
- $u$ / $m$ = 访问位 / 修改位

## Worked Examples

### 例 1：TLB与多级页表平均访存次数计算 *(来自 exam-signals 候选 3)*
*Tags: 多级页表, TLB, 访存次数*
**问题**：配置TLB的页式虚拟存储系统，TLB命中率为 $75\%$，忽略访问TLB时间，使用二级页表，求每次存取的平均访存次数。
**解答**：`[PROPOSED — verify]`
1. 分析命中情况：若TLB命中，直接得到物理地址，仅需 $1$ 次访存读取目标数据。
2. 分析未命中情况：若TLB未命中，需先访存访问一级页表（1次），再访问二级页表（1次），最后按物理地址读取数据（1次），共 $3$ 次。
3. 计算平均值公式： $$\text{平均访存次数} = \text{TLB命中率} \times 1 + (1 - \text{TLB命中率}) \times (\text{访存页表次数} + 1)$$
4. 代入数据计算： $$0.75 \times 1 + (1 - 0.75) \times 3 = 0.75 + 0.75 = 1.5 \text{ 次}$$
5. 结论：每次存取的平均访存次数为 $1.5$ 次。

### 例 2：动态分区分配算法特性判断 *(本章概念综合应用)*
*Tags: 动态分区分配, 分配算法*
**问题**：在内存分区分配存储管理中，不需要对空闲区链进行排序的分配算法是？并提供依据。
**解答**：`[PROPOSED — verify]`
1. 考查各算法对数据结构的排序要求：
   - 最佳适应法按容量递增链接。
   - 最坏适应法按容量递减链接。
2. **首次适应算法**不需要排序，它只需要将空闲分区**按地址递增**的顺序链接即可。
3. 分配时从链首开始顺序查找，找到第一个满足大小要求的空闲分区进行分配。余下部分留在链中。
4. 结论：选首次适应算法。
