11. ABC  
解析：A 选项：错误。通用操作系统通过分时或并发调度分配 CPU 时间，用户程序无需手动预可运行时间。  
B 选项：错误。操作系统无需直接控制硬件电路的信号时序（如晶体管开关、总线信号等），这些由硬件本身或底层固件处理。C 选项：错误。编译器是开发工具链的一部分，属于应用软件范畴，操作系统核心功能是提供运行环境（如进程调度、内存管理），而非直接提供编译服务。D 选项：正确。资源管理（如 CPU、内存、设备等）是操作系统的核心职能。  
考点：操作系统的定位：区分操作系统核心功能（资源管理、抽象硬件）与外围工具（编译器、调试器）。  
12. ACD  
解析：A. 正确。用户程序通过系统调用命令（如高级语言中的库函数）发起请求，编译器会将其转换为包含参数和陷入指令（Trap）的机器代码。陷入指令用于主动触发特权级切换，使 CPU 从用户态进入内核态。  
B. 错误。屏蔽中断指令是操作系统内核使用的特权指令，用户程序无法直接执行，且与系统调用的机制无关。系统调用依赖陷入指令而非屏蔽中断。  
C. 正确。创建新进程（如 fork( )）是典型的系统调用功能，用户程序必须通过操作系统提供的接口（如 API 或库函数）间接调用内核服务，无法直接操作进程管理结构。  
D. 正确。系统调用完成后，操作系统需通过特权级切换将 CPU 从内核态返回用户态，确保用户程序继续执行且无法直接访问内核数据。  
考点：1）系统调用的机制：用户态到内核态的切换（通过陷入指令）、参数传递方式、特权级管理。2）进程管理：用户程序与操作系统的交互必须通过系统调用接口，不能绕过内核直接操作硬件或内核数据结构。3）状态转换：系统调用是主动触发特权级切换的典型场景，完成后需恢复用户态以保证系统安全。
