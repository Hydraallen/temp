知识点 12：实体（Entity）
客观存在并可相互区别的事务。
知识点 13：属性（Attribute）
实体所具有的某一特性。
知识点 14：码（Key）
唯一标识实体的属性集。
知识点 15：实体型（Entity Type）
用实体名及其属性名集合来抽象和刻画同类实体。
知识点 16：实体集（Entity Set）
同一类型实体的集合。
知识点 17：联系（Relationship）
实体内部的联系；实体各个属性之间的联系；实体之间的联系；不同实体集之间的联系。
知识点 18：实体-联系方法（Entity-Relationship Approach）
用 E-R 图来描述现实世界的概念模型。E-R 方法也称为 E-R 模型。
知识点 19：数据模型的三要素
数据结构、数据操作、数据的完整性约束条件。
知识点 20：常用的数据模型
层次模型、网状模型、关系模型。当前的主要模型为关系模型。
知识点 21：关系模型的数据结构
从用户观点看，关系模型由一组关系组成。每个关系的数据结构是一张规范化的二维表。
知识点 22：关系（Relation）
一个关系对应通常说的一张表。
知识点 23：元组（Tuple）
表中的一行即为一个元组。
知识点 24：属性（Attribute）
表中的一列即为一个属性，给每一个属性起一个名称即属性名。
知识点 25：码（Key）
码也称为码键。表中的某个属性组，可以唯一确定一个元组。
知识点 26：域（Domain）
域是一组具有相同数据类型的值的集合。
知识点 27：分量
元组中的一个属性值。
术语对比
| 关系术语 | 一般表格的术语 |
| 关系名 | 表名 |
| 关系模式 | 表头（表格的描述） |
| 关系 | （一张）二维表 |
| 元组 | 记录或行 |
| 属性 | 列 |
| 属性名 | 列名 |
| 属性值 | 列值 |
| 分量 | 一条记录中的一个列值 |
| 非规范关系 | 表中有表（大表中嵌有小表） |
知识点 28：关系模型的优点
（1）关系模型与格式化模型不同，它是建立在严格的数学概念的基础上的。（2）关系模型的概念单一。（3）关系模型的存取路径对用户透明，从而具有更高的数据独立性、更好的安全保密性，也简化了程序员的工作和数据库开发建立的工作。
