# 计算机网络 · 第四章 网络层 — 复习资料

> 来源：3-5-计算机网络讲义.pdf 物理页 [66, 89]（印刷页 64–87）
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与书末统一答案标注）。

## 一、本章概览
网络层是 OSI 参考模型中的第三层，介于传输层和数据链路层之间。它在数据链路层提供的相邻端点数据帧传送功能上，进一步管理网络中的数据通信，将数据设法从源端经过若干个中间节点传送到目的端，从而向运输层提供最基本的端到端的数据传送服务。如果用尽量少的词来记住网络层，那就是“路径选择、路由及逻辑寻址”。本章主要解决两大争论（网络层应向运输层提供怎样的服务、可靠交付由谁负责），并详细阐述了两种服务（虚电路与数据报）、IP 数据报格式、IPv4 地址规划（分类、子网划分、无类别编址 CIDR）以及下一代 IP 协议 IPv6 的特点与过渡技术。

## 二、知识点详解

### 1. 网络层的概念与两种服务争论 ⭐ (练习)
网络层在数据链路层提供的两个相邻端点之间的数据帧的传送功能上，进一步管理网络中的数据通信。网络层存在两种争论（观点）：
- **观点一：面向连接的可靠交付（虚电路）**。计算机网络模仿电信网络，通信之前先建立虚电路 VC (Virtual Circuit) 以保证双方通信所需的一切网络资源。虚电路只是一条逻辑上的连接，分组都沿着这条逻辑连接按照存储转发方式传送，并不是真正建立了一条物理连接。如果再使用可靠传输的网络协议，可使所发送的分组无差错按序到达终点，不丢失、不重复。
- **观点二：无连接的、尽最大努力交付的数据报服务（因特网设计思路）**。网络层要设计得尽量简单，向其上层只提供简单灵活的、无连接的、尽最大努力交付的数据报服务。网络在发送分组时不需要先建立连接。每一个分组（即 IP 数据报）独立发送，与其前后的分组无关（不进行编号）。网络层不提供服务质量的承诺（可能出错、丢失、重复和失序，不保证时限）。由主机中的运输层负责可靠的通信。好处是使网络的造价大大降低，运行方式灵活，路由器可以做得比较简单且价格低廉。

### 2. 网络层的功能
网络层的目的是实现两个端系统之间的数据透明传送，具体功能包括寻址和路由选择、连接的建立、保持和终止等。它提供的服务使传输层不需要了解网络中的数据传输和交换技术。
- 关键词：“路径选择、路由及逻辑寻址”。
- OSI 模型中面向数据通信的下三层（通信子网）中最为复杂也最关键的一层。

### 3. 虚电路服务与数据报服务对比 ⭐ (练习)
- **虚电路服务**：可靠通信应当由网络来保证。必须有连接建立阶段。终点地址仅在连接建立阶段使用，每个分组使用短的虚电路号。属于同一条虚电路的分组均按照同一路由进行转发。当结点出故障时，所有通过出故障的结点的虚电路均不能工作。总是按发送顺序到达终点。端到端的差错处理和流量控制可以由网络负责，也可以由用户主机负责。
- **数据报服务**：可靠通信应当由用户主机来保证。不需要连接建立。每个分组都有终点的完整地址。每个分组独立选择路由进行转发。故障结点可能会丢失分组，一些路由可能会发生变化。到达终点时不一定按发送顺序。由用户主机负责差错处理和流量控制。

### 4. 路由选择及其分类
- **路由选择**：通信子网为网络源节点和目的节点提供了多条传输路径，网络节点在收到一个分组后，要确定向下一节点传送的路径。在数据报方式中，节点要为每个分组路由做出选择；在虚电路方式中，只需在连接建立时确定路由。
- **路由算法**：确定路由选择的策略。分为静态路由和动态路由。
- **理想路由算法特点**：正确和完整、计算上简单、能适应通信量和网络拓扑的变化（自适应性）、具有稳定性、公平的、最佳的。

### 5. 拥塞控制
- **拥塞**：当通信子网中有太多的分组时，网络吞吐量将随流入网络的分组的增加而下降，网络的性能会明显变差，甚至导致通信业务停顿（死锁）。
- **基本方法**：开环控制和闭环控制。
  - **开环控制**：在设计网络时事先将有关发生拥塞的因素考虑周到，力求网络在工作时不产生拥塞。
  - **闭环控制**：基于反馈环路的概念。措施包括：监测网络系统以检测拥塞何时何地发生；将拥塞发生的信息传送到可采取行动的地方；调整网络系统的运行以解决问题。
- **拥塞具体处理方法**：分组删除、流量控制、缓冲分配、抑制分组。
- **死锁两种类型**：存储转发死锁、重装死锁。

### 6. IP 数据报格式 ⭐ (练习)
IP 数据包也叫 IP 报文分组，由 IP 报文头和 IP 报文用户数据组组成。IP 报文头的长度一般在 20 到 60 个字节之间，一个 IP 分组的最大长度不能超过 65535 个字节。报文头的前 20 个字节是固定的，后面的可变。
- **版本**：占 4 位(bit)，指 IP 协议的版本号（如 IPv4），通信双方必须一致。
- **首部长度**：占 4 位(bit)，最大的长度为 15 个长度单位，每个长度单位为 4 字节。
- **服务类型(TOS)**：占 8 位(bit)，前 3 位表示报文的优先级，后面几位表示更低时延、更高吞吐量等要求。
- **总长度**：占 16 位(bit)，单位为字节。最大长度为 65535 字节。
- **标识**：标记当前分片为第几个分片，在数据报重组时有用。
- **标志**：标记该报文是否为分片，后面是否还有分片。
- **片偏移**：指当前分片在原数据报中相对于用户数据字段的偏移量。
- **生存时间(TTL)**：表明当前报文还能生存多久。每经过 1ms 或者一个网关，TTL 的值自动减 1，当为 0 时被丢弃。
- **协议**：指出上层使用的协议（如 UDP、TCP、ICMP、IGMP、IGP 等）。
- **首部校验和**：用于检验 IP 报文头部在传播中是否出错。
- **源 IP 地址 / 目的 IP 地址**：各占 32 位(bit)，4 个字节。

### 7. IP 网与 IP 地址基础 ⭐ (练习)
接入因特网的设备必须有一个全局地址，在因特网上是统一的和唯一的，且与具体的网络无关，即 **IP 地址（逻辑地址）**。MAC 地址与具体的网络有关，称为物理地址。
- IPv4 地址长度为 32 位，最多有 $2^{32}$ 个不同地址。IPv6 采用 128 位地址。
- **书写格式**：
  - 二进制格式：如 `11000000 10101000 00001010 00000001`（每 8 比特间留空格）。
  - 点分十进制格式：把 32 比特分成 4 字节，每个字节用十进制书写，加点分隔，如 `193.131.27.255`。

### 8. IPv4 有类别地址 ⭐ (练习)
IP 地址主管机构把 IP 地址分为 A、B、C、D、E 五个类别。用 4 字节地址的第 1 字节（高位）的前几个比特来区分：
- **A 类**：最高位为 `0`。第 1 字节十进制范围：0–127。
- **B 类**：最高位为 `10`。第 1 字节范围：128–191。
- **C 类**：最高位为 `110`。第 1 字节范围：192–223。
- **D 类**：最高位为 `1110`。第 1 字节范围：224–239。（用作多播/组播地址，一对多通信）
- **E 类**：最高位为 `1111`。第 1 字节范围：240–255。（保留以备后用）

**地址的层次结构**：A、B、C 类地址由网络标识 和主机标识 两个层次组成，区别在于长度划分不同：
- A 类：7 位网络标识，共有 $2^7 = 128$ 个网络；24 位主机标识，单网可有 $2^{24} = 16777216$ 台主机。
- B 类：14 位网络标识，有 $2^{14} = 16384$ 个网络；16 位主机标识，单网最多 $2^{16} = 65536$ 台主机。
- C 类：21 位网络标识，有 $2^{21} = 2097152$ 个网络；8 位主机标识，单网最多 $2^8 = 256$ 台主机。

**IPv4 有类别地址的缺点**：IP 地址满足不了因特网急剧增长的需要（地址不够分）。原因：发展迅速超出预料；地址分类不合理（A 类过大，C 类过小），造成地址浪费。
缓解办法：使用无类别地址、使用地址转换技术、使用动态地址分配方案。

### 9. 子网与子网掩码 ⭐ (练习)
通过划分子网，在内部增加一个层次，地址变成 3 层结构（网络标识、子网标识、主机标识）。
- **子网掩码**：长度为 32 比特，代表了“网络标识+子网标识”与主机标识的划分方案，用于屏蔽 IP 地址的一部分以区别网络标识和主机标识。
- **默认掩码**：A 类 `255.0.0.0`，B 类 `255.255.0.0`，C 类 `255.255.255.0`。
- **作用**：
  1. 便于网络设备区分本网段和非本网段：将 IP 地址与子网掩码进行逻辑与操作，结果相同则视为同一网络。
  2. 将子网进一步划分，缩小地址空间，便于网络管理。

### 10. IPv4 无类别地址与 CIDR ⭐ (练习)
无类别地址分配方案的特点是：需要多大地址就分配多大地址块，避免浪费。地址块有三个要求：
1. 地址块内的地址是连续的。
2. 地址的总数是 2 的幂（1, 2, 4, 8...）。
3. 块内容的第 1 个地址能被地址数整除。
- **CIDR (无类别域间路由)**：掩码中 1 的个数可以为任意，用 `/n` 表示左边 $n$ 位均为 1，右侧 $(32-n)$ 为 0。简写为 `x.y.z.t/n`。
- **计算方法**：
  - 地址块的大小为 $2^{32-n}$。
  - 首地址：将 `x.y.z.t` 的右边 $(32-n)$ 个比特均置为 0。
  - 末地址：将 `x.y.z.t` 的右边 $(32-n)$ 个比特均置为 1。
- **地址会聚**：因特网名字与地址指派公司 ICANN 将大地址块分给 ISP，ISP 再分成小块给客户。反之，从小块逐级聚集成大块的过程称为地址会聚。

### 11. IPv6 ⭐ (练习)
IPv6 是 IETF 设计的用于替代现行版本 IPv4 的下一代 IP 协议。
- **特点**：
  1. 具有更大的地址空间：长度为 128 位，有 $2^{128}-1$ 个地址。
  2. 使用更小的路由表：遵循聚类 原则，提高了转发速度。
  3. 增加增强的组播支持及对流的支持，为 QoS 提供良好平台。
  4. 加入对自动配置 的支持。
  5. 具有更高的安全性：对网络层数据进行加密并校验。
- **IPv6 安全性问题**：虽然病毒和蠕虫难以通过扫描地址段传播（因为地址空间太大犹如大海捞针），但基于应用层的病毒（电子邮件病毒）仍会存在。针对关键主机（如 DNS 服务器、DHCP 服务器）的攻击（如 DDos）、报文侦听、应用层攻击、中间人攻击、洪水攻击等依然存在。
- **寻址的文本表示形式**：
  1. 冒号十六进制形式：`n:n:n:n:n:n:n:n`（8 个 16 位元素）。
  2. 压缩形式：连续的 0 块由双冒号 `::` 表示（在地址中只能出现一次）。
  3. 混合形式：`n:n:n:n:n:n:d.d.d.d`。
- **IPv6 地址类型**：
  1. 单播地址：高序位八位字节非 `FF`。包含链路本地地址（`FE80::InterfaceID`）、站点本地地址（`FEC0::SubnetID:InterfaceID`）、全局单播地址。
  2. 多路广播地址：高序位八位字节为十六进制 `FF`。
  3. 任一广播地址：取自单播地址空间，发送到此地址的数据包传递给最近的一个接口。

### 12. IPv4 向 IPv6 过渡技术 ⭐ (练习)
1. **双栈策略**：网元中同时具有 IPv4 和 IPv6 两个协议栈，既能处理 IPv4 也能处理 IPv6 分组。
2. **隧道策略**：利用一种协议来传输另一种协议的数据技术。隧道端点通常都是双栈节点。在入口封装，在出口解封装。包括手工配置隧道和自动配置隧道（6to4 等）。

## 三、公式 / 算法 / 定理速查表

- **IP 分组最大长度**：65535 字节。
- **IPv4 地址总数**：$2^{32}$ 个。
- **IPv6 地址总数**：$2^{128}-1$ 个。
- **A 类主机数**：$2^{24} = 16777216$。
- **B 类主机数**：$2^{16} = 65536$。
- **C 类主机数**：$2^8 = 256$。
- **CIDR 地址块大小计算**：大小为 $2^{32-n}$（其中 $n$ 为前缀长度）。
- **CIDR 首地址算法**：将 IP 地址的右边 $(32-n)$ 个比特均置为 0。
- **CIDR 末地址算法**：将 IP 地址的右边 $(32-n)$ 个比特均置为 1。

## 四、重点对比与易混辨析

| 对比的方面 | 虚电路服务 | 数据报服务 |
| :--- | :--- | :--- |
| **思路** | 可靠通信应当由网络来保证 | 可靠通信应当由用户主机来保证 |
| **连接的建立** | 必须有 | 不需要 |
| **终点地址** | 仅在连接建立阶段使用，分组用短的虚电路号 | 每个分组都有终点的完整地址 |
| **分组的转发** | 按照同一路由进行转发 | 每个分组独立选择路由进行转发 |
| **结点出故障时** | 虚电路均不能工作 | 丢失分组，路由发生变化 |
| **分组的顺序** | 按发送顺序到达终点 | 不一定按发送顺序 |
| **差错处理和流量控制**| 可以由网络负责，也可以由用户主机负责 | 由用户主机负责 |

| 地址分类 | 第 1 字节最高位前缀 | 十进制范围（第 1 字节） | 用途 |
| :--- | :--- | :--- | :--- |
| **A 类** | `0` | 0 - 127 | 单播 |
| **B 类** | `10` | 128 - 191 | 单播 |
| **C 类** | `110` | 192 - 223 | 单播 |
| **D 类** | `1110` | 224 - 239 | 多播/组播 |
| **E 类** | `1111` | 240 - 255 | 保留 |

## 六、一句话记忆要点
- **网络层概念**：网络层只管路径选择、路由及逻辑寻址，提供端到端传送。
- **虚电路 vs 数据报**：虚电路靠网络保证顺序，同路走；数据报靠主机保证可靠，独立走。
- **拥塞控制**：开环事先防，闭环靠反馈，死锁分存储转发和重装两类。
- **IP 数据报格式**：TTL 过网关减 1，总长 65535，首部校验只查头部。
- **IP 分类与子网**：A 主多网少，C 网多主少；CIDR 网络前缀任意划，`/n` 定大小，`0` 全首 `1` 全末。
- **IPv6**：128 位长地址，`::`只能压一次，双栈隧道两过渡。
