## 2. 反复出现的题型模式
- **多进制/编码数值大小比较**：题干给出二进制、八进制、十进制、十六进制、BCD码混合的选项求最值 → 解题切入点：全部统一转换为十进制进行对比 `[PROPOSED — verify]`。
- **寻址方式速度与访存次数排序**：题干问最快/最慢/不需访存的寻址方式 → 解题切入点：立即寻址最快且不访存，间接寻址最慢（多次访存），直接寻址居中（一次访存） `[PROPOSED — verify]`。
- **PC值自动增量计算**：题干给出机器字长、编址方式、指令长度，求取指令后PC的值 → 解题切入点：先看编址单位（按字编址加1，按字节编址加指令字节数），双字长指令加2倍单字长 `[PROPOSED — verify]`。
- **存储器芯片数量计算**：题干给出总存储容量和单片芯片容量，求所需芯片数或组数 → 解题切入点：总容量除以单片容量得总片数，位扩展看数据线位数，字扩展看地址线位数 `[PROPOSED — verify]`。
- **IEEE754浮点数真值求解**：题干给出32位十六进制浮点数格式，求其十进制数值 → 解题切入点：十六进制转二进制，提取1位符号、8位阶码、23位尾数，阶码减127得指数，尾数补隐藏的1求小数 `[PROPOSED — verify]`。
- **相对寻址转移地址偏移量计算**：题干给出当前PC值、目标地址、指令长度，求偏移量 → 解题切入点：偏移量 = 目标地址 - (当前PC + 指令字节长度)，注意PC在取指后已自增 `[PROPOSED — verify]`。

## 3. 技巧 / 易错点
- > Caveat (第一章单选2): 存储单位换算中 1KB = 1024B，而非 1000B；1MB = 1024KB。
- > Caveat (第二章单选16): 8位补码的表示范围是 -128 ~ +127，负数比正数多表示一个，不要错记为 -127 ~ +127。
- > Caveat (第二章判断8、13): 浮点加减法对阶必须是小阶向大阶看齐，小阶尾数右移，切勿大阶向小阶看齐。
- > Caveat (第三章单选20-22): PC自动增量取决于存储器编址方式（按字还是按字节）以及指令长度，按字编址取单字长指令PC加1，按字节编址取双字长指令PC加4。
- > Caveat (第四章单选8): 取指令操作是控制器固有的硬件功能，不受当前或上一条指令的操作码控制。
- > Caveat (第四章单选15): 指令寄存器（IR）对用户完全透明；PC和状态寄存器用户不可写但可读；通用寄存器对用户不透明。
- > Caveat (第四章单选21、25): 单总线结构的CPU中，ALU不能将输入和输出端同时连到总线，必须通过暂存器锁存输入数据以防总线冲突。
- > Caveat (第五章单选24-25): DRAM读出是破坏性的，读后需立即重写；刷新是按行进行的，不是按位或按列。
- > Caveat (第六章单选14): 中断向量是中断服务程序入口地址的指示器（即存放入口地址的内存单元地址），并非入口地址本身。
- > Caveat (第六章判断13): DMA方式传送数据时，CPU并非完全不能访问主存，常采用周期挪用（窃取）使CPU与DMA交替访存。
- > Caveat (第七章单选1、13、20): 超线程技术（SMT）是在一个物理核心内模拟多个逻辑核心，共享运算器等资源，属于线程级并行，并非多核技术，性能不等于两个CPU之和。

## 5. Worked Example 候选
### 候选 1：第一章单选20·MIPS指标计算 `[PROPOSED — verify]`
- 概念 tag：CPI，MIPS，主频
- 题干（精简）：某计算机主频 1.2 GHz，指令分为 4 类，所占比例及CPI分别为：A(50%, 2), B(20%, 3), C(10%, 4), D(20%, 5)。求该机的 MIPS 数。
- 解题骨架（3-8 步）：
  1. 明确公式：MIPS = 主频 / 平均CPI。
  2. 计算平均CPI = Σ(指令比例 × 该指令CPI)。
  3. 代入数据：CPI = 2×0.5 + 3×0.2 + 4×0.1 + 5×0.2 = 1 + 0.6 + 0.4 + 1.0 = 3。
  4. 转换主频单位：1.2 GHz = 1200 MHz。
  5. 计算MIPS = 1200 / 3 = 400。

### 候选 2：第二章单选17·IEEE754单精度浮点数转换 `[PROPOSED — verify]`
- 概念 tag：IEEE754，阶码，尾数
- 题干（精简）：采用IEEE754单精度浮点数格式表示一个数为 45100000H，求该数的值。
- 解题骨架（3-8 步）：
  1. 十六进制转二进制：45100000H = 0100 0101 0001 0000 0000 0000 0000 0000。
  2. 提取各段：第1位符号位 S=0；后8位阶码 E=10001010B = 138D；后23位尾数 M=001000...0。
  3. 计算阶码真值：e = E - 127 = 138 - 127 = 11。
  4. 计算尾数真值：1.M = 1.001B = 1 + 1×2^(-3) = 1.125。
  5. 组合结果：(-1)^S × 1.M × 2^e = +1.125 × 2^11。

### 候选 3：第三章单选18·相对寻址转移指令偏移量计算 `[PROPOSED — verify]`
- 概念 tag：相对寻址，PC自增，补码偏移量
- 题干（精简）：相对寻址转移指令占2字节（操作码+偏移量），CPU每取一字节PC加1。当前PC为2000H，需转移到2008H，求第二字节偏移量内容。
- 解题骨架（3-8 步）：
  1. 确定指令长度：2字节，因此取指结束后PC自增了2。
  2. 计算取指后PC值：2000H + 2H = 2002H。
  3. 计算目标地址与当前PC的差值：2008H - 2002H = 06H。
  4. 偏移量用补码表示，06H的正数补码即为其本身，得出第二字节内容为06H。

### 候选 4：第五章单选12·内存编址范围计算 `[PROPOSED — verify]`
- 概念 tag：按字节编址，地址范围，十六进制运算
- 题干（精简）：内存按字节编址，地址从 A4000H 到 CBFFFH，共有多少个字节？
- 解题骨架（3-8 步）：
  1. 明确存储容量计算公式：末地址 - 首地址 + 1。
  2. 十六进制减法：CBFFFH - A4000H = 27FFFH。
  3. 加1得到字节数：28000H。
  4. 转换为十进制或KB表示：28000H = 2×16^4 + 8×16^3 = 131072 + 32768 = 163840 Bytes = 160 KB。
