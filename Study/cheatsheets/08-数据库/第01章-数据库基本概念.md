# 数据库 · 第01章 数据库基本概念 — 复习资料

> 来源：8-3-数据库计算机类讲义.pdf 物理页 [3, 13]（印刷页 p1–p11）
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与答案标注）。

## 一、本章概览

本章是数据库课程的入门章，解决"数据库是什么、由什么组成、如何组织数据"这一基础问题。全章围绕四大块展开：① 数据库相关基本概念（数据、DB、DBMS、DBS）；② 数据管理技术的演进（人工管理→文件系统→数据库系统）；③ 数据模型（概念模型、E-R 方法、关系模型术语与优缺点）；④ 数据库系统的体系结构（三级模式、两级映像与数据独立性）及系统组成（硬件、软件、人员）。本章是后续章节（关系代数、SQL、数据库设计等）的概念基石。

## 二、知识点详解

### 1. 数据 (Data) ⭐(练习1)

- **定义**：描述事物的符号记录称为数据。
- **要点**：描述事物的符号可以是数字也可以是文字、图形、图像、音频、视频等，数据有多种表现形式，它们都可以经过数字化后存入计算机。

### 2. 数据库 (DataBase, DB) ⭐(练习1)

- **定义**：数据库是长期存储在计算机内、有组织的、可共享的大量数据的集合。
- **要点**：数据库中的数据按一定的数据模型组织、描述和存储，具有较小的冗余度、较高的数据独立性和易扩展性，并可为各种用户共享。

### 3. 数据库管理系统 (DataBase Management System, DBMS) ⭐(练习2,3)

- **定义**：数据库管理系统是位于用户与操作系统之间的一层数据管理软件。数据库管理系统和操作系统一样是计算机的基础软件，也是一个大型复杂的软件系统。
- **主要功能**：
  1. 数据定义功能
  2. 数据组织、存储和管理
  3. 数据操纵功能
  4. 数据库的事务管理和运行管理
  5. 数据库的建立和维护功能
  6. 其他功能（包括数据库管理系统与网络中其他软件系统的通信功能，一个数据库管理系统与另一个数据库管理系统或文件系统的数据转换功能，异构数据库之间的互访和互操作功能等）

### 4. 数据库系统 (DataBase System, DBS) ⭐(练习3)

- **定义**：数据库系统是由数据库、数据库管理系统、应用程序、数据库管理员组成的存储、管理、处理和维护数据的系统。
- **层次结构**（自上而下）：
  - 用户 / 用户 / … / 用户
  - 应用系统
  - 应用开发工具
  - 数据库管理系统 / 数据库管理员
  - 操作系统
  - 数据库

### 5. 数据管理技术的三个阶段 ⭐(练习4,5)

- **三个阶段**：人工管理阶段、文件系统阶段、数据库系统阶段。

### 6. 人工管理的特点 ⭐(练习4)

- 数据不保存
- 应用程序管理数据
- 数据不共享
- 数据不具有独立性

### 7. 文件系统的特点 ⚠(练习5)

- 数据可以长期保存
- 由文件系统管理数据
- **缺点**：数据共享性差，冗余度大；数据独立性差

### 8. 数据库系统的特点 ⭐(练习4,6)

- 数据结构化
- 数据的共享度高、冗余度低且易扩充
- 数据独立性高
- 数据由 DBMS 统一管理和控制

### 9. DBMS 必须提供的数据控制功能 ⭐(练习3)

1. 数据的安全性保护
2. 数据的完整性检查
3. 并发控制
4. 数据库恢复

### 10. 数据模型 (Data Model) ⭐(练习11)

- **定义**：数据模型是对现实世界数据特征的抽象。精确地描述了系统的静态特征、动态特性、数据约束条件。
- **地位**：是数据库系统的核心和基础。

### 11. 数据模型分类 ⭐(练习10)

- 第一类：概念模型
- 第二类：逻辑模型和物理模型

### 12. 实体 (Entity)

- **定义**：客观存在并可相互区别的事物 [原文为"事务"]。

### 13. 属性 (Attribute)

- **定义**：实体所具有的某一特性。

### 14. 码 (Key) ⚠(练习6)

- **定义**：唯一标识实体的属性集。
- **要点**：码可以是一个属性，也可以是多个属性的组合。

### 15. 实体型 (Entity Type)

- **定义**：用实体名及其属性名集合来抽象和刻画同类实体。

### 16. 实体集 (Entity Set)

- **定义**：同一类型实体的集合。

### 17. 联系 (Relationship)

- 实体内部的联系
- 实体各个属性之间的联系
- 实体之间的联系
- 不同实体集之间的联系

### 18. 实体-联系方法 (Entity-Relationship Approach) ⭐(练习9,10)

- **定义**：用 E-R 图来描述现实世界的概念模型。E-R 方法也称为 E-R 模型。
- **要点**：E-R 图用于描述现实世界的概念模型，是数据库设计人员和用户之间沟通信息的桥梁。

### 19. 数据模型的三要素 ⭐(练习7)

1. 数据结构
2. 数据操作
3. 数据的完整性约束条件

### 20. 常用的数据模型 ⭐(练习7)

- 层次模型、网状模型、关系模型。
- 当前的主要模型为关系模型。

### 21. 关系模型的数据结构 ⭐(练习13)

- 从用户观点看，关系模型由一组关系组成。每个关系的数据结构是一张规范化的二维表。

### 22. 关系 (Relation)

- **定义**：一个关系对应通常说的一张表。

### 23. 元组 (Tuple)

- **定义**：表中的一行即为一个元组。

### 24. 属性 (Attribute)（关系模型中）

- **定义**：表中的一列即为一个属性，给每一个属性起一个名称即属性名。

### 25. 码 (Key)（关系模型中）⚠(练习6)

- **定义**：码也称为码键。表中的某个属性组，可以唯一确定一个元组。

### 26. 域 (Domain)

- **定义**：域是一组具有相同数据类型的值的集合。

### 27. 分量

- **定义**：元组中的一个属性值。

### 28. 关系模型的优点 ⭐(练习8)

1. 关系模型与格式化模型不同，它是建立在严格的数学概念的基础上的。
2. 关系模型的概念单一。
3. 关系模型的存取路径对用户透明，从而具有更高的数据独立性、更好的安全保密性，也简化了程序员的工作和数据库开发建立的工作。

### 29. 关系模型的缺点

- 存取路径对用户透明，查询效率往往不如格式化数据模型，为提高性能，必须对用户的查询请求进行优化，增加了开发数据库管理系统的难度。

### 30. 数据库系统的结构 ⭐(练习8)

- **从数据库应用开发人员角度看**：数据库系统通常采用三级模式结构，这是数据库系统内部的系统结构。
- **从数据库最终用户角度看**：数据库系统的结构分为单用户结构、主从式结构、分布式结构、客户-服务器、浏览器-服务器结构等。这是数据库系统外部的体系结构。

### 31. 型和值

- **型**：是指对某一类数据的结构和属性的说明。
- **值**：是型的一个具体赋值。

### 32. 三级模式 ⭐(练习8)

- 数据库系统的三级模式结构是指数据库系统是由外模式、模式和内模式三级构成。

### 33. 模式 ⭐(练习9)

- **定义**：数据库数据在逻辑级上的视图。
- **要点**：一个数据库只有一个模式。

### 34. 外模式 ⭐(练习9)

- **定义**：也称子模式或用户模式，是数据库用户（包括应用程序员和最终用户）能够看见和使用的局部数据的逻辑结构和特征的描述，是数据库用户的数据视图，是与某一应用有关的数据的逻辑表示。
- **要点**：外模式通常是模式的子集。一个数据库可以有多个外模式。

### 35. 内模式 ⭐(练习14)

- **定义**：也称存储模式，一个数据库只有一个内模式。它是数据物理结构和存储方式的描述，是数据在数据库内部的组织方式。

### 36. 数据库的两级映像 ⭐(练习15)

- 外模式/模式映像
- 模式/内模式映像

### 37. 数据的逻辑独立性（外模式/模式映像）⚠(练习16,19)

- **定义**：当模式改变时（例如增加新的关系、新的属性、改变属性的数据类型等），由数据库管理员对各个外模式/模式的映像作相应改变，可以使外模式保持不变。应用程序是依据数据的外模式编写的，从而应用程序不必修改，保证了数据与程序的逻辑独立性，简称数据的逻辑独立性。

### 38. 数据的物理独立性（模式/内模式映像）⚠(练习17,18)

- **定义**：当数据库的存储结构改变时（例如选用了另一种存储结构），由数据库管理员对模式/内模式映像作相应改变，可以使模式保持不变，从而应用程序也不必改变。保证了数据与程序的物理独立性，简称数据的物理独立性。

### 39. 数据库系统的硬件 ⭐(练习20)

1. 要有足够大的内存，存放操作系统、数据库管理系统的核心模块、数据缓冲区和应用程序。
2. 有足够大的磁盘或磁盘阵列等设备存放数据库，有足够大的磁带（或光盘）作数据备份。
3. 要求系统有较高的通道能力，以提高数据传送率。

### 40. 数据库系统的软件 ⭐(练习10)

1. 数据库管理系统。
2. 支持数据库管理系统运行的操作系统。
3. 具有与数据库接口的高级语言及其编译系统，便于开发应用程序。
4. 以数据库管理系统为核心的应用开发工具。
5. 为特定应用环境开发的数据库应用系统。

### 41. 人员 ⭐(练习10)

- 主要包括：数据库管理员、系统分析员和数据库设计人员、应用程序员和最终用户。

### 42. 数据库管理员 DBA

- **背景**：在数据库系统环境下有两类共享资源，一类是数据库，另一类是数据库管理系统软件。因此需要有专门的管理机构来监督和管理数据库系统。数据库管理员则是这个机构的一个（组）人员，负责全面管理和控制数据库系统。
- **职责**：
  - 决定数据库中的信息内容和结构
  - 决定数据库的存储结构和存取策略
  - 定义数据的安全性要求和完整性约束条件
  - 监控数据库的使用和运行
  - 数据库的改进和重组、重构

| 特征 | 人工管理 | 文件系统 | 数据库系统 |

| 数据是否保存 | 数据不保存 | 数据可以长期保存 | 长期存储在计算机内 |
| 数据管理方式 | 应用程序管理数据 | 由文件系统管理数据 | 由 DBMS 统一管理和控制 |
| 数据共享性 | 数据不共享 | 数据共享性差 | 数据的共享度高 |
| 冗余度 | [原文未述] | 冗余度大 | 冗余度低 |
| 数据独立性 | 数据不具有独立性 | 数据独立性差 | 数据独立性高 |
| 数据结构化 | [原文未述] | [原文未述] | 数据结构化 |

### 4.2 关系术语 vs 一般表格术语

| 关系术语 | 一般表格的术语 |
|----------|----------------|
| 关系名 | 表名 |
| 关系模式 | 表头（表格的描述） |
| 关系 | （一张）二维表 |
| 元组 | 记录或行 |
| 属性 | 列 |
| 属性名 | 列名 |
| 属性值 | 列值 |
| 分量 | 一条记录中的一个列值 |
| 非规范关系 | 表中有表（大表中嵌有小表） |

### 4.3 三级模式对比

| 模式 | 别名 | 数量 | 描述内容 |
|------|------|------|----------|
| 模式 | 逻辑模式 | 一个数据库只有一个 | 数据库数据在逻辑级上的视图 |
| 外模式 | 子模式 / 用户模式 | 一个数据库可以有多个 | 用户能看见和使用的局部数据的逻辑结构和特征 |
| 内模式 | 存储模式 | 一个数据库只有一个 | 数据物理结构和存储方式的描述 |

### 4.4 两级映像与数据独立性

| 映像 | 当什么改变时 | 保持什么不变 | 保证的独立性 |
|------|-------------|-------------|-------------|
| 外模式/模式映像 | 模式改变（如增加新关系、新属性、改变属性数据类型等） | 外模式不变 → 应用程序不变 | 数据的逻辑独立性 |
| 模式/内模式映像 | 存储结构改变（如选用另一种存储结构） | 模式不变 → 应用程序不变 | 数据的物理独立性 |

## 五、高频考点与易错提醒

> 以下考点信号来自本章专项练习与答案解析，仅列方向，不收录题目。

1. ⚠ **数据库的定义**是高频易错点——注意"有组织的、可共享的"两个关键词 (练习1)。
2. ⚠ **DBMS 的位置**——位于用户与操作系统之间，不是操作系统与硬件之间 (练习2)。
3. ⚠ **DB、DBMS、DBS 三者关系**——DBS 包括 DB 和 DBMS，不能搞反包含关系 (练习3)。
4. ⚠ **数据库系统区别于文件系统的核心特点**——数据结构化（不是"数据可共享"或"数据可长期保存"，后者文件系统也有） (练习4)。
5. ⚠ **文件系统的特点**——数据可以长期保存（不是"不可长期保存"），缺点是共享性差、冗余度大、独立性差 (练习5)。
6. ⚠ **数据库系统特点**——冗余度低（不是"冗余度高"） (练习6)。
7. ⚠ **当前主要数据模型**——关系模型（不是层次模型） (练习7)。
8. ⚠ **关系模型存取路径对用户透明**——程序员不需熟悉存取路径 (练习8)。
9. ⚠ **E-R 图用于设计概念模型**——不是用于设计关系模型 (练习9)。
10. ⚠ **E-R 图是设计人员和用户沟通的桥梁**——对应概念模型 (练习10)。
11. ⚠ **数据模型是数据库系统的核心和基础** (练习11)。
12. ⚠ **数据库中存储的是数据及数据之间的联系**——不仅仅是数据本身 (练习12)。
13. ⚠ **关系模型用规范化的二维表表示**——不是任意二维表，每一项不可再分 (练习13)。
14. ⚠ **内模式给出物理存储结构与存取方法** (练习14)。
15. ⚠ **二级映像的主要作用是提高数据独立性** (练习15)。
16. ⚠ **数据独立性的体现**——不会因系统数据存储结构与数据逻辑结构的变化而影响应用程序 (练习16)。
17. ⚠ **物理独立性**——物理结构改变时不影响整体逻辑结构、用户逻辑结构及应用程序 (练习17)。
18. ⚠ **磁盘存储结构改变而基本表和视图无需更改**——属于物理独立性 (练习18)。
19. ⚠ **重建视图实现数据的逻辑独立性**——视图属于外模式，外模式/模式映像体现逻辑独立性 (练习19)。
20. ⚠ **数据库系统硬件要有足够大的内存**——存放 OS、DBMS 核心模块、数据缓冲区和应用程序 (练习20)。
21. ⚠ **码可以是一个属性或多个属性的组合**——不是只能由一个字段组成 (练习6)。
22. ⚠ **关系中元组和属性的顺序均可任意交换**——行和列都没有先后顺序 (判断题8)。
23. ⚠ **数据不限于文字或数字**——可以是图形、图像、音频、视频等 (判断题1)。
24. ⚠ **数据库系统软件不包括防病毒软件** (多选题10)。
25. ⚠ **一个数据库只能有一个模式**——逻辑模式的基础是概念模式 (多选题9)。

## 六、一句话记忆要点

1. **数据/DB/DBMS/DBS**：数据是符号记录；DB 是有组织可共享的数据集合；DBMS 是用户与 OS 之间的数据管理软件；DBS = DB + DBMS + 应用程序 + DBA。
2. **三阶段演进**：人工管理（不保存/不共享/无独立性）→ 文件系统（可保存但共享差/冗余大/独立性差）→ 数据库系统（结构化/共享高/冗余低/独立性高/DBMS 统一管理）。
3. **DBMS 四大控制功能**：安全性保护、完整性检查、并发控制、数据库恢复。
4. **数据模型**：是 DBS 的核心和基础；三要素 = 数据结构 + 数据操作 + 完整性约束条件；分概念模型与逻辑/物理模型。
5. **E-R 图**：描述现实世界的概念模型，是设计人员与用户的沟通桥梁。
6. **关系模型**：规范化二维表；存取路径透明；优点是数学基础严格、概念单一、独立性好；缺点是查询效率可能不如格式化模型。
7. **三级模式**：模式（唯一，逻辑级视图）、外模式（多个，用户视图）、内模式（唯一，存储结构）。
8. **两级映像**：外模式/模式映像 → 逻辑独立性；模式/内模式映像 → 物理独立性。
9. **系统组成**：硬件（大内存/大磁盘/高通道能力）、软件（DBMS/OS/高级语言/开发工具/应用系统）、人员（DBA/系统分析员/数据库设计人员/应用程序员/最终用户）。
