## 文件系统基本概念

**文件**：指具有文件名的若干相关元素的集合。元素通常是记录，记录又是一组有意义的数据项的集合。基于文件系统的概念，可以把数据组成分为**数据项、记录和文件**三级。*(讲义 p117)*

**数据项**：文件系统中最低级的数据组织形式，分为两种——*(讲义 p117)*
- **基本数据项**：用于描述一个对象的某种属性的一个值（如姓名、日期或证件号等），是数据中可命名的最小逻辑数据单位，即原子数据。
- **组合数据项**：由多个基本数据项组成。

**记录**：一组相关的数据项的集合，用于描述一个对象在某方面的属性（如一名考生的报名记录包括考生姓名、出生日期、报考学校代号、身份证号等一系列域）。*(讲义 p117)*

**文件**（逻辑分类）：由创建者所定义的一组相关信息的集合，逻辑上可分为有结构文件（记录式文件）和无结构文件（流式文件，如二进制文件或字符文件）。*(讲义 p117)*

**文件的属性**（4项）：*(讲义 p117)*
1. 文件类型（源文件、目标文件、可执行文件等）
2. 文件长度（当前长度，单位可以是字节、字或块，也可能是最大允许长度）
3. 文件的物理位置（所在设备及所在设备中地址的指针）
4. 文件的建立时间（最后一次修改时间等）

**文件的基本操作**（6种）：*(讲义 p118)*

| 操作 | 说明 |
|---|---|
| 创建文件 | 两步：①在文件系统中为文件找到空间；②在目录中为新文件创建条目（记录文件名、位置及其他信息） |
| 写文件 | 系统调用指明文件名和待写入内容；搜索目录找文件位置；维护写位置指针，每次写操作更新指针 |
| 读文件 | 系统调用指明文件名和待读入内存位置；搜索目录找目录项；维护读位置指针。读和写使用同一指针，节省空间降低复杂度 |
| 文件重定位（文件寻址） | 按条件搜索目录，将当前文件位置设为给定值，不读、写文件 |
| 删除文件 | 从目录中找到目录项使之成为空项，回收文件所占存储空间 |
| 截断文件 | 文件所有属性不变，删除文件内容（长度设为0并释放空间） |

> Caveat (本章单选13): 删除文件时不需要删除文件所在的目录，但需要删除与此文件关联的目录项和文件控制块，并释放关联的内存缓冲区。

**文件的分类**（按4种角度）：*(讲义 p118-119)*

| 分类角度 | 类别 | 说明 |
|---|---|---|
| 按用途 | 系统文件 | 由系统软件构成，大多数只允许调用不允许读/修改，有的不直接对用户开放 |
| 按用途 | 用户文件 | 由用户的源代码、目标文件、可执行文件或数据等构成，委托系统保管 |
| 按用途 | 库文件 | 由标准子例程及常用例程构成，允许调用不允许修改 |
| 按数据形式 | 源文件 | 由源程序和数据构成，通常由ASCII码或汉字组成 |
| 按数据形式 | 目标文件 | 源程序经编译但未链接的目标代码，后缀".obj" |
| 按数据形式 | 可执行文件 | 目标代码经链接后形成，后缀".exe" [原文OCR为".cxc"，疑为OCR错误] |
| 按存取控制 | 只执行文件 | 只允许被核准的用户调用执行，不允许读和写 |
| 按存取控制 | 只读文件 | 只允许文件主及被核准的用户读，不允许写 |
| 按存取控制 | 读写文件 | 允许文件主和被核准的用户读或写 |
| 按组织形式 | 普通文件 | 由ASCII码或二进制码组成的字符文件（源程序、数据文件、OS自身代码等） |
| 按组织形式 | 目录文件 | 由文件目录组成，可检索下属文件信息，操作与普通文件一样 |
| 按组织形式 | 特殊文件 | 特指系统中的各类I/O设备，按文件方式统一管理，操作由设备驱动程序完成 |

> Caveat (本章判断31): 文件系统中源程序是**无结构文件**（流式文件），不是有结构的记录式文件。

## 文件的逻辑结构 ⭐

**逻辑结构分类**：从是否有结构来分，分为有结构文件（记录式文件）和无结构文件（流式文件）；从组织方式来分，分为顺序文件、索引文件和索引顺序文件。*(讲义 p119)*

**有结构文件**：由一个以上的记录构成的文件。每个记录用于描述实体集中的一个实体，各记录有着相同或不同数目的数据项。记录长度分为定长和不定长两类：*(讲义 p119)*

| 类型 | 说明 | 特点 |
|---|---|---|
| 定长记录 | 所有记录长度相同，各数据项处在记录中相同位置，具有相同顺序和长度，文件长度用记录数目表示 | 能有效提高检索速度和效率，便于处理和修改，目前较常用，广泛用于数据处理 |
| 变长记录 | 各记录长度不相同（数据项数目不同或数据项本身长度不定），处理前每个记录长度可知 | 检索速度慢，不便于处理修改，但适合某些场合需要，广泛用于商业领域 |

**无结构文件（流式文件）**：由字符流构成的文件，长度以字节为单位。利用读写指针指出下一个要访问的字符。可看作记录式文件的特例：一个记录仅有一个字节。系统中运行的源程序、可执行文件、库函数等采用此形式。*(讲义 p120)*

> Caveat (本章判断33-34): 文件的逻辑结构是从**用户观点**出发所观察到的文件组织形式，取决于用户需求；不是为方便OS管理而设计的。物理结构才取决于文件系统设计者针对硬件结构的措施。

**按组织方式分类**：*(讲义 p120)*

| 类型 | 说明 |
|---|---|
| 顺序文件 | 由一系列记录按某种顺序排列所形成的文件，记录可以是定长或可变长 |
| 索引文件 | 为可变长记录文件建立一张索引表，每个记录设置一个表项，以加速检索速度 |
| 索引顺序文件 | 顺序文件和索引文件相结合，为一组记录中的**第一个**记录建立索引表项 |

> Caveat (本章单选1): 不定长文件主要分为索引文件、索引顺序和**顺序文件**。

## 文件的物理结构 ⭐

**物理结构**：文件在存储介质上的组织方式，依赖于物理存储设备和物理存储空间，可看作相关物理块的集合，又称物理文件。*(讲义 p120)*

| 结构 | 说明 | 优点 | 缺点 |
|---|---|---|---|
| 顺序结构 | 把逻辑上顺序的记录分配到顺序的物理块中 | 管理简单，存储速度快 | 空间利用率低，插入/删除不方便，只能在文件末尾进行 |
| 链接结构 | 文件信息存放在非连续物理块中，每个物理块设指针指向后续物理块，链接成串联队列。分**显式链接**（指针在专门链接表中）和**隐式链接**（指针在物理块中） | 空间利用率高，易于文件扩充 | 搜索率较低 |
| 索引结构 | 为每个文件建立索引表，表项指出记录所在物理块号，按逻辑记录编号顺序或关键字顺序排列。大文件可建多级索引，索引表本身也可作为文件（索引表文件） | 满足文件动态增长要求，存取方便 | 增加存储空间开销，多级索引访问时间开销大 |

*(讲义 p120-121)*

> Caveat (本章单选21): 文件的存储结构（物理结构）中，顺序结构要求连续存放，链接结构和索引结构不必连续存放。流式结构属于逻辑结构而非物理结构。

**标准解题套路**（for 二级索引最大文件计算）*(本章单选2)*
1. 计算每个索引块可存放的地址数：索引块大小 / 每个地址占用字节数
2. 计算 $k$ 级索引可管理的物理块总数：（每个索引块地址数）$^k$
3. 计算最大文件大小：物理块总数 × 每个物理块大小

> Caveat (本章单选2): 每块2KB，每块地址4B → 一个索引块可存 $2\text{KB}/4\text{B} = 512$ 个地址；二级索引可管理 $512 \times 512 = 262144$ 块；最大文件 $= 262144 \times 2\text{KB} = 512\text{MB}$。

## 目录管理 ⭐

**目录管理的要求**（4项）：*(讲义 p120-121)*

| 要求 | 说明 |
|---|---|
| 实现"按名存取" | 最基本的功能，文件系统向用户提供的最基本服务。用户只须提供文件名，便能找到文件在外存上的存储位置 |
| 提高对目录的检索速度 | 设计大、中型文件系统的主要目标 |
| 文件共享 | 允许多个用户共享一个文件，外存中只保留一份副本，节省存储空间 |
| 允许文件重名 | 允许不同用户对不同文件采用相同名字 |

> Caveat (本章单选6): 文件目录的主要作用是实现"按名存取"。

> Caveat (本章判断38): 文件系统实现按名存取主要靠查找**文件目录**实现，不是靠位示图。

**文件控制块（FCB）**：为对系统中的大量文件施以有效管理，FCB中通常含三类信息：*(讲义 p121)*

| 类别 | 包含内容 |
|---|---|
| 基本信息 | 文件名（唯一标识）、文件物理位置（设备名、起始盘块号、文件长度）、文件逻辑结构（流式/记录式、记录数、定长/变长）、文件物理结构（顺序/链接/索引） |
| 存取控制信息 | 描述文件的存取权限，用于核准用户的存取权限 |
| 使用信息 | 建立日期和时间、上一次修改日期和时间、当前使用信息（已打开进程数、是否被锁住、是否在内存中已修改但未拷贝到盘上等） |

> Caveat (本章多选25): 文件大小、创建时间、拥有者、访问权限都需要保存在FCB中。

**索引结点（i结点）的引入**：文件目录存放在磁盘上，文件很多时目录占用大量盘块。查找时需逐盘块调入内存比较文件名。采用把**文件名与文件描述信息分开**的办法，文件描述信息单独形成索引结点数据结构（i结点），目录中每个目录项仅由**文件名和指向该文件所对应的i结点的指针**构成。*(讲义 p121)*

> Caveat (本章单选18): 将文件描述信息从目录项中分离的好处是：磁盘盘块中可存放更多目录项，**查找文件时大大减少I/O信息量**。

**内存索引结点**（文件被打开时，磁盘索引结点拷贝到内存，增加以下内容）：*(讲义 p121-122)*
- 索引结点编号（标识内存索引结点）
- 状态（指示i结点是否上锁或被修改）
- 访问计数（每有进程访问加1，访问完减1）
- 文件所属文件系统的逻辑设备号
- 链接指针（指向空闲链表和散列队列的指针）

**磁盘索引结点**（每个文件有唯一一个）：*(讲义 p122)*
- 文件主标识符（拥有该文件的个人或小组的标识符）
- 文件类型（正规文件、目录文件、特别文件）
- 文件存取权限
- 文件物理地址（含13个地址项 `iaddr(0)`~`iaddr(12)`，以直接或间接方式给出数据文件所在盘块编号）
- 文件长度（字节为单位）
- 文件连接计数（本文件系统中所有指向该文件的文件名指针计数）
- 文件存取时间（最近被进程存取的时间、最近被修改的时间、索引结点最近被修改的时间）

> Caveat (本章单选17): **访问计数值**是内存索引节点中特有的内容，磁盘索引节点中没有。

**目录结构对比**：*(讲义 p122)*

| 目录结构 | 说明 | 特点 |
|---|---|---|
| 单级文件目录 | 整个系统只建立一张目录表，每个文件占一个目录项（含文件名、扩展名、长度、类型、物理地址等），设状态位标明空闲 | 最简单；**不能解决重名问题** |
| 两级文件目录 | 为每个用户建立用户文件目录UFD（由用户所有文件的FCB组成），系统建立主文件目录MFD（每个用户目录文件占一个目录项，含用户名和指向该用户目录文件的指针） | [原文未述具体优缺点] |
| 树形结构目录 | 最通用且实用的目录。根目录唯一，每个文件和目录都只有一个父目录。数据文件称为树叶，其他目录作为树的结点（子目录） | 提高检索速度和文件系统性能；**允许文件重名** |

> Caveat (本章单选4): 多级目录结构的主要目的是**解决命名冲突**（允许不同用户建立同名文件），同时提高文件查找速度。

> Caveat (本章判断35): 单级文件目录**不能**解决文件重名问题，多级文件目录才能解决。

> Caveat (本章多选26): 二级目录结构由**主文件目录（MFD）**和**用户文件目录（UFD）**组成。

> Caveat (本章单选5): 使用绝对路径名访问文件是从**根目录**开始按目录结构访问。

**目录查询技术**：*(讲义 p123)*

| 方法 | 说明 |
|---|---|
| 线性检索法（顺序检索法） | 单级目录中用顺序查找法直接查找；树形目录中用户提供的文件名是路径名（多个分量名组成），需对多级目录逐级查找 |
| Hash方法 | 建立Hash索引文件目录，将文件名变换为目录索引值再查找，显著提高检索速度。处理"冲突"：若目录项为空→无此文件；若文件名匹配→找到；若不匹配→Hash值加常数（与目录长度互质）重新查找 |

> Caveat (本章判断36): 在顺序检索中，只要路径名的一个分量名未找到，就应停止查找。

> Caveat (本章单选15): 设置当前工作目录的主要目的是**加快文件的检索速度**，不需要从根目录一层层检索。

**标准解题套路**（for FCB访问磁盘次数计算）*(本章单选8)*
1. 计算目录项占用的盘块数：目录项数 × FCB大小 / 盘块大小
2. 一级目录平均访盘次数 = 盘块数 / 2（顺序查找，平均扫描一半盘块）

> Caveat (本章单选8): FCB占64B，盘块1KB，3200个目录项 → 占用盘块数 $= 3200 \times 64\text{B} / 1\text{KB} = 200$ 个；一级目录平均访盘 $= 200/2 = 100$ 次。

## 文件存储空间的管理 ⭐

文件存储设备分成许多大小相同的物理块，以块为单位交换信息。管理实质上是对空闲块的组织和管理，包括空闲块的组织、分配与回收。*(讲义 p123)*

| 管理方法 | 说明 | 特点 |
|---|---|---|
| 空闲表法 | 连续分配方式，为所有空闲区建立空闲表，每项含序号、第一空闲盘块号、空闲盘块数，按起始盘块号递增排列 | 类似内存动态分配；不适用于大型文件系统 |
| 空闲盘块链 | 以盘块为单位拉成一条链，每个盘块有指向后继的指针。分配从链首摘下，回收挂到链尾 | 分配回收简单但效率低，链表很长 |
| 空闲盘区链 | 以盘区（含若干盘块）为单位拉成一条链，每个盘区含指针和大小信息，采用首次适应算法，回收时与相邻空闲盘区合并 | 分配回收较复杂但效率较高，每次分配多个连续块，链表较短 |
| 位示图法 | 用二进制一位表示一个盘块使用情况（0=空闲，1=已分配），构成 $m \times n$ 位数组，$m \times n$ = 磁盘总块数 | 占用空间少可放内存，易于找相邻空闲块；常用于微型机和小型机 |
| 成组链接法 | UNIX系统采用，把顺序的n个空闲扇区地址保存在第一个空闲扇区内，其后一个扇区保存另一组地址，依次链接，系统只保存指向第一个空闲扇区的指针 | 结合空闲表和链表，克服表太大的缺点 |

*(讲义 p123-125)*

**位示图法公式** ⚠：*(讲义 p124-125)*

**分配**：顺序扫描位示图找到值为"0"的位（第 $i$ 行第 $j$ 列），转换为盘块号：
$$b = n(i - 1) + j$$
其中 $n$ 为每行的位数。

**回收**：将盘块号 $b$ 转换为行号和列号：
$$i = (b - 1) / n + 1$$
$$j = (b - 1) \% n + 1$$
然后令 `map[i,j] = 0`。

> Caveat (本章单选10): 位示图用于实现**磁盘空间的管理**，不是磁盘驱动调度或文件目录查找。

## 文件的共享和保护 ⭐⚠

**文件共享的两种方式**：*(讲义 p125-126)*

| 方式 | 别名 | 机制 | 引用计数变化 | 删除原文件后 |
|---|---|---|---|---|
| 基于有向无循环图 | 硬链接 | 共享文件链接到多个用户目录，文件物理地址等放在索引结点中，目录项只含文件名和指向i结点的指针 | 原文件引用计数 **+1** | 其他用户指针**悬空**，操作作废 |
| 利用符号链 | 软链接 | 系统创建LINK类型新文件，只含被链接文件的路径名，写入用户B的目录 | 符号链接文件引用计数**始终为1**，独立于被链接文件 | 访问失败，删除符号链即可，不会产生悬空指针 |

> Caveat (题目P68-T11): 硬链接会使原文件引用计数+1，符号链接（软链接）的引用计数始终为1（且独立于被链接文件）。删除硬链接文件，原文件引用计数-1；删除原文件，符号链接会失效（悬空）。

> Caveat (本章单选11): 建立F1的硬链接F2后，F1引用计数变为2；建立F2的符号链接F3，F3引用计数为1（不受被链接文件影响）；删除F2后，F1引用计数变为 $2-1=1$，F3仍为1。最终 F1=1, F3=1。

> Caveat (本章判断37): 在基于索引结点的共享方式中，如果用户A创建文件、用户B共享，用户A**不能**随意删除该文件，否则B的指针会悬空，在此文件上的操作就会作废。

**文件保护——访问类型**（6种）：*(讲义 p126)*

| 访问类型 | 说明 |
|---|---|
| 读 | 从文件中读 |
| 写 | 向文件中写 |
| 执行 | 将文件装入内存并执行 |
| 添加 | 将新信息添加到文件结尾部分 |
| 删除 | 删除文件，释放空间 |
| 列表清单 | 列出文件名和文件属性 |

此外还可对重命名、复制等加以控制。具有读访问权限的用户同时也就具有了复制和打印权限。

**文件保护——访问控制方法**：*(讲义 p126-127)*

| 方法 | 说明 |
|---|---|
| 访问控制列表（ACL） | 为每个文件和目录增加ACL，规定每个用户名及其允许的访问类型。优点：可使用复杂访问方法；缺点：长度无法预计，空间管理复杂 |
| 精简的访问列表 | 采用**拥有者、组、其他**三种用户类型，只需三个域列出访问权限。文件拥有者创建文件时说明用户名和组名，系统在FCB中记录。UNIX采用此方法 |
| 口令 | 另一种访问控制方法 |
| 密码 | 另一种访问控制方法 |

> Caveat (本章单选19): 口令、存取控制矩阵、用户权限表都是常用的文件保护方法。

> Caveat (本章多选30): 存取控制矩阵、用户权限表、口令或密码都可用于文件存取控制和保护；**FCB只是文件存在的标识，本身不具备保护能力**。

## 磁盘组织与管理

**磁盘结构**：磁盘设备可包括一个或多个物理盘片，每个盘片分一个或两个存储面，每个盘面上有若干个磁道，磁道之间留有必要的间隙。为使处理简单，每条磁道上可存储相同数目的二进制位。*(讲义 p127)*

**磁盘访问时间** ⭐：*(讲义 p127)*

磁盘以恒定速率旋转。为读或写，磁头须移动到指定磁道，等待指定扇区旋转到磁头下，然后开始读/写数据。总访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间。

**寻道时间** $T_s$：磁臂（磁头）移动到指定磁道所经历的时间，为启动磁臂时间 $s$ 与磁头移动 $n$ 条磁道所费时间之和：
$$T_s = m \times n + s$$
其中 $m$ 是与磁盘驱动器速度有关的常数，$n$ 是移动的磁道数，$s$ 是启动磁臂的时间。

**旋转延迟时间** $T_r$：指定扇区移动到磁头下面所经历的时间。例如硬盘转速15000r/min，每转需时4ms，平均旋转延迟时间 $T_r$ 为2ms（最小为0，最大为4，平均为2）。

**传输时间** $T_t$：把数据从磁盘读出或向磁盘写入数据所经历的时间：
$$T_t = \frac{b}{rN}$$
其中 $b$ 为每次所读/写的字节数，$r$ 为磁盘每秒钟的转数，$N$ 为一条磁道上的字节数。

**磁盘调度算法** ⭐：*(讲义 p128)*

| 算法 | 全称 | 说明 | 特点 |
|---|---|---|---|
| FCFS | 先来先服务 | 根据进程请求访问磁盘的先后次序进行调度 | 最简单 |
| SSTF | 最短寻道时间优先 | 选择要求访问的磁道与当前磁头所在磁道距离最近的进程 | 每次寻道时间最短，但不能保证平均寻道时间最短；可能导致低优先级进程"饥饿" |
| SCAN | 扫描算法 | 对SSTF略加修改以防止"饥饿"现象 | 获得较好寻道性能且防止饥饿；但刚越过的磁道需等待磁头往返后才能处理 |
| CSCAN | 循环扫描 | 对SCAN的改进，解决刚越过磁道请求被大大推迟的问题 | [原文未详述具体步骤] |

**磁盘的管理**：包括磁盘的初始化、低级格式化和逻辑格式化、引导块和坏块。*(讲义 p128)*

> Caveat (本章判断40): 一个物理硬盘可以分成多个逻辑硬盘分区进行面向用户文件系统的管理。

> Caveat (本章判断39): 单个文件的大小不仅受磁盘剩余空间大小的限制，还受**FCB和FAT表等结构**的限制。

> Caveat (本章判断32): 关闭文件是将文件当前的**控制信息**从内存写回磁盘，并不意味着将文件**数据**写回磁盘（写文件操作才会写回数据，不考虑延迟写）。

> Caveat (本章单选12): 文件被打开后，系统为每个打开的文件分配一个**文件描述符**（索引号），之后用户使用文件描述符而非文件名进行读、写、定位等操作。

> Caveat (本章单选16): `open()`调用的参数含有文件名（路径名），它在进程的**用户打开文件表**中增加一个表目并返回索引号（文件描述符）。**系统打开文件表**只有在文件实体第一次被打开时才增加表目。不同文件名互为硬链接时，打开的文件实体相同。

> Caveat (本章多选22): 多个进程共享同一文件时：各进程可以"读"或"写"方式打开（OS不保证写互斥）；系统打开文件表中同一文件只有一个表项（改变引用计数）；各进程的用户打开文件表关于同一文件的内容**不一定相同**（如读写位置指针、访问权限可能不同）；进程关闭文件时引用计数减1，变为0时才删除系统打开文件表中的表项。

## 一句话记忆要点

- **文件三级数据组织**：数据项→记录→文件，数据项是最低级（基本数据项=原子数据）。
- **逻辑结构**：有结构（记录式：定长/变长）vs 无结构（流式）；组织方式分顺序、索引、索引顺序。
- **物理结构**：顺序（快但不灵活）、链接（灵活但慢）、索引（灵活但开销大）。
- **目录管理四要求**：按名存取（最基本）、提高检索速度、文件共享、允许重名。
- **FCB三类信息**：基本信息、存取控制信息、使用信息。
- **i结点分离**：文件名与描述信息分开→减少查找I/O量；内存i结点比磁盘i结点多：访问计数、状态、链接指针等。
- **目录结构**：单级（不能重名）→两级（MFD+UFD）→树形（最通用，允许重名）。
- **位示图公式**：分配 $b=n(i-1)+j$；回收 $i=(b-1)/n+1$，$j=(b-1)\%n+1$。
- **硬链接 vs 软链接**：硬链接引用计数+1，软链接引用计数始终为1；删原文件后硬链接指针悬空，软链接访问失败可删链。
- **磁盘访问时间**：$T_s = m \times n + s$（寻道），$T_r$（旋转延迟），$T_t = b/(rN)$（传输）。
- **磁盘调度**：FCFS最简单，SSTF可能饥饿，SCAN防饥饿，CSCAN改进SCAN。

## Worked Examples

### 例 1：位示图盘块号与行列号互算 *(来自本章正文·位示图法公式改造)*
*Tags: 位示图, 空闲空间管理, 盘块号换算*
**问题**：磁盘共 200 块，位示图每行 16 位（即 $n=16$，从 1 开始编号）。求：(1) 第 3 行第 5 列对应的盘块号 $b$；(2) 盘块号 $b=100$ 对应的位示图行号 $i$ 与列号 $j$。*(讲义 p124-125)*
**解答**：`[PROPOSED — verify]`
1. 套用分配公式 $b = n(i-1)+j$，代入 $n=16$、$i=3$、$j=5$：$b = 16 \times (3-1) + 5 = 32 + 5 = 37$。
2. 套用回收公式 $i = (b-1)/n + 1$，代入 $b=100$、$n=16$：$i = (100-1)/16 + 1 = 6 + 1 = 7$。
3. 套用回收公式 $j = (b-1) \% n + 1$，代入：$j = 99 \% 16 + 1 = 3 + 1 = 4$。
4. 结论：第 3 行第 5 列对应盘块号 $37$；盘块号 $100$ 对应第 $7$ 行第 $4$ 列。

### 例 2：硬链接与软链接引用计数推算 *(来自本章单选11改造)*
*Tags: 硬链接, 软链接, 引用计数*
**问题**：用户建立文件 F1，再为 F1 建立硬链接 F2，又为 F2 建立符号链接 F3。求建立后 F1、F2、F3 的引用计数；之后删除 F2，F1 与 F3 的引用计数变为多少？*(本章单选11)*
**解答**：`[PROPOSED — verify]`
1. 初值：F1 引用计数 $=1$（仅自身目录项指向其索引结点）。
2. 建立硬链接 F2：F2 与 F1 共享同一索引结点，原文件引用计数 $+1$，故 F1=2，F2 同样指向该索引结点，计数同为 2。
3. 建立符号链接 F3：符号链接是新建的 LINK 类型文件，独立于被链接文件，F3 引用计数始终为 $1$，F1/F2 不变。
4. 删除 F2：硬链接每删一个，被链接文件引用计数 $-1$，故 F1 由 2 变为 1；F3 不受影响仍为 1。
5. 结论：建立后 F1=F2=2，F3=1；删除 F2 后 F1=1，F3=1。
