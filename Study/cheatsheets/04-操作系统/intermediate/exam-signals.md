## 2. 反复出现的题型模式
- **死锁资源分配计算**：题干给出"系统有N台资源，K个进程竞争，每个最多需要M台" → 解题切入点：利用公式 `不发生死锁的最少资源数 = N * (M - 1) + 1`，或反推发生死锁的临界进程数。`[PROPOSED — verify]`
- **调度算法平均周转时间计算**：题干给出多个作业的到达时间/运行时间，指定调度算法（SJF/RR/FCFS） → 解题切入点：画出甘特图，逐个计算完成时间。SJF在同时到达时平均周转时间最小；RR需按时间片切分交替执行。`[PROPOSED — verify]`
- **访存次数计算**：题干涉及分页/段页式、TLB命中率、多级页表 → 解题切入点：基本分页访存2次，段页式3次。引入TLB时，`平均访存次数 = TLB命中率 * 1 + (1 - TLB命中率) * (访存页表次数 + 1)`。`[PROPOSED — verify]`
- **状态转换触发条件判断**：题干描述进程发生某事件（如I/O完成、时间片用完、申请资源失败） → 解题切入点：牢记三态模型图，阻塞态只能由运行态转入，且只能转出到就绪态；时间片用完回就绪态，I/O完成从阻塞回就绪态。`[PROPOSED — verify]`
- **安全属性识别**：题干描述某种攻击或威胁（如截取数据、篡改数据、删除软件） → 解题切入点：截取对应保密性，篡改对应完整性，删除/拒绝服务对应可用性。`[PROPOSED — verify]`

## 3. 技巧 / 易错点
- > Caveat (题目P28-T8): 死锁最小资源数计算时，注意"每个进程最多需要M台"与"每个进程已占有M-1台"的等价转换，此时系统处于临界死锁状态，再多分配1个资源即可打破死锁。
- > Caveat (题目P30-T20): 时间片轮转法（RR）计算平均周转时间时，必须严格按照时间片大小交替执行，注意进程完成后从就绪队列移除，切勿按整体运行时间简单相加。
- > Caveat (题目P44-T18): 计算带有TLB的多级页表访存次数时，TLB命中只需访存1次（直接获取数据）；未命中需访问各级页表，最后还要加1次访存获取数据。不要漏掉最后获取数据的1次。
- > Caveat (题目P68-T11): 硬链接与符号链接的引用计数机制不同。硬链接会使原文件引用计数+1，符号链接（软链接）的引用计数始终为1（且独立于被链接文件）。删除硬链接文件，原文件引用计数-1；删除原文件，符号链接会失效（悬空）。
- > Caveat (题目P5-T20): 用户态与核心态指令判断，修改内存单元的值看似危险，但在用户自己的地址空间内是合法的非特权指令；而屏蔽中断、设置时钟、停机、I/O操作均为核心态特权指令。
- > Caveat (题目P42-T2): Belady异常现象（分配页框增加，缺页次数反而增加）仅出现在FIFO页面置换算法中，LRU和OPT绝不会出现。看到Belady直接选FIFO。
- > Caveat (题目P30-T18): 死锁四个必要条件中，"互斥"条件通常由设备物理特性决定（如打印机），无法通过操作系统策略破坏，而其他三个条件（请求和保持、不剥夺、环路等待）均可破坏。

## 5. Worked Example 候选（适合放进最终 cheatsheet 末尾）
### 候选 1：P28-T8·死锁临界资源计算 `[PROPOSED — verify]`
- 概念 tag：死锁预防, 资源分配
- 题干（精简）：某系统有8台打印机，K个进程竞争，每个进程最多需要3台。求可能发生死锁的K的最小值。
- 解题骨架（3-8 步）：
  1. 明确死锁发生的临界条件：每个进程都已获得部分资源，但均未达到最大需求，且此时系统已无剩余资源。
  2. 设每个进程最多需要 $m$ 台，当前已获得 $m-1$ 台。若系统总资源 $R$ 满足 $R = K \times (m-1)$，则此时系统无空闲资源，发生死锁。
  3. 代入公式：$8 = K \times (3-1)$，解得 $K = 4$。
  4. 验证：当 $K=3$ 时，最坏情况每个进程分得2台，共占6台，系统还剩2台，足够分给一个进程使其运行完毕并释放资源，不会死锁。
  5. 结论：发生死锁的最小 $K$ 值为 4。

### 候选 2：P30-T20·时间片轮转法周转时间计算 `[PROPOSED — verify]`
- 概念 tag：时间片轮转, 平均周转时间
- 题干（精简）：4个进程A、B、C、D同时到达，运行时间分别为11、7、2、4，时间片为1，求平均周转时间。
- 解题骨架（3-8 步）：
  1. 按时间片轮转原则，队列初始顺序为 A, B, C, D。
  2. 模拟执行过程：A(1), B(1), C(1), D(1) -> 剩余 A(10), B(6), C(1), D(3)。
  3. 继续轮转：C在第7个时间单位完成。D在第14个时间单位完成。B在第20个时间单位完成。A在第24个时间单位完成。
  4. 计算各进程周转时间（完成时间 - 到达时间0）：C=7, D=14, B=20, A=24。
  5. 求平均：(7 + 14 + 20 + 24) / 4 = 65 / 4 = 16.25。

### 候选 3：P44-T18·TLB多级页表平均访存次数计算 `[PROPOSED — verify]`
- 概念 tag：TLB, 多级页表, 访存次数
- 题干（精简）：配置TLB的页式虚拟存储系统，TLB命中率75%，忽略访问TLB时间，使用二级页表，求每次存取的平均访存次数。
- 解题骨架（3-8 步）：
  1. 分析命中情况：若TLB命中，直接得到物理地址，仅需1次访存读取数据。
  2. 分析未命中情况：若TLB未命中，需先访问一级页表（1次访存），再访问二级页表（1次访存），最后根据物理地址读取数据（1次访存），共3次。
  3. 计算平均值：$0.75 \times 1 + (1 - 0.75) \times 3 = 0.75 + 0.75 = 1.5$ 次。
