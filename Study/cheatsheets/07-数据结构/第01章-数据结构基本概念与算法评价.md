# 数据结构 · 数据结构基本概念与算法评价 — 复习资料

> 来源：7-3-数据结构计算机类讲义.pdf 物理页 [3, 17]
> 说明：⭐=高频考点，⚠=易错点；> Caveat: 为陷阱/技巧；末尾 Worked Examples 为代表性题目（无书内答案的标 [PROPOSED — verify]）。

## 一、数据结构基本概念

**数据结构（Data Structure）**：互相之间存在着一种或多种关系的数据元素的集合。数据结构涉及三个方面：数据的逻辑结构、数据的存储结构和数据的运算。

**数据**：信息的载体，是所有能够被计算机识别、存储和加工处理的符号的总称。可分为数值数据（整数、实数、复数，用于工程计算、科学计算、商务处理）和非数值数据（字符、文字、图形、图像、语音等）。

**数据项（Data Item）**：具有独立含义的标识单位，是数据不可分割的**最小单位**。

**数据元素（Data Element）**：数据的**基本单位**，在不同条件下可称为元素、结点、顶点、记录等。

**数据对象（Data Object）/ 数据元素类（Data Element Class）**：具有相同性质的数据元素的集合。数据元素是数据元素类的一个实例。

> Caveat (p003-T2): 数据的最小单位是**数据项**，不是数据元素；数据元素是数据的**基本单位**——二者不可混淆。

> Caveat (p003-T1): 数据结构作为一门学科研究的是：数据的**逻辑结构、物理结构（存储结构）及操作算法（运算）**，三者缺一不可。

> Caveat (多选1): 数据结构是相互之间存在一种或多种特定关系的数据元素的组合；数据元素**不是**孤立存在的；数据结构是一个二元组；有四类基本结构。

## 二、数据的逻辑结构 ⭐

**逻辑结构**：数据元素之间的逻辑关系。数据的逻辑结构包含两个要素：数据元素的集合和关系的集合。它是从具体问题抽象出来的数学模型，与数据的存储无关。

形式化表示为二元组：

$$\text{Data\_Structure} = (D, R)$$

其中 $D$ 是数据元素的有限集，$R$ 是 $D$ 上关系的有限集。

四种基本逻辑结构对比：

| 逻辑结构 | 元素间关系 | 特点 |
|---|---|---|
| 集合 | "属于同一个集合" | 元素关系极为松散，可用其它结构表示 |
| 线性结构 | 一对一 | — |
| 树形结构 | 一对多 | — |
| 图形结构（网状结构） | 多对多 | — |


> Caveat (p003-T3): 集合是数据元素之间关系**最简单**的数据结构。

> Caveat (p003-T6): 从逻辑上可以把数据结构分为**线性结构**和**非线性结构**两大类。

> Caveat (p003-T5): 线性表、栈、队列均为线性结构；二叉树属于树形结构，是**非线性结构**。

> Caveat (p003-T4): 数据的逻辑结构与数据元素本身的形式、内容、相对位置、个数**无关**。

> Caveat (p003-T7): 同一逻辑结构中的所有数据元素具有相同特性，意味着不仅数据元素所包含的数据项个数要相同，而且对应的数据项的类型要一致。

## 三、数据的存储结构 ⭐

**存储结构（物理结构）**：数据结构在计算机中的标识（又称映像），研究数据的逻辑结构在计算机中的实现方法，包括数据元素的表示及元素间关系的表示。

四种存储方法对比：

| 存储方法 | 核心原理 | 实现方式 | 特点 |
|---|---|---|---|
| 顺序存储 | 逻辑上相邻的元素存储在物理位置相邻的存储单元中，结点间逻辑关系由存储单元邻接关系体现 | 借助数组实现 | 最基本的存储表示；存储密度大；插入/删除需移动大量元素，效率低 |
| 链式存储 | 逻辑上相邻的元素不要求物理位置相邻，通过附设指针字段表示逻辑关系 | 借助指针实现 | 最基本的存储表示 |
| 索引存储 | 存储结点信息同时建立附加索引表（含关键字和地址） | — | 主要面向查找操作；针对数据内容的存储 |
| 散列存储 | 以关键字值为自变量，通过散列函数计算存储位置 | — | 针对数据内容的存储方式 |


> Caveat (p004-T8): 存储数据时，通常不仅要存储各数据元素的值，还要存储**数据元素之间的关系**。

> Caveat (判断题2): 顺序存储方式的优点是存储密度大，但插入、删除运算效率**低**（需移动大量元素）。

> Caveat (判断题4): 顺序存储结构的数据存储密度**高于**链式存储结构（链式需额外存储指针开销）。

> Caveat (判断题3): 数据的逻辑结构**不依赖**于数据的存储结构。

> Caveat (单选18): 数据的逻辑结构与存储结构**不一定一一对应**（如树和图是逻辑结构，但物理结构可用数组或链表）。

## 四、数据的运算

**运算**：对数据的处理，与逻辑结构紧密相连，每种逻辑结构都有一个运算的集合。运算定义在数据的逻辑结构上，但运算的具体实现是在数据的存储结构上进行的。

运算分类：

| 类型 | 说明 |
|---|---|
| 引用型运算 | 不改变数据结构中原有数据元素的状态，只根据需要读取某些信息 |
| 加工型运算 | 改变数据结构中原有数据的状态（如数据元素的内容、个数等） |


## 五、数据类型与抽象数据类型

**数据类型（Data Type）**：一个值的集合和定义在这个值集上的一组操作的总称。类型显式或隐含地规定了变量或表达式所有可能的取值范围，以及在这些值上允许进行的操作。

数据类型分类：

| 类型 | 说明 | 举例 |
|---|---|---|
| 原子类型 | 值不可分解 | C 语言中 int、char、float、double |
| 结构类型 | 值由若干成分按某种结构组成，可分解 | 数组（分量可以是整数，也可以是数组等） |


**抽象数据类型（Abstract Data Type，ADT）**：一个数学模型以及定义在该模型上的一组操作。其定义取决于它的一组逻辑特性，而与其在计算机内部如何表示和实现无关。不论其内部结构如何变化，只要它的数学特性不变，都不影响其外部的使用。

ADT 的三要素：**元素、关系、操作**。

ADT 的特征：使用与实现相分离，实行**封装和信息隐藏**。

> Caveat (判断题5): 抽象数据类型中，描述数据类型的方法与数据存储的物理结构**无关**。

> Caveat (多选3): 抽象数据类型包括**数据**和**操作**。

## 六、算法的概念与特性 ⭐

**算法**：对特定问题求解步骤的一种描述，是指令的有限序列。

算法的五个特性：

| 特性 | 含义 |
|---|---|
| 有穷性 | 必须在有穷步之后结束，在有限时间内完成 |
| 确定性 | 每一步必须有确切的定义，无二义性；相同输入必然有相同输出 |
| 可行性 | 每一步都可以通过已实现的基本运算的有限次执行得以实现 |
| 输入 | 具有零个或多个输入，取自特定的数据对象集合 |
| 输出 | 具有一个或多个输出，同输入之间存在某种特定关系 |


> Caveat (判断题8): 算法必须有输出，但**可以没有输入**。

> Caveat (判断题7): 算法和程序原则上没有区别 → **错误**。程序依赖具体的物理结构，而算法可以不依赖具体的物理结构，二者不能通用。

> Caveat (单选10): 计算机算法必须具备的 5 个特性：可行性、确定性、有穷性、输入、输出。

**算法的描述方法**：自然语言、流程图、盒图、伪码语言。

> Caveat (单选11): 流程图中，**菱形**表示决策或判断，方框表示处理，平行四边形表示输入，扁圆（椭圆）表示程序的开始与结束。

**算法的设计要求**：

| 要求 | 含义 |
|---|---|
| 正确 | 执行结果应当满足预先规定的功能和性能要求 |
| 可读 | 思路清晰、层次分明、简单明了、易读易懂，便于人的阅读和交流 |
| 健壮 | 输入不合法数据时，算法能作适当处理，不产生莫名其妙的输出或引起严重后果 |
| 高效 | 具有较好的时空性能 |


> Caveat (单选12): 健壮性 = 输入非法数据时算法能作适当处理，不会产生莫名其妙的输出或引起其它严重后果。

> Caveat (判断题6): "算法的优劣与描述算法的语言无关，但与所用计算机的性能有关" → **错误**。算法运行时间取决于硬件速度、书写程序的语言、编译程序生成目标代码的质量、问题的规模等因素。

> Caveat (单选9): 算法分析的目的是**分析算法的效率以求改进**。

## 七、算法的时间复杂度 ⭐⚠

**时间复杂度 $T(n)$**：算法的时间耗费，是问题规模 $n$ 的函数。

算法执行所耗费的时间 = 所有语句执行时间之和；每条语句执行时间 = 执行一次所用时间 × 重复执行次数（**频度 Frequency Count**）。

$$T(n) = \sum_{\text{语句 } i} (t_i \times c_i)$$

其中 $t_i$ 表示语句 $i$ 执行一次的时间，$c_i$ 表示语句 $i$ 的频度。

假设每条语句执行一次均为一个单位时间，则时间耗费简化为各语句频度之和：

$$T(n) = \sum_{\text{语句 } i} c_i$$


**大O记号定义**：设 $T(n)$ 是问题规模 $n$ 的函数 $f(n)$，若存在两个正常数 $c$ 和 $n_0$，使得对所有的 $n \geq n_0$，有 $T(n) \leq cf(n)$，则记为：

$$T(n) = O(f(n))$$


例如：$T(n) = 20n^3 + 25n^2 + 9$，则 $T(n) = O(n^3)$。

常见渐进时间复杂度按数量级递增排列：

$$O(1) < O(\log_2 n) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n)$$


**标准解题套路**（for 循环代码时间复杂度推导）
1. 找出与规模 $n$ 相关的最深嵌套层
2. 常数层循环直接忽略（如 `i<3`）
3. 若外层循环为 $m$ 次、内层为 $n$ 次且不牵扯累加变量依赖，直接相乘得 $O(mn)$
4. 取最高阶项，去掉常数系数，即为大O表示

> Caveat (p004-T13): 嵌套循环中，若外层循环次数为常数级（如 `i<3`），内层循环与 $n$ 相关，时间复杂度由内层决定为 $O(n)$，**不要无脑写成 $O(n^2)$**。

> Caveat (单选16): $O(n) < O(2^n)$，随着问题规模 $n$ 的增大，时间复杂度较低的算法更有效。

## 八、算法的空间复杂度

**空间复杂度（Space Complexity）**：算法运行从开始到结束所需的存储量。

算法运行所需的存储空间包括**固定部分**和**可变部分**。

> Caveat (单选14): 算法的时间复杂度和空间复杂度**没有必然关系**。

> Caveat (单选18): 算法的效率与问题的规模和数据的存储都相关；逻辑结构和存储结构不一定一一对应；时间复杂度与空间复杂度不一定相关。

## 九、递归 ⭐

**递归算法**：一种直接或者间接地调用自身算法的过程。

递归的特点：
- 每次调用在规模上都有所缩小
- 相邻两次调用之间有紧密联系，前一次要为后一次做准备
- 问题规模极小时必须直接给出解答而不再进行递归
- 递归调用都有结束的条件

递归的优点：结构清晰，可读性强，容易用数学归纳法来证明算法的正确性。

递归的缺点：运行效率较低，无论是耗费的计算时间还是占用的存储空间都比非递归算法要多。

> Caveat (单选19): 递归是将一个较复杂的（规模较大的）问题转化为一个稍微简单的（规模较小的）与原问题**同类型**的问题来解决。

> Caveat (单选20): 将递归求解过程改变为非递归求解过程的目的是**提高速度**。

> Caveat (判断题9): 递归定义的数据结构通常用递归算法来实现对它的操作。

> Caveat (判断题10): 递归调用算法与相同功能的非递归算法相比，主要问题在于重复计算太多，且调用本身需要分配额外的空间和传递数据和控制，所以时间与空间开销通常都比较大。

## 十、重点对比与易混辨析

### 逻辑结构 vs 存储结构

| 对比项 | 逻辑结构 | 存储结构 |
|---|---|---|
| 别称 | — | 物理结构（映像） |
| 研究内容 | 数据元素间的逻辑关系 | 逻辑结构在计算机中的实现方法 |
| 与存储关系 | 与存储**无关** | 依赖计算机存储 |
| 分类 | 集合、线性、树形、图形 | 顺序、链式、索引、散列 |
| 依赖方向 | 不依赖存储结构 | 实现逻辑结构 |


### 数据项 vs 数据元素 vs 数据对象

| 概念 | 定位 | 说明 |
|---|---|---|
| 数据项 | **最小**单位 | 具有独立含义的标识单位，不可分割 |
| 数据元素 | **基本**单位 | 可由多个数据项组成；可称元素/结点/顶点/记录 |
| 数据对象 | 集合 | 具有相同性质的数据元素的集合 |


### 算法 vs 程序

| 对比项 | 算法 | 程序 |
|---|---|---|
| 依赖物理结构 | 可以不依赖 | 依赖 |
| 有穷性 | 必须有穷 | 不一定有穷（如操作系统） |
| 通用性 | 二者**不能**通用 | — |


## 十一、高频考点与易错提醒

1. **数据结构三方面**：逻辑结构 + 存储结构 + 运算（算法），三者缺一不可 ⭐
2. **数据项是最小单位，数据元素是基本单位**——注意区分 ⚠
3. **逻辑结构不依赖存储结构**——逻辑结构是数学模型，与计算机无关 ⚠
4. **顺序存储密度大但插入删除效率低**——存储密度高于链式 ⚠
5. **算法五特性**：有穷性、确定性、可行性、输入、输出——可零输入但必须至少一个输出 ⭐
6. **算法 vs 程序**：程序依赖物理结构，算法可以不依赖，二者不通用 ⚠
7. **时间复杂度推导**：常数层循环忽略，取最高阶项去系数 ⭐
8. **时间复杂度与空间复杂度无必然关系** ⚠
9. **递归转非递归目的：提高速度**（递归时间和空间开销都大）⭐
10. **ADT 与物理结构无关**——封装和信息隐藏 ⚠
11. **存储数据时**不仅要存储值，还要存储数据元素之间的关系 ⚠
12. **逻辑结构与存储结构不一定一一对应** ⚠

## 十二、一句话记忆要点

- 数据结构 = 逻辑结构 + 存储结构 + 运算
- 数据项最小，数据元素基本，数据对象是同类元素集合
- 逻辑结构四类：集合（最松散）、线性（一对一）、树（一对多）、图（多对多）
- 存储结构四类：顺序（数组，密度大）、链式（指针）、索引（索引表）、散列（哈希函数）
- 算法五特性：有穷、确定、可行、输入（可零个）、输出（至少一个）
- 大O记号：取最高阶项去系数，常数层循环忽略
- 递归：清晰但低效，转非递归为提速

## Worked Examples

### 例 1：嵌套循环时间复杂度推导（常数层外循环 + $n$ 层内循环）

*Tags: 时间复杂度, 大O记号, 嵌套循环*
**问题**：分析以下程序段的时间复杂度（所有变量类型为 int）：

```c
s=0;
for(i=0;i<3;i++)
    for(j=0;j<n;j++)
        s++;
```


**解答**：
1. 外层循环 `for(i=0;i<3;i++)` 执行 **3 次**，是常数级，与问题规模 $n$ 无关。
2. 内层循环 `for(j=0;j<n;j++)` 执行 $n$ 次，与问题规模 $n$ 相关。
3. 关键语句 `s++` 的总执行次数 = $3 \times n = 3n$。
4. 常数层循环直接忽略，取最高阶项去系数：$T(n) = O(n)$。
5. **注意**：不要因为看到两层嵌套循环就无脑写 $O(n^2)$——外层是常数级时，时间复杂度由内层决定。


### 例 2：双变量嵌套循环时间复杂度推导

*Tags: 时间复杂度, 大O记号, 嵌套循环*
**问题**：分析以下程序段的时间复杂度（外层循环与 $m$ 相关，内层循环与 $n$ 相关，两循环嵌套）：

```c
for(i=0;i<m;i++)
    for(j=0;j<n;j++)
        a[i][j]=i*j;
```


**解答**：
1. 外层循环执行 $m$ 次，与问题规模 $m$ 相关。
2. 内层循环执行 $n$ 次，与问题规模 $n$ 相关。
3. 两循环为嵌套关系，关键语句 `a[i][j]=i*j` 的总执行次数 = $m \times n$。
4. 外层和内层均与规模相关且不牵扯累加变量依赖，直接相乘：$T(n) = O(m \times n)$。
