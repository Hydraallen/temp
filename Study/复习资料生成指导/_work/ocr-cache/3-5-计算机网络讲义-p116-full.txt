用户用来查看 HTML 超文本的客户机软件称为 www 浏览软件或 www 浏览器。浏览器使用户对超文本文件的链有清楚的认识，链与其他文本有颜色或字体上的差异，并有许多种叫法，如热链、超链等，顺着链，浏览器提供了从一个文件转到另一个文件的方法，www 浏览器和 www 服务器的交互作用完成了信息查询。www 浏览器本身分为文本浏览器(如 Lynx)和图形浏览器(如 Netscape)等，常用的是图形浏览器。

6.5.4 URL 地址

URL 地址是 www 中用来寻找资源地址的办法，URL 是 Uniform Resource Locator 的缩写，译为统一资源定位器或普遍资源定位器。URL 地址的思想是为了使所有的信息资源都能得到有效利用，从而将分散的孤立的信息点连接起来，实现资源的统一寻址。URL 大致由三部分组成：协议、主机名、目录或文件名，如图 6.1 所示。

协议 主机名 目录或文件名

http://daytona.raceway.com/home/www/files/race.html

协议即访问页的方式，也就是浏览器用于取得超文本文件的协议或程序。如果浏览器利用 HTTP 方式来访问文件，那么协议部分就是 http;如果用 FTP，则是 ftp;用 Gopher，就是 gopher;以此类推。协议必须与系统安装的信息服务器匹配以便能正常工作。例如，在一台没有安装 FTP 服务器的机器上，用户就无法使用 FTP 的 URL。

主机名是 Internet 上用于标识信息保存的地方，例如 www.netcom.com，ftp.apple.com 或 www.aol.com。同一个主机名因使用不同的协议而有不同的 URL，如：

http: //mysystem.com

ftp: //mysystem.com

gopher://mysystem.com

上例中同一主机安装了三个不同的信息服务器，也就使得浏览器可同时工作不会引起任何问题。

URL 的主机名中可包含一个端口号，它告诉浏览器用指定的网络端口来替代默认端口打开正确的协议连接。但是，需要指出：当服务器只提供对应于该特定端口的信息服务时才有必要在 URL 中写上端口号。
