## 线性表基础概念

**线性表的定义**：具有相同数据类型的 $n$（$n \ge 0$）个数据元素的有限序列，记为 $(a_1, a_2, \dots, a_{i-1}, a_i, a_{i+1}, \dots, a_n)$。其中 $n$ 为表长，$n=0$ 时称为空表。
- 存在次序关系：$a_{i-1}$ 是 $a_i$ 的直接前驱，$a_{i+1}$ 是 $a_i$ 的直接后继。
- 线性表特点：集合中必存在唯一的“第一元素”和“最后元素”；除最后元素外均有唯一后继；除第一元素之外均有唯一的前驱。

> Caveat (p013-T11 / 判断题11): 注意极端元素，第一个元素 $a_1$ 没有前驱，最后一个元素 $a_n$ 没有后继。

## 顺序存储与顺序表操作 ⭐

**顺序存储**：在内存中用地址连续的一块存储空间顺序存放线性表的各元素，用物理上的相邻实现逻辑上的相邻。
- **地址公式**：设 $a_i$ 的存储地址为 $\text{Loc}(a_i)$，每个元素占 $d$ 个存储单元，则第 $i$ 个数据元素的地址为：
$$\text{Loc}(a_i) = \text{Loc}(a_1) + (i-1) \times d \quad (1 \le i \le n)$$

**插入运算**：在第 $i$ 个位置插入新元素 $x$，使表长由 $n$ 变为 $n+1$（其中 $1 \le i \le n+1$）。
1. 将 $a_i \sim a_n$ 顺序向下移动，为新元素让出位置；
2. 将 $x$ 置入空出的第 $i$ 个位置；
3. 修改 last 指针（相当于修改表长），使之仍指向最后一个元素。

> Caveat (p015-T15 / 单选2): 等概率情况下，插入一个元素平均需移动表中 $n/2$ 个元素。

**删除运算**：将表中第 $i$ 个元素去掉，使表长由 $n$ 变为 $n-1$（其中 $1 \le i \le n$）。
1. 将 $a_{i+1} \sim a_n$ 顺序向前移动；
2. 修改 last 指针。

> Caveat (p015-T15 / 判断题3): 顺序表的插入和删除操作需要付出很大的时间代价，每次操作平均需移动近一半的元素。

## 链式存储及基本操作 ⭐

**链式存储结构**：不需要地址连续的存储单元，通过“链”表示数据元素之间的逻辑关系。
- **单链表结点**：包含存放自身信息的 `data` 域和存放后继地址的 `next` 域。
  ```c
  typedef struct lnode {
      datatype data;
      struct lnode *next;
  } LNode, *LinkList;
  ```


**单链表基本运算与操作**：
1. **建立单链表**：头插法（生成的元素顺序与读入顺序相反）或尾插法（顺序一致）。
2. **查找操作**：按序号查找或按值查找，均需从头指针开始顺序遍历。
3. **插入操作**：
   - 设 `p` 指向某结点，`s` 指向待插入的新结点。
   - **后插结点标准步骤**（保真操作顺序）：
     ```c
     s->next = p->next;
     p->next = s;
     ```


> Caveat (p013-T8): 单链表插入结点时操作顺序不可颠倒。若先执行 `p->next = s`，原后继结点地址丢失，`s->next = p->next` 将指向 `s` 自身形成死循环。

4. **删除操作**：设 `p` 指向单链表中某结点，删除 `*p`。

## 高级链表结构 ⭐

**循环链表**：将单链表最后一个结点的空指针改为指向链表头指针，使得链表头尾结点相连。

**双向链表**：每个结点增加一个指向前驱的指针域 `prior`，使得找前驱和后继的时间性能均为 $O(1)$。
- **结点结构**：`[prior | data | next]`
- **双向链表删除 `p` 结点标准操作**：
  ```c
  (p->prior)->next = p->next;
  (p->next)->prior = p->prior;
  ```

- **双向链表在 `*p` 之后插入 `*s` 标准操作**：
  ```c
  s->prior = p;
  s->next = p->next;
  p->next->prior = s;
  p->next = s;
  ```


> Caveat (p013-T8): 双向链表插入或删除时，同样需要保证指针修改顺序正确，防止断链后找不到原本相邻的结点地址。

**头结点及其作用**：附加在第一个元素结点之前的一个结点。
- 使对空表和非空表的处理得到统一；在链表的第一个位置上的操作和在其他位置上的操作一致，无需特殊处理。

## 顺序存储与链式存储对比 ⚠

| 特性 / 结构 | 顺序存储 | 链式存储 |
| :--- | :--- | :--- |
| **空间分配** | 需预先分配足够大的连续存储空间（静态分配） | 不需要预先分配，动态分配 |
| **空间开销** | 无需为逻辑关系增加额外存储开销 | 需增加额外指针域开销，存储密度小于1 |
| **存取方式** | 具有按元素序号随机访问的特点，时间复杂度 $O(1)$ | 不支持随机访问，只能顺序存取 |
| **插入/删除效率** | 需移动大约一半元素，效率低 | 无需移动元素，效率高 |


> Caveat (多选12 / 判断10): 链式存储查找第 $i$ 个元素的时间与 $i$ 值成正比；顺序存储查找第 $i$ 个元素的时间同 $i$ 值无关。

## Worked Examples

### 例 1：单链表插入指针操作判定 *(来自 exam-signals 候选与本章高频题型)*
*Tags: 单链表, 插入操作, 指针操作*
**问题**：在单链表指针为 `p` 的结点之后插入指针为 `s` 的结点，正确的操作是什么？
**解答**：
1. 明确单链表后插结点的核心原则：“先接管后继，再断开前驱”。
2. 必须先使待插入结点 `s` 的 `next` 指针指向原 `p` 的后继结点，即执行 `s->next = p->next;`。
3. 随后断开并重定向 `p` 的 `next` 指针，使其指向新结点 `s`，即执行 `p->next = s;`。
4. 得出结论：正确的语句序列为 `s->next=p->next; p->next=s;`。
