## Flynn 分类法（计算机系统结构分类）⭐

**Flynn 分类法**：1966年，Michael.J.Flynn 提出按指令流和数据流的多倍性对计算机系统结构进行分类。*(讲义 p104)*

- **指令流**：机器执行的指令序列*(讲义 p104)*
- **数据流**：由指令流调用的数据序列，包括输入数据和中间结果*(讲义 p104)*
- **多倍性**：在系统最受限的部件上，同时处于同一执行阶段的指令或数据的最大数目*(讲义 p104)*

| 指令流 | 数据流 | 名称 | 举例 |
|--------|--------|------|------|
| 1个 | 1个 | SISD | 传统的冯·诺依曼计算机 |
| 1个 | 多个 | SIMD | 向量计算机，阵列处理机 |
| 多个 | 1个 | MISD | [原文为问号，实际不存在] |
| 多个 | 多个 | MIMD | 多处理机，多计算机 |

*(讲义 p104)*

> Caveat: MISD 已被证明是不可能至少是不实际的，目前为止不存在这种类型的计算机。实际成为机器的类型只有 SISD、SIMD、MIMD 三种。*(题目多选2)*

**Flynn 分类法的局限**：*(讲义 p105)*
1. 分类对象主要是控制驱动方式下的串行处理和并行处理计算机，对于非控制驱动方式的计算机不适合采用 Flynn 分类法。
2. 把两个不同等级的功能并列对待，通常数据流受指令流控制从而造成 MISD 不存在。
3. 分类太粗，对流水线处理机的划分不明确：标量流水处理机为 SISD，向量流水处理机为 SIMD。

## 各类体系结构详解 ⭐

**SISD（单指令流单数据流）**：单一的指令流从存储器取指令，以单一的数据流从存储器取操作数和将结果写回存储器。*(讲义 p104)*
- 单功能部件处理机：IBM1401，VAX-11*(讲义 p104)*
- 多功能部件处理机：IBM360/370，CDC6600*(讲义 p104)*
- 流水线处理机：标量流水线处理机*(讲义 p104)*

**SIMD（单指令流多数据流）**：单一的控制部件，多个处理部件。计算机以一个控制单元从存储器取单一的指令流，一条指令同时作用到各个处理单元，控制各个处理单元对来自不同数据流的数据组进行操作。*(讲义 p104)*
- 典型代表：阵列处理机和向量流水处理机*(讲义 p104)*

**向量计算机**：可以在一个向量的每个元素上执行相同的操作，向量处理器以流水线式 ALU 为核心，实现向量各个元素的并行计算采用的是**时间重叠技术**。*(讲义 p105)*

**阵列计算机**：采用**资源重复方法**引入空间因素，在系统中设置多个相同的处理单元来开发并行性。利用并行性中的**同时性**，所有处理单元必须同时进行相同操作。*(讲义 p105-106)*

**MISD（多指令流单数据流）**：多个处理单元各配相应的控制单元，接收不同的指令，多条指令同时在一份数据上进行操作。已被证明是不可能至少是不实际的，目前为止不存在。*(讲义 p105)*

**MIMD（多指令流多数据流）**：同时有多个处理单元，每个处理单元都配有相应的控制单元，可以接收不同的指令并对不同的数据流进行操作。大多数现代并行计算机都属于这一类。*(讲义 p105)*
- 例：IBM3081、IBM3084、UNIVAC-1100/80、CRAY-2*(讲义 p105)*

**MIMD 的两大子类**：*(讲义 p106)*

| 类型 | 存储模型 | 特点 |
|------|----------|------|
| 多处理机系统 | 共享存储器 | 系统中只有唯一的地址空间，所有处理器共享该地址空间；可通过物理共享存储器或分布式存储器+软硬件支持实现 |
| 多计算机系统 | 消息传递 | 每个处理器有自己的私有存储器（局部存储器），只能被该处理器访问；处理器间通过消息形式传送数据 |

*(讲义 p106)*

> Caveat: 共享存储器的多处理机一般没有自己的（私有）存储器，"每个处理器有自己的存储器"是多计算机系统的特征，不是多处理机系统的特征。*(题目多选3)*

## 并行计算机体系结构图谱

依据 Flynn 分类法归纳的并行计算机体系结构图谱：*(讲义 p106)*

```
计算机系统结构
├── SISD ──→ 冯·诺依曼
├── SIMD ──→ 向量计算机 / 阵列计算机
├── MISD ──→ [不存在]
└── MIMD ──→ 多处理机系统（共享内存）
│           ├── UMA
│           ├── COMA
│           └── NUMA（CC-NUMA / NC-NUMA）
│               互连方式：总线 / 交换结构
└── 多计算机系统（消息传递）
    ├── MPP
    └── COW
        互连方式：网格 / 超立方体
```

*(讲义 p106)*

## 并行计算机系统发展的原因

**发展原因**（三点）：*(讲义 p106)*
1. **应用需求永远是动力**——随着计算机速度提高，人们对性能要求也越来越高（科学计算、工程和工业设计等需要高性能计算）；芯片速度受光速制约，但集成度还有发展空间，并行计算机可以处理越来越复杂的问题。
2. **大量商品化的处理器的出现**为设计并行计算机系统提供了可能。
3. 并行计算机系统获得快速发展和**处理器间通信技术的发展**密不可分。

## 互连网络概述与分类 ⭐

**互连网络**：并行计算机系统内部的互连网络，由开关元件按一定拓扑结构和控制方式构成的网络，以实现计算机系统内部多个处理器或多个功能部件间的相互连接。*(讲义 p106)*

- 与计算机网络在工作原理、概念以及术语上有许多相同或相似之处*(讲义 p106)*
- 某些并行计算机系统中的互连网络就是高速以太网和 ATM 网络*(讲义 p106)*

**互连网络的组成**（五部分）：CPU、内存模块、接口、链路和交换结点。*(讲义 p106)*

**互连网络分类**：*(讲义 p106-107)*

| 类型 | 定义 | 举例 |
|------|------|------|
| 静态网络 | 结点间有固定连接通路，程序执行期间连接保持不变 | 线性阵列、环形、全连接、树形、网格型、环网型 |
| 动态网络 | 由开关单元构成，可按应用程序要求动态改变连接状态 | 总线、交叉开关、多级交换网络 |

*(讲义 p106-107)*

> Caveat: 交叉开关是**动态**互连网络结构，不是静态互连网络。*(题目判断8)*

> Caveat: 动态互连网络中，只有在互连网络**边界上**的开关元件才能与处理机相连。*(题目多选9)*

## 互连网络参数 ⭐⚠

**互连网络参数定义**：*(讲义 p107)*

| 参数 | 定义 |
|------|------|
| 结点度 | 与结点相连接的边数 = 入度 + 出度 |
| 链路的长度 | 链路中包含的边数 |
| 距离 | 两个结点之间相连的**最少**边数 |
| 网络直径 | 网络中任意两个结点间距离的最大值 |
| 网络规模 | 网络中结点数 |
| 等分宽度 | 某一网络被切成相等的两半时，沿切口的最小边数 |
| 对称性 | 从任何结点看，拓扑结构都一样 |

*(讲义 p107)*

> Caveat: 结点距离是两个结点之间跨越的图的边的**最少**条数，不是最多条数。*(题目判断6)*

> Caveat: 互连网络总通信带宽指所有边对应的网络通信带宽**之和**。*(题目判断7)*

**常见静态互连网络拓扑参数对比**（$N$ 个结点）：*(题目多选6-8)*

| 拓扑 | 链路数 | 直径 | 结点度 | 对称性 | 等分宽度 |
|------|--------|------|--------|--------|----------|
| 线性阵列 | $N-1$ | $N-1$ | $2$ | 不对称 | $1$ |
| 双向环 | $N$ | $N/2$ | $2$ | 对称 | $2$ |
| 单向环 | $N$ | $N-1$ | $2$ | 对称 | $2$ |
| 星形 | $N-1$ | $2$ | 最大 $N-1$ | 非对称 | $1$ |

*(题目多选6-8)*

> Caveat: 线性阵列的等分宽度为 **1**（不是 2），$N$ 很大时通信效率很低。*(题目多选6)*

> Caveat: 星形实际上是一种二层树，最大结点度为 $N-1$（中心结点），非对称。*(题目多选8)*

## GPU 概述与发展

**GPU（Graphics Processing Unit）**：现代显卡中非常重要的部分，地位与 CPU 在主板上一致，负责加速图形处理速度。GPU 是一个高度并行化的多线程、多核心处理器。*(讲义 p107)*

- GPU 为图形处理设计，具有天然的并行特性*(讲义 p107)*
- NVIDIA 在 1999 年推出第一款 GPU 产品—GeForce 256*(讲义 p107)*
- 主要任务是进行图形渲染任务，缓解 CPU 压力*(讲义 p107-108)*

**GPGPU（General Purpose Graphic Process Unit）**：将通用计算问题移植到 GPU 上完成以提升效率，但开发难度较大，未被广泛接受。*(讲义 p108)*

**CUDA（Computer Unified Device Architecture）**：2007年6月 NVIDIA 推出，是一种将 GPU 作为数据并行计算设备的软硬件体系。*(讲义 p108)*
- 不再将通用计算映射到图形 API 中，开发门槛大幅降低*(讲义 p108)*
- 编程语言基于标准 C*(讲义 p108)*
- 广泛应用于石油勘测、天文计算、流体力学模拟、分子动力学仿真、生物计算、图像处理、音视频编解码等领域*(讲义 p108)*

**GPU 产品线分类**：*(题目单选7)*

| 产品线 | 面向用户 | 用途 |
|--------|----------|------|
| GeForce | 家庭和企业娱乐 | 面向游戏用户 |
| Quadro | 图形工作站 | 面向专业级用户 |
| Tesla | 高性能通用计算 | 面向研究人员 |
| Tegra | 移动设备 | [原文未述] |

*(题目单选7)*

## GPU 内部架构 ⭐

**CPU vs GPU 设计差异**：*(讲义 p108-109)*

| 特性 | CPU | GPU |
|------|-----|-----|
| 设计倾向 | 强控制弱计算 | 强计算弱控制 |
| 资源分配 | 更多资源用于缓存 | 更多资源用于数据计算 |
| 线程模型 | 重线程 | 轻线程 |
| 多核类型 | MIMD 多核 | SIMT 众核 |
| 适用场景 | 复杂逻辑运算，大容量缓存减小延迟 | 计算密度高、逻辑分支简单的大规模数据并行负载 |

*(讲义 p108-109)*

**Tesla GT200 架构**：*(讲义 p109)*
- 由**可伸缩流处理器阵列（SPA）**和**存储器系统**两部分组成，由片上互联网络连接*(讲义 p109)*
- SPA 由若干个**线程处理器群（TPC）**构成*(讲义 p109)*
- 每个 TPC 包含 2～3 个**流多处理器（SM）***(讲义 p109)*
- 每个 SM 包含 8 个**流处理器（SP）***(讲义 p109)*

**SP 与 SM 的关系**：*(讲义 p109)*
- SP 有独立的寄存器和指令指针，但**缺少取指和调度单元***(讲义 p109)*
- SM 拥有完整前端，包括取值、译码、发射等*(讲义 p109)*
- 每个 SM 相当于一个 8 路 SIMD 处理器*(讲义 p109)*

**SIMT（单指令流多线程）**：GPU 实现了自动向量机化，NVIDIA 将之命名为 Single Instruction Multiple Thread。整个 SPA 可看成由多个 SM 组成的**多单指令流多线程（MSIMT）系统**。*(讲义 p109)*

**线程组织方式**：线程被组织成多个**线程块（Thread Block）**，分配到各个 SM 上，每个线程块内的线程以 SIMT 方式交给 SP 运行。*(讲义 p109)*

## GPU 编程

**CUDA 基本思想**：支持大量的线程级并行（Thread Level Parallel），在硬件中动态地调度和执行这些线程。*(讲义 p109)*

**异构思想**：Host + Device（Coprocessor），让 GPU 运行能够被高度线程化的代码。GPU 只有在计算高度数据并行任务时才发挥作用。*(讲义 p109-110)*

**OpenCL（Open Computing Language）**：第一个面向异构系统通用目的并行编程的开放式、免费标准，也是一个统一的编程环境，广泛适用于多核心处理器（CPU）和图形处理器（GPU）。*(讲义 p110)*

## 集群技术 ⭐

**集群技术概述**：集群技术是实现系统高可用性、可靠性、稳定性、安全性并需要快速处理能力、大容量存储及系统的可管理性的重要手段。*(讲义 p110)*

**集群定义**：一组相互独立的服务器在网络中表现为单一的系统，并以单一系统的模式加以管理，为客户工作站提供高可靠性的服务。*(讲义 p110)*
- 将多台同构或异构的计算机连接起来协同完成特定任务*(讲义 p110)*
- 对外表现为一台虚拟主机*(讲义 p110)*
- 可透明地向 Cluster 中加入组件*(讲义 p110)*

> Caveat: 机群系统的节点可以是**同构或异构**的，允许使用不同型号的处理器和不同的操作系统。*(题目单选3)*

> Caveat: 机群系统的节点通常是**完整的计算机**，拥有本地磁盘、完整操作系统，且可脱离机群独立运行——这是 COW 区别于 MPP 的重要特征。*(题目单选3)*

> Caveat: 机群系统基于**消息传递**机制进行节点间通信，每个节点拥有独立的地址空间，**不通过共享内存**通信。*(题目单选3)*

> Caveat: 机群**没有共享磁盘**。*(题目单选11)*

> Caveat: 机群系统具有**单一系统映像**的特点——作为一个单独的统一资源来使用。*(题目单选10)*

> Caveat: 机群是**松耦合**结构，MPP 是另外一种紧耦合的结构。*(题目单选10)*

**集群实现方式**：*(讲义 p110)*
- 一个 Cluster 包含两台以上拥有共享数据存储空间的服务器*(讲义 p110)*
- 应用数据存储在共享的数据空间内，操作系统和应用程序文件存储在各节点本地存储空间*(讲义 p110)*
- 各节点通过内部局域网相互通讯*(讲义 p110)*
- 节点故障时应用程序在另一节点自动接管*(讲义 p110)*

**集群分类**：*(讲义 p110)*

| 分类方式 | 类型 |
|----------|------|
| 按功能侧重 | 高可用性集群、高性能集群 |
| 按设备 | 镜像服务器双机、双机与磁盘阵列柜 |

*(讲义 p110)*

**高可用性集群 vs 高性能集群**：*(讲义 p110)*

| 类型 | 目标 | 典型应用 |
|------|------|----------|
| 高可用性集群 | 提供不间断的服务 | Web 服务器、ATM、医学与军事监测仪、股票处理机 |
| 高性能集群 | 多台机器连接同时处理复杂计算问题 | 模拟星球磁场、预测龙卷风等大量数据处理 |

*(讲义 p110)*

> Caveat: 可用性用**平均无故障时间（MTTF）**来度量，即系统平均能正常运行多长时间才发生一次故障。*(题目单选9)*

> Caveat: 刀片服务器属于机群系统——每个刀片相当于一台完整的服务器，共享机箱电源/冷却/网络，通过高速网络互连。*(题目单选18)*

## 超线程 / 多核 / 超标量概念辨析 ⭐⚠

> 以下概念在讲义知识点正文中未展开定义，但在本章专项练习中高频出现，依据答案解析整理。

**超线程技术（Hyper-Threading / SMT）**：在一个物理处理器内核上实现多个逻辑处理器，使其能够同时处理多个线程。*(题目单选1、13、20)*

> Caveat: 超线程是在一个物理核心内模拟多个逻辑核心，**共享运算器**等资源（没有自己独立的运算器），涉及资源冲突时仍要串行。*(题目单选1、14)*

> Caveat: 超线程属于**线程级并行**，并行级别**高于指令级**，不是指令级并行技术。*(题目单选13)*

> Caveat: 超线程**并非多核技术**，而是在一个核心内模拟同时执行两个线程。*(题目单选20)*

> Caveat: 超线程性能**不等于**两个 CPU 的性能之和。*(题目单选1)*

> Caveat: 超线程技术可以和多核技术**同时应用**。*(题目单选13)*

**多核处理机**：在一颗处理机芯片内集成两个或两个以上完整且并行工作的计算引擎（核），核间通过某种方式互联，对外呈现为一个统一的多核处理机。*(题目单选2)*

> Caveat: 多核是**物理**内核，不是"逻辑"处理机内核（后者是超线程的描述）。*(题目单选2)*

> Caveat: 多核处理机核间耦合度高，可以在**任务级、线程级和指令级**等多个层次充分发挥程序的并行性。*(题目单选19)*

**超标量流水线技术**：多条流水线并行工作，提高指令级并行度。*(题目单选2、12)*

> Caveat: 超标量流水线能在一个时钟周期内同时发射多条指令，能结合动态调度技术提高指令执行并行性，但**不会缩短**流水线功能段的处理时间。*(题目单选12)*

> Caveat: 超标量技术让多条流水线同时运行，实质是以**空间换取时间**。*(题目单选15)*

**超线程 vs 多核 vs 超标量对比**：*(题目单选1-2、12-14、19-20)*

| 技术 | 并行级别 | 核心类型 | 独立运算器 | 成本 |
|------|----------|----------|------------|------|
| 超线程（SMT） | 线程级 | 逻辑核心 | 否（共享） | 低 |
| 多核 | 任务/线程/指令级 | 物理核心 | 是 | 高 |
| 超标量流水线 | 指令级 | [原文未述] | [原文未述] | [原文未述] |

*(题目单选1-2、12-14、19-20)*

## 并行性概念与等级 ⭐⚠

> 以下概念依据本章专项练习答案解析整理。

**并行性定义**：两个以上的事件在同一时刻发生或同一时间间隔内发生。*(题目单选4)*
- 同一时刻发生严格称为"并发"，也是并行的一种*(题目单选4)*

**并行性等级**（从执行程序角度看，由低到高）：*(题目单选5、8)*

| 等级 | 说明 |
|------|------|
| 指令内部并行 | 最低级 |
| 指令级并行 | 在处理机内部 |
| 任务级或过程级并行 | 过程是程序的一部分 |
| 作业或程序级并行 | 最高级（最宏观） |

*(题目单选5、8)*

> Caveat: 多处理机实现的是**作业或程序级**并行，指令级并行是在处理机内部的，不是处理器之间的。*(题目单选8)*

**数据并行性等级**（从处理数据角度看）：*(题目单选16)*

| 等级 | 是否存在并行性 |
|------|----------------|
| 字串位串 | 否（两级都串行） |
| 字串位并 | 是 |
| 字并位串 | 是 |
| 字并位并 | 是 |

*(题目单选16)*

## 其他高频考点

**MPP（大规模并行处理机）**：每个结点有独立存储器，耦合度低于 SMP、PVP、DSM 等紧耦合系统。*(题目单选17)*

> Caveat: MPP 属于**松耦合**结构；SMP、PVP、DSM 属于紧耦合系统。*(题目单选17)*

**NUMA 多处理器系统特点**：*(题目多选10)*
1. 所有的 CPU 都看到一个单一的地址空间
2. 使用 LOAD 和 STORE 指令访问远程内存
3. 访问远程内存比访问本地内存慢
4. NUMA 系统中的处理器可使用高速缓存

*(题目多选10)*

**SMP（对称多处理）特征**：多个 CPU 共享一个统一的内存地址空间，所有 CPU 通过共享的内存总线进行通信和数据访问，CPU 对称地共享内存和 I/O 资源。*(题目判断4)*

> Caveat: SMP **不是**以结点为单位、每个结点包含一个或多个 CPU——SMP 中 CPU 对称共享内存，没有明确结点划分。*(题目判断4)*

**存储墙**：CPU 和存储系统的性能差距。处理器跟随摩尔定律逐年提升性能，而存储器性能提升缓慢，两者差距形成"存储墙"。*(题目单选15)*

**互连结构对比**：*(题目单选19)*

| 互连结构 | 冲突 | 带宽 |
|----------|------|------|
| 总线结构 | 冲突大 | 较低 |
| 交叉开关 | 冲突小 | 较高 |

*(题目单选19)*

> Caveat: 与交叉开关结构相比，总线结构**不能**有效提高核间数据交换的带宽。*(题目单选19)*

**并行程序性能优化方法**：*(题目判断9)*
1. 减少通信量、提高通信粒度（减少通信时间是并行程序设计中首先要考虑的问题）
2. 全局通信尽量利用高效聚合通信算法
3. 挖掘算法的并行度，减少 CPU 空闲等待
4. 负载平衡
5. 通信、计算的重叠
6. 通过引入重复计算来减少通信

*(题目判断9)*

**并行程序性能评价**：加速比和效率是衡量一个并行程序性能的最基本的评价方法。*(题目判断10)*

**并行算法分类**：按运算基本对象分为数值并行算法和非数值并行算法。*(题目判断3)*
