# 操作系统 · 第五章 设备管理 — 复习资料

> 来源：4-3-操作系统-讲义.pdf 物理页 [94, 116]
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与答案标注）。

## 一、本章概览
本章主要解决操作系统如何管理计算机系统中的各类 I/O 设备及外存（磁盘）的问题。主要包含四大块：一是 I/O 设备的基本概念、控制器及 I/O 子系统的层次结构；二是 I/O 控制方式的演变（程序查询、中断驱动、DMA、通道）；三是设备分配的数据结构与策略，以及利用 SPOOLing 技术实现虚拟设备；四是磁盘的物理结构与多种磁盘调度算法、RAID 技术。本章与进程管理（中断与并发）、文件管理（磁盘存储基础）联系极其紧密，是操作系统的核心章节。

## 二、知识点详解

### 1. I/O设备 ⭐(练习2,13,22)
- **分类**：
  - **按使用特性分**：人机交互类外部设备（慢，字节单位）、存储设备（快，块单位）、网络通信设备（速度介于两者之间）。
  - **按传输速率分**：低速设备（如键盘鼠标）、中速设备（如打印机）、高速设备（如磁盘机）。
  - **按信息交换的单位分**：
    - **块设备**：信息存取以数据块为单位，属于有结构设备（如磁盘）。特征：传输速率较高、可寻址、可随机读写。
    - **字符设备**：传输基本单位为字符，属于无结构类型（如终端、打印机）。特征：传输速率低、不可寻址、常采用中断驱动方式。
  - **按共享属性分类**：
    - **独占设备**：一段时间内只允许一个进程访问（临界资源），应互斥访问。
    - **共享设备**：一段时间内允许多个进程同时访问（如磁盘），但每一时刻仍然只允许一个进程访问。
    - **虚拟设备**：通过虚拟技术将一台独占设备变换为若干台逻辑设备，供若干个进程同时使用。

- **设备控制器的基本功能**：接收和识别命令、数据交换、标识和报告设备的状态、地址识别、数据缓冲区、差错控制。
- **设备控制器的组成**：位于 CPU 与设备之间。
  - **设备控制器与处理机的接口**：含数据线、地址线和控制线。数据线连着两类寄存器：①数据寄存器（放输入输出数据）；②控制/状态寄存器（放控制信息或设备状态）。
  - **设备控制器与设备的接口**：可连多个设备，含数据、控制和状态三种信号。
  - **I/O逻辑**：用于实现对设备的控制，对 CPU 发来的地址译码并控制所选设备。

### 2. I/O子系统层次 ⭐(练习7)
整个 I/O 系统被视为具有 4 个层次的系统结构（从上到下）：
1. **用户层I/O软件**：实现与用户交互的接口，用户可直接调用与 I/O 操作有关的库函数。必须通过一组系统调用来获取操作系统服务。
2. **设备独立性软件（设备无关性软件）**：使应用程序独立于具体使用的物理设备。引入逻辑设备和物理设备概念（应用程序中使用**逻辑设备名**请求）。功能包括：实现用户程序与设备驱动器的统一接口、设备命名、保护、分配与释放，为数据传送提供存储空间。
3. **设备驱动程序**：与硬件直接相关，具体实现系统对设备发出的操作指令，驱动设备工作。
4. **中断处理程序**：保存被中断进程的 CPU 环境，转入相应的中断处理程序进行处理，完毕后恢复现场并返回被中断的进程。

### 3. I/O控制方式 ⭐(练习19,25)
- **程序查询方式**：CPU 向控制器发出 I/O 指令，置忙/闲标志 busy=1，然后不断循环测试 busy（轮询）。直到 busy=0 说明数据已送入控制器数据寄存器，CPU 取数据送入内存。
- **中断驱动方式**：
  - 允许 I/O 设备主动打断 CPU 运行并请求服务。CPU 发读命令后保存上下文去执行其他程序，在每个指令周期末尾检查中断。
  - 比程序查询有效，但数据在每个字传输时都必须经过 CPU 寄存器，消耗较多 CPU 时间。
- **DMA方式（直接存储器存取）**：
  - 基本思想：I/O 设备和内存间开辟直接的数据交换通路。
  - 特点：①基本单位是**数据块**；②数据从设备直接送入内存（或相反）；③仅在传送一个或多个数据块的开始和结束时需 CPU 干预。
  - **DMA控制器组成（4类寄存器）**：命令/状态寄存器（CR）、内存地址寄存器（MAR）、数据寄存器（DR）、数据计数器（DC）。
  - **与中断驱动方式的主要区别**：中断驱动在每个数据传输时中断 CPU，DMA 是在整批数据全部传送结束时才中断 CPU；中断驱动数据传送由 CPU 控制，DMA 由 DMA 控制器控制。
- **通道方式**：
  - I/O 通道是专门负责输入/输出的处理机，是 DMA 方式的发展。把对数据块为单位的干预减少为对一组数据块及控制管理的干预，实现 CPU、通道和 I/O 设备三者并行操作。
  - 与一般处理机区别：指令类型单一，没有自己的内存，与 CPU 共享内存。
  - 与 DMA 方式区别：通道自己控制传输数据块大小和内存位置；一个通道可控制多台设备与内存交换数据（DMA 控制器对应一台设备）。

### 4. I/O核心子系统
- **I/O调度**：确定一个好顺序来执行 I/O 请求。通过为每个设备维护请求队列，重新安排队列顺序，改善总体效率和平均响应时间。
- **高速缓存的引入**：利用内存中的存储空间暂存从磁盘读出的盘块信息。逻辑上属于磁盘，物理上驻留在内存。
- **引入缓冲区的原因**：①缓和 CPU 与 I/O 设备间速度不匹配矛盾；②减少对 CPU 的中断频率，放宽中断响应时间限制；③解决数据粒度不匹配；④提高 CPU 和 I/O 设备间的并行性。

### 5. 缓冲区机制 ⭐(练习9,10,26,36)
- **单缓冲**：设备和处理机之间设置一个缓冲区。
  - 设 $T$ 为输入一块数据到缓冲区时间，$M$ 为缓冲区传送到用户区时间，$C$ 为 CPU 处理时间。
  - 处理每块数据用时为 $\max(C,T)+M$。
- **双缓冲**：
  - I/O 设备先装填缓冲区1，装满后装填缓冲区2，处理机同时从缓冲区1取数据处理。必须等缓冲区充满才能让处理机取出数据。提高了处理机和输入设备并行操作程度。
- **缓冲池**：
  - 系统的共用资源，可供多个进程共享，既能用于输入又能用于输出。包含三种类型缓冲区：空闲缓冲区、装满输入数据的缓冲区、装满输出数据的缓冲区。

### 6. 设备的分配和回收 ⭐(练习3,21,27,34)
- **设备分配总原则**：充分发挥设备使用效率，尽可能让设备忙碌，避免不合理分配造成死锁。使用方式分为独占式使用设备、分时式共享使用设备、以 SPOOLing 方式使用设备。
- **设备分配的数据结构**：设备控制表、控制器控制表（COCT）、通道控制表（CHCT）、系统设备表（SDT）。
  - 设备映射表（DMT）作用：建立逻辑设备与物理设备的对应关系。
- **设备分配方式**：
  - **静态分配**：主要用于独占设备。作业执行前一次性分配，直到撤销。不会死锁，但设备利用率低。
  - **动态分配**：进程执行中按需请求，用完立即释放。利用率高，但算法不当可能死锁。
- **设备分配算法**：常用的有先请求先分配、优先级高者优先。
- **设备分配的安全性**：
  - **安全分配方式**：进程发出 I/O 请求后阻塞，直到完成才唤醒。特点：分配安全，但 CPU 与设备串行工作（对同一进程）。
  - **不安全分配方式**：进程发 I/O 请求后继续运行，可发多个请求。特点：可操作多设备迅速推进，但可能产生死锁。

### 7. SPOOLing技术（假脱机操作） ⭐(练习4,12,16,24,29,35,38,40)
- **相关概念**：为缓和 CPU 高速性与 I/O 设备低速性矛盾引入脱机技术。利用外围控制器，先将低速 I/O 设备数据传送到高速磁盘（或相反）。外围操作与 CPU 处理同时进行，在联机情况下实现同时外围操作称为 SPOOLing。
- **本质**：以**空间换时间**的技术。
- **SPOOLing 的组成**：
  1. **输入井和输出井**：磁盘上开辟的存储区域。模拟脱机输入输出的磁盘。收容 I/O 设备输入数据或用户程序输出数据，一般以井文件形式组织。
  2. **输入缓冲区和输出缓冲区**：内存中开辟，暂存 CPU 与磁盘间传送的数据。
  3. **输入进程和输出进程**：模拟脱机时的外围控制器（预输入进程、缓输出进程）。
  4. **井管理程序**：控制作业与磁盘井间信息交换。
- **SPOOLing 的特点**：①提高 I/O 速度；②将独占设备改造为共享设备；③实现了虚拟设备功能。
- **假脱机打印机系统**：包含磁盘缓冲区、打印缓冲区、假脱机管理进程和假脱机打印进程。实际上没有分配物理设备，只是分配磁盘空闲块和建立请求表。

### 8. 磁盘管理 ⭐(练习1,17,18,20,32,33)
- **磁盘访问时间**：分为寻道时间、旋转延迟时间、传输时间。
- **磁盘调度算法**：
  - **FCFS算法（先来先服务）**：按请求先后次序调度。公平简单，但平均寻道时间可能较长。
  - **SSTF算法（最短寻道时间优先）**：要求访问的磁道与当前磁道距离最近。不能保证平均寻道时间最短，可能导致“饥饿”。
  - **SCAN算法（电梯调度算法）**：优先考虑磁头当前移动方向。磁头从一端向另一端移动，沿途服务，无请求时反向。避免饥饿。
  - **CSCAN算法（循环扫描）**：磁头单向移动（如自里向外），移到最外并访问后，立即返回最里欲访问磁道（最小磁道号紧接最大磁道号），循环扫描。

### 9. 磁盘阵列 (RAID)
- **RAID0**：仅提供并行交叉存取，无冗余或错误修复，成本低，适合数据安全性要求不高的场合。
- **RAID1**：具有磁盘镜像功能。读请求最短寻道时间，但磁盘利用率只有 50%（牺牲容量为代价）。
- **RAID2**：数据字或字节交叉存放，使用海明校验码，适合大量顺序数据。多个冗余盘成本较高。
- **RAID3**：并行传输，RAID2的简化版。利用一台奇偶校验盘。如7盘系统，6数据1校验，利用率6/7。
- **RAID4**：独立存取，数据条带交叉存放。使用一只冗余盘存放奇偶校验码。
- **RAID5**：校验块分散到所有数据盘块中，独立传送，常用于 I/O 频繁的事务处理。
- **RAID6**：双重冗余技术，允许两个磁盘出错也能恢复。专用异步校验盘，昂贵。
- **RAID7**：优化的高速数据传送结构，所有 I/O 同步进行。每个磁盘带高速缓存，是目前最高档次的磁盘阵列。

## 三、公式 / 算法 / 定理速查表

**磁盘访问时间公式**：
- **寻道时间**：$T_s = m \times n + s$ （$m$ 为常数，与磁盘驱动器速度有关，一般 $m=0.2$，高速盘 $m \le 0.1$；$n$ 为移动磁道数；$s$ 为磁臂启动时间，约 $2\text{ms}$）。
- **旋转延迟时间**：$T_r$ （硬盘 15000r/min，平均 $T_r \approx 2\text{ms}$；软盘平均 $T_r \approx 50\sim100\text{ms}$）。
- **传输时间**：$T_t = \frac{b}{rN}$ （$b$ 为每次读写字节数；$r$ 为每秒转数；$N$ 为一条磁道上的字节数）。
- **总访问时间**：$T_a = T_s + \frac{1}{2r} + \frac{b}{rN}$

**单缓冲区处理时间公式**：
- $\max(C,T)+M$ （$C$ 为 CPU 处理时间，$T$ 为设备输入到缓冲区时间，$M$ 为缓冲区传到用户区时间）。

## 四、重点对比与易混辨析

### 1. I/O控制方式对比
| 控制方式 | 传输单位 | CPU干预程度 | 数据通路 |
| :--- | :--- | :--- | :--- |
| **程序查询** | 字（符） | 极高，需不断轮询测试状态 | 设备 $\leftrightarrow$ CPU $\leftrightarrow$ 内存 |
| **中断驱动** | 字（符） | 较高，每传一个字需 CPU 中断处理 | 设备 $\leftrightarrow$ CPU $\leftrightarrow$ 内存 |
| **DMA方式** | 数据块 | 低，仅头尾介入 | 设备 $\leftrightarrow$ 内存 (不经过 CPU 寄存器) |
| **通道方式** | 一组数据块 | 最低，通道自行控制 | 设备 $\leftrightarrow$ 内存 (通道自带指令控制) |

### 2. 磁盘调度算法对比
| 算法 | 依据 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **FCFS** | 请求先后次序 | 公平，不会出现长期得不到满足 | 平均寻道时间长 |
| **SSTF** | 当前磁头最近的距离 | 寻道时间较短 | 可能导致远端进程“饥饿” |
| **SCAN** | 移动方向 + 距离 | 避免饥饿，兼顾两端 | 中间区域响应较快，两端可能等待 |
| **CSCAN** | 单向移动的循环扫描 | 各个位置响应时间更均匀 | 返程时不服务，存在空跑 |

### 3. 缓冲区机制对比
| 缓冲类型 | 数量 | 特点 |
| :--- | :--- | :--- |
| **单缓冲** | 1 | 处理时间受限于 $\max(C,T)+M$ |
| **双缓冲** | 2 | 交替装填，提高并行性，同时间内可双向传输 |
| **缓冲池** | 多个 | 共享资源，可链成队列（空闲、装满输入、装满输出） |

## 五、高频考点与易错提醒

- ⚠ **设备分类属性辨析**是高频易错点：块设备与字符设备、独占设备与共享设备的根本区别要明确。打印机、磁带机属于独占，磁盘属于共享(练习13,22)。
- ⚠ **DMA基本概念**是易错点：DMA 是一个控制器而非处理机，需要 CPU 指出所取数据的地址和长度，数据传输不需要 CPU 干预（练习25）。
- ⚠ **设备独立性（无关性）软件**职责是易错点：用户程序使用逻辑设备名请求，由设备无关软件翻译为设备操作命令，更换物理设备无需修改应用层程序（练习30）。
- ⚠ **SPOOLing技术本质**是高频易错点：是假脱机真联机技术，空间换时间，实现虚拟设备分配，必须需要外存(磁盘)和多道程序设计支持（练习29,38）。
- ⚠ **缓冲区技术**易错点：缓冲区主要目的除了速度匹配，还有提高并行性，可由内存实现，不仅限硬件，可同时用于块和字符设备（练习26,36）。
- ⚠ **SSTF算法产生饥饿**机制是常考陷阱：因为总是寻找最近磁道，容易把磁头固定在某区域，导致远处请求长期等待(练习18)。

## 六、一句话记忆要点
1. **I/O设备**：块可寻址、字符不可寻址；共享允许多进程并发但单时刻互斥。
2. **I/O控制方式**：程序查询忙等待，中断驱动断字频，DMA 通块道宽，通道独机批处理。
3. **缓冲与SPOOLing**：缓冲解速差提并行，SPOOLing 空换时间造虚拟。
4. **设备分配**：数据结构四件套，逻辑映射看 DMT，动态静态防死锁。
5. **磁盘调度**：寻道主导时间差，FCFS 最公平，SSTF 怕饥饿，SCAN 电梯看方向，CSCAN 循环单向跑。
