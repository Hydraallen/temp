
25. D
解析：短作业优先（SJF）使平均周转时间最小。
运行时间：2h（J1）、3h（J3）、5h（J2）。正确顺序应为：2h → 3h → 5h，即 J1，J3，J2。
验证：
D（J1，J3，J2）：周转时间 = 2，(2+3)=5，(2+3+5)=10 → 平均 = (2+5+10)/3 = 17/3 ≈ 5.67
A（J1，J2，J3）：(2, 7, 10) → 平均 = 19/3 ≈ 6.33
B（J3，J2，J1）：(3, 8, 10) → 平均 = 21/3 = 7
C（J2，J1，J3）：(5, 7, 10) → 平均 = 22/3 ≈ 7.33
最小为 D。
考点：短作业优先调度算法的最优性（同时到达时）

26. C
解析：响应比 =（等待时间 + 运行时间）/ 运行时间 = 1 +（等待时间 / 运行时间）
到达时间：8:00，开始执行时间：10:00 → 等待时间 = 2h
运行时间 = 1h
响应比 = (2 + 1) / 1 = 3
考点：响应比计算公式（高响应比优先调度算法）

27. C
解析：A：高响应比优先调度算法。每次调度需计算响应比（涉及除法/浮点运算），开销较大。B：多级反馈队列调度算法。需维护多个队列，开销最大。C：先来先服务调度算法（FCFS）。只需维护一个就绪队列，按到达顺序调度，无计算开销，系统开销最小。D：时间片轮转调度算法。需定时器中断和周期性队列操作，开销大于FCFS。
考点：调度算法的系统开销比较

28. D
解析：A：时间片轮转调度算法。只考虑公平获得CPU，不考虑等待时间和执行时间。B：短进程优先调度算法。只考虑执行时间（短的优先），不考虑等待时间。C：先来先服务调度算法。只考虑等待时间（先到的优先），不考虑执行时间。D：高响应比优先调度算法。响应比 =（等待时间 + 执行时间）/ 执行时间 = 1 + 等待时间/执行时间，同时考虑了等待时间和执行时间。
考点：高响应比优先调度算法的特点

1. AD
解析：银行家算法是避免死锁的算法，故B选项错误；只要能使系统始终都处于安全状态，便可避免发生死锁，故C选项错误。
考点：银行家算法

2. BCD
解析：竞争可能会导致各进程的资源需求都得不到满足而无法继续执行，撤消进程和挂起进程实现的才能剥夺资源。系统死锁发生时，必然存在互斥条件、进程循环等待资源、不剥夺资源以及请求和保持条件。D选项破坏了死锁产生的四个必要条件中的请求和保持条件，能预防发生死锁。
考点：死锁的预防
