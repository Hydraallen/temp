## 一、本章概览

本章是操作系统的核心章节之一，解决的是**多道程序环境下如何描述、控制和管理并发执行的程序**这一问题。全章围绕「进程」这一基本概念展开，包含七大块：程序顺序/并发执行的特征对比 → 进程的概念与特征 → 进程状态及转换 → 进程控制（创建/终止/阻塞/唤醒/挂起/激活） → 进程组织（PCB结构与组织方式） → 进程通信（共享存储器/消息传递/管道/客户机-服务器） → 线程（引入动机、状态、实现方式、与进程对比）。本章是后续章节（处理机调度、同步与死锁）的基础，进程状态转换、PCB、线程概念是全章高频考点。

## 二、知识点详解

### 1. 程序的概念 ⭐(练习4)

#### 1.1 程序的顺序执行

一个应用程序由若干个程序段组成，每一个程序段完成特定的功能，它们在执行时，都需要按照某种先后次序顺序执行，仅当前一程序段执行完后，才运行后一程序段。

- 典型前趋关系：$I_i \rightarrow C_i \rightarrow P_i$（I=输入操作，C=计算操作，P=打印操作）
- 即使一个程序段内部也存在执行顺序，如：
  - $S_1: a=x+y;$
  - $S_2: b=a-5;$
  - $S_3: c=b+1;$
  - 前趋关系：$S_1 \rightarrow S_2 \rightarrow S_3$

#### 1.2 程序顺序执行的特征

1. **顺序性**：处理机严格地顺序执行，即每一操作必须在下一个操作开始之前结束。
2. **封闭性**：程序在封闭的环境[原文模糊，疑为"环境"]中运行，即程序运行时独占全机资源，资源的状态（除初始状态外）只有本程序才能改变它，程序一旦开始执行，其执行结果不受外界因素影响。
3. **可再现性**：只要程序执行时的环境和初始条件相同，当程序重复执行时，不论它是从头到尾不停顿地执行，还是"停停走走"地执行，都可获得相同的结果。

> 程序顺序执行时的这种特性，为程序员检测和校正程序的错误带来了很大的方便。

#### 1.3 程序的并发执行

1. **间断性**：程序在并发执行时，由于它们共享系统资源，以及为完成同一项任务而相互合作，致使在这些并发执行的程序之间形成了相互制约的关系。相互制约将导致并发程序具有"执行-暂停-执行"这种间断性的活动规律。
2. **失去封闭性**：当系统中存在着多个可以并发执行的程序时，系统中的各种资源将为它们所共享，而这些资源的状态也由这些程序来改变，致使其中任一程序在运行时，其环境都必然会受到其它程序的影响。
3. **不可再现性**：程序在并发执行时，由于失去了封闭性也将导致其又失去可再现性。

### 2. 进程的概念及特征 ⭐(练习4)(练习21)(练习25)

#### 2.1 进程的概念

在多道程序环境下，允许多个程序并发执行，此时它们将失去封闭性，并具有间断性及不可再现性的特征。为此引入了进程的概念，以便更好地描述和控制程序的并发执行，实现操作系统的并发性和共享性（最基本的两个特性）。

- **进程实体（进程映像）** = 程序段 + 相关数据段 + PCB（进程控制块）
- 创建进程，实质上是创建进程映像中的 **PCB**；撤销进程，实质是撤销进程的 **PCB**。
- 进程映像是**静态**的，进程则是**动态**的。
- **PCB 是进程存在的唯一标志！**

进程的典型定义：
1. 进程是程序的一次执行过程。
2. 进程是一个程序及其数据在处理机上顺序执行时所发生的活动。
3. 进程是具有独立功能的程序在一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位。

> 引入进程实体概念后的定义："进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。"

#### 2.2 进程的特征

| 特征 | 说明 |
|------|------|
| **动态性** | 进程的最基本特征。进程实质是进程实体的执行过程。"由创建而产生，由调度而执行，由撤消而消亡。"进程实体有一定的生命期，而程序只是一组有序指令的集合，是静态的。 |
| **并发性** | 多个进程实体同存于内存中，且能在一段时间内同时运行。引入进程的目的正是为了使其进程实体能和其它进程体并发执行。程序（没有建立PCB）不能参与并发执行。 |
| **独立性** | 进程实体是一个能独立运行、独立获得资源和独立接受调度的基本单位。凡未建立PCB的程序都不能作为一个独立的单位参与运行。 |
| **异步性** | 进程按异步方式运行，即按各自独立的、不可预知的速度向前推进。为使进程在并发运行时虽具有异步性，却仍保证结果的可再现，在OS中引进了进程的概念，并配置相应的进程同步机制。 |

### 3. 进程的状态及转换 ⭐⚠(练习13)(练习15)(练习16)(练习18)(练习27)(练习34)

#### 3.1 进程的三种基本状态

1. **就绪状态**：进程已处于准备好运行的状态，即进程已分配到除CPU以外的所有必要资源后，只要再获得CPU，便可立即执行。多个就绪进程通常按一定策略（如优先级策略）排成就绪队列。
2. **执行状态**：进程已获得CPU，其程序正在执行。单处理机系统中，任一时刻只有一个进程处于执行状态；多处理机系统中，则有多个进程处于执行状态。
3. **阻塞状态**（等待状态/封锁状态）：正在执行的进程由于发生某事件（如I/O请求、申请缓冲区失败等）暂时无法继续执行时的状态。此时引起进程调度，OS把处理机分配给另一个就绪进程，而让受阻进程处于暂停状态。通常将阻塞进程排成阻塞队列；较大系统中根据阻塞原因设置多个阻塞队列。

#### 3.2 三种基本状态的转换

| 转换 | 触发条件 |
|------|----------|
| 就绪 → 执行 | 调度程序为就绪状态的进程分配处理机 |
| 执行 → 阻塞 | 正在运行的进程因需要等待某事件（如I/O请求）而无法运行，让出处理机 |
| 阻塞 → 就绪 | 进程所等待的事件发生了（如I/O完成） |
| 执行 → 就绪 | 时间片用完被暂停运行；或可抢先式调度中，高优先级进程到来，低优先级进程被强制撤下处理机 |

> **关键：阻塞状态不能直接到执行状态。** 阻塞→就绪→执行。

#### 3.3 创建状态和终止状态

在三种基本状态基础上，增加：
- **创建状态**：进程正在被创建的过程。
- **终止状态**：进程已结束执行，但尚未完全撤销。

形成**五状态模型**：创建 → 就绪 → 执行 → 终止，以及执行 ↔ 阻塞。

#### 3.4 挂起操作

引起挂起的原因：
1. **终端用户的需要**：用户希望暂停程序运行，以便研究执行情况或修改程序。
2. **父进程请求**：父进程希望挂起某个子进程，以便考查和修改，或协调各子进程间的活动。
3. **负荷调节的需要**：实时系统工作负荷较重时，挂起一些不重要的进程，保证系统正常运行。
4. **操作系统的需要**：检查运行中的资源使用情况或进行记账。

#### 3.5 引入挂起原语后的状态转换

引入挂起原语 Suspend 和激活原语 Active 后：

| 转换 | 说明 |
|------|------|
| 活动就绪 → 静止就绪 | 用 Suspend 将 Readya 挂起为 Readys，处于 Readys 状态的进程不再被调度执行 |
| 活动阻塞 → 静止阻塞 | 用 Suspend 将 Blockeda 挂起为 Blockeds；其期待事件出现后，变为静止就绪 |
| 静止就绪 → 活动就绪 | 用 Active 激活 Readys 为 Readya |
| 静止阻塞 → 活动阻塞 | 用 Active 激活 Blockeds 为 Blockeda |

#### 3.6 引入挂起操作后五个进程状态的转换

增加考虑：
1. NULL → 创建
2. 创建 → 活动就绪
3. 创建 → 静止就绪
4. 执行 → 终止

### 4. 进程控制 ⭐(练习11)(练习12)(练习32)

#### 4.1 进程的原子操作

**原语即原子操作**，要么全做，要么不做，一般通过屏蔽中断来完成。

| 原语 | 功能 |
|------|------|
| create | 进程创建 |
| halt | 进程撤销 |
| block | 进程阻塞 |
| wakeup | 进程唤醒 |
| suspend | 进程挂起 |
| active | 进程激活 |

#### 4.2 进程的创建

**引起创建进程的事件（四类）：**
1. **用户登录**：分时系统中，用户登录成功后，系统为该用户建立一个进程，插入就绪队列。
2. **作业调度**：多道批处理系统中，作业调度程序按一定算法调度到某作业时，将其装入内存，为其创建进程，插入就绪队列。
3. **提供服务**：运行中的用户程序提出请求后，系统专门创建一个进程来提供服务（如打印进程）。
4. **应用请求**：由用户进程自己创建新进程，以便使新进程以同创建者进程并发运行的方式完成特定任务。

**进程的创建过程（Creat 原语步骤）：**
1. **申请空白 PCB**：为新进程申请获得唯一的数字标识符，并从 PCB 集合中索取一个空白 PCB。
2. **为新进程分配资源**：包括各种物理和逻辑资源，如内存、文件、I/O设备和CPU时间等。资源从操作系统或从其父进程获得。
3. **初始化进程控制块（PCB）**：
   - (I) 初始化标识信息：将系统分配的标识符和父进程标识符填入新PCB中；
   - (II) 初始化处理机状态信息：使程序计数器指向程序的入口地址，使栈指针指向栈顶；
   - (III) 初始化处理机控制信息：将[原文模糊，疑为"将"]进程的状态设置为就绪状态或静止就绪状态，优先级通常设置为最低优先级（除非用户以显式方式提出高优先级要求）。
4. **插入就绪队列**：如果进程就绪队列能够接纳新进程，便将新进程插入就绪队列。

#### 4.3 进程的终止

引起进程终止的事件：
1. **正常结束**：进程的任务已完成并准备退出运行。
2. **异常结束**：如存储区越界、保护错、非法指令、特权指令错、运行超时、算术运算错、I/O故障等。
3. **外界干预**：如操作员或操作系统干预、父进程请求和父进程终止。

#### 4.4 进程的阻塞和唤醒

**引起进程阻塞和唤醒的事件：**
- 向系统请求共享资源失败
- 等待某种操作的完成
- 新数据尚未到达
- 等待新任务的到达

**进程阻塞过程**：正在执行的进程发生上述事件后，通过调用阻塞原语 block 将自己阻塞。**阻塞是进程自身的一种主动行为**。进入 block 过程后，由于该进程还处于执行状态，所以应先立即停止执行，把进程控制块中的现行状态由"执行"改为阻塞，并将 PCB 插入阻塞队列。

**进程唤醒过程**：当被阻塞进程所期待的事件发生时，由有关进程调用唤醒原语 wakeup，将等待该事件的进程唤醒。wakeup 执行过程：首先把被阻塞的进程从等待该事件的阻塞队列中移出，将其 PCB 中的现行状态由阻塞改为就绪，然后再将该 PCB 插入到就绪队列中。

#### 4.5 进程的挂起与激活

- **挂起**：当出现引起进程挂起的事件时，系统利用挂起原语 suspend() 将指定的进程或处于阻塞状态的进程挂起。
- **激活**：当发生激活进程的事件时，系统利用激活原语 active() 将指定进程激活。

### 5. 进程组织 ⭐(练习14)(练习29)

进程是一个独立的基本单位[原文模糊]，由以下三部分组成，其中最核心的是进程控制块（PCB）。

#### 5.1 进程控制块 PCB

进程创建时，操作系统为它新建一个 PCB，该结构之后常驻内存，任意时刻都可以存取，并在进程结束时删除。PCB 是进程实体的一部分，是**进程存在的唯一标志**。

#### 5.2 PCB 的作用（五点）

1. **作为独立运行基本单位的标志**：程序配置了 PCB 后，就表示它是一个能在多道程序环境下独立运行的、合法的基本单位。PCB 已成为进程存在于系统中的唯一标志。
2. **能实现间断性运行方式**：进程因阻塞暂停运行时，必须保留 CPU 现场信息；再次被调度运行时，还需恢复 CPU 现场信息。系统将 CPU 现场信息保存在被中断进程的 PCB 中。
3. **提供进程管理所需要的信息**：PCB 中记录程序和数据在内存或外存中的始址指针；进程运行过程中访问文件或 I/O 设备也需借助 PCB 中的信息；PCB 中的资源清单记录进程所需的全部资源。
4. **提供进程调度所需要的信息**：PCB 中提供进程处于何种状态的信息；优先级调度算法需要知道进程的优先级；较为公平的调度算法需要知道进程的等待时间和已执行的时间等。
5. **实现与其它进程的同步与通信**：在采用信号量机制时，每个进程中都设置有相应的用于同步的信号量。PCB 中还具有用于实现进程通信的区域或通信队列指针等。

#### 5.3 PCB 的信息（四个方面）

1. **进程标识符**：用于唯一地标识一个进程。
   - 内部标识符：操作系统为每个进程赋予的惟一数字标识符（通常是进程序号），方便系统使用。
   - 外部标识符：由创建者提供，由字母、数字组成，用户（进程）访问该进程时使用。还可设置父进程标识及子进程标识、用户标识。
2. **处理机状态**：主要由处理机的各种寄存器中的内容组成。处理机被中断时，所有信息保存在 PCB 中，以便进程重新执行时从断点继续执行。
3. **进程调度信息**：进程的状态及有关进程调度的信息。
4. **进程控制信息**。

#### 5.4 程序段与数据段

- **程序段**：能被进程调度程序调度到 CPU 执行的程序代码段。程序可被多个进程共享，即多个进程可以运行同一个程序。
- **数据段**：可以是进程对应的程序加工处理的原始数据，也可以是程序执行时产生的中间或最终结果。

#### 5.5 进程控制的组织方式

| 方式 | 说明 | 特点 |
|------|------|------|
| **线性方式** | 所有 PCB 组织在一张线性表中，首址存放在内存专用区域 | 实现简单、开销小；每次查找需扫描整张表；适合进程数目不多的系统 |
| **链接方式** | 具有相同状态进程的 PCB 通过链接字链接成队列（就绪队列、若干阻塞队列、空白队列等）。就绪队列往往按优先级从高到低排列；阻塞队列根据阻塞原因排成多个 | 适合进程较多的系统 |
| **索引方式** | 根据所有进程状态建立几张索引表（就绪索引表、阻塞索引表等），各索引表首地址记录在内存专用单元中。每个索引表表目记录相应状态 PCB 在 PCB 表中的地址 | — |

### 6. 进程通信 ⭐(练习19)(练习22)

进程通信是指进程之间的信息交换。后续 P、V 操作称为**低级通信**。**高级通信**是指进程之间以较高的效率传送大量数据的通信方式。高级通信方式可分为三大类：共享内存、消息传递以及管道机制。

#### 6.1 共享存储器

在通信的进程之间存在一块可直接访问的共享空间，通过对这片共享空间进行写/读操作实现进程之间的信息交换。

> 注意：用户进程空间一般是独立的，进程运行期间不能访问其他进程的空间。要想让两个用户进程共享空间，必须通过特殊的系统调用实现，而进程内的线程是自然共享进程空间的。

#### 6.2 消息传递

在消息传递系统中，进程间的数据交换是以**格式化的消息**为单位的。若通信的进程之间不存在可直接访问的共享空间，则必须利用操作系统提供的消息传递方法实现进程通信。进程通过系统提供的**发送消息**和**接收消息**两个原语进行数据交换。

1. **直接通信方式**：发送进程直接把消息发送给接收进程，并将它挂在接收进程的消息缓冲队列上，接收进程从消息缓冲队列中取得消息。
2. **间接通信方式**：发送进程把消息发送到某个中间实体，接收进程从中间实体取得消息。这种中间实体一般称为**信箱**，又称**信箱通信方式**。广泛应用于计算机网络中，相应的通信系统称为电子邮件系统。

#### 6.3 管道通信

管道通信是消息传递的一种特殊方式。所谓"管道"，是指用于连接一个读进程和一个写进程以实现它们之间通信的一个共享文件，又名 **pipe 文件**。

- 写进程以字符流形式将大量数据送入（写）管道；
- 读进程从管道中接收（读）数据。
- 管道机制必须提供三方面的协调能力：**互斥、同步和确定对方的存在**。

#### 6.4 客户机-服务器系统

在网络环境的各种应用领域已成为当前主流的通信实现机制，主要实现方法分为三类：**套接字**、**远程过程调用**和**远程方法调用**。

### 7. 线程 ⭐⚠(练习2)(练习8)(练习23)(练习24)(练习26)(练习31)(练习35)(练习36)

#### 7.1 线程的引入

引入进程的目的是使多个程序能并发执行，提高资源利用率和系统吞吐量。引入线程，是为了**减少程序在并发执行时所付出的时空开销**，使 OS 具有更好的并发性。

**进程的两个基本属性：**
1. 进程是一个**可拥有资源的独立单位**：要能独立运行，必须拥有一定的资源（磁盘和内存地址空间、I/O设备、已打开的文件、信号量等）。
2. 进程是一个**可独立调度和分派的基本单位**：每个进程在系统中有唯一的 PCB，系统根据 PCB 感知进程存在并进行调度。

**线程——调度和执行的基本单位**：将进程的上述两个属性分开，由 OS 分开处理——不把作为调度和分派的基本单位也同时作为拥有资源的单位，做到"轻装上阵"；对于拥有资源的基本单位，不为之施以频繁的切换。

#### 7.2 线程的状态和线程控制块

**线程运行的三个状态：**
1. **执行状态**：线程已获得处理机而正在运行。
2. **就绪状态**：线程已具备了各种执行条件，只须再获得 CPU 便可立即执行。
3. **阻塞状态**：线程在执行中因某事件受阻而处于暂停状态。

**线程控制块 TCB** 包含：
- 线程标识符
- 寄存器状态（包括程序计数器 PC 和堆栈指针中的内容；堆栈中保存局部变量和返回地址）
- 线程运行状态
- 优先级
- 线程专有存储器（保存线程自己的局部变量拷贝）
- 信号屏蔽

#### 7.3 线程的实现方式

##### (1) 内核支持线程 KST

在内核的支持下运行，创建、阻塞、撤消和切换等都在内核空间实现。内核空间为每一个内核线程设置了一个线程控制块。当前大多数 OS 都支持内核支持线程。

**优点：**
- 在多处理器系统中，内核能够同时调度同一进程中的多个线程并行执行；
- 如果进程中的一个线程被阻塞了，内核可以调度该进程中的其它线程占有处理器运行，也可以运行其它进程中的线程；
- 内核支持线程具有很小的数据结构和堆栈，线程的切换比较快，切换开销小；
- 内核本身也可以采用多线程技术，可以提高系统的执行速度和效率。

**缺点：**
- 对于用户的线程切换而言，其模式切换的开销较大。在同一个进程中，从一个线程切换到另一个线程时，需要从用户态转到核心态进行，系统开销较大。

##### (2) 用户级线程 ULT

在用户空间中实现。对线程的创建、撤消、同步与通信等功能，都无需内核的支持，即用户级线程是与内核无关的。

**优点：**
- 线程切换不需要转换到内核空间；
- 调度算法可以是进程专用的；
- 用户级线程的实现与 OS 平台无关。

**缺点：**
- **系统调用的阻塞问题**：当线程执行一个系统调用时，不仅该线程被阻塞，而且进程内的所有线程会被阻塞。
- 在单纯的用户级线程实现方式中，多线程应用不能利用多处理机进行多重处理的优点，内核每次分配给一个进程的仅有一个 CPU，进程中仅有一个线程能执行。

##### (3) 组合方式

有些 OS 把用户级线程和内核支持线程两种方式进行组合，提供 ULT/KST 线程。内核支持多个内核支持线程的建立、调度和管理，同时也允许用户应用程序建立、调度和管理用户级线程。

> **轻型进程 LWP**：每个进程可拥有多个 LWP，每个 LWP 都有自己的数据结构（如 TCB），包括线程标识符、优先级、状态、栈和局部存储区等。LWP 可共享进程所拥有的资源。当一个用户级线程运行时，只须将它连接到一个 LWP 上，此时它便具有了内核支持线程的所有属性。

#### 7.4 进程与线程的比较

| 比较维度 | 进程 | 线程 |
|----------|------|------|
| **调度的基本单位** | 传统OS中，进程既是资源拥有单位也是调度单位 | 引入线程后，线程是调度和执行的基本单位。同一进程中线程切换不引起进程切换；跨进程线程切换会引起进程切换。切换代价低 |
| **并发性** | 进程间可并发执行 | 同一进程中的多个线程之间亦可并发执行，使OS具有更好的并发性 |
| **拥有资源** | 进程是拥有资源的基本单位 | 线程自己不拥有系统资源（也有一点必不可少的资源），但可以访问其隶属进程的资源 |
| **独立性** | 独立性高 | 相对进程而言，独立性低 |
| **系统开销** | 切换代价远高于线程 | 切换代价低。同一进程中的多个线程具有相同的地址空间，在同步和通信的实现方面线程也比进程容易 |
| **支持多处理机系统** | — | 多线程可利用多处理器 |

## 四、重点对比与易混辨析

### 表1：程序顺序执行 vs 并发执行

| 特征 | 顺序执行 | 并发执行 |
|------|----------|----------|
| 顺序性 | 严格顺序执行，每一操作在下一操作开始前结束 | 具有"执行-暂停-执行"的间断性 |
| 封闭性 | 封闭环境，独占全机资源，结果不受外界影响 | 失去封闭性，资源共享，环境受其它程序影响 |
| 可再现性 | 环境和初始条件相同则结果相同 | 失去可再现性 |

### 表2：进程 vs 程序

| 维度 | 程序 | 进程 |
|------|------|------|
| 静态/动态 | 静态的，一组有序指令的集合，存放于介质上 | 动态的，进程实体的执行过程 |
| 生命期 | 无生命期 | 有生命期：由创建而产生，由调度而执行，由撤消而消亡 |
| PCB | 无 | 有，是进程存在的唯一标志 |
| 并发性 | 不能参与并发执行 | 能和其它进程并发执行 |
| 资源 | [原文未述] | 系统进行资源分配和调度的独立单位 |

### 表3：内核支持线程 KST vs 用户级线程 ULT

| 维度 | 内核支持线程 KST | 用户级线程 ULT |
|------|------------------|----------------|
| 实现位置 | 内核空间 | 用户空间 |
| 内核是否感知 | 是（内核为每个线程设TCB） | 否（与内核无关） |
| 线程切换 | 需从用户态转到核心态，模式切换开销大 | 不需要转换到内核空间 |
| 系统调用阻塞 | 仅该线程阻塞，进程内其它线程仍可运行 | 整个进程（含所有线程）被阻塞 |
| 多处理机利用 | 可利用多处理机并行执行 | 不能利用多处理机（内核每次仅分配一个CPU给进程） |
| 调度算法 | 内核调度 | 可以是进程专用的 |
| 平台无关性 | [原文未述] | 与OS平台无关 |

### 表4：三种进程通信方式对比

| 维度 | 共享存储器 | 消息传递 | 管道通信 |
|------|------------|----------|----------|
| 通信原理 | 通过共享空间读写交换信息 | 以格式化消息为单位交换数据 | 通过共享文件（pipe文件）连接读写进程 |

| 子分类 | — | 直接通信、间接通信（信箱） | — |

### 表5：进程控制组织方式对比

| 方式 | 原理 | 优点 | 缺点/适用 |
|------|------|------|-----------|

## 五、高频考点与易错提醒

1. ⚠ **进程与程序的根本区别是动态性**——动态性是进程最重要的特性，以此区分静态程序。(练习4)
2. ⚠ **进程状态转换中，阻塞态不能直接到执行态**——必须经过就绪态。这是高频易错点。(练习16)
3. ⚠ **时间片用完，进程从执行态回到就绪态（不是阻塞态/等待队列）**。(练习1)
4. ⚠ **I/O请求完成，进程从阻塞态变为就绪态（不是运行态）**。(练习18)
5. ⚠ **进程申请处理器得不到满足时，状态为就绪态（不是阻塞态）**——阻塞是等待某事件（如I/O），就绪是等待CPU。(练习6)
6. ⚠ **原语在执行过程中不允许被中断**——原语是原子操作，不可分割。(练习32)
7. ⚠ **进程创建原语不为进程分配CPU**——分配CPU由进程调度实现，创建原语只负责申请PCB、分配资源（内存等）、初始化PCB、插入就绪队列。(练习11)
8. ⚠ **PCB是进程存在的唯一标志**——进程控制块中主要包括四方面信息：描述信息、控制信息、资源管理信息和CPU现场保护结构。(练习14)
9. ⚠ **资源分配的基本单位是进程，CPU调度的基本单位是线程**——引入线程后两者分离。(练习8)(练习24)
10. ⚠ **用户级线程被阻塞时，整个进程（含所有线程）都被阻塞**——这是ULT的主要缺点。(练习36)
11. ⚠ **同一进程或不同进程内的线程均可并发执行**——不同进程的线程并非只能串行执行。(练习31)
12. ⚠ **线程又称"轻量级进程"，但并非所有线程都比进程小**——当进程只含一个线程时，二者一样大。(练习35)
13. ⚠ **同一进程中线程切换不会引起进程切换；跨进程线程切换才会引起进程切换**。(练习23)
14. ⚠ **信箱通信属于间接通信方式**——借助中间实体（信箱），收发双方不直接建立联系。(练习19)
15. ⚠ **进程通信方式包括共享内存、消息传递、管道**——临界区不是进程通信方式。(练习22)
16. ⚠ **挂起操作可由终端用户需要、父进程请求、负荷调节需要、操作系统需要引起**。(练习27)
17. ⚠ **CPU不会因竞争而产生死锁**——进程经分配调度后独占CPU。(练习10)
18. ⚠ **死锁最少资源数公式 $n \times (m-1)+1$**（n个进程，每个需m个资源）——(练习17，反推)
19. ⚠ **V操作后信号量值 $\leq 0$ 时，应释放一个等待该信号量的进程**——(练习3，反推)
20. ⚠ **信号量值为负时，其绝对值表示等待该资源的进程个数**——(练习20，反推)
21. ⚠ **P操作后 S.value < 0 时，才表示有进程进入等待队列**——(练习33，反推)
22. ⚠ **资源静态分配法破坏了死锁产生的"保持和请求"条件**——(练习7，反推)
23. ⚠ **抢占方式基于的原则：优先权原则、短进程优先原则、时间片原则**——(练习38，反推)
24. ⚠ **实时调度算法必须采用抢占式调度才能保证实时性**——(练习37，反推)
25. ⚠ **若用信号量作为同步工具，多个P和V顺序不当也会产生死锁**——(练习40，反推)

## 六、一句话记忆要点

1. **程序顺序执行**：顺序性、封闭性、可再现性；**并发执行**：间断性、失去封闭性、不可再现性。
2. **进程 = 程序段 + 数据段 + PCB**，PCB 是进程存在的唯一标志，进程是动态的、程序是静态的。
3. **进程三态**：就绪、执行、阻塞；核心转换——阻塞不能直接到执行，必须经就绪。
4. **进程控制六原语**：create/halt/block/wakeup/suspend/active，原语不可中断、要么全做要么不做。
5. **进程创建四步**：申请PCB → 分配资源 → 初始化PCB → 插入就绪队列（不分配CPU）。
6. **PCB 四方面信息**：进程标识符、处理机状态、进程调度信息、进程控制信息；组织方式三种：线性、链接、索引。
7. **高级通信三类**：共享存储器、消息传递（直接/间接信箱）、管道（互斥+同步+确定对方存在）。
8. **线程**：调度和执行的基本单位，不拥有资源（访问隶属进程资源）；KST内核感知可并行但模式切换开销大，ULT内核不感知但系统调用阻塞整个进程。
9. **进程 vs 线程**：进程=资源分配单位，线程=CPU调度单位；同进程线程切换不引起进程切换，跨进程则引起。
10. **死锁与信号量**：CPU不竞争不死锁；静态分配破坏"保持和请求"；信号量负值绝对值=等待进程数；V操作后值≤0则唤醒。
