# 计算机网络 · 第3章 数据链路层 — 复习资料

> 来源：5-5-计算机网络讲义.pdf 物理页 [51, 65]（印刷页 p49–p63）
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与答案标注）。

## 一、本章概览
本章介绍数据链路层的基本概念、功能与服务，以及几种核心的数据链路层协议。数据链路层在物理层提供的服务基础上向网络层提供服务，将物理线路改造成逻辑上无差错的数据链路。本章重点涵盖数据链路层的帧同步、差错控制、流量控制等基本功能，详细讨论了停止等待协议、连续ARQ协议、选择重传ARQ协议及滑动窗口机制，并深入解析了HDLC协议和因特网点对点协议PPP的帧结构与特点。

## 二、知识点详解
### 1. 数据链路层模型
- **定义**：
  - **链路**：一条无源的点到点的物理线路段，中间没有任何交换结点，也称物理链路。
  - **数据链路**：也称逻辑链路，是指物理链路加上必要的通信规程。
- **要点**：
  - 在一条线路上传送数据时，除了必须有一条物理线路外，还必须有一些必要的规程来控制这些数据的传输。
  - 简化通信模型：把数据链路层以上的各层抽象为一个主机，而把物理层和传输媒体则抽象成一条简单的链路。
- **公式 / 关系**：
  - 数据链路（逻辑链路） = （物理）链路 + 通信规程

### 2. 数据链路层的功能 ⭐(练习1,2,3,16,17,18)
- **链路管理**：数据链路的建立、维持和释放。OSI/RM中数据链路层面向连接；Internet中数据链路层并不提供可靠的服务。
- **帧同步**：收方应当能从收到的比特流中准确地区分出一帧的开始和结束。主要方法有：字节计数法、字符填充首尾定界符法、比特填充首尾定界符法、违法编码法。
  - **比特填充的首尾定界符法**：以特定比特模式（如01111110）标志帧起止。发送端检测到连续5个1时在其后加0，接收端将5个1后的0取走还原。HDLC规程采用此法。
- **流量控制**：发送方发送数据的速率必须使接收方来得及接收。数据链路层上的流量控制，控制的是相邻两节点之间数据链路上的流量。
- **差错控制**：ARQ自动反馈重发是最常用的差错控制方法。分为：空闲重发/停止等待法、连续重发（GO-BACK-N策略和选择重发策略）。
- **透明传输**：不管所传数据是什么样的比特组合，都应当能够在链路上传送。措施如零比特填充法和DEL字符填充法。
- **寻址**：多点式链路上传输数据时，必须保证每一帧都能送到正确的接收方，接收方也应知道发送方是哪一个。

### 3. 数据链路层提供的服务 ⭐(练习5)
- **无确认、无连接服务**：目的结点不作确认，差错由上层负责。
- **有确认的无连接服务**：目的节点对收的帧作确认，发送节点可以知道已发出的帧是否安全到达目的节点。
- **面向连接服务**：可靠地传送数据的服务，为网络层实体间的交互提供了最可靠的数据传输服务。

### 4. 数据链路层协议与停止等待协议 ⭐(练习8,9)
- **两个假定**：①链路是理想的传输信道，所传送的任何数据既不会出错也不会丢失。②不管发方以多快的速率发送数据，收方总是来得及收下，并及时上交主机。
- **简单的停止等待协议**：前提是满足假定①。发送端每发完一帧必须停下来，只有收到接收端发来的确认才能继续发送下一帧。
- **实用的停止等待协议**：前提是不满足两个假定，需考虑数据帧出错、数据帧丢失、应答帧丢失。
  - **解决方法**：对数据帧进行校验，出错重发；对数据帧丢失，设置超时定时器，超时重发；应答帧丢失的处理，设置超时定时器，以序号标识数据帧（解决重复帧问题）。
  - **特点**：优点是比较简单，缺点是信道利用率不高。

### 5. 连续ARQ与选择重传ARQ协议 ⭐⚠(练习9,10,11)
- **连续ARQ(Go-Back-N)协议**：
  - **工作原理**：发送端可连续发送若干帧。若收到对其中某一帧的否认帧，或某个帧对应的应答帧超时，则从该帧开始的后续帧全部重发。
  - **缓冲区要求**：发送端需缓存多个数据帧，接收端只需容纳一个数据帧。
  - **特点**：因连续发送提高了传输效率，但重传时会把原来已正确传送过的数据帧进行重传，误码率较高时不一定优于停止等待协议。
- **选择重传ARQ协议**：
  - **工作原理**：只重传出错的数据帧或定时器超时的数据帧。接收端先缓存那些正确的数据帧（发送序号可能不连续），等到所缺序号的数据帧收到之后再一并送交主机。
  - **缓冲区要求**：收发双方缓冲区都必须能缓存多个数据帧。
  - **特点**：避免重传已正确到达的帧，但代价是接收端需要设置具有相当容量的缓存空间。

### 6. 滑动窗口 ⭐(练习9,16,19)
- **定义**：在发送方和接收方各自维持一个随时间变化向前滑动的窗口（发送窗口和接收窗口）。允许发送方发送多个分组而不需等待确认。TCP的滑动窗口是以字节为单位的。
- **发送窗口四个概念**：已发送并收到确认的数据、已发送但未收到确认的数据、允许发送但尚未发送的数据、发送窗口之外的暂时不允许发送的数据。
- **接收窗口四个概念**：已发送确认并交付主机的数据、未按序收到的数据、允许的数据、不允许接收的数据。
- **规则**：
  1. 凡是已经发送过的数据，在未收到确认之前，都必须暂时保留，以便在超时重传时使用。
  2. 只有当发送方收到了接收方的确认报文段时，发送方窗口才可以向前滑动几个序号。
  3. 当发送方发送的数据经过一段时间没有收到确认（由超时计时器控制），就要使用回退N步协议，回到最后接收到确认号的地方，重新发送这部分数据。
- **公式**：发送窗口 $S_{size}=2^{n}-1$。

### 7. HDLC协议 ⭐(练习4,14,15)
- **概述**：High level Data Link Control，高级数据链路控制。面向位的数据链路层协议，由IBM SDLC研发出的ISO标准。功能包括差错释放和流量控制。支持全双工点对点操作，提供完整的数据透明度。
- **帧结构**：标志F(8bit) + 地址A(8bit) + 控制C(8bit) + 信息Info(可变) + 帧检验序列FCS(16bit) + 标志F(8bit)。
  - **标志域F**：6个连续1加两边各一个0（01111110）。采用零比特填充法（发送端5个连续1填0，接收端删除0）。连续传两帧时前一帧结束F兼作后一帧起始F；无信息时连续发F保持同步。
  - **地址域A**：8 bit。多终端线路区分终端；点到点线路区分命令和响应。接收站地址为命令帧，发送站地址为响应帧。
  - **帧检验序列FCS**：16 bit。检验范围从地址字段第一比特起，到信息字段最末一比特为止。
  - **控制域C**：8 bit。帧类型分为信息帧、监控帧、无序号帧。
    - **信息帧**：3位序号，滑动窗口大小为7。捎带确认，指第一个未收到的帧序号。P/F位中命令帧置“P”，响应帧置“F”。
    - **监控帧**：Type 0为RR(确认帧)，1为REJ(返回N帧重传)，2为RNR(要发方停)，3为SREJ(选择性重传)。
    - **无序号帧**：传控制信息，或不可靠无连接服务中传数据。

### 8. 因特网的点对点协议PPP ⭐⚠(练习12,13,14)
- **组成**：将IP数据报封装到串行链路的方法、链路控制协议LCP、网络控制协议NCP。
- **帧格式**：标志(01111110) + 地址(11111111) + 控制(00000011) + 协议(2字节) + 净荷(≤500字节，默认1500) + 校验和(2字节) + 标志(01111110)。
  - **标志F**：01111110 (0x7E)。异步传输用字节填充法，同步传输用零比特填充法。
  - **地址A**：11111111 (0xFF)，表示所有站都可以接受该帧。
  - **控制C**：00000011 (0x03)，无编号的帧，默认无序号和确认。
  - **协议**：0x21为IP数据报，0xC021为LCP数据，0x8021为网络层控制数据。默认2字节。
  - **净荷**：变长，默认1500字节。
  - **校验和**：CRC的FCS，通常2字节。
- **特点**：不可靠的传输协议。不纠错、不流量控制，不使用序号和确认机制。原因：数据链路层差错概率不大；IP数据报可靠传输不能保证网络层可靠；FCS可保证无差错接受。

## 三、公式 / 算法 / 定理速查表
- **数据链路** = 物理链路 + 通信规程
- **发送窗口大小公式**：$S_{size}=2^{n}-1$ （其中 $n$ 为序号位数）
- **HDLC零比特填充法**：发送端遇5个连续1填入0，接收端遇5个连续1删除其后0。
- **PPP协议字段值**：0x21 (IP数据报), 0xC021 (LCP), 0x8021 (NCP)

## 四、重点对比与易混辨析

| 特性 | 停止等待协议 | 连续ARQ (Go-Back-N) | 选择重传ARQ |
|---|---|---|---|
| 发送窗口 | 1 | >1 | >1 |
| 接收窗口 | 1 | 1 | >1 |
| 重传机制 | 超时重传当前帧 | 从出错帧起后续帧全部重发 | 只重传出错或超时的帧 |
| 缓冲区要求 | 发送/接收各1帧 | 发送多帧，接收1帧 | 收发双方均需多帧 |
| 优缺点 | 简单，信道利用率低 | 效率高，但重传量大 | 避免无效重传，但接收端缓存大 |

| 特性 | HDLC | PPP |
|---|---|---|
| 面向类型 | 面向比特 | 面向字节 |
| 透明传输 | 零比特填充法 | 异步用字节填充，同步用零比特填充 |
| 可靠性 | 使用序号和确认实现可靠传输 | 默认不可靠（无序号和确认） |
| 协议位置 | 数据链路层 | 数据链路层 |

## 五、高频考点与易错提醒
- ⚠ 数据链路层的功能（帧定界、差错控制、流量控制、链路管理等）是高频易错点，注意电路管理不属于数据链路层功能 (练习1,2,3,16,17,18)。
- ⚠ 数据链路层提供的服务不包括无确认的面向连接服务 (练习5)。
- ⚠ 停止等待协议中，解决死锁现象（帧丢失永远等待）的办法是超时机制 (练习8)。
- ⚠ 发送窗口为1、接收窗口为1时，相当于停止-等待方式 (练习9)。
- ⚠ GBN协议中，发送窗口大小与所需序列号位数关系是高频考点 (练习10，反推)。
- ⚠ 选择重传协议中，5位帧序列号对应的最大接收窗口大小是高频考点 (练习11，反推)。
- ⚠ PPP协议不仅支持IP协议，还支持动态分配IP地址和身份验证 (练习12)。
- ⚠ PPP透明传输：对字符数据使用字符填充，对非字符数据使用位填充 (练习13)。
- ⚠ PPP是面向字节的协议，HDLC是面向比特的协议；PPP是数据链路层协议 (练习14)。
- ⚠ 滑动窗口的作用是流量控制，接收窗口保存的是可接收的帧序号 (练习16,19)。
- ⚠ ICMP和HDLC不属于TCP/IP协议簇 (练习4,15)。

## 六、一句话记忆要点
- 数据链路 = 物理链路 + 通信规程。
- 帧同步找首尾，5个1填0是HDLC。
- 流量控制看窗口，停止等待最简单，连续ARQ全重发，选择重传只发错。
- 滑动窗口向前滑，收到确认才滑动，超时回退N步走。
- HDLC面向比特，FCS检验不含标志位。
- PPP面向字节，默认不可靠，异步字符同步比特。
