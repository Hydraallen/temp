## 3. 全系 13 款共性参数（一次性列出，下方不再重复）

以下参数经逐款核对，**13 款取值完全一致**（多为 AMD 对 X870E 的硬性规定或全系统一设计），**不构成选板依据**。为避免同一数值在文中重复 13 遍，集中列此；**第 4 节三张对比表与 4.4 逐款明细已删除对应行，不再赘述**。

### 3.1 全系完全一致（已从下方表格 / 明细中删除，仅在此列出）

- **SATA：统一 4 × SATA 6Gb/s**（想要 6 口以上 SATA 的，X870E 全系都满足不了）。唯一细节差异：**DARK HERO** 的 4 口为「2 × 芯片组 + 2 × ASM1162 控制器」，其余各款为芯片组直出。
- **CPU 供电接口：全系 EPS 8Pin × 2（8+8，`2 x 8-pin +12V`）**——唯 **APEX** 额外多 1 个 PCIe 8Pin 辅助供电接口。下方各表 / 明细的「供电」项只保留**相数 / 每相电流**（各款不同），不再重复 8+8。
- **USB4 40Gbps × 2**：全系标配（下方后置 USB 计数里的「2×USB4 40G」即此项）。
- **CPU 直连 PCIe 5.0 x16 主显卡槽 × 1**：全系第一条 x16 均为 CPU 直连、满血 PCIe 5.0 x16。
- **至少 1 条 CPU 直连 PCIe 5.0 x4 M.2**：全系保证。
- **无线：全系 Wi-Fi 7**（版本一致；具体**模块型号各款不同**：MT7927 / MT7925 / RTL8922AE / QCNCM865 / 通用 802.11be，见各表与明细）。
- **芯片组：双 Promontory 21** → 全系最小板型 ATX，无 mATX、无 ITX。

### 3.2 「几乎一致但有例外」——不抽离，仍完整保留在下方表里

- **内存**：除 **ROG Crosshair X870E APEX（2 DIMM / 128GB）** 外，其余 12 款均为 **4 DIMM / 最大 256GB DDR5**；因 APEX 例外、且各款超频标称频率不同，**内存行保留在各表**（APEX 也因此被 6.3 的 ≥192GB 硬性筛选排除）。
- **板型尺寸**：11 款为 ATX 30.5×24.4cm，仅 **GLACIAL / EDITION 20 / EXTREME** 为 E-ATX 30.5×27.7cm，**保留在表**。
- **其余存在差异、全部保留**：供电相数 / 每相电流、M.2 总数与构成、后置 / 前置 USB 分速度计数、有线网卡速率、音频编解码芯片、WiFi 模块型号、视频输出（HDMI 有无）、价格。

## 5. 三个真正的分水岭

在 SATA 数量、USB4、第一条 PCIe 5.0 x16 全系相同的前提下，**只有下面 3 项真正区分档次**：

### 分水岭 1：第二条 x16 是否 CPU 直连 PCIe 5.0
- **是真 x8/x8 的**：ROG Crosshair 全系 + **ProArt X870E-Creator**
- **不是**：Strix / TUF 的第二条 x16 只是**物理 x16、电气 PCIe 4.0 x4，且走芯片组**
- 影响对象：**双计算卡 / 双 5090**、PCIe 5.0 扩展卡（高速采集卡、网卡、M.2 扩展卡）用户——这类第二张卡才真吃带宽
- **不影响**：RTX 3060 之类**副卡**（额外显示输出/编码/轻量推理），它不吃带宽，插芯片组 4.0 x4 就够（详见 [6.5](#65-多卡场景未来加第二张卡的通道与拆分逻辑)）

### 分水岭 2：CPU 直连 PCIe 5.0 M.2 的数量
- **3 条**：Hero / Hero BTF / Strix X870E-E / Extreme / Apex
- **2 条**：其余全部
- 注意：第 3 条 CPU M.2 是**从显卡的 16 条通道里借的**（见下节）

### 分水岭 3：万兆（10G）有线网口
- **有**：GLACIAL（双 10G）、EDITION 20（双 10G）、EXTREME（10G+5G）、DARK HERO（10G+5G）、ProArt（10G+2.5G）
- **没有**：其余全部（5G / 2.5G）
- 现实提醒：家里没有 10G 交换机 / NAS 的话，这一项等于白花钱

## 6. 9950X + RTX 5090 专项建议（重点）

### 6.1 平台事实

**AMD Ryzen 9 9950X**
- Zen 5 架构，16 核 32 线程
- TDP **170W**，PPT ≈ 230W
- **无 3D V-Cache**
- AM5 接口，PCIe 5.0
- 总计 **28 条可用 PCIe 通道** = 16（显卡）+ 4（CPU 直连 M.2）+ 4（芯片组链路）+ 4（额外 CPU M.2 / 其他）

### 6.2 五条关键推论

#### ① 供电相数：16+2+1 的 80A 方案余量充足，别为相数付钱

9950X 满载 PPT 约 230W（170W TDP，比 9900X 高约 40%）。以 TUF / Strix X870E-H 的 **16 相 × 80A ≈ 1280A** 理论供电能力计算，喂 230W PPT 的 9950X 依然绰绰有余，实际负载占用率仍低、温度余量充足——这类 80A 级甜点板稳喂 9950X 没有问题。

Crosshair 系列 20~24 相 110A（2000A+）的供电堆料，是给 **液氮/极限超频、长时间 AVX-512 全核满载**这类极端场景准备的。对一颗默认 170W TDP 的 9950X，日常与全核生产力负载下这些多出来的相数依旧是过剩的，**不会带来任何频率、温度或稳定性上的可感知收益**，只有拉到极限超频才用得上。

> 结论：供电这一项，Strix / TUF 档位对 9950X 依然完全够用，不需要往上加钱。

#### ② RTX 5090 单卡只需要一条 PCIe 5.0 x16 → 第二条 CPU 直连 x16 现在用不上，但「未来加卡」会改变结论

单卡状态下，RTX 5090 占用第一条 CPU 直连 PCIe 5.0 x16 即可满血运行。**当前**这台机器不需要第二条 CPU 直连 x16。

而 **Crosshair 全系和 ProArt 相对 Strix 最贵的那部分溢价，恰恰就是「第二条 CPU 直连 x16，支持 x8/x8 拆分」**。

> 结论（重要，已随需求更新）：这条卖点**对单卡价值为 0，但对未来加卡的两种场景价值截然不同**——
> - **加第二张 5090（生产力/AI/渲染双卡）→ 这条卖点是刚需**：第二张卡必须走 CPU 直连才有足够带宽，芯片组 4.0 x4 会成瓶颈（详见新增的 [6.5 多卡场景](#65-多卡场景未来加第二张卡的通道与拆分逻辑)）。
> - **加一张 RTX 3060 之类副卡（额外显示输出/编码/轻量推理）→ 这条卖点仍然无关紧要**：副卡对带宽不敏感，插芯片组 4.0 x4 槽即可，Strix/TUF 的布局完全够用。

#### ③ 通道冲突警告：别把 3 条 CPU 直连 M.2 插满

Ryzen 9000 只有 28 条通道。在 Hero / Strix X870E-E / Extreme 这类「3 条 CPU 直连 5.0 M.2」的板子上，**插满第 3 条 CPU M.2 会触发 x8+x4+x4 拆分，显卡直接从 x16 掉到 x8**。

对 RTX 5090（实测数据，见来源）：
- PCIe 5.0 x8 的带宽（约 32GB/s 单向）**等于 PCIe 4.0 x16**，游戏实测平均损失仅约 **1%**（TechPowerUp / GamersNexus 的 5090 PCIe scaling 测试）；作为对比 PCIe 3.0 x16 平均掉约 **4%**
- **但生产力/内容创作不同**：DaVinci Resolve 等场景下 5.0 x8 ≈ 4.0 x16 几乎无损，然而在**带宽更受限**的配置（更低代次或更少通道，如降到 4.0 x8 甚至 4.0 x4）下，个别 3D / 视频工作流可损失高达 **约 25%**（Puget Systems / Tom's Hardware）。这正是「副卡 vs 计算卡」要区别对待的根源

> 建议：**只用 2 条 CPU 直连 M.2（这两条走的是独立的 4 条通道，不动显卡），第 3 块及以后的 SSD 走芯片组 PCIe 4.0 x4 槽**。PCIe 4.0 x4 的 SSD 在游戏和日常场景下与 5.0 的体感差异极小，保住显卡满 x16 更划算。
>
> 各板 lane sharing 规则不同，**装机前务必查该型号说明书的 lane sharing 表**。

#### ④ RTX 5090 是三槽以上厚卡 → 优先看 M.2 散热布局和免工具卸载

5090 公版/非公普遍 3–4 槽厚，实际后果：
- **会完全压住第一条 M.2 散热片**（位于显卡正下方），高负载时 SSD 温度上升明显
- **会挡住第二条 PCIe 槽**，甚至连第三条也够不着（对单卡无所谓，但影响声卡/采集卡）
- 拆装显卡时，PCIe 卡扣被显卡散热鳍片挡住，非常难按

选板要点：
1. 第一条 M.2 是否有**独立厚散热马甲**，最好带背面散热（Dark Hero 的 3D VC 均热板是这一档最好的）
2. 是否有 **Q-Release / Q-Release Slim 免工具显卡卸载**——对 5090 这种重卡是刚需体验
3. 优先把主力 SSD 装在**不被显卡覆盖**的 M.2 位

#### ⑤ 电源与供电接口

- RTX 5090 TGP **575W**，+ 9950X 满载约 230W（PPT）+ 其余
- **单卡整机建议 1000W 以上电源**（推荐 1000–1200W ATX 3.1 / 原生 12V-2×6 接口）
- **若未来上双 5090**：两卡实测整机可拉到约 **1.5kW** 负载（hostkey 双 5090 实测），电源需 **1600W 级起步、留头建议 2000W**，且需两路独立 12V-2×6 供电——这是把双 5090 从「想法」变成「现实」的第一道硬门槛
- 主板端需确认 **CPU 供电为双 8Pin（EPS 8+8）**——本文列出的 Strix / TUF / ProArt / Crosshair 全系均满足

### 6.3 选购建议（先过硬性筛选，再按「装几张卡」对号入座）

选板分两步：**第一步用硬性条件筛掉出局的板，第二步按你未来装几张卡对号入座。**

#### 硬性筛选条件（任一不满足直接出局）

1. **最大内存容量 ≥ 192GB**（用户硬性要求）。X870E 全系除 APEX 外均为 **4×DIMM / 最大 256GB**（即 4×64GB，ASUS 官方规格页逐款明确标称），256GB > 192GB，**全部满足**；**唯 ROG Crosshair X870E APEX 是 2×DIMM / 128GB，低于 192GB，直接排除**——即便它是最强的内存超频板（DDR5-9600+）、别的方面很合超频胃口，内存上限这一条就把它挡在门外。**本文所有推荐均不含 APEX。**
2. **CPU 供电为双 8Pin（EPS 8+8）**。本文列出的 Strix / TUF / ProArt / Crosshair 全系均满足，不构成差异。

> 关于 ≥192GB 如何达成：4-DIMM 板最大 256GB 即 **4×64GB**，天然覆盖 192GB 需求；若想正好 192GB 也可用 **4×48GB**。**关键结论：凡 4-DIMM / 256GB 的板都满足 ≥192GB（含所有推荐机型），只有 2-DIMM 的 APEX 不满足。**（各板对 48GB/64GB 单条模组的具体兼容型号以官方 QVL 为准，但「最大 256GB」本身已是 ASUS 对 4×64GB 支持的官方承诺。）

#### 你未来会装几张卡？

- **情况一：只装 1 张 5090**（现状，且大概率一直如此，或最多再加一张 RTX 3060 类副卡）→ 看下方「情况一」。**结论先行：甜点板 Strix X870E-H / X870E-A NEO 就是最优解，不用多花钱。**
- **情况二：确定要装 2 张卡**（尤其双 5090 做渲染 / AI / 生产力）→ 看下方「情况二」。需要第二条 CPU 直连 x16、可 x8+x8 拆分的板，且先过三道现实门槛。

#### 情况一：只装 1 张 5090（现状 / 大概率）

> **首选：ROG Strix X870E-H GAMING WIFI7（¥2,949）**
> **平价备选：ROG Strix X870E-A GAMING WIFI7 NEO（¥2,999，白色、供电略强 16+2+2、内存标称 8600+、WiFi 模块 MT7925 略低一档）**

**为什么单卡下这些甜点板已经够用：**
- X870E 芯片组的强制项它们一个不少：**USB4 40Gbps × 2、CPU 直连 PCIe 5.0 x16 显卡槽、2 条 CPU 直连 PCIe 5.0 x4 M.2、双 8Pin CPU 供电**——这些才是 5090 满血运行真正需要的，全系一致，堆料更贵的板在这几项上并不比它们强。
- **内存 4×DIMM / 256GB ≥ 192GB**，满足本次硬性要求。
- 16+2+1（或 16+2+2）× 80A 供电对 170W TDP / 230W PPT 的 9950X 仍绰绰有余（详见 6.2 ①）。
- 单卡状态下 5090 独占第一条 CPU 直连 PCIe 5.0 x16，稳定满 x16；2 条 CPU 直连 M.2 走独立通道、**天然不跟显卡抢通道**。
- 后置 USB 多（X870E-H 达 14 个，这一档最多）、WiFi7、Realtek 5G 网口。
- 省下的 ¥1,000~2,000 直接加到电源 / SSD / 内存，收益远高于主板堆料。喜欢初音未来联名外观的，同规格 X870E-H 版本 ¥3,199，贵 ¥250 纯买外观。

**⚠️ 如果你在这些（单卡定位的）板上，后来又插了第二张卡，会遇到什么：**

Strix / TUF 这类板的**第二条 x16 槽是「物理 x16、电气 PCIe 4.0 x4、且走芯片组（PCH）」**——不是 CPU 直连，**不能 x8+x8 拆分**。后果按第二张卡的类型分两种：

- **插第二张 5090 当计算卡 → 明显瓶颈 + 基本装不下**：
  - **带宽**：第二张卡只能跑 **PCIe 4.0 x4（≈ PCIe 3.0 x8 带宽）**，对吃带宽的渲染 / AI 数据吞吐是硬瓶颈（带宽受限时部分 3D / 视频工作流可损失约 **25%**，见 6.2 ②③ 的 Puget / Tom's 数据）；而且这条 4.0 x4 还要和芯片组下的部分 M.2 / USB **共享上行链路**，可能被进一步挤占。
  - **物理**：两张 3.5~4 槽厚的 5090 在这些 ATX 板的槽间距下**基本装不下**——第二条 x16 常在第一张卡正下方 3~4 槽处，**位置直接被第一张卡挡住**，两张风冷公版叠不下。
- **插一张 RTX 3060 类副卡 → 完全没问题**：副卡（额外显示输出 / 编码 / 轻量推理）不吃 PCIe 带宽，4.0 x4 对它**几乎无损**，Strix / TUF 的第二条槽正好拿来插它。这是单卡定位板最合理的「第二张卡」用法。
- **通道共享提醒**：插第二张卡时，该槽可能与某些 M.2 / USB 抢通道（触发禁用或降速），**具体以该型号说明书的 lane sharing 表为准**。

**结论**：**单卡，或未来最多再加一张副卡 → 情况一的甜点板（X870E-H / X870E-A NEO）就是最优解，不用为「第二条 CPU 直连 x16」多花钱。** 想给双卡留一条模糊后路、又不愿现在就上 ProArt 的，可以接受，但请认清那是「买保险」而非「买性能」。

#### 情况二：确定要装 2 张卡（尤其双 5090）

只有当你**明确计划**装两张真正跑计算的卡（双 5090 做渲染农场 / 并行 AI 推理训练 / 显存池化）时，才需要为第二条 CPU 直连 x16 付费。

> **首选：ProArt X870E-Creator（¥4,699）** —— 双卡生产力最对口
> **备选：ROG Crosshair X870E DARK HERO（¥4,899）** —— 偏好 ROG 顶级用料 / 3D VC 均热板 M.2 散热 / BIOS

**为什么必须是这一档**：这两块（连同 Crosshair 全系）是本平台具备**第二条 CPU 直连 PCIe 5.0 x16、可真 x8+x8 拆分**、同时又满足 **≥192GB 内存**的板子。双卡时两张各拿 PCIe **5.0 x8（约 32GB/s，等于满血 4.0 x16）**，不会被芯片组 4.0 x4 卡到（避免上面情况一里的 ~25% 损失）。
- **ProArt**：CPU 直连双 x16 + Marvell 10G 网口 + 8K DP Alt，定位内容创作，比 Dark Hero 便宜 ¥200 且更对口 → **双卡生产力首选**。
- **Dark Hero**：纯双卡带宽上并不比 ProArt 强，选它是为 ROG 顶级用料 / 散热 / BIOS，或已有 10G 网络。

**⚠️ APEX 虽然也有 CPU 直连双 x16、可 x8+x8 拆分，但因内存上限 128GB < 192GB 被硬性排除，不在双卡候选内。**

**双卡的现实门槛（掏钱买板前先确认能不能过，过不了就退回情况一）：**
1. **无 NVLink，不能游戏叠加**：40/50 系消费卡取消 SLI/NVLink，两张 5090 无法合并渲染同一帧游戏；第二张卡只对**能吃多 GPU 的计算任务**有意义（渲染农场、并行 AI、2×32GB=64GB 显存池化，可跑单卡放不下的大模型）。
2. **功耗与电源**：双卡实测整机约 **1.5kW**，需 **1600W+ 电源**（留头 2000W）、两路独立 12V-2×6 供电。
3. **物理空间与散热**：5090 普遍 3.5~4 槽厚，两张 4 槽风冷卡在标准 ATX 上几乎叠不下，现实里要么用涡轮 / 水冷 / 2 槽专业卡，要么上 E-ATX / 开放式机箱。
4. **CPU 供数**：9950X 是 16 核 32 线程，在多卡计算供数上比 9900X 更从容，不再是明显短板；若真要追求双卡满血生产力，长远看更高端的 CPU / 平台（HEDT / EPYC）在超大规模并行下才可能成为瓶颈，而非主板贵不贵。

> **坦诚判断**：双 5090 是小概率、门槛极高的**系统级工程**（电源 / 机箱 / 散热 / 平台缺一不可），不是「买块贵主板就成」。**若你没有明确的双卡生产力计划，不建议现在就为它多掏 ¥1,700+ 买 ProArt / Dark Hero**——那笔钱加到电源 / SSD / 内存上收益高得多。原理详见 [6.5](#65-多卡场景未来加第二张卡的通道与拆分逻辑)。

- **X870E APEX**：已在硬性筛选阶段因内存 128GB < 192GB 排除（见 6.3 开头）。

### 6.4 一句额外的话：CPU 搭配

如果这台机器的**主要用途是游戏**，9950X 配 RTX 5090 偏生产力——9950X 同样无 3D V-Cache，**纯游戏帧数仍明显低于同为 16 核但带 3D 缓存的 9950X3D（以及 8 核的 9800X3D）**，在 1080p/1440p 高刷场景下差距尤为明显，而 5090 恰恰是最容易被 CPU 拖后腿的卡。

9950X 的优势在于 16 核 32 线程带来的强生产力/多线程性能，配 5090 属于**游戏 + 生产力兼顾、偏生产力**的搭配（多核渲染、编译、虚拟机）。如果你的实际使用确实是这种混合场景，那这个搭配是合理的；如果纯游戏，建议在下单前重新考虑 CPU。

### 6.5 多卡场景：未来加第二张卡的通道与拆分逻辑

这一节解释「为什么副卡随便插、计算卡必须走 CPU 直连」，让你理解 6.3 两种情况背后的原理。

#### 通道从哪来：CPU 直连 vs 芯片组（PCH）

Ryzen 9000（Zen 5）对外一共 **28 条可用 PCIe 通道**：其中 **16 条**是给显卡的 PCIe 5.0，另外几条给 CPU 直连 M.2 和芯片组链路。主板上第二条 x16 槽的电气规格，取决于它接在哪：

- **接 CPU（Crosshair 全系 + ProArt）**：那 16 条显卡通道可**对半拆成 x8 + x8**（BIOS 里叫 bifurcation）。两张卡各拿 PCIe **5.0 x8**，每条约 32GB/s——等于满血 4.0 x16，带宽充足。
- **接芯片组（Strix / TUF 的第二条 x16）**：物理是 x16 长度，但**电气只有 PCIe 4.0 x4**，而且要和其他 PCH 设备（部分 M.2、USB）**共享那条上行 4.0 x4 链路**。带宽只有 CPU 直连 x8 的约 1/4，且可能被别的设备挤占。

> 一句话：**「x8+x8」是把显卡的通道分一半给第二张卡（CPU 直连，快）；芯片组 x4 是从一条细管子里再分（慢、且要抢）。**

#### 情况一延伸：加 RTX 3060 之类副卡 → 芯片组 4.0 x4 完全够

副卡的用途是额外显示输出 / 视频编码 / 轻量推理，本身**不吃 PCIe 带宽**。RTX 3060 虽是 x16 卡，但其算力早在到达 PCIe 瓶颈前就先饱和了——即便降到 4.0 x4，游戏/日常几乎无可感损失（3060 本身是瓶颈，不是通道）。所以副卡插 **Strix/TUF 第二条（芯片组 4.0 x4）槽即可**，不需要为 CPU 直连双 x16 付一分钱。主推荐 Strix X870E-H / X870E-A NEO 依然成立。

#### 情况二延伸：加第二张 5090 做生产力 → 必须 CPU 直连，且门槛很高

**带宽层面**：第二张 5090 是真正在跑计算的卡，PCIe 带宽会实打实影响渲染/AI 数据吞吐。若它只有芯片组 4.0 x4，等于把一张 575W 的卡喂在一根细管子上，部分 3D/视频工作流可损失约 25%（见 ②③ 的 Puget/Tom's 数据）。因此**只有 CPU 直连 x8/x8 的板（Crosshair 全系 / ProArt）才适合双 5090**。

**但在掏钱买板之前，先确认能不能过这三道现实门槛**：

1. **无 NVLink，不能游戏叠加**：40/50 系消费卡早已取消 SLI/NVLink，两张 5090 **无法合并渲染同一帧游戏**。第二张卡只对**能吃多 GPU 的计算任务**有意义——渲染农场、并行 AI 推理/训练、多卡显存池化（2×32GB=64GB，可跑单卡放不下的大模型）。想靠它提升游戏帧数是无效的。
2. **功耗与电源**：双卡实测整机可达约 **1.5kW**，需 **1600W+ 电源**（留头 2000W）、两路独立 12V-2×6。
3. **物理空间与散热**：5090 普遍 **3.5~4 槽厚**。两张 4 槽风冷卡在标准 ATX 上，第二条 x16 通常在第一张卡正下方 3~4 槽处——**两张风冷公版几乎叠不下**，现实里要么用**涡轮/水冷/2 槽版**专业卡，要么上 EATX/开放式机箱。
4. **CPU 供数**：9950X 是 16 核 32 线程，在多卡计算供数上比 9900X 更从容，不再是明显短板。不过在超大规模并行下，消费级平台仍可能吃紧——实测有消费级平台带不动双 5090、换到 HEDT/服务器平台（如 EPYC）才稳的案例。追求双卡满血生产力，长远看 CPU/平台才是瓶颈，而不是主板贵不贵。

> 结论：双 5090 技术上可行、且在**显存受限的 AI/渲染**上收益明显，但它是一套**围绕电源、机箱、散热、平台**的系统级工程，不是「买块贵主板就成」。没有明确计划时，为它多花的主板钱是「保险费」，不是「性能」。

### 通用注意事项

**Ryzen 9000 系列仅 28 条可用 PCIe 通道**。多款主板的第 3 条 CPU 直连 M.2 会与显卡共享通道（触发 x8+x4+x4 拆分）。本文表格中的槽位数量是**物理槽位**，不代表可以同时满速使用。

**装机前必做**：下载目标型号的官方 User Manual，查阅其中的 **lane sharing / bandwidth allocation 表**，确认你的 SSD + 显卡 + 扩展卡组合不会互相降速或直接禁用某个槽位。
