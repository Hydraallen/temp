## 1. 三条铁律（最重要，先读这里）

1. **保真，不改写。** 公式逐字保留（用 LaTeX：`$...$` 或 `$$...$$`）；算法/伪代码/代码用代码块保留；定义尽量贴原文。**不要**把「快速排序」缩成「一种排序方法」——那会丢掉学生真正要看的东西。

2. **零编造。** 只写页面上真实存在的内容。看不清的字、拿不准的公式，用 `[?]` 或 `[原文模糊]` 标注，**绝不猜一个填上去**。这些是扫描件，手写/公式最易出错——**慢一点，逐字核对**。

3. **只摘「知识点」，练习题只做考点信号。** 每章由三部分组成：`知识点` → `专项练习` → `参考答案及解析`。
   - **知识点**：这是唯一要摘录进复习资料的正文主体。
   - **专项练习 + 参考答案**：**只用来判断「哪些知识点是高频考点、哪些是易错点」**，用来给知识点打 ⭐/⚠ 标记。**不要把题目或例题解答抄进复习资料。**

## 2. 每章的执行流程（对任务表里每一行重复一次）

任务表 `01-章节任务表.md` 每一行给出：**专题、输出文件夹、章号、章名、PDF 文件名、物理页范围 `[start, end]`**（页码偏移已替你算好，直接用物理页）。

### Step 1 — 渲染该章页面为图片

扫描件用 PyMuPDF 渲染后视觉阅读最可靠。在工作目录执行（把 `<PDF>`、`<start>`、`<end>` 换成任务表里的值；物理页从 1 开始计，脚本里减 1）：

```bash
/opt/anaconda3/bin/python - <<'PY'
import fitz, os
PDF   = "<PDF文件名>"      # 例：4-3-操作系统-讲义.pdf
START = <start>           # 物理页，从1开始（任务表给的值）
END   = <end>
OUT   = "/private/tmp/cheatsheet-render"
os.makedirs(OUT, exist_ok=True)
doc = fitz.open(PDF)
for p in range(START-1, END):      # fitz 页索引从0开始
    doc[p].get_pixmap(dpi=150).save(f"{OUT}/p{p+1:03d}.png")
print("rendered", START, "to", END)
PY
```

> DPI 用 150（清晰且不过大）。若某页文字仍偏小看不清，对该页单独用 dpi=200 重渲。

### Step 2 — 逐页阅读

用 `Read` 工具依次读取 `p{页码}.png`。**一次读几页即可，读完就在脑子里归纳，不要囤积。**

- 先定位本章「知识点」部分的起止（通常从「第X章 知识点」开始，到「第X章 专项练习」前结束）。
- **只精读「知识点」页**：逐个「知识点 N: xxx」提取定义、要点、公式、算法、对比。
- **快速扫「专项练习 + 参考答案」页**：不抄题，只记录「哪些知识点被反复考」「答案解析里点出的常见错误/陷阱」，用于 Step 3 的 ⭐/⚠ 标注。

计算机网络（`3-5`）特殊：每章末尾是「课后习题」，**统一答案在全书最后**（见任务表备注）。做该专题时，考点信号主要看每章课后习题覆盖的知识点即可，答案可不逐章回溯。

### Step 3 — 按模板写出该章复习资料

用 `Write` 工具写到对应路径，例如
`cheatsheets/04-操作系统/第01章-操作系统基础.md`。
文件名格式：`第0X章-<章名>.md`（章号两位补零，章名去掉空格）。

模板见第 3 节。**目标篇幅：详尽复习版，正文约 3 页以上**（知识点全覆盖、讲透，不是速查单页）。

### Step 4 — 自检后再进入下一章

写完立即对照第 4 节「完成前自检清单」核对。通过再处理下一章。

## 3. 每章复习资料模板

> 下面是**结构模板**，实际内容按本章真实知识点填充。章内知识点有几个就写几个，不要硬凑也不要漏。

```markdown

> 来源：<PDF文件名> 物理页 [a, b]（印刷页 p–p）
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与答案标注）。

## 一、本章概览
（3–6 句：这一章在整门课里解决什么问题、包含哪几大块、与前后章的关系。）

## 二、知识点详解
### 1. <知识点标题>  ⭐/⚠（如适用）
- **定义**：……（保真，必要时贴原文）
- **要点**：……
- **公式 / 定理**：$……$（逐字保真，不改写）
  - 适用条件：……
- **算法 / 步骤**（如有）：
  ```
  <伪代码或代码，保真>
  ```
### 2. <下一个知识点>
……
（逐个覆盖本章全部知识点，这是主体，占最大篇幅）

## 三、公式 / 算法 / 定理速查表
（把本章所有公式、定理、关键算法集中列一处，便于速记与冲刺。）

## 四、重点对比与易混辨析
（用表格：如 X vs Y、几种调度算法对比、几种存储管理对比等。没有可省略。）

## 五、高频考点与易错提醒
（来自专项练习+答案的信号：哪些知识点反复考、答案解析里点名的常见错误与陷阱。**只列考点/错点，不放题目**。）

## 六、一句话记忆要点
（每个大知识点浓缩成一句，末尾冲刺用。）
```

**填写要求：**
- 数学/公式类（数据结构、计算机组成里的运算、数据库范式等）：公式、复杂度、进制/补码运算**必须 LaTeX 保真**。
- 代码/算法类（数据结构算法、SQL 语句）：**用代码块保真**，SQL 保留完整语句。
- 对比性强的内容（进程调度算法、页面置换算法、锁协议、范式等）**尽量做成表格**。
- 每个知识点若能判断考频，就在标题后标 ⭐（高频）或 ⚠（易错）；判断不了就不标。

## 5. 进度与容错

- **一次做一章，做完一章写一个文件**，不要试图一次性把整本读进来（扫描件很大，会撑爆上下文）。
- 建议维护一个进度文件 `cheatsheets/_progress.md`，每完成一章记一行（专题/章号/状态），便于中断后续跑。
- 若某章页面范围读下来发现与任务表标注对不上（页码错位、章节跨页），**以页面实际内容为准**，并在该章文件顶部「说明」里记一句实际页码，然后继续。
- 渲染出的临时 PNG 在 `/private/tmp/cheatsheet-render`，处理下一章前可覆盖，不必保留。
