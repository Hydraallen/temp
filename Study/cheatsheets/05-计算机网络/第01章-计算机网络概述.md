## 一、本章概览
本章是计算机网络的入门概述，主要解决“计算机网络是什么、怎么分类、架构如何”的问题。内容涵盖计算机网络的发展历程、未来趋势，按地理范围和拓扑结构等多种维度的分类方法，以及最为核心的网络体系架构（OSI 七层模型与 TCP/IP 四层模型）。本章是后续各章学习的基础，特别是 OSI 各层功能及设备对应关系、TCP/IP 协议栈的层次结构，是考试的高频考查区域。

## 二、知识点详解

### 1. 计算机网络概念与发展历程
- **网络共同特点**：连线和结点；网络作用：连通和共享。
- **以主机为中心的网络**：早期计算机通过终端连接到大型机上，集中式控制。
  - 缺点：主机负荷重；可靠性差。
  - 终端：是主机的一个远程的输入/输出设备（不是一个自治系统），本身不具备计算能力。
- **ARPAnet**：1969年，4个结点，网络从“主机-终端”转为“主机-主机”。
- **发展历程**：
  - 第一时期：联机系统。中央处理机连接大量分散计算机，前端处理机和通信控制器解决了运行速度问题。
  - 第二时期：20世纪60年代，计算机互联网。特点：分散交换和控制、资源多向共享、网络分层协议。具有独立和封闭的特点。
  - 第三时期：20世纪80年代，标准化网络。微处理器和集成电路推动微型计算机发展，局域网迅速发展，通过路由器和调制解调器实现信息共享。
  - 第四时期：20世纪90年代，互连和高速网络时代。核心为互联网，全球资源得到共享。
- **未来趋势**：移动通信（3G、4G、5G）、无线通信（WiFi）、网络融合（电信网、广播电视网、计算机通信网）、卫星通信、“六”化（数字化、宽带化、综合化、融合化、智能化、个人化）。

### 2. 计算机网络的分类 ⭐(练习4,10)
- **通信网络定义**：是由一定数量的节点（包括终端节点和交换节点）和连接这些节点的传输系统有机地组织在一起的，按约定的信令或协议，完成任意用户间信息交换的通信体系。
- **按业务分类**：电信网（话音业务）、计算机网络（数字业务）和广播电视网（广播和图像业务）。
- **电信网分类（按 ITU-T 标准）**：
  - 业务网：向用户提供各种电信业务（如PSTN、PLMN、ISDN等）。
  - 传送网：完成用户信号传输（如接入网AN、SDH）。
  - 支撑网：使业务网正常运行、增强网络功能、提供全网服务质量（如7号信令网、同步网、管理网）。
- **计算机网络按地理范围分类**：
  - **局域网 (LAN)**：地理覆盖范围较小(1-5km)，数据传输速率较高(10Mbps以上)，延迟小，具有较高的可靠性。目前多采用集线器或交换机连接的星形结构，以“以太网”应用最广泛。
  - **城域网 (MAN)**：介于局域网与广域网之间，覆盖一个城市，通常覆盖几十公里（分布距离一般为5到50km）。属于宽带网络，为整个城市服务。建设集中在通信子网上（城市骨干网、城市接入网）。
  - **广域网 (WAN)**：实现不同地区的LAN或MAN的互连，覆盖范围从几十公里到几千公里。由资源子网和通信子网组成。通信子网由传输线路和交换单元（路由器）组成，采用点到点的传输方式和“交换”原理。整个网络是一个网状结构。
- **互联网、因特网、万维网的关系**：
  - 互联网⊃因特网⊃万维网。
  - **互联网**：凡是能彼此通信的设备组成的网络就叫互联网（小写 `internet`）。
  - **因特网**：互联网的一种，由上千万台设备组成，使用 TCP/IP 协议（大写 `Internet`）。
  - **万维网 (WWW)**：只要应用层使用的是 HTTP 协议，就称为万维网。

### 3. 计算机网络的拓扑结构 ⭐(练习2,12,14,17)
- **定义**：从拓扑学来看网络可以看成是由若干结点和连接这些结点的链路组成的几何图形。一般所讲的拓扑结构指的是物理拓扑结构。
- **总线型**：
  - 通过一条公共线路把各个节点连接起来，无中心节点，信息发送是广播式的。总线两端需安装端接器。同一时间内只允许一个节点发送数据，可能出现“冲突”。
  - 优点：结构简单、经济、易于扩充、可靠性高。
  - 缺点：传输距离有限、故障诊断与排除困难，不具有实时功能。站点必须是智能的，要有媒体访问控制功能。
- **环型**：
  - 所有节点连接形成一个封闭的环，点对点通信。信号在环上以固定的方向传输，单向逐节点传输。
  - 优点：电缆长度短、增减工作站只需简单连接、可用光纤、网络初始安装和控制简单。
  - 缺点：节点故障容易引起全网故障、故障难检测、媒体访问协议采用令牌传递方式，负载轻时信道利用率低。
- **星型**：
  - 以一台设备作为中心节点（中继器或集线器），其他外围节点都与中心节点相连接。目前局域网系统中多采用星型拓扑结构。
  - 优点：控制简单、故障诊断与隔离容易、易于扩充、服务方便。
  - 缺点：电缆长度和安装工作量可观；中央节点负担较重；各站点分布处理能力较低。
- **树型**：
  - 总线型或星型结构的扩展，又叫层次结构。信息广播式传送。
  - 优点：易于扩充、故障隔离较容易。
  - 缺点：节点对根依赖性太大，根发生故障，则全网瘫痪。
- **网状型**：
  - 计算机之间通过多条线路连接。一般用于骨干网。
  - 优点：健壮性（可靠性高）。
  - 缺点：结构复杂、协议复杂、成本高。
- **按通信介质划分**：有线网（同轴电缆、双绞线、光纤）、无线网（红外线、微波、激光）。
- **按信息传播方式划分**：点对点通信（如Internet中远程主机通信）、广播式通信（如局域网通信）。

### 4. 计算机网络体系架构：标准化组织 ⭐(练习24)
- **ISO**：国际标准化组织，世界上最大的非政府性标准化专门机构，制定国际技术标准。
- **ITU**：国际电信联盟，负责分配和管理全球无线电频谱与卫星轨道资源。分为ITU-T、ITU-R、ITU-D。
- **IEC**：国际电工委员会，侧重硬件方面的标准（ISO侧重软件）。
- **IEEE**：电子及电器工程师协会，目前全球最大的非营利性专业技术学会。
- **EIA**：电子工业联盟。
- **ECMA**：欧洲计算机制造商协会。

### 5. OSI 参考模型 ⭐(练习22,23,26,27,31,32,34,35,36,37,38,39,40)
- **分层结构**：将网络结构划分为七层。第一层到第三层属于低三层，负责创建网络通信连接的链路；第四层到第七层为高四层，负责端到端的数据通信。
- **分层的优点**：各层间相互独立，某一层的变化不会影响其他层；促进标准化工作；使网络易于实现和维护。
- **分层结构工作原理**：纵向通信（低层为高层提供服务）；横向通信（对应的分层协同工作完成通信）。
- **各层功能及设备**：
  1. **物理层**：在物理连接上传输Bit流；定义网络的物理结构、传输介质的电气、机械规格。设备：中继器、集线器。
  2. **数据链路层**：分为LLC（逻辑链路控制）和MAC（媒体访问控制）子层。将比特组织成数据帧；硬件地址/物理地址寻址；差错检测与流量控制。设备：网桥、二层交换机。
  3. **网络层**：基于网络层地址进行不同网络系统间的路径选择与中继；将数据包以最佳路径通过通信子网到达目的主机；分割和重新组合数据包。设备：路由器、三层交换机。
  4. **传输层**：将数据组织成数据段；提供端到端的信息传送；通过纠正和流量控制机制提供可靠且有序列的数据包传送；用端口号来标识特定的应用程序。提供面向连接和无连接服务。
  5. **会话层**：建立、管理和终止会话。
  6. **表示层**：解决用户信息的语法表示问题；数据格式转换、数据压缩和解压缩、加密和解密。
  7. **应用层**：对软件提供接口以使程序能使用网络服务（文件传输、文件管理、电子邮件等）。

### 6. 数据封装与 OSI 模型特点
- **PDU (Protocol Data Units)**：在OSI模型的每一层，通过封装使每个PDU被附加到数据上。
  - 物理层：比特流
  - 链路层：数据帧
  - 网络层：数据包
  - 传输层：数据段
- **数据封装过程**：
  1. 用户信息转换为数据。
  2. 数据转换为数据段，建立可靠连接。
  3. 数据段转换为数据包或数据报，放上逻辑地址。
  4. 数据包转换为帧，放上硬件地址。
  5. 帧转换为比特流，采用数字编码和时钟方案。
- **OSI模型特点**：关键概念是虚电路。每一层都有一虚电路直接连接目的主机上的对应层。层不知道传输数据的实际细节，只利用上下层服务维持虚电路。

### 7. TCP/IP 模型 ⭐(练习21,25,33)
- **四层结构**：因特网是4层结构，协议栈由几十个协议组成，用两个最重要的协议TCP和IP表达。
- **各层功能及协议**：
  1. **主机—网络层（网络接口层）**：没有定义任何专用的协议，支持现有的局域网和广域网标准和协议（如IEEE802.3、X.25）。
  2. **互联网层**：将分组传送到因特网上的任一台主机上。
     - IP协议：提供无连接服务。
     - 支持协议：ARP（地址解析协议）、RARP（逆地址解析协议）、ICMP（因特网控制报文协议）、IGMP（因特网组管理协议）。
  3. **传输层**：
     - UDP：无连接服务。
     - TCP：面向连接服务。
     - SCTP：流控制传输协议。
  4. **应用层**：包含SMTP、FTP、HTTP、DNS、SNMP、TELNET等协议。

## 四、重点对比与易混辨析

### 1. 网络分类按地理范围对比
| 特性 | 局域网 (LAN) | 城域网 (MAN) | 广域网 (WAN) |
| :--- | :--- | :--- | :--- |
| 覆盖范围 | 1-5km (房间、大楼、校园) | 5到50km (一个或几个城市) | 几十到几千公里 (国家/全球) |
| 传输速率 | 10Mbps以上，延迟小 | 几十Kbps到100Mbps | 高速链路，较大通信容量 |
| 通信方式 | 广播式通信为主 | [原文未述具体方式，多采用以太网技术] | 点对点通信技术，采用“交换” |
| 结构特点 | 拓扑稳定规范(星型等) | 集中在通信子网(骨干网+接入网) | 网状结构，有资源子网与通信子网界定 |

### 2. 网络拓扑结构对比
| 拓扑类型 | 通信方式 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| 总线型 | 广播式 | 结构简单、经济、易于扩充、可靠性高 | 传输距离有限、故障诊断困难、有冲突 |
| 环型 | 点对点、单向 | 电缆短、增减站简单、可用光纤 | 节点故障引起全网故障、轻载利用率低 |
| 星型 | [原文未述] | 控制简单、诊断隔离容易、易扩充 | 电缆长、中心节点负担重、分布处理低 |

| 网状型 | 多条线路 | 健壮性（可靠性高） | 结构复杂、协议复杂、成本高 |

| OSI 模型 | TCP/IP 模型 | 典型协议 |
| :--- | :--- | :--- |

## 五、高频考点与易错提醒
- ⚠ **OSI 各层功能及对应设备**是高频易错点：物理层（中继器、集线器）、数据链路层（网桥、二层交换机）、网络层（路由器、三层交换机）。(练习35,38,39)
- ⚠ **OSI 传输层与网络层区分**：传输层负责端到端信息传送，用端口号寻址；网络层负责路径选择，用逻辑地址寻址。(练习22,23,32,38)
- ⚠ **OSI 表示层功能**：数据格式转换、数据压缩和解压缩、加密和解密。(练习31,40)
- ⚠ **TCP/IP 模型层次对应**：TCP/IP 的应用层对应 OSI 的上三层；ARP/RARP 属于互联网层（非应用层）。(练习21,25,33)
- ⚠ **拓扑结构特点及故障影响**：星型中心节点故障全网瘫痪，树型根节点故障全网瘫痪，环型节点故障易引起全网故障。(练习2,12)
- ⚠ **广域网拓扑结构**：广域网通常是网状结构。(练习17)
- ⚠ **计算机网络资源**：主要指硬件、软件与数据。(练习8,11)
- ⚠ **互联网与因特网大小写区别**：互联网 `internet` (小写)，因特网 `Internet` (大写)。
- ⚠ **传输介质特性**：光纤带宽最宽，信号传输衰减最小，抗干扰能力最强。(练习7)

## 六、一句话记忆要点
- **网络分类**：局域网小而快广播多，广域网大而广点对点，城域网居中连骨干。
- **拓扑结构**：总线广播易冲突，环型单向怕断点，星型靠中心，树型靠根，网状最可靠但最贵。
- **OSI模型**：物集网传会表应（物理层传比特集线器，链路层传帧网桥交换机，网络层传包路由器，传输层段端到端，会话层建会话，表示层翻译加密，应用层服务）。
- **TCP/IP模型**：网接互传应（网络接口、互联网、传输、应用），IP无连接，TCP面向连接，ARP/RARP在网络层。
