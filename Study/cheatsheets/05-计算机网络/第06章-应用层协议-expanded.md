## 一、域名系统 DNS ⭐

**DNS 概念**：域名系统 DNS (Domain Name System) 负责将主机名转换为 IP 地址。Internet 的域名系统被设计成一个联机分布式数据库系统，并采用客户／服务器模式。在专设的结点上运行域名服务器程序，运行该程序的计算机称为域名服务器。*(讲义 p108)*
- 域名是一个逻辑概念，用于标识主机，并不反映主机所在的物理地点。*(讲义 p108)*

**域名结构**：域名采用层次结构：`三级域名．二级域名．顶级域名`。每一级域名不超过 63 个字符（不分大小写），完整的域名不超过 255 个字符。*(讲义 p108)*

**顶级域名分类**：顶级域名由 Internet 的有关机构管理，有三大类：*(讲义 p108)*

| 类别 | 说明 | 例子 |
|---|---|---|
| 国家顶级域名 | 以英文缩写表示 | cn、us、uk |
| 国际顶级域名 | 用于国际性组织 | int |
| 通用顶级域名 | 表示机构性质 | com（公司企业）、net（网络服务机构）、org（非赢利性组织）、edu（教育机构）、gov（政府部门，美国专用）、mil（军事部门，美）、firm、shop、web、arts、rec、info、nom |

**中国二级域名**：中国将二级域名划分为"类别域名"和"行政区域域名"。*(讲义 p109)*
- 类别域名 6 个：ac（科研机构）；com（企业）；edu（教育）；gov（政府）；net（网络信息中心）；org（非赢利性组织）。*(讲义 p109)*
- 行政区域名 34 个：省、自治区、直辖市，如 bj（北京），sh（上海）。*(讲义 p109)*
- 三级域名注册：在二级域名 edu 下申请注册三级域名由中国教育和科研网网络中心负责；在其他二级域名下申请注册三级域名，应向中国互联网网络信息中心(CNNIC)申请。*(讲义 p109)*

**域名与 IP 的转换过程**：源主机的某个应用进程，将待转换的域名放在 DNS 请求报文中，以 **UDP 数据报**方式发给本地域名服务器。本地域名服务器在查找域名后，将对应的 IP 地址放在回答报文中，返回给请求进程。若本地域名服务器不能回答该请求，则该域名服务器就暂时成为 DNS 中的另一个客户，去寻找能够回答该请求的域名服务器，直到找到为止。*(讲义 p109)*

**解析方法** ⭐：*(讲义 p110)*
- **递归查询**：逐级查询，逐级回答。*(讲义 p110)*
- **递归与迭代结合**：根域名服务器在收到查询后，直接将下属的授权域名服务器的 IP 地址返回给发出查询的本地域名服务器，由本地域名服务器直接向授权域名服务器查询，从而可减轻根域名服务器的负担。*(讲义 p110)*

**高速缓存**：每个域名服务器都维护一个高速缓存，存放着最近使用过的域名以及从何处获得域名的 IP 地址映射信息的记录，用以优化域名查询。*(讲义 p110)*

## 二、文件传输协议 FTP 与 TFTP ⭐

**FTP（文件传输协议）**：基于 **TCP**。使用端口号 **20（数据连接）**和 **21（控制连接）**。传送文件类型分为两种：ASCII（文本）文件和二进制（非文本）文件。*(讲义 p111)*
- FTP 服务器进程由两部分组成：一个主进程，负责接受请求；若干个从属进程，负责处理单个请求。*(讲义 p111)*
- 工作时需要建立两个连接：**控制连接**和**数据连接**。*(讲义 p111)*
- 匿名 FTP：使用默认的帐号(anonymous)、口令即可登录访问的 FTP 服务。*(讲义 p111)*
- 特点：必须先获得文件的副本，才能修改文件，然后将该副本传回到服务器。*(讲义 p111)*

**TFTP（简单文件传送协议）**：是一个很小且易于实现的文件传送协议。使用客户服务器方式和使用 **UDP 数据报**，因此 TFTP 需要有自己的差错改正措施。*(讲义 p111)*
- 只支持文件传输而不支持交互。*(讲义 p111)*
- 没有一个庞大的命令集，没有列目录的功能，也不能对用户进行身份鉴别。*(讲义 p111)*
- 每次传送的数据 PDU 中有 512 字节的数据，但最后一次可不足 512 字节。*(讲义 p111)*
- 数据 PDU 也称为文件块(block)，每个块按序编号，从 1 开始。*(讲义 p111)*
- 支持 ASCII 码或二进制传送；可对文件进行读或写；使用很简单的首部。*(讲义 p111)*

**FTP 与 TFTP 对比**：*(讲义 p111)*

| 特性 | FTP | TFTP |
|---|---|---|
| 传输层协议 | TCP | UDP |
| 端口号 | 20（数据）、21（控制） | [原文未述] |
| 差错控制 | 依赖 TCP 可靠传输 | 需有自己的差错改正措施 |
| 交互性 | [原文未述] | 只支持文件传输，不支持交互 |
| 命令集 | [原文未述] | 无庞大命令集，无列目录，无身份鉴别 |
| 数据 PDU | [原文未述] | 512 字节/块，块编号从 1 开始 |
| 文件类型 | ASCII、二进制 | ASCII、二进制 |

## 三、远程终端协议 Telnet

**Telnet 概念**：远程登录是为用户提供的以终端方式与 Internet 上的主机建立在线连接的一种服务。连接建立后，用户的计算机就可以作为远程主机的一台终端来使用该主机提供的各种资源。*(讲义 p112)*
- 用户所在的计算机被称为本地(Local)计算机或本地主机(Host)；用户所连接的另一台计算机称为远程(Remote)计算机或远程主机。*(讲义 p112)*

## 四、电子邮件 SMTP、POP3 和 IMAP ⭐

**电子邮件概况**：电子邮件是 Internet 最早开发的而且是使用最广泛的应用之一。1982 年制订了 ARPANET 的"简单电子邮件协议"SMTP 和"电子邮件报文格式"MAIL。CCITT 制订了"报文处理系统"MHS 的标准 X.400。为传送多媒体数据，1993 年又制订了"通用因特网扩充"MIME 标准。目前因特网的电子邮件系统已成为事实上的标准，而 X.400 则由于过于复杂而基本上被淘汰。*(讲义 p113)*

**电子邮件系统组成**：一般由两个子系统组成：*(讲义 p113)*
- **用户代理 UA**：用户与系统的接口，本地程序，提供收发邮件的界面。*(讲义 p113)*
- **邮件服务器**：运行在后台的服务器软件，负责消息的管理、传送与提交。*(讲义 p113)*

**电子邮件系统的基本功能**：撰写、传输、报告、呈现、处理。*(讲义 p113)*

**SMTP 协议**：在 Internet 电子邮件系统中，使用简单邮件传送协议 SMTP 进行邮件传送。SMTP 规定在两个通信的 SMTP 客户进程和服务器进程之间如何交换信息。*(讲义 p113)*
- 使用客户/服务器模式，在 **TCP 连接**上传送 **ASCII 码**邮件。*(讲义 p113)*
- 负责发送邮件的 SMTP 进程是 SMTP 客户，负责接收邮件的 SMTP 进程是 SMTP 服务器。*(讲义 p113)*
- 发送前要先建立 TCP 连接，SMTP 客户与目的主机的 SMTP 服务器（使用周知端口 **25**）建立 TCP 连接。可以发送多个电子邮件，发送完后 TCP 连接被释放。*(讲义 p113)*

**POP3 协议**：邮局协议版本 3 (POP3)很简单，但功能有限。客户 POP3 软件安装在收信人的计算机上；服务器 POP3 软件安装在邮件服务器上。*(讲义 p113)*
- POP 客户与 POP 服务器（使用 TCP 端口 **110**）建立 TCP 连接。POP 客户发送用户名和口令，访问邮箱，可列出清单并逐个读取邮件报文。*(讲义 p113)*

**SMTP 与 POP3 工作过程**：*(讲义 p114)*
- **SMTP**：负责电子邮件的发送，电子邮件在 Internet 上的传递通过 SMTP 来实现。*(讲义 p114)*
- **POP3**：用于接收 Internet 上的电子邮件，电子邮件在客户 PC 机与服务提供商之间的传递通过 POP3 来完成。*(讲义 p114)*

> Caveat (Ch6-判8): SMTP 只能传输 ASCII 文本，不能直接传多媒体。MIME 扩展后才支持多媒体。*(题目Ch6-判8)*
> Caveat (Ch6-判10): 基于万维网的电子邮件，用户↔邮件服务器用 HTTP，但邮件服务器之间仍用 SMTP。*(题目Ch6-判10)*

## 五、万维网 WWW ⭐

**WWW 概念**：www 是 World Wide Web 的简称，译为万维网。www 使用超文本(Hypertext)组织、查找和表示信息，利用了从一个站点到另一个站点的链接。超文本文件使用超文本标记语言(HTML)格式写成。*(讲义 p115)*
- 热链含有指向其他 Internet 信息 URL 地址(Uniform Resource Locator)。*(讲义 p115)*
- 链接可指向声音、视频等多媒体超文本，与多媒体一起形成超媒体(Hypermedia)。*(讲义 p115)*

**WWW 的工作原理**：www 由三部分组成：客户机(Client)、服务器(Server)和 HTTP 协议。以 Client/Server 方式工作。*(讲义 p115)*
- 客户机向服务器发送请求并得到响应；服务器负责管理信息和对请求作出回答。*(讲义 p115)*
- 客户机与服务器使用 HTTP 协议（超文本传送协议）传送文本信息。*(讲义 p115)*
- 服务器的基本信息单位叫网页(Web 页)，由 HTML 写成。客户机请求采用 URL 形式，Web 页中有许多超链指向其他 URL 地址。*(讲义 p116)*

**URL 地址**：URL 是 Uniform Resource Locator 的缩写，译为统一资源定位器。URL 大致由三部分组成：协议、主机名、目录或文件名。*(讲义 p116)*
- 协议即访问页的方式（如 http、ftp、gopher）。*(讲义 p116)*
- 主机名是 Internet 上用于标识信息保存的地方（如 www.netcom.com）。*(讲义 p116)*
- URL 的主机名中可包含一个端口号，写在主机名和目录之间（如 `http://my-public-access-unix.com:150/pub/file`）。*(讲义 p117)*
- URL 中允许使用的字符包括：字母 A-Z, a-z，数字 0-9，美元符号($)，加号(+)，减号(－)，下划线(_)和句点(.)。其他字符须用 URL 换码字符代替。*(讲义 p117)*

**URL 换码序列**：以百分号加上两个字符形成的 ISO-Latin-1 字符集。*(讲义 p117)*
- `%20` 是空格符，`%3f` 是问号，`%2f` 是斜杠。*(讲义 p117)*

**常见 URL 类型**：*(讲义 p117-118)*

| 类型 | 说明 | 示例 |
|---|---|---|
| HTTP | Internet 上最流行的 URL 类型，超文本传输协议，www 服务器用于传输 HTML 页的协议 | `http://www.foo.com/home/foo/` |
| 匿名 FTP | 用于访问 FTP 服务器上的文件，用户用 anonymous 作为登录名，电子邮件地址作为口令 | `ftp://ftp.com/home/foo` |
| 非匿名 FTP | 加入了账号、密码的 FTP URL，格式中无法对口令进行隐藏 | `ftp://username:password@ftp.foo.com/home/foo/homepage.html` |
| 文件(file) | 索引本地盘上的文件，引用浏览器所在系统上的文件；第一部分是空主机名(有 3 个而非 2 个斜杠)或用 localhost 作主机名 | `file://localhost/dirl/dir2/file` |

## 六、动态主机配置协议 DHCP ⭐

**DHCP 概念**：*(讲义 p119)*
- **DHCP Client**：DHCP 客户端，通过 DHCP 协议请求 IP 地址的客户端。接口级概念，每个接口都可配置成一个 DHCP 客户端。*(讲义 p119)*
- **DHCP Server**：DHCP 服务端，负责为 DHCP 客户端提供 IP 地址，并管理分配的 IP 地址。*(讲义 p119)*
- **DHCP Relay**：DHCP 中继器，DHCP 客户端跨网段申请 IP 地址时，实现 DHCP 报文的转发功能。*(讲义 p119)*
- **DHCP Security**：DHCP 安全特性，实现合法用户 IP 地址表的管理功能。*(讲义 p119)*
- **DHCP Snooping**：DHCP 监听，记录通过二层设备申请到 IP 地址的用户信息。*(讲义 p119)*

**DHCP 工作原理**：*(讲义 p119-121)*

1. **发现阶段**：DHCP 客户端寻找 DHCP 服务端的过程，对应于客户端发送 DHCP Discovery。因为 DHCP Server 对客户端是未知的，所以 DHCP Discovery 报文是**广播包**，源地址为 `0.0.0.0`，目的地址为 `255.255.255.255`。网络上所有支持 TCP/IP 的主机都会收到该报文，但只有 DHCP Server 会响应。如果网络中存在多个 DHCP Server，则均会回复。*(讲义 p119)*
   - DHCP Relay：如果同一个 vlan 内没有 DHCP Server，而该 VlanIf 配置了 DHCP Relay 功能，则该 VlanIf 即为 DHCP 中继。中继会将报文源 IP 地址修改为该 VlanIf 的 IP 地址，[目的?]地址则为 DHCP Relay 配置的 DHCP Server 的 IP 地址。同时修改报文中 giaddress 为 VlanIf 的 IP 地址，并以**单播**将 DHCP Discovery 发送到 DHCP Server 端。*(讲义 p119-120)*

2. **DHCP Server 提供阶段**：即 DHCP Server 响应 DHCP Discovery 所发的 DHCP Offer 阶段。*(讲义 p120)*
   - Server 收到 Discovery 报文后，解析请求 IP 地址所属的 Subnet，从匹配的 subnet 中取出一个可用 IP 地址。*(讲义 p120)*
   - **冲突检测机制**：选择 IP 地址后，首先发送 ICMP 报文来 ping 该 IP 地址，如果收到回应则抛弃该 IP 地址，重新选择并继续测试，直到找到网络中没有人使用的 IP 地址。*(讲义 p120)*
   - 将分配的 IP 地址设置在 Discovery 报文中 yiaddress 字中，并为该 Lease 设置 Subnet 配置 Option（如默认租期、最大租期、router 等信息）。*(讲义 p120)*

3. **DHCP Client 选择阶段**：Client 收到若干个 DHCP Offer 报文后，选择其中一个 DHCP Server 作为目标（通常选择第一个响应的 Offer 所属的 Server）。然后以**广播**方式回答一个 DHCP Request 报文，包含向目标 DHCP 请求的 IP 地址等信息。广播方式是为了通知其他 DHCP Server 自己的选择。*(讲义 p120)*

4. **DHCP Server 确认阶段**：Server 收到 Request 后，确认要为该 Client 提供的 IP 地址，便向 Client 响应一个包含该 IP 地址以及其他 Option 的 **ACK** 报文。Client 即可将该 IP 地址与网卡绑定。其他 DHCP Server 都将收回之前为该 Client 提供的 IP 地址。*(讲义 p120)*

5. **DHCP Client 重新登录网络**：Client 重新登录后，发送一个包含之前分配的 IP 地址信息的 DHCP Request 报文。Server 收到后会尝试让客户端继续使用该 IP 地址，并回答一个 **ACK** 报文。如果该 IP 地址无法再次分配，Server 回复 **NAK** 报文，Client 收到后重新发送 DHCP Discovery 报文来获取 IP 地址。*(讲义 p120-121)*

6. **DHCP Client 更新租约**：DHCP 获取到的 IP 地址都有租约，租约过期后 Server 将回收。如果 Client 想继续使用，必须更新租约。更新方式：当前租约期限过了一半后，Client 发送 **DHCP Renew** 报文来续约租期。*(讲义 p121)*

**DHCP Server 从地址池中选择 IP 地址的优先级**：*(讲义 p120)*
1. 当前已经存在的 IP-MAC 的对应关系*(讲义 p120)*
2. Client 以前的 IP 地址*(讲义 p120)*
3. 读取 Discovery 报文中的 Requested IP Address Option 的值，如果存在并且 IP 地址可用*(讲义 p120)*
4. 从配置的 Subnet 中选择 IP 地址*(讲义 p120)*

## 八、高频考点与易错提醒

- **DNS** ⭐：域名转换使用 UDP 数据报；递归查询与迭代查询的区别（根服务器是否直接返回下属授权服务器 IP 给本地服务器）；高速缓存优化查询。*(讲义 p109-110)*
- **FTP** ⭐：两个连接（控制连接端口 21、数据连接端口 20）；基于 TCP；匿名 FTP 帐号 anonymous。*(讲义 p111)*
- **TFTP** ⚠：基于 UDP，需自己的差错改正措施；每次 512 字节，块编号从 1 开始；无交互、无命令集、无身份鉴别。*(讲义 p111)*
- **SMTP** ⭐：发送邮件，TCP 端口 25，只能传 ASCII 码邮件；邮件服务器之间用 SMTP。*(讲义 p113-114)*
- **POP3** ⭐：接收邮件，TCP 端口 110。*(讲义 p113)*
- **WWW** ⭐：三部分（客户机、服务器、HTTP 协议）；URL 三部分（协议、主机名、目录/文件名）。*(讲义 p115-116)*
- **DHCP** ⭐：Discovery 广播包源 `0.0.0.0` 目的 `255.255.255.255`；Client 选择阶段广播 Request 通知其他 Server；租期过半发 Renew 续约。*(讲义 p119-121)*
- **应用层协议归属** ⭐：FTP、SMTP、POP3、HTTP、DNS、Telnet、DHCP 均为应用层协议。*(讲义 p121-123（课后习题）)*

## 九、一句话记忆要点

- **DNS**：UDP 传查询，层次结构查域名，递归逐级答、迭代根返回。*(讲义 p108-110)*
- **FTP**：TCP 双连接，21 控制 20 数据，匿名 anonymous。*(讲义 p111)*
- **TFTP**：UDP 简单传，512 字节块从 1 编，无交互无鉴别。*(讲义 p111)*
- **Telnet**：远程登录变终端，本地主机操控远程资源。*(讲义 p112)*
- **SMTP/POP3**：SMTP 发送 TCP 25 传 ASCII，POP3 接收 TCP 110。*(讲义 p113-114)*
- **WWW**：客户机+服务器+HTTP，URL 找资源（协议+主机+路径）。*(讲义 p115-116)*
- **DHCP**：广播 Discovery 找 Server，Offer 提供选第一个，广播 Request 通知全网，ACK 确认绑定 IP，租期过半 Renew 续约。*(讲义 p119-121)*
