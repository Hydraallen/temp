# 信息技术 · 物联网基础 — 复习资料

## 一、物联网架构与基本概念

**物联网概念**：物联网是一个基于互联网、传统电信网等信息载体，让所有能被独立寻址的普通物理对象实现互联互通的网络。

**物联网的三个重要特征**：普通对象设备化、自治终端互联化和普适服务智能化。

**物联网核心技术（四层模型）**：根据信息生成、传输、处理和应用，物联网分为感知识别层、网络构建层、管理服务层和综合应用层。
> Caveat (题目2): 感知识别层位于物联网四层模型的最底端，是所有上层结构的基础，考点经常询问最底层是哪一层。

**物联网感知层传输数据的要求**：无障碍、高可靠性、高安全性。

## 二、自动识别技术与 RFID ⭐

**自动识别技术分类**：光符号识别、语音识别、虹膜识别、指纹识别、IC 技术、条形码技术。

**IC 卡原理与特点**：实际上是一种数据存储系统，如有必要还可附加计算能力。
> Caveat (单5): IC 卡不一定都是无线通信，接触式 IC 卡通过表面金属触点与读卡器物理连接通信，非接触式才采用无线通信。

**RFID 系统三大组件**：标签、阅读器、天线。

**RFID 标签分类**：依据内部供电有无，分为被动式（无源）、半被动式（半主动式）、主动式（有源）。

| 频段类别 | 频率 / 典型频点 | 通信距离 | 特点及适用场景 | 常用标签类型 |
| :--- | :--- | :--- | :--- | :--- |
| 低频 (LF) | 125-134 kHz | 短（几厘米） | 抗干扰能力强，适合动物识别、门禁系统 | 无源标签 |
| 高频 (HF) | 13.56 MHz | 适中（几厘米到一米） | 适合智能卡、图书管理 | 无源标签 |

> Caveat (单3): 低频 125KHz 和高频 13.56MHz 频点上一般采用无源标签。

**主动式标签特性**：因内部携带电源又被称为有源标签。主动式标签通信距离更远，可达上百米。工作模式有主动模式和唤醒模式。

## 三、无线传感器网络 (WSN)

**WSN 基本概念**：由大量分布式传感器节点组成的自组织网络，用于感知、采集和处理环境数据。

**WSN 节点结构**：由感知单元、处理单元、通信单元、电源单元组成。

**传感器按工作原理分类**：
1. 电参数传感器（电阻式、电感式、电容式）
2. 压电式传感器
3. 光电式传感器（一般光电式、光纤式、激光式、红外式等）
4. 热电式传感器
5. 半导体式传感器
6. 波式和辐射式传感器

**微处理器关键特性**：功耗特性、唤醒时间、供电电压、运算速度、内存大小。

**通信芯片重要指标**：睡眠功耗、唤醒时间、发射功率、接收灵敏度、接受功耗。

**常见的节点操作系统**：TinyOS 系统、Contiki OS 系统、Mantis OS 系统、Nanno-RK 系统、SenSpire OS 系统等。

**制约传感器性能提升的因素**：功耗的制约、价格的制约、体积的制约。

## 四、定位系统与定位技术 ⭐

**位置信息组成要素**：
1. 所在的地理位置
2. 处在该地理位置的时间
3. 处在该地理位置的对象（人或设备）

**现存的主流定位系统**：
1. 卫星定位（GPS）
2. 蜂窝基站定位
3. 室内精确定位
4. WiFi 基站定位

**GPS 系统组成**：GPS 导航系统由三部分组成：
1. 地面控制部分（主控站、地面天线、随机测量站和通信辅助系统）
2. 空间部分（分布在 6 个轨道平面的 24 颗卫星）
3. 用户设备（GPS 接收机和卫星天线）
> Caveat (单9): 冲突检测部分不属于 GPS 系统的构成组件。

**GPS 功能**：主要功能是定位、导航、授时等。
> Caveat (单10): GPS 的功能不包含通信功能。

**常见的定位技术及特点对比**：

| 定位技术 | 特点 / 要求 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| ToA (基于距离/时间) | 需要接收端特殊装置；需要参考点和测量目标时钟同步 | 测距原理直观（基于信号传播时间），在视距 (LOS) 条件下定位精度较高 | 需要参考点和测量目标时钟同步 |
| TDoA (基于距离差) | 需要接收端特殊装置 | 不需要参考点和测量目标时钟同步 | 参考点之间仍然需要时钟同步 |
| RSS (基于信号特征) | 基于接收信号强度指示 (RSSI) 测距，无需特殊硬件，利用已有无线信号 | 硬件成本低、部署简单，不需时钟同步 | 需要事先建立精细的特征地图，不能应对动态变化 |

**室内定位技术方案特点**：
1. 一般不利用 TOA、TDoA、AOA 等技术（需较贵硬件支持），一般采取基于 RSS 技术。
2. 利用已有设备与网络（蓝牙，WiFi，ZigBee）：部署方便、成本低、精度有限。
3. 利用专门设备与网络（红外线、超声波、RFID、超宽带 UWB 等）：精度高、部署成本高。

**LANDMARC 定位法**：布置一系列 RFID 标签作为参考标志；随时记录收到的 RSS 信号强度特征向量；将目标测得的信号特征向量与参考标志特征向量比对确定位置，误差在 1m 范围以内。

## 五、无线宽带网络、无线低速网络与物联网协议 ⭐

**无线网络组成元素**：无线网络用户、基站、自组网、无线连接。

**无线网络的难点**：信号强度衰减、非视线传输、信号干扰、多径传播。

**CSMA/CA 协议工作原理**：即使侦听到信道为空，也为了避免冲突而等待一小段随机时间以后再发送数据帧。

**常见无线低速网络协议**：红外、蓝牙通信协议、IEEE 协议、体域网协议、容迟网络协议。

**蓝牙设备的工作模式**：主设备和从设备。同一个蓝牙设备可以在这两种角色之间转换。

**Zigbee 标准**：基于 IEEE 802.15.4 标准。
> Caveat (单12): Zigbee 基于 IEEE 802.15.4 标准，不要与 Wi-Fi (IEEE 802.11) 混淆。

**Zigbee 特点**：低功耗、低成本、时延短、网络容量大、可靠、安全。

**ZigBee 设备类型及差别**：精简功能设备(RFD)、全功能设备(FFD)。FFD 有能力成为协调者并建立新网络，RFD 只能注册连结使用网络。必须属于 FFD 才可向协调者要求保证传输时槽使用权。

**ZigBee 设备角色**：
1. 端设备 End device (RFD or FFD)
2. 协调器 Coordinator (FFD)
3. 网络协调器 PAN Coordinator (FFD)

**ZigBee 网络拓扑结构类型**：星形网、网状网、树状网。

**ZigBee 降低能耗机制**：
1. 低功率侦听协议 (LPL)：采样侦听、链路层调度。
2. 采样侦听：无线收发模块在没有数据的时候关闭、定时采样获取信道的信息。

**ZigBee 网络层路由协议**：按需距离矢量路由协议（AODV）。

**物联网常见协议特点对比**：

| 协议名称 | 特点 | 优点 | 缺点 | 典型应用场景 |
| :--- | :--- | :--- | :--- | :--- |
| MQTT | 轻量级、基于发布/订阅模式 | 适合低带宽不稳定网络，支持异步通信 | 依赖中间代理 (Broker)，存在单点瓶颈；规范未规定消息格式，需双方约定 | 远程监控、智能家居、工业物联网 (IIoT) |
| CoAP | 基于 REST 架构，专为低功耗设备设计 | 支持 UDP 传输，适合资源受限的设备 | 基于 UDP，可靠性弱于 TCP；资源发现与安全机制相对简单 | 智能城市、环境监测、低功耗传感网 |
| HTTP | 基于请求/响应模式，广泛使用的 Web 协议 | 兼容性好，易于实现 | 开销较大，不适合低功耗设备 | 物联网设备与云平台的数据交互 |
| AMQP | 面向消息的中间件协议 | 高可靠性，支持可靠的消息传递 | 协议较重、报文开销大；不适合极度资源受限设备 | 金融交易、企业级物联网应用 |
| Zigbee | 基于 IEEE 802.15.4 标准短距离通信 | 低功耗、自组网能力强 | 通信距离短、速率低；跨厂商互操作性受限 | 智能家居、工业自动化、传感器网络 |
| LoRaWAN | 低功耗广域网 (LPWAN) 技术 | 覆盖范围广，功耗低 | 速率低、延迟大；不适合实时/大流量应用 | 智慧农业、智能城市、远程监测 |
| NB-IoT | 基于蜂窝网络的低功耗广域网技术 | 覆盖广、穿透性强 | 上下行速率低、不支持语音；依赖运营商网络部署 | 智能电表、智慧停车、环境监测 |
| Bluetooth | 短距离无线通信技术 | 低功耗、易于部署 | 通信距离短、可连接节点数有限 | 可穿戴设备、智能家居、健康监测 |
| Modbus | 工业通信协议，基于主从架构 | 简单、可靠，广泛用于工业设备 | 无内置认证/加密，安全性弱；数据模型缺乏统一标准 | 工业自动化、设备监控 |
| WebSocket | 全双工通信协议，基于 TCP | 实时性强，适合需持续连接的场景 | 需维持长连接，服务器并发资源占用高；断线重连需应用层处理 | 实时监控、在线游戏、即时通讯 |

**物联网协议场景选择总结**：
- 低功耗设备：CoAP、MQTT、Zigbee、LoRaWAN、NB-IoT。
- 实时性要求高：WebSocket、MQTT。
- 工业场景：Modbus、AMQP。
- 短距离通信：Bluetooth、Zigbee。
- 长距离通信：LoRaWAN、NB-IoT。

**体域网 (WBSN/BSN)**：是基于无线传感器网络 (WSN) 的，人体上的或移植到人体内的生物传感器共同形成的一个无线网络，是物联网重要的感知及组成部分。

**容迟网络 (DTN)**：网络中端到端路径通常很难建立，消息传播具有很大延时，使得基于 TCP/IP 的协议失效。

## 六、移动通信网络

**蜂窝网络工作原理（GSM系统模块）**：
1. 基站系统（基站和相关控制器）
2. 网络和交换系统（核心网，负责衔接）
3. GPRS 核心网（可选，基于报文的互联网连接）
4. 身份识别模块（SIM 卡）

**CDMA 技术原理**：利用编码技术区分并分离多个同时传输的信号。允许用户任何时刻在任何频段发送信号，从混合信号中提取期望数据并拒绝噪音信号。

**3G 技术标准与归属**：
1. TD-SCDMA：中国移动获得（我国自主研究制定）。
2. WCDMA：中国联通获得。
3. CDMA2000：中国电信获得。

**移动通信系统基本结构**：由移动终端、接入网与核心网三部分组成。
> Caveat (单21): 核心网与基站一般通过光纤等有线网通信，而非通过空中接口通信。

## 七、数据存储与数据中心

**三种网络存储对比**：

| 存储类型 | 全称 / 描述 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| DAS | 直接附加存储 | 管理容易，结构简单 | 集中式体系结构，不能满足大规模数据访问需求；资源利用率低，造成“信息孤岛” |
| NAS | 网络附加存储 | 网络的存储实体，容易实现文件级别共享 | 性能严重依赖网络流量，用户多、读写频繁时受限 |
| SAN | 存储区域网络 | 存储共享、支持服务器从 SAN 直接启动，容量利用率高 | 无直接文件级别的访问能力，但可在 SAN 基础上建立文件系统 |

**物联网数据的特点**：海量性、多态性、关联性及语义性。

**数据中心概念**：维基百科定义其为包含计算机系统、配套设备、冗余连接、环境控制及安全装置的复杂设施；Google 定义其为容纳服务器与通信设备的多功能建筑物。

**数据中心相关技术**：GFS、Hadoop、HDFS、MapReduce、Hbase 等。

**主流关系数据库**：SQL Server、Sybase、Oracle、DB2。

**NoSQL 数据库分类**：键值存储数据库、列存储数据库、文档型数据库、图形数据库。

**云存储概念**：云存储提供按需使用的专业化仓储服务，实现可扩展的海量存储，用户不能明确指明主机（由系统自动分配）。

## Worked Examples

### 例 1：物联网 LPWAN 与局域网通信技术辨析 *(来自 exam-signals 候选 1)*
*Tags: 物联网通信技术, ZigBee, NB-IoT, LoRa*
**问题**：以下哪种技术基于蜂窝网络？/ 哪种技术工作于非授权频段且与NB-IoT齐名？/ 哪种技术具有低功耗、低速率、自组网特点？
**解答**：`[PROPOSED — verify]`
1. 判断频段授权：NB-IoT 基于蜂窝网（授权频段）；LoRa 工作于非授权频段。
2. 判断传输距离：NB-IoT 与 LoRa 属于 LPWAN（长距离广域网）；ZigBee 与蓝牙属于短距离局域网。
3. 判断组网与速率特征：ZigBee 具有低功耗、低速率、自组网特征，常用于传感器网络；蓝牙适合短距离但组网能力弱。
4. 匹配选项：蜂窝网 → NB-IoT；非授权长距离 → LoRa；自组网短距离 → ZigBee。
