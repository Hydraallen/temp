## 一、本章概览

本章是数据库课程的核心章节，系统讲解关系数据库标准语言 SQL 的全部主要功能。SQL 不仅仅用于查询，而是涵盖数据定义（模式/表/视图/索引的创建、修改、删除）、数据查询（单表查询、连接查询、嵌套查询、集合查询）、数据更新（插入/修改/删除）以及视图的定义与操作。本章与第三章关系数据库（关系代数）紧密衔接——SQL 是关系代数在工程上的标准实现；同时为后续章节（数据库安全性、完整性、数据库设计等）提供操作基础。

## 二、知识点详解

### 一、SQL 基础概念

#### 1. 结构化查询语言 ⭐ (练习单选1)

- **定义**：结构化查询语言（Structured Query Language，SQL）是关系数据库的标准语言，也是一个通用的、功能极强的关系数据库语言。
- **要点**：其功能不仅仅是查询，而是包括数据库模式创建、数据库数据的插入与修改、数据库安全性完整性定义与控制等一系列功能。

#### 2. SQL 的基本概念 ⭐ (练习多选1)

- **要点**：支持 SQL 的关系数据库管理系统同样支持关系数据库三级模式结构。
  - **外模式**：包括若干视图和部分基本表
  - **数据库模式**：包括若干基本表
  - **内模式**：包括若干存储文件
- 用户可以用 SQL 对基本表和视图进行查询或其他操作，基本表和视图一样，都是关系。
- **基本表**：本身独立存在的表，在关系数据库管理系统中一个关系就对应一个基本表。一个或多个基本表对应一个存储文件，一个表可以带若干索引，索引也存放在存储文件中。
- **存储文件**：存储文件的逻辑结构组成了关系数据库的内模式。存储文件的物理结构对最终用户是隐蔽的。
- **视图**：从一个或几个基本表导出的表。本身不独立存储在数据库中，即数据库中只存放视图的定义而不存放视图对应的数据。这些数据仍存放在导出视图的基本表中，因此视图是一个虚表。视图在概念上与基本表等同，用户可以在视图上再定义视图。

#### 3. SQL 语言的特点 ⭐ (练习单选1,2；判断1,2)

- **要点**：
  1. **综合统一**：集数据定义语言、数据操纵语言、数据控制语言的功能于一体，语言风格统一，可以独立完成数据库生命周期中的全部活动。
  2. **高度非过程化**：进行数据操作时，只要提出"做什么"，而无须指明"怎么做"，无须了解存取路径。
  3. **面向集合的操作方式**：不仅操作对象、查找结果可以是元组的集合，而且一次插入、删除、更新操作的对象也可以是元组的集合。
  4. **一种语法，多种方式**：SQL 既是独立的语言，又是嵌入式语言。作为独立的语言，能够独立地用于联机交互的使用方式；作为嵌入式语言，SQL 语句能够嵌入到高级语言程序中。
  5. **语言简洁**：完成核心功能只用了 9 个动词。SQL 接近英语口语，因此易于学习和使用。

- **SQL 的动词表**：

| SQL 功能 | 动词 |
|----------|------|
| 数据查询 | SELECT |
| 数据定义 | CREATE，DROP，ALTER |
| 数据操纵 | INSERT，UPDATE，DELETE |
| 数据控制 | GRANT，REVOKE |

#### 4. 数据定义 ⭐ (练习单选28；多选1；判断2)

- **要点**：包括模式定义、表定义、视图和索引的定义。
- 一个关系数据库管理系统的实例中可以建立多个数据库，一个数据库中可以建立多个模式，一个模式下通常包括多个表、视图和索引等数据库对象。

- **SQL 的数据定义语句表**：

| 操作对象 | 创建 | 删除 | 修改 |
|----------|------|------|------|
| 模式 | CREATE SCHEMA | DROP SCHEMA | — |
| 表 | CREATE TABLE | DROP TABLE | ALTER TABLE |
| 视图 | CREATE VIEW | DROP VIEW | — |
| 索引 | CREATE INDEX | DROP INDEX | ALTER INDEX |

#### 5. 模式的定义与删除

- **定义模式**：
  ```sql
  CREATE SCHEMA <模式名> AUTHORIZATION <用户名>；
  ```
- **删除模式**：
  ```sql
  DROP SCHEMA <模式名> <CASCADE | RESTRICT>；
  ```

### 二、基本表与索引

#### 6. 基本表的相关操作 ⭐ (练习单选3,4,5,6；多选2)

- **（1）定义基本表**：
  ```sql
  CREATE TABLE <表名>
  （<列名><数据类型>[<列级完整性约束条件>]
  [，<列名><数据类型>[<列级完整性约束条件>]]
  …
  [，<表级完整性约束条件>]）；
  ```

- **（2）修改基本表**：
  ```sql
  ALTER TABLE <表名>
  [ ADD[COLUMN] <新列名><数据类型>[完整性约束] ]
  [ ADD <表级完整性约束>]
  [ DROP [ COLUMN ] <列名> [CASCADE| RESTRICT] ]
  [ DROP CONSTRAINT<完整性约束名>[ RESTRICT | CASCADE ] ]
  [ ALTER COLUMN <列名><数据类型> ]；
  ```

- **（3）删除基本表**：
  ```sql
  DROP TABLE <表名> [RESTRICT|CASCADE] ；
  ```

#### 7. 索引 ⭐⚠ (练习判断3)

- **要点**：
  - 常见索引有顺序文件上的索引、B+树索引、散列（hash）索引、位图索引等。
  - 索引虽然能够加速数据库查询，但需要占用一定的存储空间，当基本表更新时，索引要进行相应的维护，这些都会增加数据库的负担，因此要根据实际应用的需要有选择地创建索引。
  - 关系数据库管理系统在执行查询时会自动选择合适的索引作为存取路径，用户不必也不能显式地选择索引。
  - 索引是关系数据库管理系统的内部实现技术，属于内模式的范畴。

#### 8. 索引的建立与删除

- **（1）建立索引**：
  ```sql
  CREATE [UNIQUE] [CLUSTER] INDEX <索引名>
  ON <表名>（<列名>[<次序>][, <列名>[<次序>]…]）；
  ```

- **（2）修改索引**：
  ```sql
  ALTER INDEX <旧索引名> RENAME TO <新索引名>
  ```

- **（3）删除索引**：
  ```sql
  DROP INDEX <索引名>；
  ```

#### 9. 数据字典

- **定义**：数据字典是关系数据库管理系统内部的一组系统表，它记录了数据库中所有的定义信息，包括关系模式定义、视图定义、索引定义、完整性约束定义、各类用户对数据库的操作权限、统计信息等。
- **要点**：关系数据库管理系统在执行 SQL 的数据定义语句时，实际上就是在更新数据字典表中的相应信息。在进行查询优化和查询处理时，数据字典中的信息是其重要依据。

### 三、数据查询

#### 10. 数据查询 ⭐ (练习单选7)

- **定义**：数据查询是数据库的核心操作。SQL 提供了 SELECT 语句进行数据查询。
- **一般格式**：
  ```sql
  SELECT [ALL|DISTINCT] <目标列表表达式>[, <目标列表表达式>] …
  FROM <表名或视图名>[, <表名或视图名>…]| (SELECT 语句) [AS] <别名>
  [WHERE <条件表达式>]
  [GROUP BY <列名1> [HAVING <条件表达式>]]
  [ORDER BY <列名2> [ASC|DESC]];
  ```

#### 11. 单表查询 ⭐⚠ (练习单选7–18；多选3–6；判断4)

- **定义**：单表查询是指仅涉及一个表的查询。

- **（1）选择表中的若干列**：①查询指定列 ②查询全部列

- **（2）选择表中的若干个元组**：①消除取值重复的行 ②查询满足条件的元组
  - 查询满足指定条件的元组可以通过 WHERE 子句实现。

- **常用的查询条件**：

| 查询条件 | 谓词 |
|----------|------|
| 比较 | `=`, `>`, `<`, `>=`, `<=`, `!=`, `<>`, `\|>`, `\|<`; NOT+上述比较运算符 |
| 确定范围 | BETWEEN AND, NOT BETWEEN AND |
| 确定集合 | IN, NOT IN |
| 字符匹配 | LIKE, NOT LIKE |
| 空值 | IS NULL, IS NOT NULL |
| 多重条件（逻辑运算） | AND, OR, NOT |

- **字符匹配**：谓词 LIKE 可以用来进行字符串的匹配。
  - 通配符 `%`（百分号）代表任意长度的字符串。
  - 通配符 `_`（下横线）代表任意单个字符。

- **（3）ORDER BY 子句**：用户可以用 ORDER BY 子句对查询结果按照一个或多个属性列的升序（ASC）或降序（DESC）排列，默认值为升序。

- **（4）聚集函数**：当聚集函数遇到空值时，除 `COUNT(*)` 外，都跳过空值而只处理非空值。`COUNT(*)` 是对元组进行计数，某个元组的一个或部分列取空值不影响 COUNT 的统计结果。**注意，WHERE 子句中是不能用聚集函数作为条件表达式的。聚集函数只能用于 SELECT 子句和 GROUP BY 中的 HAVING 子句。**

- **SQL 主要的聚集函数**：

| 函数 | 功能 |
|------|------|
| `COUNT(*)` | 统计元组个数 |
| `COUNT([DISTINCT\|ALL] <列名>)` | 统计一列中值的个数 |
| `SUM([DISTINCT\|ALL] <列名>)` | 计算一列值的总和（此列必须是数值型） |
| `AVG([DISTINCT\|ALL] <列名>)` | 计算一列值的平均值（此列必须是数值型） |
| `MAX([DISTINCT\|ALL] <列名>)` | 求一列值中的最大值 |
| `MIN([DISTINCT\|ALL] <列名>)` | 求一列值中的最小值 |

- **（5）GROUP BY 子句**：将查询结果按某一列或多列的值分组，值相等的为一组。对查询结果分组的目的是为了细化聚集函数的作用对象。如果分组后还要求按一定的条件对这些组进行筛选，最终只输出满足指定条件的组，则可以使用 HAVING 短语指定筛选条件。

#### 12. 连接查询 ⭐ (练习单选19,20；多选7)

- **定义**：若一个查询同时涉及两个以上的表，则称之为连接查询。
- **要点**：连接查询是关系数据库中最主要的查询，包括等值连接查询、自然连接查询、非等值连接查询、自身连接查询、外连接查询和复合条件连接查询、多连接等。
- 连接查询的 WHERE 子句中用来连接两个表的条件称为**连接条件**或**连接谓词**。连接条件中的列名称称为**连接字段**。
- 一般格式为：`[<表名1>]<列名1> <比较运算符> [<表名2>]<列名2>`
- 还可以使用：`[<表名1>]<列名1> BETWEEN [<表名2>]<列名2> AND [<表名2>]<列名3>`
- 当连接运算符为 `=` 时，称为**等值连接**。使用其他运算符称为**非等值连接**。

#### 13. 自身连接 (练习判断5)

- **定义**：一个表与其自己进行连接。
- **要点**：需要给表起别名以示区别。因为所有属性名都是同名属性，必须使用"别名"。

#### 14. 外连接 ⭐⚠ (练习单选21；多选7)

- **要点**：外连接 OUTER JOIN，左外连接 LEFT OUTER JOIN，右外连接 RIGHT OUTER JOIN。

#### 15. 多表连接 (练习单选20)

- **定义**：两个以上的表进行连接。

#### 16. 嵌套查询 ⭐ (练习单选22)

- **定义**：在 SQL 语言中，一个 SELECT-FROM-WHERE 语句称为一个查询块。将一个查询块嵌套在另一个查询块的 WHERE 子句或 HAVING 短语的条件中的查询称为嵌套查询。
- **要点**：嵌套查询使用户可以用多个简单查询构成复杂的查询，从而增强 SQL 的查询能力。以层层嵌套的方式来构造程序正是 SQL 中"结构化"的含义所在。

#### 17. 父查询、子查询 ⭐⚠ (练习单选22；判断6)

- **定义**：
  - 嵌套查询上层的查询块称为**外层查询**或**父查询**。
  - 下层查询块称为**内层查询**或**子查询**。
- **要点**：
  - SQL 语言允许多层嵌套查询，即一个子查询中还可以嵌套其他子查询。
  - 子查询的 SELECT 语句中**不能使用 ORDER BY 子句**，ORDER BY 子句只能对最终查询结果排序。
  - **不相关子查询**：子查询的查询条件不依赖于父查询。
  - **相关子查询**：子查询的查询条件依赖于父查询。

#### 18. 常用的子查询 ⭐ (练习单选23)

- **要点**：带有 IN 谓词的子查询、带有比较运算的子查询、带有 ANY/ALL 谓词的子查询、带有 EXISTS 谓词的子查询。

#### 19. 集合查询 ⭐⚠ (练习多选8；判断7)

- **要点**：SELECT 语句的查询结果是元组的集合，所以多个 SELECT 语句的结果可进行集合操作。
- 集合操作主要包括：并 UNION、交 INTERSECT、差 EXCEPT。

#### 20. 基于派生表的查询

- **要点**：子查询不仅可以出现在 WHERE 子句中，还可以出现在 FROM 子句中，这时子查询生成的临时派生表成为主查询的查询对象。

### 四、数据更新

#### 21. 数据更新 ⭐ (练习单选2,24)

- **要点**：数据更新操作有三种：插入数据 INSERT、修改数据 UPDATE、删除数据 DELETE。

#### 22. 插入数据 ⭐⚠ (练习单选4,25)

- **要点**：插入语句 INSERT 通常有两种形式，一种是插入一个元组，另一种是插入子查询结果。

- **（1）插入元组**：
  ```sql
  INSERT INTO <表名> [ (<属性列1>[, <属性列2>...]) ] VALUES ( <常量1>[, <常量2>]... ) ;
  ```

- **（2）插入子查询结果**：
  ```sql
  INSERT INTO <表名> [ (<属性列1>[, <属性列2>...]) ] 子查询;
  ```

#### 23. 修改数据 ⭐⚠ (练习单选24,27；多选9)

- **定义**：修改操作又称为更新操作。
- **一般格式**：
  ```sql
  UPDATE <表名> SET <列名>=<表达式>[, <列名>=<表达式>]... [WHERE <条件>];
  ```
- **要点**：其功能是修改指定表中满足 WHERE 子句条件的元组。其中 SET 子句给出 `<表达式>` 的值用于取代相应的属性列值。如果省略 WHERE 子句，则表示要修改表中的所有元组。
- 三种修改方式：修改某一个元组的值，修改多个元组的值，带子查询的修改语句。

#### 24. 删除数据 ⭐⚠ (练习单选26,28；判断8)

- **一般格式**：
  ```sql
  DELETE FROM <表名> [WHERE <条件>]；
  ```
- **要点**：DELETE 语句的功能是从指定表中删除满足 WHERE 子句条件的所有元组。如果省略 WHERE 子句则表示删除表中全部元组，但**表的定义仍在字典中**。也就是说 DELETE 语句删除的是表中的数据，而不是关于表的定义，**注意与 DROP 的区别**。
- 三种删除方式：删除某一个元组的值，删除多个元组的值，带子查询的删除语句。

#### 25. 空值的处理 ⭐⚠ (练习单选15)

- **定义**：所谓空值就是"不知道"或"不存在"或"无意义"的值。
- **一般有以下几种情况**：该属性应该有一个值，但目前不知道它的具体值；该属性不应该有值；由于某种原因不便于填写。
- **要点**：空值是很特殊的值，含有不确定性，对关系运算带来特殊的问题，需要做特殊的处理。
- **相关操作**：（1）空值的产生（2）空值的判断（3）空值的约束条件（4）空值的运算。

### 五、视图

#### 26. 视图 ⭐ (练习多选10；判断10)

- **定义**：视图是从一个或几个基本表（或视图）导出的表。它与基本表不同，是一个虚表。
- **要点**：
  - 数据库中只存放视图的定义，而不存放视图对应的数据，这些数据仍存放在原来的基本表中。
  - 基本表中的数据发生变化，从视图中查询出的数据也就随之改变。
  - 视图一经定义，就可以和基本表一样被查询、被删除。
  - 可以在一个视图之上再定义新的视图，但对视图的更新（增、删、改）操作则有一定的限制。

#### 27. 建立视图 ⭐⚠ (练习单选29)

- **一般格式**：
  ```sql
  CREATE VIEW <视图名> [（<列名> [，<列名> ] ）] AS <子查询> [WITH CHECK OPTION]；
  ```
- **WITH CHECK OPTION**：表示对视图进行 UPDATE、INSERT 和 DELETE 操作时要保证更新、插入或删除的行满足视图定义中的谓词条件（即子查询中的条件表达式）。
- **列名规则**：组成视图的属性列名或者全部省略或者全部指定，没有第三种选择。如果省略了视图的各个属性列名，则隐含该视图由子查询中 SELECT 子句目标列中的诸字段组成。
- **必须明确指定组成视图的所有列名的三种情况**：
  1. 某个目标列不是单纯的属性名，而是聚集函数或列表达式
  2. 多表连接时选出了几个同名列作为视图的字段
  3. 需要在视图中为某个列启用新的更合适的名字
- **要点**：关系数据库管理系统执行 CREATE VIEW 语句的结果只是把视图的定义存入数据字典，并不执行其中的 SELECT 语句。只是在对视图查询时，按视图的定义从基本表中将数据查出。
- 视图可以建立在单个基本表上，建立在多个基本表上，建立在一个或多个已定义好的视图上，还可以建立在基本表与视图上。

#### 28. 行列子集视图

- **定义**：若一个视图是从单个基本表导出的，并且只是去掉了基本表的某些行和某些列，但保留了主码，我们称这类视图为行列子集视图。

#### 29. 删除视图

- **一般格式**：
  ```sql
  DROP VIEW <视图名>[CASCADE]；
  ```
- **要点**：视图删除后视图的定义将从数据字典中删除。如果该视图上还导出了其他视图，则使用 CASCADE 级联删除语句把该视图和由它导出的所有视图一起删除。
- 基本表删除后，由该基本表导出的所有视图均无法使用了，但是视图的定义没有从字典中清除。删除这些视图定义需要显式地使用 DROP VIEW 语句。

#### 30. 查询视图

- **要点**：用户就可以像对基本表一样对视图进行查询。

#### 31. 视图消解 (练习判断9)

- **定义**：关系数据库管理系统执行对视图的查询时，首先进行有效性检查，检查查询中涉及的表、视图等是否存在。如果存在，则从数据字典中取出视图的定义，把定义中的子查询和用户的查询结合起来，转换成等价的对基本表的查询，然后再执行修正了的查询。这一转换过程称为**视图消解**。

#### 32. 更新视图 ⭐ (练习单选29)

- **定义**：更新视图是指通过视图来插入（INSERT）、删除（DELETE）和修改（UPDATE）数据。
- **要点**：由于视图是不实际存储数据的虚表，因此对视图的更新最终要转换为对基本表的更新。对视图的更新操作也是通过视图消解，转换为对基本表的更新操作。
- 为防止用户通过视图对数据进行增加、删除、修改时，有意无意地对不属于视图范围内的基本表数据进行操作，可在定义视图时加上 WITH CHECK OPTION 子句。关系数据库管理系统会检查视图定义中的条件，若不满足条件则拒绝执行该操作。

#### 33. 更新视图的注意事项 ⭐⚠

- **要点**（以下情况视图不允许更新）：
  1. 若视图是由两个以上基本表导出的，则此视图不允许更新。
  2. 若视图的字段来自字段表达式或常数，则不允许对此视图执行 INSERT 和 UPDATE 操作，但允许执行 DELETE 操作。
  3. 若视图的字段来自聚集函数，则此视图不允许更新。
  4. 若视图定义中含有 GROUP BY 子句，则此视图不允许更新。
  5. 若视图定义中含有 DISTINCT 短语，则此视图不允许更新。
  6. 若视图定义中有嵌套查询，并且内层查询的 FROM 子句中涉及的表也是导出该视图的基本表，则此视图不允许更新。

#### 34. 视图的作用 ⭐ (练习单选30；多选10)

- **要点**：
  1. 视图能够简化用户的操作
  2. 视图使用户能以多种角度看待同一数据
  3. 视图对重构数据库提供了一定程度的逻辑独立性
  4. 视图能够对机密数据提供安全保护
  5. 适当利用视图可以更清晰地表达查询

## 五、高频考点与易错提醒

- ⚠ **BETWEEN AND 包含边界值**（练习单选9）——"BETWEEN 20 AND 25"包括 20 岁和 25 岁。
- ⚠ **空值判断必须用 IS NULL，不能用 `=`**（练习单选15）——`出生日期 IS NULL` 正确，`出生日期=NULL` 错误。
- ⚠ **WHERE 子句中不能用聚集函数作为条件表达式**（练习单选17,18）——聚集函数只能用于 SELECT 子句和 GROUP BY 中的 HAVING 子句；分组后筛选用 HAVING 而非 WHERE。
- ⚠ **`_` 代表单个字符，`%` 代表任意长度**（练习单选14）——`DB_` 表示长度为 3 且以 DB 开头的字符串。
- ⚠ **左外连接保留左表全部记录**（练习单选21，多选7）——LEFT JOIN 结果集中左表字段不会出现空值，右表字段可能出现空值。
- ⚠ **DELETE 删除数据不删定义，DROP 删除定义**（练习单选26,28，判断8）——DELETE 删表中元组但表定义仍在；DROP 删除表/视图/数据库本身。
- ⚠ **视图不提高查询效率**（练习多选10）——视图是逻辑结构的虚表，不独立存储，不提升查询性能。
- ⚠ **索引不是越多越好**（练习判断3）——索引占用存储空间且基本表更新时需维护，增加数据库负担。
- ⚠ **选择、投影、连接不是集合操作**（练习判断7）——SQL 集合操作仅指并 UNION、交 INTERSECT、差 EXCEPT。
- ⚠ **WITH CHECK OPTION 保证视图更新满足条件**（练习单选29）——对视图进行 UPDATE/INSERT/DELETE 时检查是否满足视图定义中的谓词条件。
- ⚠ **数据类型决定存储空间**（练习单选5）——不同的数据类型占用不同的存储空间。
- ⚠ **学号等数字标识宜用 CHAR 而非 INT**（练习单选6）——学号本质上是字符串，用字符类型更合适。
- ⚠ **相关子查询的查询条件依赖于父查询**（练习判断6）——注意与不相关子查询区分。
- ⚠ **UNION 要求列数相同、对应列数据类型相同**（练习多选8）——参加集合操作的各查询结果列数必须相同，对应项数据类型也必须相同。
- ⚠ **SQL 可嵌入高级语言**（练习判断1）——SQL 既是独立的语言，又是嵌入式语言。
- ⚠ **数据定义 ≠ 数据操纵**（练习判断2）——数据定义包括定义表/视图/索引；增删改查属于数据操纵功能。
- ⚠ **SELECT 结果是表（元组的集合）**（练习单选7）——查询语句针对表操作，查询结果为表。
- ⚠ **DISTINCT 用于去重**（练习单选10,16）——SELECT 中使用 DISTINCT 可消除重复元组。
- ⚠ **ORDER BY 默认升序 ASC**（练习单选12）——DESC 为降序，ASC 为升序。
- ⚠ **UPDATE 语法：`UPDATE <表名> SET <列名>=<表达式> [WHERE <条件>]`**（练习单选27）——注意 SET 在表名之后。
- ⚠ **INSERT 中数值型不加单引号，字符型需加单引号**（练习单选25）——数值型数据不应带单引号。

## 六、一句话记忆要点

1. **SQL**：综合统一、非过程化、面向集合、一种语法多种方式、语言简洁（9 个动词）。
2. **三级模式**：外模式 = 视图 + 部分基本表；模式 = 基本表；内模式 = 存储文件。
3. **数据定义**：CREATE / DROP / ALTER 操作模式、表、视图、索引（模式与视图无 ALTER）。
4. **基本表操作**：CREATE TABLE 定义、ALTER TABLE 修改（ADD/DROP/ALTER COLUMN）、DROP TABLE 删除。
5. **索引**：加速查询但占空间需维护，RDBMS 自动选择，属内模式范畴。
6. **SELECT**：`SELECT … FROM … [WHERE] [GROUP BY … HAVING] [ORDER BY …]`。
7. **单表查询**：列选择、WHERE 条件（比较/BETWEEN/IN/LIKE/IS NULL/AND/OR）、ORDER BY 排序、聚集函数、GROUP BY + HAVING 分组筛选。
8. **聚集函数**：COUNT/SUM/AVG/MAX/MIN，遇空值跳过（COUNT(*) 除外），只能用于 SELECT 和 HAVING，不能用于 WHERE。
9. **连接查询**：等值（`=`）/非等值/自身连接（需别名）/外连接（左/右）/多表连接。
10. **嵌套查询**：子查询在 WHERE 或 HAVING 中；不相关子查询不依赖父查询，相关子查询依赖父查询；子查询不能用 ORDER BY。
11. **集合查询**：UNION（并）、INTERSECT（交）、EXCEPT（差）；要求列数相同、类型匹配。
12. **数据更新**：INSERT（元组/子查询）、UPDATE（SET + WHERE）、DELETE（FROM + WHERE）。
13. **DELETE vs DROP**：DELETE 删数据留定义，DROP 删定义本身。
14. **空值**："不知道"/"不存在"/"无意义"；判断用 IS NULL 而非 `=`。
15. **视图**：虚表，只存定义不存数据；可查询、有限更新；WITH CHECK OPTION 约束更新。
16. **视图消解**：查询视图时将视图定义的子查询与用户查询结合，转换为等价的基本表查询。
17. **视图更新限制**：多表导出/表达式/聚集函数/GROUP BY/DISTINCT/特定嵌套查询 → 不可更新。
18. **视图作用**：简化操作、多角度看待数据、逻辑独立性、安全保护、清晰表达查询。
