9. A
解析：缓冲池是系统的共用资源，可供多个进程共享，并且既能用于输入又能用于输出。其一般包含三种类型的缓冲：①空闲缓冲区；②装满输入数据的缓冲区；③装满输出数据的缓冲区。为了管理上的方便，可将相同类型的缓冲区链成一个队列。B、C、D属于专用缓冲。
考点：缓冲技术的概念

10. C
解析：两个通信进程之间若只设置单缓冲区，则同一时刻只能实现单向传输，但可在一段时间内用于发送数据，另一段时间内用于接收数据。若设置双缓冲区，则同一时刻可以实现双向传输。
考点：缓冲技术的概念

11. A
解析：缓冲技术的缓冲池在内存中。
考点：数据缓冲技术

12. C
解析：SPOOLing 技术实现将一台物理 I/O 设备虚拟为多台逻辑 I/O 设备。
考点：SPOOLing 技术

13. C
解析：共享设备是指在一段时间间隔内可被多个进程同时访问的设备，只有磁盘满足。打印机在一段时间间隔内被多个进程访问时打印出来的文档就会错乱；磁带机旋转到所需的读写位置需要较长的时间，若一段时间间隔内被多个进程访问，磁带机就只能一直旋转，没时间读写。
考点：共享设备的概念

14. A
解析：使用磁盘高速缓冲区的目的是匹配磁盘和内存速度的差异，减少磁盘 I/O 次数。当有一进程请求访问某个盘块中的数据时，先去查看磁盘高速缓冲器，若有，直接从中提取数据，若无，再访问磁盘并且把访问内容写入高速缓冲，之后再访问该内容时就可以直接访问高速缓冲区。
考点：磁盘高速缓冲

15. D
解析：缓冲技术可缓和 CPU 与 I/O 设备间速度不匹配的矛盾。
考点：缓冲的概念

16. B
解析：通过 SPOOLing 技术可将一台物理 I/O 设备虚拟为多台逻辑 I/O 设备，同样允许多个用户共享一台物理 I/O 设备，所以 SPOOLing 技术并不是将物理设备真的分配给用户进程。
考点：SPOOLing 技术的实现原理

17. D
解析：电梯调度、FCFS 算法、CLOCK 算法都可以是磁盘调度算法，而银行家算法是死锁的避免算法。
考点：磁盘调度算法的种类
