> 来源：2-3-计算机组成计算机类讲义.pdf 物理页 [80, 103]
> 说明：⭐=高频考点，⚠=易错点；> Caveat: 为陷阱/技巧；末尾 Worked Examples 为代表性题目（无书内答案的标 [PROPOSED — verify]）。

## 一、总线与互连结构
**互连结构**：连接计算机各个模块（CPU、存储器、I/O）的通路的集合。目前最普遍的是总线和各种多总线结构（包括层次式总线）。*(讲义 p80)*

**总线**：是连接两个或多个设备的公共通信通路，构成计算机系统的互联机构。其关键特征是**共享传输媒质**。*(讲义 p80)*

**总线的四大特性***(讲义 p80)*
- **机械特性**：规定总线在机械连接方式上的性能，保证机械上的可靠连接。
- **电气特性**：规定总线的每一根传输线上信号的传递方向和有效的电平范围。
- **功能特性**：规定总线中每根传输线的功能。
- **时间特性**：规定总线中的任一条线在什么时间内有效。

**总线性能指标***(讲义 p80)*
- **总线宽度**：指数据总线的根数。
- **标准传输率**：总线上每秒能传输的最大字节数。
- **时钟同步/异步**：分为同步总线（与时钟同步工作）和异步总线（与时钟不同步工作）。
- **总线复用**：地址总线和数据总线共用一组物理线路。
- **信号线数**：地址总线、数据总线和控制总线三种总线数的总和。

**按层次分类的总线结构***(讲义 p80)*
| 分类 | 定义 / 适用范围 | 特点与对应关系 |
|---|---|---|
| **内部总线** | CPU 内部连接各寄存器及运算部件之间的总线 | CPU 芯片向外引出的信号*(题目单选1)* |
| **系统总线** | CPU 同计算机系统的其他具有高速传输功能的部件 | ISA、EISA、PCI 等标准；PCI 属于此列*(题目单选9)* |
| **I/O总线** | 中、低速 I/O 设备之间互相连接的总线 | [原文未述] |
| **通信总线** | 多处理器系统中的连接总线 | [原文未述] |

**系统总线按功能分类**：由数据总线（提供数据传送路径）、地址总线（指定数据来源和去向）、控制总线（控制对数据/地址线的访问和使用）组成，此外还有电源线。*(讲义 p81)*

**总线连接方式对比***(讲义 p82)*
| 结构类型 | 核心组件与布局 | 特点与优劣 |
|---|---|---|
| **单总线结构** | CPU、主存、所有 I/O 设备均通过**一组总线**连接 | 结构简单，易于扩展；只能支持信息传送量较小的系统。外设可以和主存单元统一编址。*(题目单选4)* |
| **双总线结构** | 带**存储总线**（CPU与主存间局部总线）；或带 **IOP**（I/O处理机连接外设） | [原文未述具体优劣] |
| **中间层结构** | 处理器总线、系统总线、高速总线之间通过**总线桥接器（桥）**连接 | 高速总线独立于处理器；桥接器内含总线控制器、数据缓冲区及加速器，起仲裁作用。 |

## 二、总线设计与传输控制 ⭐
**总线仲裁**：分为**集中式**（有中央控制器/仲裁器）和**分布式**（无中央控制器，各模块共同作用共享总线）。*(讲义 p83)*

**集中式仲裁三种方式对比***(讲义 p83)*
| 仲裁方式 | 工作原理 | 优缺点 |
|---|---|---|
| **链式查询方式** | 离中央仲裁器最近的设备优先权最高 | 优点：连线少，易扩充设备；缺点：对电路故障敏感，优先级固定。 |
| **计数器定时查询方式** | 通过计数器轮询寻找有传输需求的设备 | 优点：可方便地改变优先级。*(题目单选5)* |
| **独立请求方式** | 每个设备有独立的请求线和授权线 | 优点：**响应时间最快**，优先次序控制极为灵活。*(题目判断4)* |

**总线定时协议**：分同步时序与异步时序，用于同步主方与从方的操作。*(讲义 p84)*
| 定时方式 | 工作原理 | 优缺点与适用场景 |
|---|---|---|
| **同步定时** | 事件出现的时刻由总线**时钟信号**确定 | 传输频率较高；适用于各功能模块速度相差不多的情况。 |
| **异步定时** | 应答方式或**互锁机制**（握手） | 总线周期长度可变；适用于快速、慢速设备连接到同一总线。 |

**异步通信的互锁机制对比***(题目单选6)*
| 类型 | 机制 | 速度与可靠性 |
|---|---|---|
| **不互锁方式** | 发送方不等待接收方确认即发送下一数据 | 传输速度**最快**，但可靠性较低。 |
| **半互锁方式** | 只需单方确认 | 速度与可靠性介于两者之间。 |
| **全互锁方式** | 通信双方互相确认 | 可靠性高，但因每次传输都需双方确认，速度最慢。 |

**总线数据传输模式***(讲义 p84)*
- **读、写**：所有总线系统都必须支持的最基本形式。
- **读—修改—写**：用于共享存储器的保护（如 PV 原语、管程）。
- **写后读**：用于数据校验。
- **成块传输**：现代总线特征，实现与高速设备的信息传输。

**专用与复用总线**：专用总线冲突少吞吐量高，但规模大成本高；复用总线节省空间和成本，但控制复杂，有性能降低危险。*(讲义 p84)*
> *Caveat* (判断题2): 复用传送是指分时使用总线（如 PCI 总线中地址和数据传送可以复用 32 根总线）。*(题目判断2)*

## 三、I/O接口结构与寻址 ⭐
**I/O接口的必要性**：外设操作方法不同、速度慢于处理器、数据格式和字长不同，无法直接连系统总线。*(讲义 p84)*

**接口的功能与传递信息***(讲义 p85)*
- 接口需实现：控制与定时、处理器通信、设备通信、数据缓冲、检错。
- 主机与外设间传送的信息分为：**数据信息、控制信息、状态信息**。*(题目单选19)*
- 外部设备的状态信息被视为**输入数据**，控制信息被视为**输出数据**，分时使用数据总线传送到各自的寄存器。

**接口与端口的区别**：端口是指接口电路中可以被 CPU 直接访问的**寄存器**；若干个端口加上相应的控制逻辑才组成接口。*(讲义 p85)*

**端口类型与寻址方式***(讲义 p85)*
| 编址方式 | 特点与对应关系 |
|---|---|
| **统一编址**（存储器映射） | 端口地址与存储器统一编址；优点：指令格式统一（访存指令即可访问外设）；缺点：占用主存地址空间。 |
| **独立编址**（I/O映射） | 两者独立编址；优点：主存和外设有各自独立空间；缺点：访问外设和主存需使用不同指令。 |

## 四、程序查询与程序中断方式 ⭐
**程序直接控制方式**：由 CPU 执行 I/O 程序实现数据传送。分为无条件传送（适用简单设备）和程序查询方式（适用低速外设）。*(讲义 p86)*

**程序查询方式的缺点**：CPU 长期踏步等待导致效率低；一段时间内只能与一台外设交换信息；不能发现和处理无法估计的错误/异常。*(讲义 p86)*

**程序中断的基本思想**：外设主动向 CPU 报告自己的状态。在一定程度上实现了 CPU 和外设的并行工作。*(讲义 p87)*

**中断分类与内/外中断对比***(讲义 p87)*
| 分类 | 信号来源 | 举例与特点 |
|---|---|---|
| **内中断**（异常） | CPU 内部 | 指令执行时发生（如除零、页面失效、溢出）。包含陷阱、故障、终止。*(题目单选10)* |
| **外中断**（中断） | CPU 外部 | I/O 中断请求、人工干预。分为可屏蔽中断（INTR）和非屏蔽中断（NMI）。 |

**内中断（异常）的三种类型***(讲义 p87)*
| 类型 | 原因与处理 | 典型例子 |
|---|---|---|
| **陷阱** | 有意的异常，处理完返回**下一条指令** | 系统调用*(题目多选9)* |
| **故障** | 错误条件引起，可能被修复，返回**原指令**重新执行 | 缺页中断 |
| **终止** | 不可恢复的致命错误，不返回 | 硬件故障*(题目多选9)* |

**中断隐指令（硬件完成）***(讲义 p88)*
- **关中断**：不再允许被打断。
- **保存断点**：将程序计数器（PC）的内容压入堆栈保存。
- **引出中断服务程序**：将中断服务程序入口地址送入 PC。

> *Caveat* (单选14): **中断向量地址**是**中断服务程序入口地址的指示器**（即存放入口地址的内存单元地址），并不是入口地址本身。*(题目单选14)*

**多重中断与屏蔽***(讲义 p88)*
- 现场保护最有效的方法是采用**堆栈**。*(题目单选11)*
- 多重中断系统在**保护现场时关中断**，在**执行中断处理程序时开中断**，在**恢复现场前关中断**。*(题目多选6)*
- 中断屏蔽：通过中断屏蔽寄存器有选择地封锁部分中断请求。*(讲义 p89)*

## 五、DMA方式与通道控制方式 ⭐
**直接存储器访问 (DMA)**：完全由硬件执行 I/O 交换。在外设和内存之间开辟“直接数据通道”，不需要 CPU 干预也不需要软件介入。*(讲义 p87)*

**DMA与中断方式的区别对比***(讲义 p87)*
| 特性 | 中断方式 | DMA方式 |
|---|---|---|
| **CPU干预与资源占用** | 需要保护和恢复现场，占用CPU资源 | 仅在开始和结尾介入，不占用CPU资源 |
| **响应时间点** | 只能在**每条指令执行完毕**时响应 | 可以在**每个机器周期结束**时响应 |
| **数据传送机制** | 需要 CPU 干预（软件完成） | 不需 CPU 干预，数据传送速率非常高（硬件完成） |
| **优先级** | 较低 | **高于中断请求** |
| **适用场景** | 具有处理异常事件的能力 | 仅限于高速外设成组传送信息块 |

**DMA控制器与CPU共享内存的三种方法***(讲义 p90)*
| 方法 | 工作原理 | 优缺点 |
|---|---|---|
| **停止CPU访存** | DMA 要求 CPU 放弃总线控制权，由 DMA 接管直到传送完毕 | 控制简单；CPU可能处于空闲，效率低。 |
| **周期挪用** | I/O 发出 DMA 请求时挪用一个或多个内存周期，CPU暂停访存 | I/O周期 > 内存存储周期时较好发挥效率。*(题目判断13)* |
| **DMA与CPU交替访存** | 一个 CPU 周期分为两部分，分供 DMA 和 CPU 访存 | 发挥 CPU 最高效率；需复杂硬件逻辑。 |

> *Caveat* (判断13): DMA 传送数据时，CPU 并非完全不能访问主存，常采用**周期挪用（窃取）**使 CPU 与 DMA 交替访存。*(题目判断13)*

**DMA方式的传送过程***(题目单选20)*
1. **预处理**：CPU 初始化 DMA 控制器并启动外设（如磁盘）。
2. **数据传送**：外设与内存间直接进行成块数据传输。
3. **后处理**：DMA 控制器发出中断请求，CPU 执行“DMA结束”中断服务程序。

**通道控制方式**：通过执行通道程序，具有更强的独立处理数据输入输出的功能。用于大、中型计算机系统中连接大量 I/O 设备。*(讲义 p89)*

**通道的分类***(讲义 p92)*
| 通道类型 | 传送基本单位 | 特点与适用设备 |
|---|---|---|
| **字节多路通道** | 字节 | 各子通道分时使用；用于连接**大量低速**设备。 |
| **选择通道** | 数据块（物理上只能有一个工作） | 又称高速通道；用于连接**高速外围**设备。 |
| **数组多路通道** | 数据块 | 多个设备可同时工作（但只能有一个传输，其余控制型）；结合了上面两者的优点。 |

**I/O控制方式CPU介入程度对比***(题目判断10)*
程序方式（最大） > 中断方式 > DMA方式 > 通道方式（最小）。

## Worked Examples

### 例 1：DMA数据读盘传送顺序 *(来自 exam-signals 候选脉络)*
*Tags: DMA方式, 传送过程, 中断*
**问题**：系统将数据从磁盘读到内存的过程包括以下操作：①DMA 控制器发出中断请求；②初始化 DMA 控制器并启动磁盘；③从磁盘传输一块数据到内存缓冲区；④执行“DMA 结束”中断服务程序。正确的执行顺序是什么？*(题目单选20)*
**解答**：
1. 明确 DMA 传送过程的三个基本阶段：**预处理（准备阶段）、数据传送、后处理（结束阶段）**。
2. 对应操作：预处理对应“初始化 DMA 并启动”（操作②）；数据传送对应“传输数据块到内存”（操作③）；后处理由于 CPU 未知何时传送完毕，需要 DMA 控制器先发中断请求（操作①），随后 CPU 响应并执行结束服务程序（操作④）。
3. 排列顺序为：② → ③ → ① → ④。

### 例 2：外中断与内中断的判定 *(来自 exam-signals 候选脉络)*
*Tags: 中断分类, 异常, 内部中断*
**问题**：下列选项中，能引起外部中断（外中断）的事件是哪些？（备选可能包含：键盘输入、除数为 0、浮点运算下溢、访存缺页）。*(题目多选4)*
**解答**：
1. **外中断（中断）**的信号来源是 CPU 外部的事件（外部硬件信号）。如键盘输入属于外部设备发出的中断请求，属于外中断。
2. **内中断（异常）**的信号来源是 CPU 内部，与当前执行的指令直接相关。如“除数为0”、“访存缺页”等是 CPU 在执行指令时检测到的异常，属于内中断。
3. “浮点运算下溢”在计算机中通常按机器零处理，不会产生中断请求。
4. 结论：只有“键盘输入”能引起外部中断。
