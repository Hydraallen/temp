## 一、本章概览

本章是数据结构课程的总纲，解决三个核心问题：**数据结构是什么**（数据结构、逻辑结构、存储结构、运算等基本概念）、**如何描述数据**（数据类型与抽象数据类型）、**如何评价算法**（算法的特性、设计要求、时间复杂度与空间复杂度、递归）。全章共 13 个知识点，概念密集、定义多，是后续各章（线性表、栈、队列、树、图等）的概念基石。与后续章节的关系：逻辑结构的四分类（集合/线性/树/图形）直接对应后续各章的具体结构；存储结构的四方法（顺序/链式/索引/散列）贯穿全课程；时间复杂度分析是所有算法评价的工具。

## 二、知识点详解

### 1. 数据结构的定义和相关概念 ⭐（练习1、2、3、4、5、7，多选1、2、7）

- **定义**：数据结构（Data Structure）是指互相之间存在着一种或多种关系的数据元素的集合。数据结构涉及数据元素之间的逻辑关系，数据在计算机中的存储方式和这些数据上定义的一组运算，一般称这三个方面为：**数据的逻辑结构、数据的存储结构和数据的运算**。
- **数据**：信息的载体，是所有能够被计算机识别、存储和加工处理的符号的总称。它是计算机程序加工的原料。可以是数值数据（整数、实数、复数，用于工程计算、科学计算和商务处理等），也可以是非数值数据（字符、文字、图形、图像、语音等）。
- **数据项（Data Item）**：具有独立含义的标识单位，是数据**不可分割的最小单位**。
- **数据元素（Data Element）**：数据的**基本单位**。在不同的条件下，又可称为元素、结点、顶点、记录等。例如：学生成绩表中的一个记录、棋盘布局问题中状态树的一个状态、教学计划编排问题中的一个顶点等。
- **数据对象（Data Object）/ 数据元素类（Data Element Class）**：具有相同性质的数据元素的集合。在某个具体问题中，数据元素都具有相同的性质（元素值不一定相等），属于同一数据对象。数据元素是数据元素类的一个实例。例如：交通网中所有顶点是一个数据元素类，顶点A和顶点B各自代表一个城市，是该类中的两个实例。

### 2. 数据的逻辑结构 ⭐（练习4、5、6、7、8，判断1、3）

- **定义**：在任何问题中，数据元素之间都不会是孤立的，在它们之间都存在着这样或那样的逻辑关系，这种数据元素之间的关系称为**逻辑结构**。数据的逻辑结构包含两个要素：数据元素的集合、关系的集合。
- **形式表示**：数据的逻辑结构通常采用一个二元组来表示：

  $$Data\_Structure = (D, R)$$

  其中，$D$ 是数据元素的有限集，$R$ 是 $D$ 上关系的有限集。
- **四类逻辑结构**：
  1. **集合**：数据元素间的关系是"属于同一个集合"。元素关系极为松散的一种结构。
  2. **线性结构**：数据元素之间存在**一对一**的关系。
  3. **树形结构**：数据元素之间存在**一对多**的关系。
  4. **图形结构**：数据元素之间存在**多对多**的关系，又称**网状结构**。
- **要点**：
  - 集合是元素关系极为松散的一种结构，也可用其它结构来表示。
  - 数据的逻辑结构可以看作是从具体问题抽象出来的数学模型，**它与数据的存储无关**。
  - 从逻辑上可以把数据结构分为**线性结构**和**非线性结构**两大类。

### 3. 数据的存储结构 ⭐⚠（练习8、18，判断2、3、4）

- **定义**：数据结构在计算机中的标识（又称映像）称为数据的**物理结构**，又称**存储结构**。它研究的是数据的逻辑结构在计算机中的实现方法，包括数据结构中**元素的表示**及**元素间关系的表示**。
- **四种存储方法**：

  | 方法 | 说明 | 实现方式 |
  |------|------|----------|
  | **顺序存储** | 把逻辑上相邻的元素存储在物理位置相邻的存储单元中，结点间的逻辑关系由存储单元的邻接关系来体现 | 通常借助于程序设计语言中的**数组**来实现 |
  | **链式存储** | 对逻辑上相邻的元素不要求其物理位置相邻，元素间的逻辑关系通过附设的**指针字段**来表示 | 通常借助于程序设计语言中的**指针**来实现 |
  | **索引存储** | 在存储结点信息的同时，还建立附加的**索引表**。索引表中的每一项包含**关键字**和**地址**，关键字唯一标识一个数据元素，地址指示出数据元素所在的存储位置 | 主要面向**查找**操作 |
  | **散列存储** | 以数据元素的关键字的值为自变量，通过某个函数（**散列函数**）计算出该元素的存储位置 | 针对数据内容的存储方式 |

- **要点**：
  - 顺序存储和链式存储是最基本最常用的；索引存储和散列存储在具体实现时需要用到前两种方法。
  - 一种逻辑结构可以有不同的存储方法，选用何种存储结构要视具体情况而定，主要考虑运算的实现及算法的时空要求。
  - 顺序存储方式的优点是存储密度大，但插入、删除运算都需要移动大量元素，效率比较低。
  - 使用顺序存储结构无需为存储关系而花费额外的开销，所以数据存储密度比链式存储结构要高。

### 4. 数据的运算

- **定义**：运算是对数据的处理。运算与逻辑结构紧密相连，每种逻辑结构都有一个运算的集合。
- **分类**（根据操作结果）：
  1. **引用型运算**：不改变数据结构中原有的数据元素的状态，只根据需要读取某些信息。
  2. **加工型运算**：结果会改变数据结构中原有数据的状态，如数据元素的内容、个数等。
- **要点**：
  - 数据的运算是定义在数据的**逻辑结构**上的，但运算的具体实现是在数据的**存储结构**上进行的。
  - 在数据的逻辑结构和存储结构给定之后，如果定义的运算集合及运算的性质不同，也会导致完全不同的数据结构（例如线性表、栈和队列等）。

### 5. 数据类型

- **定义**：数据类型（Data Type）是和数据结构密切相关的一个概念。它最早出现在高级程序设计语言中，用以刻划程序中操作对象的特性。数据类型是一个**值的集合**和定义在这个值集上的**一组操作**的总称。
- **分类**：
  1. **原子类型**：值不可分解。如 C 语言中整型（`int`）、字符型（`char`）、浮点型（`float`）、双精度型（`double`）等基本类型。
  2. **结构类型**：值由若干成分按某种结构组成，是可分解的。成分可以是非结构的，也可以是结构的。例如数组的值由若干分量组成，每个分量可以是整数，也可以是数组等。
- **要点**：在某种意义上，数据结构可以看成是"一组具有相同结构的值"，而数据类型则可被看成是由一种数据结构和定义在其上的一组操作所组成的。

### 6. 抽象数据类型 ⭐（多选3，判断5）

- **定义**：抽象数据类型（Abstract Data Type，简称 ADT）是指一个数学模型以及定义在该模型上的一组操作。抽象数据类型的定义取决于它的一组逻辑特性，而**与其在计算机内部如何表示和实现无关**。即不论其内部结构如何变化，只要它的数学特性不变，都不影响其外部的使用。
- **三要素**：抽象数据类型一般可以由**元素、关系及操作**三种要素来定义。
- **特征**：使用与实现相分离，实行**封装**和**信息隐藏**。在抽象数据类型设计时，把类型的定义与其实现分离开来。

### 7. 算法的概念 ⭐（练习9、18，判断6、7）

- **定义**：算法是对特定问题求解步骤的一种描述，是指令的有限序列。
- **特性概述**：算法具有有穷性、确定性、可行性、有输入和输出的特性。
- **要点**（来自练习信号）：
  - 算法可以不依赖具体的物理结构，而程序要依赖具体的物理结构，二者不能通用。（练习判断7）
  - 算法的优劣与描述算法的语言无关，与所用计算机的性能也无关（算法本身的性能只依赖于问题规模）。（练习判断6）

### 8. 算法的特性 ⭐（练习10、14，多选4，判断8）

1. **有穷性**：一个算法必须在有穷步之后结束，即必须在有限时间内完成。
2. **确定性**：算法的每一步必须有确切的定义，无二义性。算法的执行对应着相同的输入仅有唯一的一条路径，即相同的输入必然有相同的输出。
3. **可行性**：算法中的每一步都可以通过已经实现的基本运算的有限次执行得以实现。
4. **输入**：一个算法具有**零个或多个**输入，这些输入取自特定的数据对象集合。
5. **输出**：一个算法具有**一个或多个**输出，这些输出同输入之间存在某种特定的关系。

### 9. 算法的描述 ⭐（练习11，多选5）

- **描述方法**：算法可以使用各种不同的方法来描述，主要有**自然语言、流程图、盒图、伪码语言**。
- **要点**（来自练习信号）：
  - 流程图中：扁圆表示程序的开始与结束，方框表示要执行的处理，平行四边形表示资料输入，**菱形**表示决策或判断，箭头表示程序执行的方向。（练习11）
  - 伪码语言介于高级程序设计语言和自然语言之间，忽略高级语言中一些严格的语法规则与描述细节，比程序设计语言容易描述和被人理解，比自然语言更接近程序设计语言，虽不能直接执行但很容易被转换成高级语言。（多选5解析）

### 10. 算法的设计要求 ⭐（练习12、14）

1. **正确**：算法的执行结果应当满足预先规定的功能和性能要求。
2. **可读**：一个算法应当思路清晰、层次分明、简单明了、易读易懂。算法不仅仅是让机器来执行的，更重要的是便于人的阅读和交流。
3. **健壮**：当输入不合法数据时，算法能够作适当处理，而不会产生莫名其妙的输出或引起其它严重的后果。
4. **高效**：指算法应具有较好的时空性能。
- **要点**：这些指标一般很难做到十全十美，因为它们常常相互矛盾，在实际的算法评价中应根据需要有所侧重。

### 11. 算法的时间复杂度 ⭐⚠（练习13、14、15、16、17、18，多选6）

- **影响因素**：将一个算法转换成程序并在计算机上执行时，其运行所需要的时间取决于下列因素：
  1. 硬件的速度。
  2. 书写程序的语言。
  3. 编译程序所生成目标代码的质量。
  4. 问题的规模。
- **核心思想**：为了突出算法本身的性能，将各种与计算机相关的软、硬件因素都确定下来，这样一个特定算法的运行工作量的大小就只依赖于**问题的规模**（通常用正整数 $n$ 表示），或者说它是问题规模的函数。
- **时间复杂度定义**：一个算法的时间复杂度（Time Complexity）就是指算法的时间耗费，用 $T(n)$ 表示。
- **语句频度**：一个语句重复执行的次数称为语句的频度（Frequency Count）。每条语句的执行时间是该语句执行一次所用时间与该语句重复执行次数的乘积。
- **公式**：

  $$T(n) = \sum_{\text{语句 } i} (t_i \times c_i)$$

  其中 $t_i$ 表示语句 $i$ 执行一次的时间，$c_i$ 表示语句 $i$ 的频度。

  假设每条语句执行一次的时间均为一个单位时间，则：

  $$T(n) = \sum_{\text{语句 } i} c_i$$

- **大O记号（渐进时间复杂度）**：

  **定义**：设 $T(n)$ 是问题规模 $n$ 的函数 $f(n)$，若存在两个正常数 $c$ 和 $n_0$，使得对所有的 $n$，$n \geq n_0$，有：

  $$T(n) \leq c \cdot f(n)$$

  则记为：

  $$T(n) = O(f(n))$$

  例如，一个程序的实际执行时间为 $T(n) = 20n^3 + 25n^2 + 9$，则 $T(n) = O(n^3)$。

- **常见渐进时间复杂度排序**（按数量级递增）：

  $$O(1) < O(\log_2 n) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n)$$

- **要点**：
  - 使用大O记号表示的算法的时间复杂度，称为算法的**渐进时间复杂度**（Asymptotic Complexity），简称时间复杂度。
  - 通常用 $O(1)$ 表示常数计算时间。

### 12. 算法的空间复杂度 ⭐（练习14、15、18）

- **定义**：算法的空间复杂度（Space Complexity）是指算法运行从开始到结束所需的存储量。
- **存储空间组成**：算法运行所需的存储空间包括**固定部分**和**可变部分**。
- **要点**（来自练习信号）：
  - 算法的时间复杂度和空间复杂度没有必然关系。（练习14、18）
  - 随着计算机硬件性能的提高，一般情况下，算法所需要的额外空间已不是算法关注的重点。（练习9解析）

### 13. 递归 ⭐（练习19、20，判断9、10）

- **定义**：递归算法是一种直接或者间接地调用自身算法的过程。在程序设计中，递归算法对解决一大类问题是十分有效的，它往往使算法的描述简洁而且易于理解。
- **递归的特点**：
  - 递归的每次调用在规模上都有所缩小。
  - 相邻两次调用之间有紧密的联系，前一次要为后一次做准备。
  - 在问题的规模极小时，必须用直接给出解答而不再进行递归。
  - 递归调用都是有结束的条件的。
- **优点**：结构清晰，可读性强，而且容易用数学归纳法来证明算法的正确性。
- **缺点**：运行效率较低，无论是耗费的计算时间还是占用的存储空间都比非递归算法要多。
- **要点**（来自练习信号）：
  - 递归是将一个较复杂的（规模较大的）问题转化为一个稍微简单的（规模较小的）与原问题**同类型**的问题来解决。（练习19）
  - 将递归求解过程改变为非递归求解过程的目的是**提高速度**。（练习20）
  - 对于递归定义的数据结构而言，使用递归算法来实现对它的操作是最为方便的。（判断9）
  - 递归调用算法与相同功能的非递归算法相比，主要问题在于重复计算太多，而且调用本身需要分配额外的空间和传递数据和控制，所以时间与空间开销通常都比较大。（判断10）

|------|------|------|
| 原子类型 | 值不可分解 | C 语言 `int`、`char`、`float`、`double` |
| 结构类型 | 值由若干成分按某种结构组成，可分解 | 数组（分量可以是整数，也可以是数组等） |

## 五、高频考点与易错提醒

1. ⚠ **数据结构研究的三个方面**（逻辑结构、存储结构、运算）是高频考点，注意数据结构不仅研究逻辑结构和物理结构，还包括操作算法。（练习1）
2. ⚠ **数据项是数据的最小单位**，数据元素是数据的基本单位，二者不可混淆。（练习2，多选7）
3. ⚠ **集合是关系最简单的数据结构**，四种逻辑结构中关系最松散。（练习3）
4. ⚠ **逻辑结构与数据元素本身的形式、内容、相对位置、个数无关**，逻辑结构不依赖于存储结构。（练习4，判断3）
5. ⚠ **线性表、栈、队列均为线性结构，二叉树为非线性结构**——注意区分线性与非线性。（练习5，多选2）
6. ⚠ **存储数据时不仅要存储各数据元素的值，还要存储数据元素之间的关系**。（练习8）
7. ⚠ **算法的五个特性**（有穷性、确定性、可行性、输入、输出）是高频考点，注意输入可以是零个，输出至少一个。（练习10，判断8）
8. ⚠ **算法的健壮性**指输入不合法数据时能作适当处理，不产生莫名其妙输出——注意与正确性、可读性区分。（练习12）
9. ⚠ **时间复杂度的计算**：嵌套循环取乘积，常数层循环不影响阶数——如外层常数级+内层 $n$ 则为 $O(n)$；双层嵌套分别为 $m$ 和 $n$ 则为 $O(m \times n)$。（练习13、17）
10. ⚠ **时间复杂度与空间复杂度没有必然关系**，逻辑结构与存储结构不一定一一对应。（练习14、18）
11. ⚠ **算法效率与问题的规模和数据的存储都相关**，不仅与问题规模有关。（练习18）
12. ⚠ **递归转化为非递归的目的是提高速度**（提高执行效率），而非改善可读性或健壮性。（练习20）
13. ⚠ **算法和程序不能通用**：程序依赖具体的物理结构，算法可以不依赖。（判断7）
14. ⚠ **抽象数据类型（ADT）包括数据和操作**，其定义与计算机内部如何表示和实现无关。（多选3，判断5）
15. ⚠ **顺序存储优点是存储密度大，但插入/删除效率低**——不要误以为顺序存储插入删除效率高。（判断2）
16. ⚠ **算法描述方法**包括自然语言、流程图、盒图、伪码语言，流程图中菱形表示判断。（练习11，多选5）

## 六、一句话记忆要点

1. **数据结构** = 数据元素 + 关系 → 研究逻辑结构、存储结构、运算三方面。
2. **数据项是最小单位，数据元素是基本单位**，数据对象是同性质元素的集合。
3. **逻辑结构四类**：集合（最松散）、线性（一对一）、树（一对多）、图（多对多）；分两大类即线性与非线性。
4. **存储结构四法**：顺序（数组，密度高但插删慢）、链式（指针）、索引（索引表，面向查找）、散列（散列函数算位置）。
5. **运算分两类**：引用型（不改状态）vs 加工型（改状态）；定义在逻辑结构上，实现在存储结构上。
6. **数据类型** = 值的集合 + 操作；分原子类型（不可分解）和结构类型（可分解）。
7. **ADT** = 数学模型 + 一组操作，与内部实现无关，封装 + 信息隐藏，三要素：元素、关系、操作。
8. **算法** = 求解步骤的指令有限序列，五特性：有穷、确定、可行、输入（≥0）、输出（≥1）。
9. **算法描述**：自然语言、流程图（菱形=判断）、盒图、伪码语言。
10. **算法设计四要求**：正确、可读、健壮、高效（常相互矛盾）。
11. **时间复杂度** $T(n) = O(f(n))$：取最高阶项、忽略常数；排序 $O(1) < O(\log_2 n) < O(n) < O(n\log_2 n) < O(n^2) < O(n^3) < O(2^n)$。
12. **空间复杂度** = 算法运行所需存储量（固定部分 + 可变部分）；与时间复杂度无必然关系。
13. **递归**：调用自身，规模递减，有终止条件；优点清晰易证，缺点时空开销大；转非递归可提高速度。
