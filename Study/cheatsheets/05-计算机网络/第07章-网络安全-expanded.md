## 网络安全概述

**网络安全定义**：计算机网络安全是指利用网络管理控制和技术措施，保证在一个网络环境里，数据的保密性、完整性及可使用性受到保护。*(讲义 p125)*

计算机网络安全包括两个方面：*(讲义 p125)*
- **物理安全**：指系统设备及相关设施受到物理保护，免于破坏、丢失等。
- **逻辑安全**：包括信息的完整性、保密性和可用性。

安全三要素（CIA）：*(讲义 p125)*
- **保密性**（Confidentiality）
- **完整性**（Integrity）
- **可用性**（Availability）

> Caveat (课后习题1): 拒绝服务攻击是对**可用性**的破坏，不是保密性或完整性。*(题目课后习题1)*

**网络系统面临的威胁**（四类）：*(讲义 p125)*
1. 对硬件实体的威胁和攻击
2. 对信息的威胁和攻击
3. 同时攻击软、硬件系统
4. 计算机犯罪

**安全威胁的来源**（三类）：*(讲义 p125)*
- 天灾
- 人祸
- 系统本身的原因

**威胁的具体表现形式**：*(讲义 p125-126)*
伪装、非法连接、非授权访问、拒绝服务、抵赖、信息泄露、业务流分析、改动信息流、篡改或破坏数据、推断或演绎信息、非法篡改程序。

**网络安全的重要性**（要点）：*(讲义 p125)*
- 网络成为敌对势力、不法分子的攻击目标。
- 存取控制、逻辑连接数量不断增加，软件规模空前膨胀，隐含的缺陷、失误能造成巨大损失。
- 计算机系统使用场所转向恶劣环境（工业、野外、天空、海上、宇宙空间、核辐射环境），出错率和故障增多导致可靠性和安全性降低。
- 操作人员、编程人员和系统分析人员的失误或缺乏经验造成安全功能不足。
- 从认识论高度看，人们往往先关注系统需要和功能，然后才被动地从现象注意安全问题。

## 常见攻击手段 ⭐

本章介绍了 **8 种**常见攻击手段：*(讲义 p127-129)*

### 1. 口令入侵

使用合法用户的帐号和口令登录到目的主机，再实施攻击活动。前提是先获得合法用户帐号，再进行口令破译。*(讲义 p127)*

获得普通用户帐号的方法：*(讲义 p127)*
- 利用目标主机的 Finger 功能（查询时主机系统会显示用户资料如用户名、登录时间等）
- 利用目标主机的 x.500 服务（目录查询服务）
- 从电子邮件地址中收集（有些用户邮件地址会透露帐号）
- 查看主机是否有习惯性的帐号（很多系统会使用习惯性帐号，造成泄露）

### 2. 放置特洛伊木马程式

特洛伊木马程式能直接侵入用户的计算机并进行破坏，常被伪装成工具程式或游戏等诱使用户打开邮件附件或从网上下载。一旦执行，会在计算机中隐藏一个能在 Windows 启动时悄悄执行的程式，连接因特网时通知攻击者报告 IP 地址及预先设定的端口。攻击者可任意修改计算机参数设定、复制文件、窥视硬盘内容，达到控制目的。*(讲义 p127)*

### 3. WWW 的欺骗技术 ⭐

黑客将用户要浏览的网页的 URL 改写为指向黑客自己的服务器，用户浏览目标网页时实际向黑客服务器发出请求。*(讲义 p127)*

两种技术手段：*(讲义 p127)*
- **URL 地址重写技术**：攻击者将自己的 Web 地址加在所有 URL 地址前面，使用户进入攻击者的服务器
- **相关信息掩盖技术**：一般用 JavaScript 程式来重写地址栏和状态栏，以达到掩盖欺骗的目的

### 4. 电子邮件攻击

攻击者使用邮件炸弹软件或 CGI 程式向目的邮箱发送大量内容重复、无用的垃圾邮件，使目的邮箱被撑爆而无法使用。当垃圾邮件发送流量特别大时，可能造成邮件系统反应缓慢甚至瘫痪。特点是简单、见效快。*(讲义 p127-128)*

### 5. 网络监听 ⭐⚠

网络监听是主机的一种工作模式，在此模式下主机能接收到本网段在同一条物理通道上传输的所有信息，而不管信息的发送方和接收方是谁。*(讲义 p128)*

- 系统进行密码校验时，用户输入的密码需从用户端传送到服务器端，攻击者可在两端之间进行数据监听。
- 若通信信息没有加密，使用网络监听工具（如 NetXRay for Windows 95/98/NT、Sniffit for Linux/Solaris 等）可截取包括口令和帐号在内的信息资料。
- 监听者往往能获得其所在网段的**所有用户帐号及口令**。

> Caveat (课后习题11): 口令嗅探（网络监听）属于**被动攻击**；重放、拒绝服务、物理破坏属于主动攻击。*(题目课后习题11)*

### 6. 利用黑客软件攻击

利用黑客软件（如 Back Orifice 2000、冰河等著名的特洛伊木马）非法取得用户计算机的终极用户级权利，可进行完全控制（文件操作、桌面抓图、取得密码等）。*(讲义 p128)*

- 黑客软件分为**服务器端**和**用户端**，攻击时使用用户端程式登陆已安装好服务器端程式的计算机。
- 服务器端程式较小，一般附带于某些软件上（如下载小游戏并运行时即安装完成）。
- 大部分黑客软件重生能力较强，清除困难。
- 出现 TXT 文件欺骗手法：表面是 TXT 文本文件，实际是附带黑客程式的可执行程式；也可伪装成图片和其他格式文件。

### 7. 安全漏洞攻击 ⭐

利用系统安全漏洞（Bugs）进行攻击，分为两类：*(讲义 p128)*

**（1）操作系统或应用软件本身的漏洞**：*(讲义 p128)*
- 典型：**缓冲区溢出攻击**。系统不检查程式和缓冲之间变化的情况，任意接受任意长度的数据输入，把溢出的数据放在堆栈里，系统照常执行命令。攻击者发送超出缓冲区所能处理的长度的指令，系统进入不稳定状态。特别设置的攻击字符甚至可访问根目录，拥有对整个网络的绝对控制权。

**（2）利用协议漏洞进行攻击**：*(讲义 p128)*
- 如 POP3 必须在根目录下运行的漏洞 → 破坏根目录 → 获得终极用户权限。
- 如 ICMP 协议被用于发动**拒绝服务攻击**：向目的服务器发送大量数据包，几乎占取该服务所有的网络带宽，使其无法对正常的服务请求进行处理，导致网站无法进入、响应速度大大降低或服务器瘫痪。
- 蠕虫病毒通过 Microsoft Outlook 向众多邮箱发出带有病毒的邮件，使邮件服务器瘫痪。

> Caveat (课后习题19): 死亡之 ping 属于**拒绝服务攻击**。*(题目课后习题19)*

> Caveat (课后习题20): 在 DDoS 攻击中，通过非法入侵并被控制，但并不向被攻击者直接发起攻击的计算机称为**主控端**。*(题目课后习题20)*

### 8. 端口扫描攻击 ⭐

利用 Socket 编程和目标主机的某些端口建立 TCP 连接、进行传输协议的验证等，从而侦知目标主机的扫描端口是否处于激活状态、主机提供了哪些服务、提供的服务中是否含有某些缺陷等。*(讲义 p129)*

常用扫描方式：*(讲义 p129)*
- Connect() 扫描
- Fragmentation 扫描

> Caveat (课后习题9): 端口扫描技术既可以作为**攻击工具**，也可以作为**防御工具**（检查系统漏洞）。*(题目课后习题9)*

## 数据加密 ⭐

### 加密的概念

数据加密的基本过程就是对原来为明文的文件或数据按某种算法进行处理，使其成为不可读的一段代码，通常称为"密文"，使其只能在输入相应的密钥之后才能显示出本来内容，通过这样的途径来达到保护数据不被非法人窃取、阅读的目的。该过程的逆过程为**解密**，即将该编码信息转化为其原来数据的过程。*(讲义 p130)*

### 对称式加密 ⭐

对称式加密就是加密和解密使用**同一个密钥**，通常称之为"Session Key"。*(讲义 p130)*

- 典型代表：美国政府采用的 **DES** 加密标准，Session Key 长度为 **56 Bits**。*(讲义 p130)*
- 优点：被广泛采用，技术成熟。
- 缺点：如果是在网络上传输加密文件，就很难把密钥告诉对方，不管用什么方法都有可能被别人窃听到。

> Caveat (Ch7-判1): 对称密钥体制中**算法是公开的**，只有密钥保密（Kerckhoffs 原则）。*(题目Ch7-判1)*

> Caveat (课后习题2): 课后习题考了 DES 加密算法的**密文分组长度**和**有效密钥长度**。OCR 正文仅提及密钥长度 56 Bits，密文分组长度未在知识点正文中出现，需结合题目确认。*(题目课后习题2)*

> Caveat (出处): DES 密钥考试口径——密钥**名义长度 64 位**，其中每字节第 8 位为奇偶校验位（共 8 位），**有效密钥 56 位**；密文分组长度 64 位。**3DES = 三次 DES**（加密-解密-加密，EDE），常用 2 个密钥（K1=K3）共 112 位有效强度，3 个密钥共 168 位。*(考点补充)*

### 非对称式加密 ⭐⚠

非对称式加密就是加密和解密所使用的**不是同一个密钥**。通常有两个密钥，称为"公钥"和"私钥"，它们两个必需配对使用，否则不能打开加密文件。*(讲义 p131)*

- **公钥**：可以对外公布。
- **私钥**：不能公开，只能由持有人一个人知道。
- 公钥和私钥**不可相互推导**。
- 优越性：对称式加密在网络上传输加密文件时难以把密钥告诉对方，而非对称式的"公钥"可以公开，不怕别人知道，收件人解密时用自己的私钥即可，很好地避免了密钥的传输安全性问题。

> Caveat (Ch7-判4): CA 证书绑定的是**公钥**而非私钥，私钥永远不离开持有者。*(题目Ch7-判4)*

> Caveat (课后习题3): 非对称加密的加解密处理速度**不快**（比对称加密慢），"加解密处理速度快"是双钥密码体制的错误说法。*(题目课后习题3)*

> Caveat (课后习题7): 双钥密码体制中加解密密钥不相同，从一个很难计算出另一个。*(题目课后习题7)*

> Caveat (课后习题12,17): **RSA** 是双钥（非对称）加密算法；**DES** 属于对称加密，不属于非对称密码算法。*(题目课后习题12,17)*

### 对称式 vs 非对称式加密对比

| 特性 | 对称式加密 | 非对称式加密 |
|---|---|---|
| 密钥数量 | 1 个（Session Key） | 2 个（公钥 + 私钥） |
| 加解密密钥关系 | 相同（两者相等） | 不相同，不可相互推导 |
| 典型算法 | DES（56 Bits） | RSA |
| 密钥传输 | 网络传输密钥困难，可能被窃听 | 公钥可公开，避免密钥传输安全问题 |
| 加解密速度 | [原文未述] | 较慢（来自课后习题考点信号） |

*(讲义 p130-131)*

## IPSec / SSL / TLS ⭐ *(考点补充)*

讲义 OCR 未覆盖 IPSec/SSL/TLS 这一高频考点（exam-signals 显示该主题被考查 ×11 次），以下为标准知识补充。

**IPSec（Internet Protocol Security，IP 安全协议）**：工作在 **网络层（IP 层）**，对 IP 数据报进行加密和/或认证，保护 IP 数据报的机密性、完整性、来源认证和防重放。*(考点补充)*

**IPSec 两大子协议对比**：*(考点补充)*

| 子协议 | 提供服务 | 是否加密 | AH 头位置 |
|---|---|---|---|
| **AH（Authentication Header，认证头）** | 数据完整性、来源认证、防重放 | **不加密**（仅认证） | 插在 IP 头与上层协议之间 |
| **ESP（Encapsulating Security Payload，封装安全载荷）** | 数据完整性、来源认证、防重放 + **机密性** | **加密 + 认证** | 可同时对 payload 加密 |

*(考点补充)*

**IPSec 两种工作模式**：*(考点补充)*

| 模式 | 保护范围 | 应用场景 |
|---|---|---|
| **传输模式（Transport）** | 只保护 IP 上层载荷（如 TCP/UDP），原 IP 头保留 | 主机到主机（端到端通信） |
| **隧道模式（Tunnel）** | 保护整个原始 IP 数据报，外加新 IP 头 | 网关到网关（VPN 站点间）、主机到网关 |

*(考点补充)*

**IPSec 关键概念**：*(考点补充)*
- **SA（Security Association，安全关联）**：发送方与接收方之间为通信而建立的单向逻辑连接，由三元组（SPI、目的 IP、协议标识 AH/ESP）唯一标识。**双向通信需要两个 SA**（一去一回）。
- **IKE（Internet Key Exchange，密钥交换协议）**：基于 **Diffie-Hellman** 在通信双方之间自动协商密钥材料，建立 SA，默认运行在 **UDP 端口 500**。
- **SAD（SA 数据库）**：存放已建立的 SA 参数；**SPD（安全策略数据库）**：定义哪些流量需要 IPSec 保护、如何保护。

**SSL / TLS（安全套接层 / 传输层安全）**：工作在 **传输层之上、应用层之下**（介于 TCP 与应用之间），为基于 TCP 的应用层协议（如 HTTP→HTTPS、SMTP/TLS、FTP/TLS）提供加密与完整性保护。SSL 已废弃，TLS 为继任标准（TLS 1.0 ≈ SSL 3.1）。*(考点补充)*

**SSL/TLS 双层协议**：*(考点补充)*

| 子协议 | 作用 |
|---|---|
| **握手协议（Handshake）** | 协商加密算法、利用非对称密码（RSA/DH）交换对称密钥、双向身份认证（数字证书） |
| **记录协议（Record）** | 对上层报文做**压缩→加密→加 MAC**，再封装为 TLS 记录交 TCP 传输；提供机密性与完整性 |
| **警告协议（Alert）** | 传递告警与错误信息（如证书失效、握手失败） |
| **密码变更规范（ChangeCipherSpec）** | 通知对端后续报文启用新协商的密钥与算法 |

*(考点补充)*

**SSL/TLS 握手四件套（简化步骤）**：*(考点补充)*
1. **ClientHello**：客户端发送支持的密码套件与随机数。
2. **ServerHello + 证书**：服务器选定套件、回送随机数与数字证书（含公钥）。
3. **密钥交换**：客户端用服务器公钥加密预主密钥（RSA 模式），或双方经 DH 协商——共同导出对称的**主密钥**。
4. **Finished**：双方切换到加密通道，握手结束。

**IPSec vs SSL/TLS 对比**：*(考点补充)*

| 对比项 | IPSec | SSL/TLS |
|---|---|---|
| 工作层次 | 网络层（IP 层） | 传输层与应用层之间 |
| 保护对象 | IP 数据报 | 应用层报文（HTTP/SMTP 等） |
| 配置粒度 | 网络级（一对网关/主机） | 应用级（一个 TCP 连接） |
| 典型应用 | 站点到站点 VPN、远程接入 | HTTPS、SMTPS、FTPS |

*(考点补充)*

> Caveat (Ch7-选4~5): **AH 只认证不加密**，**ESP 既认证又加密**；**SA 是单向的**，双向通信需建两条 SA；IPSec 工作在**网络层**，TLS 工作在**传输层之上**。*(考点补充)*

## 防火墙技术 ⭐

### 防火墙的定义

防火墙是在网络之间执行安全控制策略的系统，它包括硬件和软件，采用由系统管理员定义的规则，对一个安全网络和一个不安全网络之间的数据流加以控制。*(讲义 p132)*

### 防火墙的功能

*(讲义 p132)*
- 过滤进、出网络的数据
- 管理进、出网络的访问行为
- 封堵某些禁止行为
- 记录通过防火墙的信息内容和活动
- 对网络攻击进行检测和告警

### 防火墙的优点（5 点）

*(讲义 p132-133)*
1. **强化安全策略**：防火墙是"交通警察"，执行站点的安全策略，仅容许"认可的"和符合规则的请求通过。
2. **有效记录 Internet 上的活动**：所有进出信息都必须通过防火墙，非常适用收集关于系统和网络使用和误用的信息。
3. **限制暴露用户点**：用来隔开网络中一个网段与另一个网段，防止影响一个网段的问题通过整个网络传播。
4. **安全策略的检查站**：所有进出的信息都必须通过防火墙，使可疑的访问被拒绝于门外。
5. **有效防止内部信息的外泄**：通过防火墙对内部网络的划分，可实现内部网络重点网段的隔离；可隐蔽那些透露内部细节的安全线索。此外，防火墙还支持具有 Internet 服务特性的企业内部网络技术体系 **VPN**（虚拟专用网）。

### 防火墙的不足之处（3 点）⚠

*(讲义 p133)*
1. **不能防范恶意的知情者**。
2. **不能防范不通过它的连接**。
3. **不能防备全部的威胁**。

> Caveat (课后习题5): 防火墙对**绕过防火墙的访问和攻击**无能为力。*(题目课后习题5)*

> Caveat (课后习题8): 防火墙**不能**防止受病毒感染的文件的传输。*(题目课后习题8)*

> Caveat (课后习题13): 防火墙**会削弱**计算机网络系统的性能。*(题目课后习题13)*

### 防火墙的主要类型 ⭐

三种主要类型对比：

| 特性 | 包过滤防火墙 | 应用代理防火墙 | 状态检测防火墙 |
|---|---|---|---|
| 工作层次 | 网络层 | 应用层 | 网络层（检查引擎截获数据包） |
| 工作原理 | 检查数据流中每一个 IP 数据包的源地址、目的地址、所用端口号、协议状态等因素或其组合 | 将所有跨越防火墙的网络通信链路分为两段，内外计算机系统间应用层的连接由两个代理服务器之间的连接来实现 | 由检查引擎截获数据包并抽取与应用状态有关的信息，维护动态的状态信息表并对后续数据包进行检查，发现连接参数意外变化则终止连接 |
| 优点 | 速度快、逻辑简单、成本低、易于安装和使用、网络性能和透明度好 | 外部计算机的网络链路只能到达代理服务器，起到隔离防火墙内外计算机系统的作用 | 克服了包过滤防火墙和应用代理防火墙的局限性，能根据协议、端口及 IP 地址具体情况决定是否通过 |
| 缺点 | 配置困难、容易出现漏洞、为特定服务开放的端口存在潜在危险 | 执行速度慢、操作系统容易遭到攻击 | [原文未述具体缺点] |
| 部署方式 | 可直接集成在路由器上（路由选择同时完成过滤），也可由单独计算机完成 | 代理服务器上安装代理服务软件（如 WinGate Pro、Microsoft Proxy Server 等） | [原文未述] |
| 别名 | [原文未述] | [原文未述] | 动态包过滤防火墙 |

*(讲义 p133-134)*

> Caveat (课后习题6): 防火墙代理技术的缺点是**处理速度较慢**。*(题目课后习题6)*

> Caveat (课后习题14): 包过滤技术防火墙在过滤数据包时，一般**不关心数据包的内容**（只关心源地址、目的地址、端口号、协议状态等）。*(题目课后习题14)*

> Caveat (课后习题4): 个人防火墙既可以对单机提供保护，也**可以**对网络提供保护——此说法**错误**（个人防火墙主要对单机提供保护）。*(题目课后习题4)*

实际使用中，一般综合采用以上几种技术，使防火墙产品满足安全性、高效性、适应性和易管性的要求，再集成防毒软件的功能来提高系统的防毒能力和抗攻击能力。*(讲义 p134)*

## 入侵检测

### 入侵检测的概念

入侵检测是对发生在计算机系统或网络中的事件进行监控及对入侵信号的分析过程。使监控和分析过程自动化的软件或硬件产品称为**入侵检测系统**（Intrusion Detection System），简称 **IDS**。*(讲义 p135)*

一个完整的入侵检测系统必须具有：**经济性、时效性、安全性、可扩展性**的特点。*(讲义 p135)*

### 入侵检测的系统结构 — CIDF 模型

CIDF（Common Intrusion Detection Frame，公共入侵检测框架）提出了通用模型，将入侵检测系统分为四个基本组件：*(讲义 p135)*

| 组件 | 功能 |
|---|---|
| 事件产生器（Event generators） | 负责原始数据采集，对数据流、日志文件等进行追踪，将收集到的原始数据转换为事件，向系统其他部分提供此事件 |
| 事件分析器（Event analyzers） | 接收事件信息并分析，判断是否是入侵行为或异常现象，把判断结果转换为警告信息 |
| 事件数据库（Event database） | 存放各种中间和最终数据的地方 |
| 响应单元（Response units） | 根据警告信息做出反应（切断连接、改变文本属性等强烈反应，或简单地报警），是入侵检测系统中的**主动武器** |

*(讲义 p135)*

### 入侵检测系统的分类

根据目标系统类型的不同，分为两类：*(讲义 p135-136)*

**（1）基于主机（host-based）的入侵检测系统**：*(讲义 p135)*
- 检测系统、事件和操作系统下的安全记录以及系统记录。
- 当文件发生变化时，将新的记录条目与攻击标记相比较，看是否匹配。
- 匹配则向管理员报警。
- 基本结构：审计数据收集器 → 过滤器 → 分析器 → 安全管理员。

**（2）基于网络（network-based）的入侵检测系统**：*(讲义 p136)*
- 使用原始网络数据包作为数据源。
- 利用一个运行在**混杂模式**下的网络适配器来实时监视并分析网络的所有通信业务。
- 基本结构：数据包捕获模块 → 网络协议分析模块 → 入侵事件检测模块（配合规则解析模块、攻击特征库、存取模块、数据库）→ 告警。

### 入侵检测系统的功能

*(讲义 p136)*
- 监视并分析用户和系统的行为
- 审计系统配置和漏洞
- 评估敏感系统和数据的完整性
- 识别攻击行为、对异常行为进行统计
- 自动收集与系统相关的补丁
- 审计、识别、跟踪违反安全法规的行为
- 使用诱骗服务器记录黑客行为

## 网络管理技术

### 概况

第一个实用的网管协议是因特网的**简单网管协议 SNMP**，是一个临时的、简单的、为了解决当时急需的协议。此后又设计了两个网管协议：*(讲义 p137)*
- **SNMPv2**：比 SNMP 增加了不少特性以克服原 SNMP 的缺陷。
- **CMIP**（公共管理信息协议）：是一个比 SNMPv2 更好的协议。

由于因特网的发展，**SNMP 已成为事实上的网管标准**。*(讲义 p137)*

### ISO 定义的网管五个功能域 ⭐

| 功能域 | 说明 |
|---|---|
| 配置管理 | 定义、识别、监控被管对象，改变其操作特性，报告其状态变化 |
| 故障管理 | 检测、定位、排除被管对象的故障 |
| 性能管理 | 收集、分析网络的工作性能参数，决定有关控制门限以便及时报告和处理 |
| 计费管理 | 记录用户使用网络资源的情况和统计资源利用率 |
| 安全管理 | [原文仅列名称，未展开说明] |

*(讲义 p137)*

### 简单网络管理协议 SNMP ⭐

**SNMP 的工作原理**：*(讲义 p137)*
- SNMP 采用特殊的客户机/服务器模式，即**代理/管理站模型**。
- 对网络的管理与维护是通过管理工作站与 SNMP 代理间的交互工作完成的。
- 每个 SNMP 从代理负责回答 SNMP 管理工作站（主代理）关于 **MIB** 定义信息的各种查询。

**MIB 的作用**：*(讲义 p137)*
- 管理站和代理端使用 MIB 进行接口统一，MIB 定义了设备中的被管理对象。
- 管理站和代理都实现相应的 MIB 对象，使得双方可以识别对方的数据，实现通信。
- 管理站向代理请求 MIB 中定义的数据，代理端识别后，将管理设备提供的相关状态或参数等数据转换成 MIB 定义的格式，最后将该信息返回给管理站，完成一次管理操作。

**SNMP 传输层协议**：使用 **UDP**。*(讲义 p137)*

> Caveat (课后习题15): SNMP 的中文含义为**简单网络管理协议**。*(题目课后习题15)*

### 易错点

- ⚠ 非对称加密**速度不快**，不要误选"加解密处理速度快"为双钥密码体制的特点。*(题目课后习题3)*
- ⚠ 防火墙**不能**防止受病毒感染的文件传输，不要误认为防火墙能防病毒。*(题目课后习题8)*
- ⚠ 包过滤防火墙一般**不关心数据包的内容**，只看源/目的地址、端口号、协议状态等。*(题目课后习题14)*
- ⚠ DES 属于**对称**加密算法，不属于非对称密码算法。*(题目课后习题17)*
- ⚠ 口令嗅探是**被动攻击**，不是主动攻击。*(题目课后习题11)*
- ⚠ 个人防火墙主要对**单机**提供保护，不能对网络提供保护。*(题目课后习题4)*

## 一句话记忆要点

- **网络安全三要素**：保密性、完整性、可用性（CIA）— 拒绝服务破坏的是可用性。
- **8 种攻击手段**：口令入侵、特洛伊木马、WWW 欺骗、邮件攻击、网络监听（被动）、黑客软件、安全漏洞攻击、端口扫描（攻防两用）。
- **对称加密**：一把密钥，DES（56 Bits），算法公开密钥保密，密钥传输是难点。
- **非对称加密**：两把密钥（公钥+私钥），不可互推，RSA，公钥可公开解决密钥传输问题，但速度慢。
- **防火墙**：网络间的安全控制策略系统（硬件+软件），能强化安全策略、记录活动、隔离网段，但不能防内部攻击、不能防绕过它的攻击、不能防全部威胁。
- **防火墙三类型**：包过滤（网络层，快但只看地址/端口不看内容）、应用代理（应用层，隔离但慢）、状态检测（动态包过滤，维护状态表，克服前两者局限）。
- **入侵检测 IDS**：监控+分析事件，CIDF 四组件 = 产生器→分析器→数据库→响应单元；分基于主机和基于网络两类。
- **网络管理**：SNMP 是事实标准，代理/管理站模型，MIB 统一接口，UDP 传输；ISO 五功能域 = 配置、故障、性能、计费、安全。

## Worked Examples

### 例 1：非对称密钥加密概念辨析 *(来自本章例题2)*
*Tags: 非对称加密, 密码体制*
**问题**：( )密钥加密，是指加密解密双方拥有不同的密钥。*(讲义 p138)*
**解答**：
1. 关键词"加密解密双方拥有**不同**的密钥"→ 非对称加密的定义特征
2. 对称加密使用同一密钥，非对称加密使用不同密钥（公钥+私钥）
3. 答案：**B．非对称**

### 例 2：防火墙定义辨析 *(来自本章例题3)*
*Tags: 防火墙, 安全控制策略*
**问题**：( )是在网络之间执行安全控制策略的系统，它包括硬件和软件。*(讲义 p138)*
**解答**：
1. 关键词"网络之间执行安全控制策略""包括硬件和软件"→ 防火墙的定义
2. 防火墙由系统管理员定义规则，对安全网络和不安全网络之间的数据流加以控制
3. 答案：**B．防火墙**

### 例 3：计算机病毒定义 *(来自本章例题4)*
