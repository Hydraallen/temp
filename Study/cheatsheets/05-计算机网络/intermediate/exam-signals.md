## 2. 反复出现的题型模式

- **设备→OSI 层次映射**：题干给出设备名（集线器/交换机/路由器/网卡/网桥）→ 判断工作层次 + 能否划分冲突域/广播域。切入点：物理层设备（中继器/集线器）不隔离冲突域；链路层设备（网桥/交换机）隔离冲突域不隔离广播域；网络层设备（路由器/三层交换机）两者均隔离 `[PROPOSED — verify]`
- **VLAN 配置辨析**：题干涉及 Access/Trunk/Hybrid 端口或 VLAN Tag → 判断 Tag 添加/剥离规则、VLAN 间通信需路由器/三层交换机。切入点：Access 口只属一个 VLAN，Trunk 口可承载多 VLAN 带 Tag `[PROPOSED — verify]`
- **TCP seq/ack 数值推算**：题干给出 seq、ack、数据长度 → 求确认报文中的 seq/ack。切入点：确认号 = 收到报文 seq + 数据长度；确认报文 seq = 对方 ack `[PROPOSED — verify]`
- **IPv6 地址缩写**：题干给出完整 IPv6 地址 → 选正确缩写。切入点：连续零组用 `::` 只能出现一次，前导零可省略 `[PROPOSED — verify]`
- **路由协议封装协议判断**：题干问 RIP/OSPF/BGP 分别封装在什么之上 → RIP→UDP、OSPF→IP、BGP→TCP。切入点：BGP 需可靠传输故用 TCP，RIP 简单故用 UDP `[PROPOSED — verify]`
- **最长前缀匹配**：题干给出目的 IP + 多条路由 → 选匹配路由。切入点：掩码越长（前缀越长）优先级越高，先判断目的 IP 是否在某网段内 `[PROPOSED — verify]`
- **PDU 单位对应**：题干问各层 PDU 名称 → 物理层比特、链路层帧、网络层数据报/分组、传输层报文段。切入点：按层序背诵对应 `[PROPOSED — verify]`
- **SDN/MPLS 概念辨析**：题干涉及 SDN 特征或 MPLS 标签结构 → 判断体系结构属性、标签字段位数、LDP 协议功能。切入点：SDN = 控制转发分离 + 集中控制 + 开放接口；MPLS 标签 32bit 中 Label 占 20bit `[PROPOSED — verify]`

## 3. 技巧 / 易错点（Caveat callout）

- > Caveat (Ch1-选9): OSI 传输效率计算——400B 数据经 5 层封装（除物理层和应用层），每层加 20B 开销 = 100B 额外开销，效率 = 400/(400+100) = 80%。易错点：物理层不加开销、应用层不加开销，只算中间 5 层 `[PROPOSED — verify]`
- > Caveat (Ch2-选16): 海明码校验位公式——信息位 m=12，求 r 使 2^r ≥ m+r+1，即 2^4=16 ≥ 12+4+1=17 不成立，2^5=32 ≥ 17 成立，但答案给 B(4)。需注意题目可能用不同公式版本 `[PROPOSED — verify]`
- > Caveat (Ch3-选4): CRC 计算——信息位 1101011011，生成多项式 G(x)=x⁴+x+1 = 10011，模 2 除法求余数。余数为 1110，选 D。易错：模 2 除法不借位，异或运算 `[PROPOSED — verify]`
- > Caveat (Ch3-选6): 二进制后退算法——第 11 次碰撞时，k = min(10, 16) = 10，随机数范围 [0, 2^10-1] = [0, 1023]，选 C。易错：k 值上限为 10（即最大 1023），不是碰撞次数直接取幂 `[PROPOSED — verify]`
- > Caveat (Ch3-判1): MTU 是指数据字段长度上限，不是整个帧长度上限。帧长上限 = MTU + 首部 + 尾部 `[PROPOSED — verify]`
- > Caveat (Ch3-判14): 网桥和二层交换机只划分冲突域，不划分广播域。广播域的划分需要三层设备（路由器/三层交换机） `[PROPOSED — verify]`
- > Caveat (Ch4-选14): RIP 封装在 UDP（端口 520），OSPF 封装在 IP（协议号 89），BGP 封装在 TCP（端口 179）。三者封装协议各不相同 `[PROPOSED — verify]`
- > Caveat (Ch4-判4): IPv6 基本首部是 40 字节，不是 20 字节（20 字节是 IPv4 固定首部） `[PROPOSED — verify]`
- > Caveat (Ch4-判5): IPv6 没有校验和字段（取消以提升转发效率），不是"只检验首部" `[PROPOSED — verify]`
- > Caveat (Ch4-判11): 255.255.255.255 是受限广播地址，只在本地网络广播，不能穿越路由器实现整个互联网广播 `[PROPOSED — verify]`
- > Caveat (Ch5-判10): TCP 释放连接使用四次挥手，不是三次握手。三次握手是连接建立 `[PROPOSED — verify]`
- > Caveat (Ch5-选6): TCP 确认号 = 对方发送的 seq + 数据长度。A 发 seq=300, 数据 7 字节，B 确认 ack=307；B 的 seq = A 的 ack = 101 `[PROPOSED — verify]`
- > Caveat (Ch6-判8): SMTP 只能传输 ASCII 文本，不能直接传多媒体。MIME 扩展后才支持多媒体 `[PROPOSED — verify]`
- > Caveat (Ch6-判10): 基于万维网的电子邮件，用户↔邮件服务器用 HTTP，但邮件服务器之间仍用 SMTP `[PROPOSED — verify]`
- > Caveat (Ch7-判4): CA 证书绑定的是公钥而非私钥，私钥永远不离开持有者 `[PROPOSED — verify]`
- > Caveat (Ch7-判1): 对称密钥体制中算法是公开的，只有密钥保密。Kerckhoffs 原则 `[PROPOSED — verify]`
- > Caveat (Ch8-选10): CSMA/CA 与 CSMA/CD 的关键区别——CA 不能检测碰撞（无线环境硬件困难），需 ACK 确认；CD 可以检测碰撞但不需要 ACK。CSMA/CA 也有随机退避机制 `[PROPOSED — verify]`

## 5. Worked Example 候选（适合放进最终 cheatsheet 末尾）

### 候选 1：Ch1-选9 · OSI 封装传输效率计算 `[PROPOSED — verify]`
- 概念 tag：OSI 分层、PDU 封装、传输效率
- 题干（精简）：应用层发送 400B 数据（无拆分），除物理层和应用层外各层封装引入 20B 额外开销，求传输效率。
- 解题骨架（4 步）：
  1. 确定参与封装的层数：OSI 共 7 层，除去物理层（第 1 层）和应用层（第 7 层），中间 5 层各加 20B
  2. 计算总开销：5 × 20B = 100B
  3. 计算总数据量：400B + 100B = 500B
  4. 效率 = 400 / 500 = 80%，选 A

### 候选 2：Ch3-选4 · CRC 循环冗余校验计算 `[PROPOSED — verify]`
- 概念 tag：CRC、模 2 除法、生成多项式
- 题干（精简）：信息位 1101011011，生成多项式 G(x)=x⁴+x+1，求校验位。
- 解题骨架（5 步）：
  1. G(x)=x⁴+x+1 → 二进制 10011，最高次幂 r=4
  2. 信息位后补 4 个零：11010110110000
  3. 用 11010110110000 模 2 除以 10011（异或运算，不借位）
  4. 余数 = 1110
  5. 校验位 = 1110，选 D

### 候选 3：Ch4-选6 · 最长前缀匹配路由选择 `[PROPOSED — verify]`
- 概念 tag：最长匹配原则、路由表、子网掩码
- 题干（精简）：目的 IP 172.16.10.1，候选路由：A)172.16.10.2/32 B)172.16.10.0/24 C)172.16.0.0/16 D)172.17.10.0/24，选匹配路由。
- 解题骨架（4 步）：
  1. 逐一检查目的 IP 是否在候选网段内
  2. A) /32 是主机路由，172.16.10.2 ≠ 172.16.10.1，不匹配
  3. B) /24：172.16.10.0/24 包含 172.16.10.1，匹配；C) /16 也匹配但前缀更短
  4. 最长前缀优先 → 选 B（/24 > /16）

### 候选 4：Ch5-选6 · TCP 序号与确认号推算 `[PROPOSED — verify]`
- 概念 tag：TCP seq/ack、确认机制
- 题干（精简）：A 向 B 发送 TCP 报文段，seq=300，ack=101，数据 7 字节，求 B 确认报文段的 seq 和 ack。
- 解题骨架（4 步）：
  1. B 的确认号 ack = A 的 seq + 数据长度 = 300 + 7 = 307
  2. B 的序号 seq = A 期望收到的下一个字节 = A 的 ack 值 = 101
  3. 注意：确认号表示"期望收到的下一个字节序号"
  4. 选 D：seq=101, ack=307

### 候选 5：Ch2-选16 · 海明码校验位数计算 `[PROPOSED — verify]`
- 概念 tag：海明码、校验位公式、纠错码
- 题干（精简）：信息位 12 位，使用海明码发现并纠正 1 位错误，需要多少校验位？
- 解题骨架（3 步）：
  1. 海明码公式：2^r ≥ m + r + 1，其中 m=12（信息位），r=校验位
  2. 代入 r=4：2^4 = 16 ≥ 12+4+1 = 17？不成立
  3. 代入 r=5：2^5 = 32 ≥ 12+5+1 = 18，成立 → 需要 5 位
  4. 注意：答案给 B(4)，可能题目使用不同公式版本或条件不同，需核实 `[PROPOSED — verify]`
