## 二、知识点详解

### 第一节 物联网

#### 知识点1：物联网概念 ⭐(练习1)

- **定义**：物联网是一个基于互联网、传统电信网等信息载体，让所有能被独立寻址的普通物理对象实现互联互通的网络。

#### 知识点2：物联网的特征

普通对象设备化、自治终端互联化和普适服务智能化是其三个重要特征。

具体表现为五个方面：

1. **联网终端规模化**：物联网时代每一件物品均具通信功能成为网络终端，5-10年内联网终端规模有望突破百亿。
2. **感知识别普通化**：无所不在的感知和识别将传统上分离的物理世界和信息世界高度融合。
3. **异构设备互联化**：各种异构设备利用无线通信模块和协议自组成网，异构网络通过"网关"互通互联。
4. **管理处理智能化**：物联网高效可靠组织大规模数据，运筹学、机器学习、数据挖掘、专家系统等决策手段广泛应用于各行各业。
5. **应用服务链条化**：以工业生产为例，物联网技术覆盖从原材料引进、生产调度、节能减排、仓储物流到产品销售、售后服务等各个环节。

#### 知识点3：物联网的核心技术 ⭐(练习2)

根据信息生成、传输、处理和应用将物联网分为四层：

| 层次 | 说明 |
|------|------|
| 感知识别层 | 最底端，所有上层结构的基础；信息生成设备包括RFID电子标签、传感器、定位系统及智能手机、PDA等智能设备 |
| 网络构建层 | 负责信息的传输 |
| 管理服务层 | 负责信息的处理与管理 |
| 综合应用层 | 面向具体应用场景 |

#### 知识点4：物联网的发展趋势 ⭐(练习11)

1. **更广泛的互联互通**：互联互通的对象从人延伸到物体，互联互通方式的扩展。
2. **更透彻的感知**：通信功能使传感器能够协同工作。
3. **更深入的智能**：多传感器实现"人多力量大"的智能；多维感知数据实现"防患于未然"的智能；大数据挖掘实现"见微知著"的智能。

#### 知识点5：物联网的应用领域

1. **智能交通**：通过在基础设施和交通工具中广泛应用信息、通讯技术来提高交通运输系统的安全性、可管理性、运输效能，同时降低能源消耗和对地球环境的负面影响。
2. **智能物流**：利用RFID设备、感应器或全球定位系统等装置与互联网结合形成巨大网络，实现智能化的物流管理。
3. **智能建筑**：通过建立以节能为目标的建筑设备监控网络，将各种设备和系统融合在一起，形成以智能处理为中心的物联网应用系统，有效为建筑节能减排提供支撑。
4. **环境监测**：通过对各种物质含量、排放量及环境状态参数的检测，跟踪环境质量变化，为环境管理、污染治理、防灾减灾等工作提供基础信息。

### 第三节 无线传感器网

#### 知识点1：无线传感器网络的基本概念 ⭐(练习8)

- **定义**：由大量分布式传感器节点组成的自组织网络，用于感知、采集和处理环境数据。
- **组成**：传感器节点、汇聚节点（Sink Node）、管理节点。
- **特点**：
  - **自组织性**：节点可自动组网，无需人工干预。
  - **动态性**：网络拓扑可能因节点移动或失效而变化。
  - **资源受限**：节点能量、计算能力和存储空间有限。

#### 知识点2：传感器节点的结构

| 单元 | 功能 | 典型实例 |
|------|------|----------|
| 感知单元 | 负责采集环境数据 | 温度、湿度、光照等 |
| 处理单元 | 对采集的数据进行处理和存储 | 微控制器、微处理器 |
| 通信单元 | 负责节点间的数据传输 | 射频模块、天线 |
| 电源单元 | 为节点提供能量 | 电池、太阳能 |

#### 知识点3：无线传感器网络的通信协议

**MAC层协议：**

| 协议 | 特点 |
|------|------|
| S-MAC | 通过周期性睡眠降低能耗 |
| T-MAC | 动态调整活动时间，进一步节能 |
| Z-MAC | 结合CSMA和TDMA的优点，提高效率 |

**路由协议：**

| 协议 | 特点 |
|------|------|
| LEACH | 基于分簇的路由协议，均衡能量消耗 |
| AODV | 按需路由协议，适用于动态网络 |
| GEAR | 基于地理位置的路由协议，适合大规模网络 |

#### 知识点4：制约传感器性能提升的因素

1. **功耗的制约**：无线传感节点一般被部署在野外，不能通过有线供电。硬件设计必须以节能为重要设计目标。
2. **价格的制约**：无线传感节点一般需要大量组网。硬件设计必须以廉价为重要设计目标。
3. **体积的制约**：无线传感节点一般需要容易携带，易于部署。硬件设计必须以微型化为重要设计目标。

#### 知识点5：无线传感器网络的应用场景

- 环境监测：森林火灾监测、水质监测、气象监测。
- 智能农业：土壤湿度监测、作物生长监控、精准灌溉。
- 智能城市：交通监控、智能照明、垃圾管理。
- 工业物联网：设备状态监控、生产线优化、能源管理。
- 医疗健康：远程病人监控、健康数据采集、智能医疗设备。

#### 知识点6：传感器的按工作原理的分类

1. 电参数传感器（电阻式、电感式、电容式传感器）
2. 压电式传感器
3. 光电式传感器（一般光电式、光纤式、激光式、红外式等）
4. 热电式传感器
5. 半导体式传感器
6. 波式和辐射式传感器

#### 知识点7：微处理器的关键特性、通信芯片的重要指标 ⚠(练习16)

- **微处理器关键特性**：功耗特性、唤醒时间、供电电压、运算速度、内存大小。
- **通信芯片的重要指标**：睡眠功耗、唤醒时间、发射功率、接收灵敏度、接受功耗。
- **常见的节点操作系统**：TinyOS系统、Contiki OS系统、Mantis OS系统、Nanno-RK系统、SenSpire OS系统等。

> ⚠ 易错点：传感器节点周期性地进行数据采集、处理和休眠，其中**休眠占据了大部分时间**（非小部分）。`(练习16)`

#### 知识点8：无线传感器网络的挑战

1. **能量限制**：节点依赖电池供电，能量有限，需优化能耗。
2. **网络拓扑动态性**：节点移动或失效可能导致网络拓扑变化。
3. **数据安全与隐私**：无线通信易受攻击，需加密和认证机制。
4. **数据处理与存储**：节点计算和存储能力有限，需高效算法。
5. **通信干扰**：无线信号可能受到环境干扰，影响通信质量。

#### 知识点9：无线传感器网络的能量管理技术

1. **低功耗硬件设计**：采用低功耗芯片和传感器。
2. **睡眠调度机制**：节点在空闲时进入睡眠模式以节省能量。
3. **数据压缩与聚合**：减少数据传输量，降低通信能耗。
4. **能量采集技术**：利用太阳能、振动能等为节点供电。

#### 知识点10：无线传感器网络的拓扑结构

| 拓扑结构 | 特点 | 适用场景 |
|----------|------|----------|
| 平面结构 | 所有节点地位平等 | 小规模网络 |
| 分簇结构 | 节点分为簇，簇头负责数据聚合和传输 | 大规模网络 |
| 链式结构 | 节点按链式连接，数据逐跳传输 | 线性部署场景 |

### 第四节 定位系统

#### 知识点1：位置信息

位置信息包括三个要素：（1）所在的地理位置（2）处在该地理位置的时间（3）处在该地理位置的对象（人或设备）。

#### 知识点2：现存的主流定位系统及GPS定位系统的组成部分 ⭐(练习9)

- **主流定位系统**：（1）卫星定位：GPS（2）蜂窝基站定位（3）室内精确定位（4）WiFi基站定位。
- **GPS系统组成部分**（三部分）：
  1. 地面控制部分：由主控站、地面天线、随机测量站和通信辅助系统组成。
  2. 空间部分：由分布在6个轨道平面的24颗卫星组成。
  3. 用户设备：由GPS接收机和卫星天线组成。

#### 知识点3：常见的定位技术

1. 基于距离（时间）的定位（Time of Arrival, ToA）
2. 基于距离（时间）差的定位（Time Difference of Arrival, TDoA）
3. 基于信号特征的定位（Received Signal Strength, RSS）

#### 知识点4：每一种定位技术的特点及优缺点 ⚠(练习10)

| 技术 | 特点 | 优点 | 缺点 |
|------|------|------|------|
| ToA | 需要接收端特殊装置 | — | 需要参考点和测量目标时钟同步 |
| TDoA | 需要接收端特殊装置 | 不需要参考点和测量目标时钟同步 | 参考点之间仍然需要时钟同步 |
| RSS | — | — | 需要事先建立精细的特征地图，不能应对动态变化 |

#### 知识点5：LANDMARC定位法的特点

- 除了信号发送源，再布置一系列RFID标签作为参考标志。
- 每个标志随时记录自己收到的RSS信号强度特征向量。
- 将目标测得的信号特征向量与参考标志此时的特征向量进行比对，确定位置。
- 误差在1m范围以内。

#### 知识点6：室内定位技术方案

1. 一般不利用ToA、TDoA、AoA等技术（需要较贵的硬件支持），而一般采取基于RSS技术。
2. **利用已有设备与网络**：蓝牙、WiFi、ZigBee（部署方便、成本低、精度有限）。
3. **利用专门设备与网络**：红外线、超声波、RFID、超宽带（UWB）等（精度高、部署成本高）。

### 第五节 无线宽带网络与无线低速网络

#### 知识点1：无线网络的组成元素

（1）无线网络用户（2）基站（3）自组网（4）无线连接。

#### 知识点2：无线网络的类别、带宽分布等

无线网络的类别：（1）个域网（2）城域网（3）局域网（4）广域网。

#### 知识点3：无线网络的难点，多径传播等

1. **信号强度衰减**：无线信号能量随着传输距离增长而减弱。
2. **非视线传输**：若发送者与接收者之间的路径部分被阻挡，则称其为非视线传输。无线信号可能会被阻挡物吸收或迅速衰减。
3. **信号干扰**：相同无线频段的信号会相互干扰，例如外部环境的电磁噪声（微波炉、汽车、高压电线）。
4. **多径传播**：无线信号由于阻挡物的反射和折射，到达接收端的时间可能略微不同。

#### 知识点6：蓝牙技术的特点

蓝牙设备有两种可能的角色，分别为主设备和从设备。同一个蓝牙设备可以在这两种角色之间转换。

#### 知识点8：蓝牙系统的组成

（1）天线单元（2）链路控制（固体）单元（3）软件（协议栈）单元（4）链路管理单元。

#### 知识点9：Zigbee标准及特点 ⭐(练习12)

- **标准**：IEEE 802.15.4
- **特点**：（1）低功耗（2）低成本（3）时延短（4）网络容量大（5）可靠（6）安全。

#### 知识点10：ZigBee的设备类型及角色 ⭐(练习17)

**两种设备类型：**

| 类型 | 全称 | 能力 |
|------|------|------|
| RFD | 精简功能设备 | 功能受限，只能向协调者注册后使用网络 |
| FFD | 全功能设备 | 有能力成为协调者并选取频道建立新网络 |

**FFD与RFD的关键差别：**
- FFD有能力成为协调者并选取一个频道建立新的网络，而RFD只能透过向协调者注册并连结后才能使用网络。
- 并不是任何节点都有资格向协调者提出GTS（保证传输时槽）使用申请，此节点必须属于FFD才可向协调者要求保证传输时槽使用权。

**三种设备角色：**

| 角色 | 设备类型 | 功能 |
|------|----------|------|
| 端设备 End device | RFD or FFD | 只具有简单的收发功能，不能进行分组的转发 |
| 协调器 Coordinator | FFD | 通过发送信标实现与周围节点的同步，且具有转发分组的功能 |
| 网络协调器 PAN Coordinator | FFD | 整个网络的主控节点，每个IEEE网络只能有一个 |

#### 知识点11：ZigBee的网络拓扑结构类型

（1）星形网（2）网状网（3）树状网。

#### 知识点12：ZigBee协议栈中MAC层设计，如何降低能耗

1. **低功率侦听协议（LPL, low power listening）**：采样侦听、链路层调度。
2. **采样侦听**：无线收发模块在没有数据的时候关闭、定时采样获取信道的信息。

#### 知识点13：ZigBee无线收发模块的工作特点

1. 无线收发模块占据大部分的能量消耗。
2. 无线收发器件（radio transceiver）工作时通常处于三种状态：发送、侦听和空闲状态。
3. **空闲侦听（idle listening）**：节点处于侦听状态，但是并未侦听到任何数据，从而浪费掉了能量。
4. **空闲（idle）**：节点物理地关闭一些硬件功能，从而达到较低的能耗。

#### 知识点14：ZigBee网络层路由协议

网络层采用的基本路由协议是按需距离矢量路由协议（AODV）。

#### 知识点15：体域网概念及标准

体域网（wireless body sensor network，WBSN或BSN）是基于无线传感器网络（WSN）的，是人体上的或移植到人体内的生物传感器共同形成的一个无线网络。它不仅是一种新的普适医疗保健、疾病监控和预防的解决方案，还是物联网的重要感知及组成部分。

#### 知识点16：容迟网概念及典型应用领域

- **概念**：容迟网络（Delay Tolerant Network，DTN）是指网络中端到端的路径通常很难建立，网络中消息传播具有很大延时，使得传统因特网上基于TCP/IP的协议、无线Ad Hoc网络中面向端到端的路由协议变得失效。DTN并不侧重于构建一种新的数据链路层，而是根据网络应用的特点，抽象出的一种新的网络结构。
- **典型应用领域**：深空监测、野生斑马研究、湖水质量检测、乡村通信、移动机会网络。

## 五、高频考点与易错提醒

> 以下考点信号来自专项练习与答案解析，仅指出考点方向，不收录题目与解答。

### 高频考点（⭐）

1. ⭐ **物联网核心目标**：实现物与物、物与人之间的互联。`(练习1)`
2. ⭐ **物联网四层模型**：感知识别层位于最底端，是所有上层结构的基础。`(练习2)`
3. ⭐ **物联网三种基本应用模式**：对象的智能标签、环境监控和对象跟踪、对象的智能控制。⚠ 注意"购买数据分析类应用"不属于三种应用模式。`(练习4)`
4. ⭐ **IC卡通信方式**：接触式IC卡通过金属触点物理连接通信，非接触式IC卡通过无线通信。⚠ "IC卡不需要与读卡器物理连接"的说法不适用于接触式IC卡。`(练习5)`
5. ⭐ **RFID系统三大组件**：标签、阅读器、天线。⚠ 五组件（传送器、接收器、微处理器、天线、标签）中传送器/接收器/微处理器封装在一起统称阅读器。`(练习6, 多选1, 判断9)`
6. ⭐ **RFID标签分类**：被动式（无源）、主动式（有源）、半主动式（半被动式）。主动式标签内建电池，通信距离更远（可达上百米），但体积大、价格贵。`(练习7, 判断8)`
7. ⭐ **低频RFID（125kHz/13.56MHz）采用无源标签**。`(练习3)`
8. ⭐ **传感器硬件设计考虑因素**：微型化、低功耗、扩展性和灵活性、稳定性和安全性、低成本。⚠ 不需要考虑"持续性"。`(练习8)`
9. ⭐ **GPS三部分组成**：地面控制部分、空间部分（24颗卫星/6轨道）、用户设备。⚠ 不含"冲突检测部分"。`(练习9)`
10. ⭐ **GPS功能**：定位、导航、授时。⚠ GPS不包含通信功能。`(练习10)`
11. ⭐ **物联网智能设备三趋势**：更深入的智能化、更透彻的感知、更全面的互联沟通。⚠ "更智能的异步通信"不属于三趋势。`(练习11)`
12. ⭐ **Zigbee基于IEEE 802.15.4标准**（非802.11/802.3/802.1Q）。`(练习12)`
13. ⭐ **ZigBee MAC层职责**：负责设备间无线数据链路的建立、维护和结束。`(练习17)`
14. ⭐ **NB-IoT是基于蜂窝网络的物联网通信技术**；LoRaWAN和Sigfox是非蜂窝LPWAN技术。`(练习20, 多选9)`
15. ⭐ **物联网数据处理方式**：边缘计算、云计算、分布式计算、集中式计算。`(多选6)`
16. ⭐ **感知层设备**：二维码标签、摄像头、GPS、RFID标签和读卡器均属于感知层。`(多选5)`
17. ⭐ **传感器精度和可靠性直接影响物联网系统性能**。`(多选7)`
18. ⭐ **传感网络可使用多种协议**（不仅限于Zigbee，还有LoRa、BLE等）。`(多选8)`

### 易错点（⚠）

1. ⚠ **传感器节点休眠占比**：传感器节点周期性进行数据采集、处理和休眠，其中**休眠占据大部分时间**（非小部分）。`(练习16)`
2. ⚠ **动态特性定义**：动态特性是指传感器输入量随时间变化时，**输出量**随时间变化的特性（非精度、输出阻抗或长期稳定度）。`(练习13)`
3. ⚠ **生理特征识别**：虹膜识别、掌纹识别、红外温谱图属于生理特征识别；步态识别属于行为特征识别，按键识别和签名识别不属于生理特征。`(练习14)`
4. ⚠ **EPC码解析**：连接在互联网中的**服务器**（非DNS服务器）完成对EPC码所涵盖内容的解析。EPC为每一个产品（而非一类产品）分配唯一电子标识符。`(练习15)`
5. ⚠ **IPv6地址空间**：地址长度128位，地址空间 $2^{128} \approx 3.4 \times 10^{38}$，地球表面每平方米约 $10^{26}$ 个地址（非6650个）。`(练习18，反推)`
6. ⚠ **TCP特点**：TCP是面向连接的、可靠传输、有流量控制和拥塞控制。⚠ TCP有流控制（非"非流控制"）。`(练习19，反推)`
7. ⚠ **移动通信系统结构**：核心网与基站通过光纤等**有线网**通信（非空中接口）。空中接口用于移动终端与基站之间。`(练习21)`
8. ⚠ **云存储**：用户不能明确指明使用的主机、数据库、存储设备，由系统自动分配。`(练习22)`
9. ⚠ **二维条码**：二维条码有发展前途且应用广泛。UPC码是一维条码，不属于二维条码类别。二维条码类别包括线性堆叠式二维码、矩阵式二维码、邮政码。`(判断4, 判断10)`
10. ⚠ **RFID系统组件封装**：传送器、接收器和**微处理器**（非天线）通常封装在一起统称阅读器。`(判断9)`
11. ⚠ **LoRaWAN不是蜂窝通信协议**，而是基于LoRa（长距离低功耗）技术。`(多选9)`

| 物联网三趋势 | 更广泛互联互通、更透彻感知、更深入智能 |
| IC卡 | 数据存储系统；接触式靠金属触点，非接触式靠无线；CPU卡可加解密 |
| RFID频率 | LF（125kHz）近距离抗干扰，HF（13.56MHz）适中距离 |
| RFID标签 | 被动式（无源）、主动式（有源，距离远体积大）、半主动式 |
| WSN特点 | 自组织、动态、资源受限（能量/计算/存储） |
| 传感器节点四单元 | 感知→处理→通信→电源 |
| WSN三大制约 | 功耗、价格、体积 |
| WSN拓扑 | 平面（小规模）、分簇（大规模）、链式（线性） |
| GPS三部分 | 地面控制 + 空间24颗卫星 + 用户设备 |
| GPS功能 | 定位、导航、授时（无通信） |
| 定位技术 | ToA需双向同步，TDoA只需参考点同步，RSS需特征地图 |
| LANDMARC | RFID参考标签+RSS特征向量比对，误差≤1m |
| 室内定位 | 一般用RSS（成本低），不用ToA/TDoA/AOA（硬件贵） |
| CSMA/CA | 侦听空闲后仍等随机时间再发，避免冲突 |
| 蓝牙 | 全球适用、语音数据同传、临时对等、近距离、抗干扰、低功耗小体积 |
| ZigBee标准 | IEEE 802.15.4，低功耗低成本短时延大容量可靠安全 |
| ZigBee设备 | RFD（精简，只能注册用网）vs FFD（全功能，可建网可当协调器） |
| ZigBee路由 | AODV（按需距离矢量） |
| ZigBee节能 | LPL低功率侦听 + 采样侦听（无数据时关闭收发模块） |
| 体域网BSN | 人体上/体内生物传感器形成的无线网络，普适医疗方案 |
| 容迟网DTN | 端到端路径难建立、延时大，TCP/IP失效；用于深空/野生/乡村 |
| IoT协议选择 | 低功耗选CoAP/MQTT/Zigbee/LoRaWAN/NB-IoT；实时选WebSocket/MQTT；工业选Modbus/AMQP |
| NB-IoT | 基于蜂窝网络的LPWAN，覆盖广穿透强 |
| 蜂窝网络 | GSM四模块（基站/核心网/GPRS/SIM）；CDMA用编码区分信号 |
| 3G标准 | TD-SCDMA（移动）、WCDMA（联通）、CDMA2000（电信） |
| 网络存储 | DAS简单但信息孤岛，NAS文件级共享但依赖网络，SAN存储共享但无文件级访问 |
| 物联网数据 | 海量性、多态性、关联性及语义性 |
| NoSQL四类 | 键值存储、列存储、文档型、图形数据库 |
| 云存储 | 用户不能指定主机/数据库/存储设备，由系统自动分配 |
