# 操作系统 · 第六章 文件管理 — 复习资料

> 来源：4-3-操作系统-讲义.pdf 物理页 [117, 139]（印刷页 p115–p137）
> 说明：仅摘录「知识点」正文；⭐=高频考点，⚠=易错点（依据本章专项练习与答案标注）。

## 一、本章概览

文件管理是操作系统的核心功能之一，解决的是辅存上程序与数据的组织、存储、检索、共享与保护问题。本章涵盖七大知识块：文件系统基本概念（文件、数据项、记录的层级关系及文件分类）、文件的逻辑结构（记录式 vs 流式；顺序/索引/索引顺序）、文件的物理结构（顺序/链接/索引三种存储方式）、目录管理（FCB、索引结点、单级/两级/树形目录、目录查询技术）、文件存储空间管理（空闲表/空闲链表/位示图/成组链接法）、文件共享与保护（硬链接/软链接、访问控制）、磁盘组织与管理（磁盘结构、访问时间公式、磁盘调度算法）。与进程管理（资源分配）和设备管理（I/O）紧密关联。

## 二、知识点详解

### 1. 文件系统 ⭐(练习3,9,23)

- **定义**：文件系统的管理功能是将其管理的程序和数据通过组织为一系列文件的方式实现的。文件是指具有文件名的若干相关元素的集合。元素通常是记录，记录又是一组有意义的数据项的集合。基于文件系统的概念，可以把数据组成分为**数据项、记录和文件**三级。

- **相关概念**：
  - **数据项**：文件系统中最低级的数据组织形式。
    - 基本数据项：用于描述一个对象的某种属性的一个值（如姓名、日期或证件号等），是数据中可命名的最小逻辑数据单位，即**原子数据**。
    - 组合数据项：由多个基本数据项组成。
  - **记录**：一组相关的数据项的集合，用于描述一个对象在某方面的属性。
  - **文件**：由创建者所定义的一组相关信息的集合，逻辑上可分为有结构文件（记录式文件）和无结构文件（流式文件）。

- **文件的属性**：
  - 文件类型（源文件、目标文件、可执行文件等）
  - 文件长度（当前长度，单位可以是字节、字或块，也可能是最大允许长度）
  - 文件的物理位置（指示文件所在的设备及所在设备中地址的指针）
  - 文件的建立时间（最后一次的修改时间等）

- **文件的基本操作**（6 种）：
  1. **创建文件**：在文件系统中为文件找到空间；在目录中为新文件创建条目（记录文件名称、位置及其他可能信息）。
  2. **写文件**：系统调用，指明文件名称和待写入内容；系统搜索目录查找文件位置，维护写位置指针，每次写操作更新写指针。
  3. **读文件**：系统调用，指明文件名称和待读入文件块的内存位置；维护读位置指针。读和写操作使用同一指针，节省空间、降低复杂度。
  4. **文件重定位（文件寻址）**：按条件搜索目录，将当前文件位置设为给定值，不读、写文件。
  5. **删除文件**：从目录中找到要删除文件的目录项使之成为空项，回收文件所占用的存储空间。
  6. **截断文件**：允许文件所有属性不变，删除文件内容（长度设为 0 并释放空间）。

- **文件的分类**：

  | 分类维度 | 类别 | 说明 |
  |---|---|---|
  | 按用途 | 系统文件 | 由系统软件构成，大多数只允许调用不允许读/修改 |
  | | 用户文件 | 由用户的源代码、目标文件、可执行文件或数据等构成 |
  | | 库文件 | 由标准子例程及常用例程构成，允许调用不允许修改 |
  | 按数据形式 | 源文件 | 由源程序和数据构成，通常由 ASCII 码或汉字组成 |
  | | 目标文件 | 源程序经编译但未链接的目标代码，后缀 `.obj` |
  | | 可执行文件 | 目标代码经链接后形成，后缀 `.exe`[原文 OCR 为".cxc"，疑为".exe"] |
  | 按存取控制属性 | 只执行文件 | 只允许被核准的用户调用执行，不允许读和写 |
  | | 只读文件 | 只允许文件主及被核准的用户读，不允许写 |
  | | 读写文件 | 允许文件主和被核准的用户读或写 |
  | 按组织形式和处理方式 | 普通文件 | 由 ASCII 码或二进制码组成的字符文件 |
  | | 目录文件 | 由文件目录组成，可对其下属文件信息进行检索 |
  | | 特殊文件 | 特指系统中的各类 I/O 设备，系统将所有 I/O 设备视为文件 |

### 2. 文件的逻辑结构 ⭐(练习1,14,31,33,34)

- **定义**：文件的逻辑结构从是否有结构来分，可分为两大类——有结构文件（记录式文件）和无结构文件（流式文件）。从文件的组织方式来分，可分为顺序文件、索引文件和索引顺序文件。

- **按文件是否有结构分类**：

  | 类型 | 子类 | 特点 |
  |---|---|---|
  | 有结构文件（记录式文件） | 定长记录 | 所有记录长度相同，各数据项位置/顺序/长度一致；文件长度用记录数目表示；能有效提高检索速度和效率，方便处理和修改；目前较常用 |
  | | 变长记录 | 各记录长度不相同（数据项数目不同或数据项本身长度不定）；处理前每个记录长度可知；检索速度慢，不便于处理修改；但适合某些场合需要，广泛用于商业领域 |
  | 无结构文件（流式文件） | — | 文件长度以字节为单位；利用读写指针指出下一个要访问的字符；可看作记录式文件的特例（一个记录仅一个字节）；大量源程序、可执行文件、库函数采用此形式 |

- **按文件组织方式分类**：
  - **顺序文件**：由一系列记录按某种顺序排列所形成的文件，记录可以是定长或可变长记录。
  - **索引文件**：为可变长记录文件建立一张索引表，为每个记录设置一个表项，以加速对记录的检索速度。
  - **索引顺序文件**：顺序文件和索引文件相结合的产物，为一组记录中的第一个记录建立一个索引表项（而非每一个记录）。

### 3. 文件的物理结构 ⭐(练习2,21)

- **定义**：文件物理结构是指文件在存储介质上的组织方式，它依赖于物理的存储设备、物理存储空间，可以看作是相关物理块的集合，又称物理文件。

- **顺序结构**：把一个逻辑上顺序的记录构成的文件分配到顺序的物理块中。
  - 优点：管理简单，存储速度快。
  - 缺点：空间利用率低，文件记录插入或删除操作不方便，只能在文件末尾进行。

- **链接结构**：把文件信息存放在非连续的物理块中，每个物理块设有一个指针指向其后续的另一个物理块，使存放同一文件的物理块链接成一个串联队列。链接方式分**显式链接**（指针在专门的链接表中）和**隐式链接**（指针在存放文件信息的物理块中）。
  - 优点：空间利用率高，易于文件扩充。
  - 缺点：对文件的搜索率较低。

- **索引结构**：为每个文件建立一个索引表，每一个表项指出文件记录所在的物理块号，表项按逻辑记录编号，按顺序或按记录内某一关键字顺序排列。对于大文件，可建立多级索引，还可把文件索引表也作为一个文件（索引表文件）。
  - 优点：满足文件动态增长的要求且存取方便。
  - 缺点：建立索引表增加了文件存储空间的开销，对于多级索引，访问时间开销较大。

### 4. 目录管理 ⭐⚠(练习3,4,5,6,8,12,15,16,17,18,25,26,29,35,36,38)

#### 4.1 目录管理的要求

1. **实现"按名存取"**：用户只须提供文件名字，便能快速准确地找到指定文件在外存上的存储位置。这是目录管理中最基本的功能，也是文件系统向用户提供的最基本的服务。
2. **提高对目录的检索速度**：通过合理地组织目录结构加快检索速度，从而提高对文件的存取速度。这是设计大、中型文件系统时所追求的主要目标。
3. **文件共享**：允许多个用户共享一个文件，只须在外存中保留一份副本，节省存储空间。
4. **允许文件重名**：允许不同用户对不同文件采用相同的名字。

#### 4.2 文件控制块（FCB）

文件控制块通常含三类信息：

| 类别 | 包含内容 |
|---|---|
| 基本信息 | 文件名（唯一标识）、文件物理位置（设备名、起始盘块号、盘块数/字节数）、文件逻辑结构（流式/记录式、记录数、定长/变长）、文件物理结构（顺序/链接/索引） |
| 存取控制信息 | 描述文件的存取权限，用于核准用户的存取权限 |
| 使用信息 | 建立日期和时间、上一次修改日期和时间、当前使用信息（当前已打开该文件的进程数、是否被其它进程锁住、是否在内存中已被修改但尚未拷贝到盘上等） |

#### 4.3 索引结点

- **引入原因**：文件目录存放在磁盘上，文件很多时目录占用大量盘块。查找目录时需将目录盘块逐一调入内存比较文件名。采用把**文件名与文件描述信息分开**的办法，使文件描述信息单独形成一个称为**索引结点**的数据结构（简称 **i 结点**）。目录中每个目录项仅由**文件名**和**指向该文件所对应的 i 结点的指针**构成。

- **内存索引结点**：文件被打开时，将磁盘索引结点拷贝到内存。增加以下内容：
  - 索引结点编号（标识内存索引结点）
  - 状态（指示 i 结点是否上锁或被修改）
  - 访问计数（每当有进程访问此 i 结点时加 1，访问完减 1）
  - 文件所属文件系统的逻辑设备号
  - 链接指针（分别指向空闲链表和散列队列的指针）

- **磁盘索引结点**：每个文件有唯一的一个磁盘索引结点，主要包括：
  - 文件主标识符（拥有该文件的个人或小组的标识符）
  - 文件类型（正规文件、目录文件或特别文件）
  - 文件存取权限（各类用户对该文件的存取权限）
  - 文件物理地址（每个索引结点中含有 **13 个地址项**，即 `iaddr(0)`~`iaddr(12)`，以直接或间接方式给出数据文件所在盘块的编号）
  - 文件长度（以字节为单位）
  - 文件连接计数（本文件系统中所有指向该文件的文件名的指针计数）
  - 文件存取时间（最近被进程存取的时间、最近被修改的时间及索引结点最近被修改的时间）

#### 4.4 简单文件目录

- **单级文件目录**：整个文件系统中只建立一张目录表，每个文件占一个目录项（含文件名、扩展名、文件长度、文件类型、文件物理地址及其它文件属性），设置状态位表明目录项是否空闲。
- **两级文件目录**：为每个用户建立一个单独的**用户文件目录 UFD**（由用户所有文件的 FCB 组成），系统中再建立一个**主文件目录 MFD**（每个用户目录文件占一个目录项，包括用户名和指向该用户目录文件的指针）。

#### 4.5 树形结构目录

在现代 OS 中最通用且实用。主目录称为**根目录**，每个文件系统中只能有一个根目录，每个文件和每个目录都只能有一个父目录。数据文件称为**树叶**，其它目录作为树的**结点**（子目录）。明显提高对目录的检索速度和文件系统的性能。

#### 4.6 目录查询技术

- **线性检索法（顺序检索法）**：在单级目录中，利用用户提供的文件名，用顺序查找法直接从文件目录中找到指名文件的目录项。在树形目录中，用户提供的文件名是由多个文件分量名组成的路径名，需对多级目录进行查找。
- **Hash 方法**：建立 Hash 索引文件目录，利用用户提供的文件名将其变换为文件目录的索引值，再利用该索引值到目录中查找，显著提高检索速度。
  - **冲突处理规则**：
    - (a) 如果目录表中相应的目录项是空的，则表示系统中并无指定文件。
    - (b) 如果目录项中的文件名与指定文件名匹配，则该目录项即为所要寻找的，可从中找到文件的物理地址。
    - (c) 如果目录项中的文件名与指定文件名不匹配，则发生"冲突"，须将 Hash 值再加上一个常数（该常数应与目录的长度值互质），形成新的索引值，再返回第一步重新查找。

### 5. 文件存储空间的管理 ⭐(练习10,20,24)

文件存储设备分成许多大小相同的物理块，以块为单位交换信息。文件存储设备的管理实质上是对**空闲块**的组织和管理，包括空闲块的组织、分配与回收等问题。

#### 5.1 空闲表法

属于**连续分配方式**，与内存的动态分配方式雷同。为外存上所有空闲区建立一张空闲表，每个空闲区对应一个空闲表项（含表项序号、该空闲区第一个盘块号、该区的空闲盘块数），按起始盘块号递增排列。

#### 5.2 空闲链表法

将所有空闲盘区拉成一条空闲链，分两种形式：

| 类型 | 说明 | 优点 | 缺点 |
|---|---|---|---|
| 空闲盘块链 | 以盘块为单位拉成一条链，每个盘块有指向后继盘块的指针 | 分配和回收一个盘块的过程非常简单 | 为文件分配盘块可能要重复操作多次，效率较低；空闲盘块链很长 |
| 空闲盘区链 | 以盘区（每个盘区可含若干盘块）为单位拉成一条链，每个盘区含指向下一个空闲盘区的指针和本盘区大小信息 | 分配和回收效率可能较高，每次为文件分配多个连续的块，空闲盘区链较短 | 分配与回收过程比较复杂（通常采用首次适应算法，回收时需与相邻空闲盘区合并） |

#### 5.3 位示图法 ⭐(练习10)

- **位示图**：利用二进制的一位表示磁盘中一个盘块的使用情况。值为"0"表示空闲，为"1"表示已分配（有的系统相反，本质相同）。由所有盘块对应的位构成一个集合，通常用 $m \times n$ 位数构成，使 $m \times n$ 等于磁盘总块数。可描述为二维数组 `map[m,n]`。

- **盘块的分配**（三步）：
  1. 顺序扫描位示图，找出一个或一组其值为"0"的二进制位。
  2. 将找到的二进制位转换成相应的盘块号。假定找到的其值为"0"的二进制位位于位示图的第 $i$ 行、第 $j$ 列，则相应盘块号为：

     $$b = n(i - 1) + j$$

     式中，$n$ 代表每行的位数。
  3. 修改位示图，令 `map[i,j] = 1`。

- **盘块的回收**（两步）：
  1. 将回收盘块的盘块号转换成位示图中的行号和列号：

     $$i = (b - 1) / n + 1$$

     $$j = (b - 1) \% n + 1$$

  2. 修改位示图，令 `map[i,j] = 0`。

- **优点**：很容易找到一个或一组相邻接的空闲盘块；位示图很小占用空间少，可保存在内存中，每次分配时无需先把盘区分配表读入内存，节省磁盘启动[原文模糊，疑为"启停"]。常用于微型机和小型机中（如 CP/M、Apple-DOS 等）。

#### 5.4 成组链接法

空闲表法和空闲链表法都不适用于大型文件系统（空闲表或链表太大）。**UNIX 系统**采用成组链接法，结合空闲表和空闲链表两种方法。大致思想：把顺序的 $n$ 个空闲扇区地址保存在第一个空闲扇区内，其后一个空闲扇区内保存另一顺序空闲扇区的地址，如此继续，直至所有空闲扇区均予以链接。系统只需要保存一个指向第一个空闲扇区的指针。

### 6. 文件的共享和保护 ⭐⚠(练习11,19,22,30,37)

#### 6.1 文件的共享

| 共享方式 | 别名 | 机制 | 特点 |
|---|---|---|---|
| 基于有向无循环图实现文件共享 | 硬链接 | 共享文件或子目录链接到两个或多个用户的目录中；文件物理地址及其他文件属性放在索引结点中，目录中只设置文件名及指向相应索引结点的指针 | 多个目录项指向同一索引结点；文件主删除文件后，其他用户的指针会悬空 |
| 利用符号链实现文件共享 | 软链接 | 由系统创建一个 LINK 类型的新文件（取同名），写入用户目录中；新文件中只包含被链接文件的路径名 | 只有文件拥有者拥有指向索引结点的指针；共享者只有路径名；文件主删除文件后，其他用户通过符号链访问会失败，将符号链删除即可，不会产生悬空指针 |

#### 6.2 文件的保护

- **访问类型**（可加以控制的访问类型）：
  - 读：从文件中读。
  - 写：向文件中写。
  - 执行：将文件装入内存并执行。
  - 添加：将新信息添加到文件结尾部分。
  - 删除：删除文件，释放空间。
  - 列表清单：列出文件名和文件属性。
  - 此外还可以对重命名、复制等加以控制。保护可以只在低层提供（如复制可利用一系列读请求完成，具有读权限的用户同时具有复制和打印权限）。

- **访问控制**：
  - 最常用方法：根据用户身份进行控制。
  - 实现方法：为每个文件和目录增加一个**访问控制列表（ACL）**，规定每个用户名及其所允许的访问类型。
    - 优点：可以使用复杂的访问方法。
    - 缺点：长度无法预计且可能导致复杂的空间管理。
  - **精简的访问列表**：采用**拥有者、组和其他**三种用户类型，只需三个域列出访问权限。文件拥有者创建文件时说明创建者用户名及所在组名，系统将文件主名字、所属组名列在该文件的 FCB 中。用户访问时：若为拥有者按拥有者权限；若同组按同组权限；否则按其他用户权限。**UNIX 操作系统即采用此种方法**。
  - **口令和密码**是另外两种访问控制方法。

### 7. 磁盘组织与管理 ⭐(练习7,40)

#### 7.1 磁盘的结构

- **数据的组织和格式**：磁盘设备可包括一个或多个物理盘片，每个磁盘片分一个或两个存储面，每个盘面上有若干个磁道，磁道之间留有必要的间隙。为使处理简单，每条磁道上可存储相同数目的二进制位。

- **磁盘访问时间**：磁盘以恒定速率旋转。为读或写，磁头须移动到指定磁道上，等待指定扇区的开始位置旋转到磁头下，然后开始读或写数据。

  - **(a) 寻道时间 $T_s$**：磁臂（磁头）移动到指定磁道上所经历的时间，是启动磁臂的时间 $s$ 与磁头移动 $n$ 条磁道所费时间之和：

    $$T_s = m \times n + s$$

    其中，$m$ 是一常数，与磁盘驱动器的速度有关。

  - **(b) 旋转延迟时间 $T_r$**：指定扇区移动到磁头下面所经历的时间。如硬盘旋转速度为 15000 r/min，每转需时 4 ms，平均旋转延迟时间 $T_r$ 为 2 ms（最小为 0，最大为 4，平均为 2）。

  - **(c) 传输时间 $T_t$**：把数据从磁盘读出或向磁盘写入数据所经历的时间：

    $$T_t = \frac{b}{rN}$$

    其中，$r$ 为磁盘每秒钟的转数；$N$ 为一条磁道上的字节数。

#### 7.2 磁盘调度算法

| 算法 | 全称 | 机制 | 特点 |
|---|---|---|---|
| FCFS | 先来先服务 | 根据进程请求访问磁盘的先后次序进行调度 | 最简单的磁盘调度算法 |
| SSTF | 最短寻道时间优先 | 选择要求访问的磁道与当前磁头所在磁道距离最近的进程 | 每次寻道时间最短，但不能保证平均寻道时间最短；可能导致低优先级进程"饥饿" |
| SCAN | 扫描算法 | 对 SSTF 略加修改，防止低优先级进程出现"饥饿"现象 | 既能获得较好的寻道性能，又能防止"饥饿"现象 |
| CSCAN | 循环扫描算法 | 对 SCAN 的改进 | 解决 SCAN 中磁头刚越过某磁道时恰好有请求该磁道的进程需等待的问题 |

磁盘的初始化，低级格式化和逻辑格式化，引导块和坏块。

### 4.4 硬链接 vs 软链接

| 特性 | 硬链接（基于索引结点） | 软链接（符号链） |
|---|---|---|
| 实现方式 | 多个目录项指向同一索引结点 | 创建 LINK 类型新文件，只含被链接文件的路径名 |

## 五、高频考点与易错提醒

- ⭐⚠ **索引文件结构最大文件计算**是高频易错点：二级索引最大文件 = (块大小/地址大小)² × 块大小 (练习2，反推)。
- ⭐ **多级目录结构的目的**是解决命名冲突（允许不同用户使用相同文件名）(练习4)。
- ⭐ **文件目录的主要作用**是实现"按名存取"(练习6)。
- ⚠ **位示图实现的是磁盘空间管理**，不是文件目录查找或按名存取；按名存取靠文件目录实现(练习10,38)。
- ⭐⚠ **硬链接与软链接的引用计数变化**是高频易错点：硬链接影响引用计数，软链接（符号链接）的引用计数始终为 1，不受被链接文件影响(练习11)。
- ⚠ **文件被打开后使用文件描述符**访问，不再使用文件名(练习12)。
- ⚠ **删除文件不删除文件所在的目录**，只删除关联目录项和 FCB，释放关联缓冲区(练习13)。
- ⭐ **按逻辑结构划分文件**为记录式文件和流式文件(练习14)。
- ⭐ **设置当前工作目录的主要目的**是加快文件的检索速度(练习15)。
- ⚠ **open() 调用**：参数含文件名；系统打开文件表只在文件实体第一次被打开时增加表目；不同文件名可能互为硬链接指向同一文件实体(练习16)。
- ⭐⚠ **内存索引结点有而磁盘索引结点没有的内容**是访问计数值(练习17)。
- ⭐ **文件描述信息从目录项中分离的好处**是减少查找文件时的 I/O 信息量(练习18)。
- ⚠ **源程序是无结构文件（流式文件）**，不是有结构的记录式文件(练习31)。
- ⚠ **关闭文件**是将文件控制信息写回磁盘，不是将文件数据写回磁盘（写文件操作才会写回数据）(练习32)。
- ⚠ **逻辑结构取决于用户需求，物理结构取决于文件系统设计者针对硬件的措施**(练习34)。
- ⚠ **单级文件目录不能解决重名问题**，多级目录才能(练习35)。
- ⚠ **基于索引结点的共享方式中，文件主不能随意删除共享文件**，否则其他用户指针悬空(练习37)。
- ⚠ **单个文件大小不仅受磁盘剩余空间限制，还受 FCB 和 FAT 表等结构的限制**(练习39)。
- ⚠ **文件保护方法**包括口令、存取控制矩阵、用户权限表；FCB 本身不具备保护能力(练习30)。
- ⚠ **文件存储结构不必连续存放的**是链接结构和索引结构；顺序结构要求连续(练习21)。
- ⚠ **文件在磁带上采用连续存放，在硬盘上通常不采用连续存放，在内存上采用随机存放**(练习28)。

## 六、一句话记忆要点

1. **文件系统**：数据组织分三级——数据项（原子）、记录、文件；文件六种基本操作（创建/写/读/重定位/删除/截断）；四种分类维度（用途/数据形式/存取控制/组织形式）。
2. **逻辑结构**：有结构（记录式：定长/变长）vs 无结构（流式：字节为单位）；组织方式三种——顺序/索引/索引顺序。
3. **物理结构**：顺序（快但浪费）、链接（省且易扩充但搜索慢）、索引（动态增长方便但开销大）。
4. **目录管理**：四大要求（按名存取/提高检索速度/文件共享/允许重名）；FCB 含基本/存取控制/使用三类信息；索引结点将文件名与描述信息分离；单级→两级（MFD+UFD）→树形目录；查询用线性检索或 Hash 法。
5. **存储空间管理**：空闲表（连续分配）、空闲链表（盘块链/盘区链）、位示图（$b=n(i-1)+j$，$i=(b-1)/n+1$，$j=(b-1)\%n+1$）、成组链接法（UNIX）。
6. **文件共享与保护**：硬链接（索引结点，引用计数变化，删除后悬空）vs 软链接（符号链，路径名，删除后安全）；保护靠访问类型控制 + ACL/精简访问列表（拥有者/组/其他，UNIX 采用）+ 口令密码。
7. **磁盘管理**：访问时间 = 寻道时间 $T_s=mn+s$ + 旋转延迟 $T_r$ + 传输时间 $T_t=b/(rN)$；调度算法 FCFS→SSTF（可能饥饿）→SCAN（防饥饿）→CSCAN（改进等待问题）。
