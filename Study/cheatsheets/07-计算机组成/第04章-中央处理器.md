## 一、本章概览

本章围绕中央处理器（CPU）的组织结构展开，涵盖六大板块：CPU 的功能与组成、寄存器组织、控制器组织（含控制方式与时序系统）、时序产生器组织、指令流水、RISC 硬件结构。核心线索是：CPU 由运算器、控制器和寄存器三大部分组成，控制器通过产生时序控制信号来协调指令的取指—译码—执行—写回各阶段；不同控制器（硬布线 vs 微程序）采用不同时序体制；指令流水通过时间并行提高执行速度，但需解决三类相关冲突。本章与前三章（运算方法、指令系统、存储器）紧密衔接，是计算机组成原理的核心章节。

## 二、知识点详解

### 第一节 CPU 组织

#### 知识点 1 CPU 的功能 ⭐（练习单选1）

- **定义**：CPU 的基本功能包括：
  - **指令控制**
  - **操作控制**
  - **时间控制**
  - **数据加工**

#### 知识点 2 CPU 的组成 ⭐（练习单选1、12、13）

- **定义**：传统的 CPU 由**运算器**、**控制单元**和**寄存器**三大部分组成。
- **要点**：随着高密度集成电路技术的发展，早期放在 CPU 芯片外部的一些逻辑功能部件，如浮点运算器、Cache 等，现已集成到 CPU 内部。
- **一个简单 CPU 的具体组成部分**：ALU、操作控制器、时序产生器、指令译码器、控制和状态寄存器以及通用寄存器。

### 第二节 寄存器组织

#### 知识点 1 寄存器的分类 ⭐⚠（练习单选14、15）

- **分类**：CPU 中的寄存器按功能可分为如下两类：
  - **用户可见寄存器**：允许机器语言或汇编语言的编程人员使用。
  - **控制和状态寄存器**：用来控制 CPU 的操作并被特权的操作系统程序用于控制程序的执行。（原文 OCR 识别为"GPU"，应为"CPU" `[原文疑为OCR误识]`）

- **用户可见寄存器可分类如下**：
  - **通用寄存器**：可被程序员使用。
  - **数据寄存器**：用于保持数据。
  - **地址寄存器**：用于寻址方式中，如 SI、DI 等。
  - **条件代码寄存器**：部分可供用户使用，如溢出标志、中断屏蔽等。

#### 知识点 2 几种重要的寄存器 ⭐⚠（练习单选2、9、10、11、多选4、7、判断2）

- **4 种至关重要的寄存器**（对于指令执行）：

| 寄存器 | 缩写 | 功能 |
|---|---|---|
| 程序计数器 | PC | 其内容为主存地址，指向下一条要执行的指令 |
| 指令寄存器 | IR | 存放当前正在执行的指令 |
| 存储地址寄存器 | MAR | 存放待访问的存储器地址 |
| 存储缓冲寄存器 | MBR | 存放写入存储器的数据字或最近读出的字 |

- **程序状态字（PSW）**：一般含有条件代码加上其他状态信息。普遍包括如下字段或标志：
  - **符号**：容纳最后算术运算结果的符号位。
  - **零**：当结果是零时被置位。
  - **进位**：若操作导致最高位有向上的进位（加法）或借位（减法）时被置位。
  - **等于**：若逻辑比较的结果是相等，则置位。
  - **溢出**：用于指示算术溢出。
  - **中断允许/禁止**：用于允许或禁止中断。
  - **监督**：指出 CPU 是执行监督模式还是在用户模式。

### 第三节 控制器组织

#### 知识点 1 控制器的功能 ⭐（练习单选3、6、8、判断4）

- **数据通路**：CPU 中各个部件之间传送信息的通路称为数据通路。
- **控制器的功能**：对数据通路实施控制，以便信息能够正确传输，从而完成指令的执行。

#### 知识点 2 控制器的分类 ⭐⚠（练习多选2）

根据设计方法不同，操作控制器可分为：

| 类型 | 别名 | 实现方式 |
|---|---|---|
| 时序逻辑型 | 硬布线控制器 | 采用时序逻辑技术实现 |
| 存储逻辑型 | 微程序控制器 | 采用存储逻辑实现 |
| 时序逻辑与存储逻辑结合型 | — | 前两种方式的组合 |

#### 知识点 3 基本概念 ⭐⚠（练习单选4、7、多选1、8、判断5、6）

- **指令周期**：CPU 每次取出并执行一条指令，都要完成系列的操作，这一系列操作所需的时间通常叫做一个指令周期。更简单地说，指令周期就是取出并执行一条指令的时间。一般来讲，指令周期是随指令不同而变化的，**不是一个固定值**。

- **CPU 周期**（也称为**机器周期**）：根据指令执行的基本过程划分。指令周期通常包含若干个 CPU 周期。CPU 周期的长度在有些计算机中是固定的（如等于主存的存取周期），而在有些计算机中是可变的。

- **节拍周期**：完成 CPU 内部一些最基本操作所需要的时间，如数据从一个寄存器可靠传送到另外一个寄存器的时间或 ALU 完成一次算逻运算的时间。节拍周期的长度在任何机器中都是**固定的**。

- **微周期**：执行一条微指令所需要的时间叫做一个微周期。

- **节拍脉冲**：常作为触发器的打入脉冲与节拍周期相配合完成一次数据传送。
  - 在有些计算机中，节拍脉冲与节拍周期一一对应，节拍脉冲的频率就是脉冲源的频率，即**机器主频**；
  - 但有些计算机的节拍周期包含若干个节拍脉冲，这样一个节拍周期中实现的操作也就多一些。

#### 知识点 4 硬布线控制器 ⭐（练习多选3）

硬布线控制器与微程序控制器的时序系统不同，具体表现为：

| 控制器类型 | 指令周期的组成 |
|---|---|
| 硬布线控制器 | 指令周期 → CPU 周期 → 节拍周期 → 节拍脉冲 |
| 微程序控制器 | 指令周期 → 微周期 → 节拍脉冲 |

- **现代计算机的硬布线控制器时序系统**多为：CPU 周期、节拍周期（复杂指令集）**两级**。精简指令集一般就**一级**。
- **微程序控制器的时序系统**多为：微周期**一级**。

#### 知识点 5 控制方式 ⭐⚠（练习单选16、17、19、20、多选6）

控制方式即产生微操作命令序列所用的时序控制方式。

**（一）同步方式**

按照定宽定距的时钟/节拍信号控制下，产生控制信号。同步方式有三种机器周期的模式：

1. **定长机器周期**
   - 按照最长的微操作序列（通常是访存操作相关）所需的时钟/节拍次数来确定所有机器周期的长度。
   - 保证了充足的节拍数量，但操作简单的子阶段（如取反指令执行周期只需一个取反信号）会浪费多出的时钟周期。

2. **不定长机器周期**
   - 允许不同的子阶段/机器周期内包含不同的节拍数量。
   - 一般有最低节拍数量标准，如果操作复杂可在此基础上延长相应的节拍数。

3. **中央控制与局部控制相结合**
   - 针对访存类操作反馈时间不确定、导致执行周期时间较长的问题。
   - 在执行这类操作时转为局部节拍控制，直到某个时机重新接入中央节拍。
   - 要求：局部控制节拍宽度与中央控制节拍宽度一致。

**（二）异步控制方式**

- 无基准时标，采用**请求—应答/握手**方式进行控制。
- 针对一些操作完成/反馈时间不确定的情况，异步控制实现较同步控制实现更为简单，但异步控制较为松散不利于效率的提高。

**（三）联合控制方式**

- 对于大部分操作时间/反馈时间较为固定的微操作，在机器周期内采用时钟频率调制，**同步控制**；
- 对于部分操作/反馈时间不确定（通常是访存类）的微操作，机器周期内采用**异步控制**的方式。

**（四）人工控制方式**

- 通常用于程序调试，例如清 0 重置、单步运行、终止停机。

### 第四节 时序产生器组织

#### 知识点 1 时序产生器的组织构成 ⭐（练习多选3、判断7）

- **时间标志**：计算机的协调动作需要时间标志，而时间标志则是用时序信号来体现的。
- **控制信号的本质**：控制器发出的各种控制信号都是**时间因素**（时序信号）和**空间因素**（部件位置）的函数。
- **最基本的体制**：组成计算机硬件的器件特性决定了时序信号最基本的体制是**电位—脉冲制**。
- **硬布线控制器**中，时序信号往往采用**主状态周期—时钟周期**两级体制。
- **微程序控制器**中，时序信号比较简单，一般采用**时钟周期**一级体制。
- **微程序控制器中使用的时序信号产生器**由以下部分组成：
  - 时钟源
  - 环形脉冲发生器
  - 节拍脉冲和读写时序译码逻辑
  - 启停控制逻辑

### 第五节 指令流水

#### 知识点 1 指令并行 ⭐（练习单选22、多选9、10）

- **传统冯·诺依曼计算机**的特点是：指令的**串行执行**。为了提高计算机的速度，现代计算机往往采用并行处理技术，使得多种操作能够同时进行。

- **并行性的含义**：
  - **同时性**：两个以上事件在**同一时刻**发生；
  - **并发性**：两个以上事件在**同一时间间隔内**发生。

- **并行处理技术的三种形式**：
  - **时间并行**：时间重叠。多个处理过程在时间上相互错开，轮流重叠地使用同一套硬件设备的各个部分，以加快硬件周转而赢得速度。
  - **空间并行**：资源重复。
  - **时间并行＋空间并行**。

#### 知识点 2 常用术语

指令流水类似于工厂中装配线的使用。流水中常用术语：

- **加速比**：使用流水线后指令执行的速度比串行执行快多少。
- **效率**：流水线各功能部件的利用率。

#### 知识点 3 流水线实现的关键 ⭐⚠（练习多选9）

- **实现流水线的关键**：
  - 若想使流水线正常工作，必须保证**各功能段的执行时间要几乎相等**；
  - 若想使整个处理速度加快，应设置**更多的处理阶段**。

- **"断流"**会导致流水线处理速度下降。

- **流水过程中会出现以下三种相关冲突，导致流水线断流**：
  - **资源相关冲突**
  - **数据相关冲突**
  - **控制相关冲突**

### 第六节 RISC 的硬件结构

#### 知识点 1 RISC 特点 ⭐（练习多选10）

大多数（不是所有）RISC 设计都采用的关键点：

- 一个**有限的和简单的指令集**；
- **大量的通用寄存器**或使用编译器技术来优化寄存器的使用；
- 强调**指令流水的优化**。

## 三、公式 / 算法 / 定理速查表

本章以概念定义为主，无数学公式。以下为本章关键概念/定义速查：

| 概念 | 定义/要点 |
|---|---|
| 指令周期 | CPU 取出并执行一条指令所需的时间；随指令不同而变化，不是固定值 |
| CPU 周期（机器周期） | 根据指令执行基本过程划分；指令周期包含若干个 CPU 周期；长度可固定可变 |
| 节拍周期 | 完成 CPU 内部最基本操作所需时间；长度在任何机器中固定 |
| 微周期 | 执行一条微指令所需的时间 |
| 节拍脉冲 | 作为触发器打入脉冲与节拍周期配合完成数据传送；频率=机器主频（当与节拍周期一一对应时） |
| 数据通路 | CPU 中各个部件之间传送信息的通路 |
| 电位—脉冲制 | 时序信号最基本的体制（由硬件器件特性决定） |
| 加速比 | 使用流水线后指令执行的速度比串行执行快多少 |
| 效率（流水线） | 流水线各功能部件的利用率 |
| 同时性 | 两个以上事件在同一时刻发生 |
| 并发性 | 两个以上事件在同一时间间隔内发生 |

## 四、重点对比与易混辨析

### 1. 硬布线控制器 vs 微程序控制器

| 对比项 | 硬布线控制器 | 微程序控制器 |
|---|---|---|
| 设计类型 | 时序逻辑型 | 存储逻辑型 |
| 实现方式 | 时序逻辑技术 | 存储逻辑 |
| 时序系统组成 | 指令周期→CPU周期→节拍周期→节拍脉冲 | 指令周期→微周期→节拍脉冲 |
| 现代时序体制 | CPU周期、节拍周期两级（CISC）；一级（RISC） | 微周期一级 |
| 时序信号产生器体制 | 主状态周期—时钟周期两级 | 时钟周期一级 |

### 2. 同步控制方式的三种模式对比

| 模式 | 特点 | 缺点 |
|---|---|---|
| 定长机器周期 | 按最长微操作序列确定所有机器周期长度 | 简单操作子阶段浪费时钟周期 |
| 不定长机器周期 | 不同子阶段可含不同节拍数，有最低标准可延长 | [原文未述] |
| 中央控制与局部控制结合 | 访存类操作转局部节拍控制，完成后回接中央节拍 | 要求局部节拍宽度与中央一致 |

### 3. 三种控制方式对比

| 控制方式 | 基准时标 | 特点 |
|---|---|---|
| 同步控制 | 有统一基准时标 | 定宽定距的时钟/节拍信号控制 |
| 异步控制 | 无基准时标 | 请求—应答/握手方式；较松散 |
| 联合控制 | 部分有、部分无 | 大部分操作同步，少数不确定操作异步 |
| 人工控制 | — | 用于程序调试（清0重置、单步运行、终止停机） |

### 4. 四种关键寄存器对比

| 寄存器 | 缩写 | 存放内容 | 要点 |
|---|---|---|---|
| 程序计数器 | PC | 主存地址 | 指向下一条要执行的指令 |
| 指令寄存器 | IR | 指令 | 存放当前正在执行的指令 |
| 存储地址寄存器 | MAR | 存储器地址 | 存放待访问的存储器地址 |
| 存储缓冲寄存器 | MBR | 数据字 | 存放写入或最近读出的数据字 |

### 5. 并行性两种含义对比

| 含义 | 定义 |
|---|---|
| 同时性 | 两个以上事件在**同一时刻**发生 |
| 并发性 | 两个以上事件在**同一时间间隔内**发生 |

## 五、高频考点与易错提醒

> 以下考点信号来自专项练习与答案解析，仅指出方向，不收录题目与解答。

1. ⚠ **CPU 的组成**是高频易错点：传统 CPU = 运算器 + 控制单元 + 寄存器（练习单选1）。注意"地址译码器"不属于 CPU，它属于存储器（练习单选12）。
2. ⚠ **PC 的位数取决于存储器容量**，而非机器字长或指令字长（练习单选9，反推）。
3. ⚠ **IR 的位数取决于指令字长**（练习单选10，反推）。
4. ⚠ **通用寄存器的位数取决于机器字长**（练习单选11，反推）。
5. ⚠ **IR 对用户完全透明**（用户程序不能读写）；PC 和状态寄存器虽不可写但可读，不完全透明；通用寄存器对用户不透明（练习单选15）。
6. ⚠ **取指令操作是控制器固有功能**，无需操作码控制即可完成——因为取了指令才有操作码（练习单选8）。
7. ⚠ **指令周期 > 机器周期**：一条指令的执行分为取指令、译码、执行和写结果等阶段，每个阶段需要一个机器周期（练习单选7）。
8. ⚠ **微程序控制器的速度通常较慢**（不是较快），硬布线控制器速度更快；微程序控制器优点是结构规整、复杂性低、增加/修改指令容易（练习多选2）。
9. ⚠ **转移指令时 PC 的值不一定总是修改为转移目标地址**：条件转移需根据标志位确定（练习多选7）。
10. ⚠ **不是所有指令的执行子周期一样长**：不同指令操作复杂度不同（练习多选8）。
11. ⚠ **流水线三种相关冲突**（资源相关、数据相关、控制相关）是高频考点（练习多选9）。
12. ⚠ **垂直型微指令**用较长的微程序结构换取较短的微指令结构（练习判断10）。
13. ⚠ **产生后继微地址的方法**主要有计数器方式、断定方式（练习判断9，此内容知识点正文未展开）。
14. ⚠ **CPU 中的译码器主要用于指令译码**，地址译码是存储器的功能（练习单选13）。
15. ⚠ **通用寄存器既能存放数据又能存放地址**，但不能代替指令寄存器（练习单选14）。
16. ⚠ **时钟频率是操作信号发出的基准**，不是用来"记忆时间"的（练习判断7）。
17. ⚠ **同步控制中机器周期内时钟周期个数可以不同**（定长/不定长/中央局部结合均属同步）；联合控制也允许不同；异步控制一般不计算时钟周期个数（练习多选6）。

## 六、一句话记忆要点

1. **CPU 功能**：指令控制、操作控制、时间控制、数据加工——四控一加。
2. **CPU 组成**：运算器 + 控制单元 + 寄存器，三大件；地址译码器在存储器不在 CPU。
3. **四关键寄存器**：PC 指下一条、IR 存当前、MAR 放地址、MBR 放数据。
4. **PSW 七标志**：符号、零、进位、等于、溢出、中断允许/禁止、监督。
5. **控制器分类**：硬布线（时序逻辑）、微程序（存储逻辑）、混合型。
6. **周期层次**：指令周期 > CPU周期(机器周期) > 节拍周期；微程序用微周期替代 CPU周期+节拍周期。
7. **节拍周期固定、指令周期不固定、CPU周期可固定可变**。
8. **硬布线四级时序**：指令周期→CPU周期→节拍周期→节拍脉冲；**微程序三级**：指令周期→微周期→节拍脉冲。
9. **同步三模式**：定长（浪费）、不定长（可延长）、中央+局部结合（访存专用）。
10. **异步**：无基准时标，握手应答；**联合**：大部分同步 + 少数异步。
11. **时序最基本体制**：电位—脉冲制。
12. **并行性**：同时性=同一时刻，并发性=同一时间间隔。
13. **并行三形式**：时间并行（重叠）、空间并行（重复）、时间+空间并行。
14. **流水线关键**：各段执行时间几乎相等 + 更多处理阶段；断流由三种相关冲突引起。
15. **RISC 三特点**：简单指令集 + 大量通用寄存器 + 指令流水优化。
