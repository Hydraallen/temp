## 栈的定义与核心特性

**栈的定义**：栈是限制在表的一端进行插入、删除运算的线性表。允许插入、删除的这一端被称为栈顶，另一个固定端称为栈底。当表中没有元素时称为空栈。

理解栈的定义有以下要点：
1. **线性结构**：栈元素具有线性（即前驱后继）关系
2. **限制操作**：限制了线性表插入和删除的位置
3. **单向延伸性**：栈底是固定的，栈顶随着插入和删除操作的进行而变化
4. 插入也称为进栈、压栈、入栈，删除也称为出栈、弹栈
5. 栈是一种**后进先出（LIFO — Last In First Out）**的数据结构

**出栈序列的多样性**：栈只对线性表插入和删除的位置进行了限制，并没有限定插入和删除操作进行的时间。由于出栈可随时进行，只要某个元素位于栈顶就可以出栈，所以元素最终的出栈次序可能是多样的。

> Caveat (p025-T6): 判断出栈序列合法性时，不能只看局部后进先出。若 $a$ 先于 $b$ 入栈且均在栈内，则 $a$ 必须在 $b$ 之后出栈；若 $a$ 已出栈，则不受后续入栈元素限制。

> Caveat: 栈也可被称为"先进后出"，栈可以对输入序列部分或全局起求逆作用。

> Caveat: 栈限定在一端进行插入和删除操作，但并不一定在表尾，也可以限定在表头。

> Caveat: 堆栈适合解决处理顺序与输入顺序**相反**的问题（而非相同）。

**标准解题套路**（for 出栈序列合法性判定）
1. 给定入栈序列，模拟入栈/出栈过程。
2. 逐个检查目标出栈序列中的元素：若该元素尚未入栈，则将入栈序列中该元素之前的所有未入栈元素依次入栈，再将该元素出栈。
3. 若该元素已在栈中但不在栈顶，则该出栈序列非法（因为栈顶之上的元素必须先出栈）。
4. 若所有元素都能按此过程成功出栈，则序列合法。

## 栈的存储实现

### 顺序栈 ⭐

**顺序栈**：用顺序存储方式实现的栈。栈中的数据元素用一个预设足够大的一维数组来实现：`datatype data[MAXSIZE]`，栈底位置可以设置在数组的任一个端点，栈顶随插入和删除而变化，需要变量 `int top` 作为栈顶指针标示当前栈顶位置。

顺序栈的类型描述：
```c
#define MAXSIZE 1024 //设定栈的最大长度为 1024，可根据实际情况进行修改
typedef struct
{
    datatype data[MAXSIZE];
    int top;
}SeqStack /* [OCR 原文如此，末尾应有分号] */
```
定义一个指向顺序栈的指针：
```c
SeqStack *s;
```

> Caveat: 顺序存储结构会为栈分配一个固定大小的连续空间，入栈操作会引起空间的占用，需要考虑"溢出"情况。

### 链栈

**链栈**：用链式存储结构实现的栈。通常链栈用单链表表示，其结点结构与单链表的结构相同。

链栈的类型描述：
```c
typedef struct snode
{
    datatype data;
    struct snode *next;
}StackNode, * LinkStack;
```
定义 top 为栈顶指针变量：
```c
LinkStack top;
```

> Caveat: 采用链式存储结构存储栈时，系统会动态分配空间，除非系统内存不够，一般不需要考虑"溢出"情况。

**标准解题套路**（for 链栈插入结点操作）
1. 将新结点 `p` 的 next 指向当前栈顶：`p->next = top;`
2. 更新栈顶指针为新结点：`top = p;`
3. 注意顺序不可颠倒——必须先链接新结点到旧栈顶，再修改栈顶指针。

### 上溢与下溢 ⚠

**上溢**：栈满时再做进栈运算产生的空间溢出。

**下溢**：栈空时再做退栈运算产生的无元素可退。

### 两栈共享空间 ⭐

> Caveat: 两个栈共享一片连续内存空间 $S[1,m]$ 时，为提高内存利用率、减少溢出机会，应把两个栈的栈底分别设在这片内存空间的两端。栈 1 的底在 $S[1]$（`top[1]` 从低位向高位增长），栈 2 的底在 $S[m]$（`top[2]` 从高位向低位增长），栈满条件为 `top[1]+1 == top[2]`。

## 队列的定义与核心特性

**队列的定义**：队列是仅允许在表的一端进行插入运算，而在表的另一端进行删除运算的受限的线性表。允许插入的一端称为队尾，允许删除的一端称为队头。

理解队列的定义有以下要点：
1. **线性结构**：相邻元素具有前驱后继关系
2. **限制操作**：限制了线性表插入和删除的位置
3. 插入也称为入队、进队，删除也称为出队
4. 队列是一种**先进先出（FIFO — First In First Out）**的数据结构

> Caveat: 队列的"先进先出"指的是先入队的元素最先出队。队列并不能在两端的每一端都进行插入和删除操作（插入只在队尾，删除只在队头）。

## 队列的存储实现

### 顺序队列

**顺序队列**：顺序存储的队。因为队的队头和队尾都是活动的，因此除了队列的数据区外还有队头、队尾两个指针。

顺序队的类型定义：
```c
define  MAXSIZE  1024          //队列的最大容量 /* [OCR 原文如此，应为 #define] */
typedef  struct
{datatype  data[MAXSIZE];      //队员的存储空间
  int rear,front;              //队头队尾指针
}SeQueue;
```
定义一个指向队的指针变量：
```c
SeQueue  *q;
```
申请一个顺序队的存储空间：
```c
q= (SeQueue *)malloc(sizeof(SeQueue));
```
队列的数据区为 `q->data[0]` ～ `q->data[MAXSIZE-1]`，队头指针 `q->front`，队尾指针 `q->rear`。

> Caveat: 顺序存储结构会为队列分配一个固定大小的连续空间，入队操作会引起空间的占用，需要考虑"溢出"情况。

### 链队列

**链队列**：链式存储的队。用单链表来实现链队列，根据队的 FIFO 原则，为了操作上的方便，分别需要一个头指针和尾指针。

头指针 `front` 和尾指针 `rear` 封装在一个结构中，用指向头尾指针结点的变量 `q` 标示这个链队。

链队的描述如下：
```c
typedef struct node
{
    datatype  data;
    struct  node *next;
} QNode;          //链队结点的类型
typedef struct
{
    QNode *front,*rear;
}LQueue;          //将头尾指针封装在一起的链队
```
定义指向链队的指针：
```c
LQueue  *q;
```

> Caveat: 用单链表表示链式队列时，队尾最好设在链表的尾位置，最方便做入队操作。

> Caveat: 在用循环单链表表示的链式队列中，可以不设队头指针，仅在链尾设置队尾指针——因为找队头元素的操作可由队尾指针指向的最后一个结点的 next 域得到。

> Caveat: 用不带头结点的单链表存储队列时，进行删除运算时头、尾指针**可能都要修改**（当队列中只有一个元素时，删除后头、尾指针都要修改；多于一个元素时只修改头指针）。

> Caveat: 采用链式存储结构存储队列时，系统会动态分配空间，一般不需要考虑"溢出"情况。

**标准解题套路**（for 链队列入队操作）
1. 将新结点 `p` 链接到队尾：`r->next = p;`
2. 更新队尾指针为新结点：`r = p;`
3. 注意：入队操作只修改尾指针，不修改头指针。

### 假上溢 ⚠

**假上溢**：在顺序队列中由于队的头尾位置只增不减，导致队尾位置已到队列空间的上界不能再做入队操作，而随着出队操作的进行，队首位置增加，使队列空间尚有空闲单元，这种队列已满的假象被称为假上溢。

### 循环队列 ⭐

**循环队列**：将顺序队列的向量空间视为一个首尾相接的圆环进行运算的队列；克服了假上溢现象，充分利用了向量空间。

> Caveat: 引入循环队列的目的就是为了克服**假溢出问题**（即假上溢），而非真溢出问题。

## 栈与队列对比

| 对比项 | 栈 | 队列 |
|---|---|---|
| 逻辑结构 | 线性结构 | 线性结构 |
| 操作受限位置 | 仅在同一端（栈顶）插入和删除 | 一端（队尾）插入，另一端（队头）删除 |
| 操作特性 | 后进先出（LIFO） | 先进先出（FIFO） |
| 别称 | 先进后出 | [原文未述] |
| 插入操作名称 | 进栈/压栈/入栈 | 入队/进队 |
| 删除操作名称 | 出栈/弹栈 | 出队 |
| 固定端 | 栈底固定，栈顶变化 | 队头和队尾均活动 |
| 顺序存储溢出 | 需考虑上溢/下溢 | 需考虑假上溢 |
| 链式存储溢出 | 一般不需考虑 | 一般不需考虑 |
| 典型应用 | 函数调用/递归、括号匹配、撤销、表达式求值 | 打印机调度、缓冲区、多道程序设计、排队服务 |


> Caveat: 栈和队列都是**操作受限的线性结构**（限制存取点的线性结构），不是非线性结构。

## 栈与队列的应用场景 ⭐

**栈的典型应用**（来自专项练习考点信号）：
- 符号匹配 / 括号配对检查（含 HTML 标签匹配校验）
- 文本编辑器中的撤销（undo）序列
- 函数调用 / 递归过程（处理参数及返回地址）
- 表达式求值与后缀表达式转换

> Caveat: 客户在呼叫中心的等待时间**不是**栈的应用（应使用队列）。

**队列的典型应用**（来自专项练习考点信号）：
- 模拟现实世界中售票柜台服务（先来先服务）
- 多道程序设计
- 打印机任务调度 / 打印数据缓冲区（主机与打印机速度不匹配）
- 任何符合"先进先出"原则的排队场景

> Caveat: 实现函数调用需要使用**栈**，不是队列。

## 高频考点与易错提醒

1. **出栈序列合法性判定** ⭐⚠：给定入栈序列判断某出栈序列是否可能，是本章最高频题型。核心原则：若 $a$ 先于 $b$ 入栈且均在栈内，$a$ 必须在 $b$ 之后出栈。
2. **栈/队列的操作特性辨析** ⭐：LIFO vs FIFO 是必考概念，注意"先进后出"是栈的等价说法。
3. **链栈/链队列指针操作顺序** ⚠：必须"先链接后更新指针"，顺序不可颠倒。
4. **两栈共享空间** ⭐：栈满条件 `top[1]+1 == top[2]`，栈底分别设在数组两端。
5. **循环队列的目的** ⭐：克服假溢出（假上溢），不是真溢出。
6. **顺序存储 vs 链式存储的溢出** ⚠：顺序存储需考虑溢出，链式存储一般不需要。
7. **栈和队列的本质** ⭐：都是操作受限的线性结构（限制存取点），不是非线性结构。
8. **栈的应用场景识别** ⭐：括号匹配、函数调用、撤销、表达式求值用栈；排队、缓冲区、调度用队列。

## 一句话记忆要点

- **栈**：一端进出，后进先出（LIFO），底固定顶变化。
- **队列**：一端进一端出，先进先出（FIFO），头尾都活动。
- **顺序栈**：数组 + `top` 指针，满则上溢、空则下溢。
- **链栈**：单链表 + `top` 指针，动态分配一般不溢出。
- **链队列**：单链表 + 封装的 `front`/`rear`，入队改尾、出队改头（仅一个元素时两头都改）。
- **假上溢**：顺序队列头尾只增不减，有空位却报满。
- **循环队列**：首尾相接的圆环，专治假上溢。
- **两栈共享**：底在两端，相向生长，相邻即满。

## Worked Examples

### 例 1：出栈序列合法性判定 *(来自 第三章·专项练习·单选 6)*

*Tags: 栈, LIFO, 出栈序列*

**问题**：一个栈的入栈序列是 a, b, c, d, e，判断输出序列 dceab 是否可能。

**解答**：
1. d 第一个出栈 → a, b, c, d 均已入栈，栈内（底→顶）为 a, b, c, d。
2. d 出栈后栈内为 a, b, c。c 出栈 → 栈内为 a, b。符合。
3. e 出栈 → e 需入栈。e 入栈后栈内为 a, b, e。e 出栈 → 栈内为 a, b。符合。
4. 下一个应出栈 a，但栈内为 a, b（b 在 a 上方），根据 LIFO 原则，b 必须先于 a 出栈。
5. a 不可能在 b 之前出栈，故 **dceab 是不可能的输出序列**。
