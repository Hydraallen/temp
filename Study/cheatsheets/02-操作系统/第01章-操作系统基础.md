## 二、知识点详解

### 1. 操作系统的定义 ⭐(练习14)

- **定义**：操作系统（OS，Operating System）是配置在计算机硬件上的第一层软件，是对硬件系统的首次扩充。其主要作用是管理好这些设备，提高它们的利用率和系统的吞吐量，并为用户和应用程序提供一个简单的接口，便于用户使用。

### 2. 计算机软件系统构成

- **构成**：计算机软件系统由**系统软件**和**应用软件**构成。
- 操作系统是现代计算机系统中最基本和最重要的**系统软件**。
- 其他系统软件包括编译程序、数据库管理系统等。
- 在系统软件之上运行着解决人们具体问题的各类**应用软件**。

### 3. 操作系统的目标 ⭐(练习23) ⚠(练习32)

- **（1）方便性**：未配置 OS 的计算机极难使用，用户必须用机器语言书写程序。配置 OS 后，系统可以使用编译命令将高级语言程序翻译成机器代码，或通过 OS 提供的各种命令操纵计算机，使计算机变得易学易用。
- **（2）有效性**：
  - 第一层含义：**提高系统资源的利用率**（处理机、I/O 设备等在未配置 OS 时常处于空闲状态）。
  - 第二层含义：**提高系统的吞吐量**（OS 通过合理组织工作流程，加速程序运行，缩短运行周期）。
  - 方便性和有效性是设计 OS 时最重要的两个目标。过去因计算机昂贵，有效性特别重要；近年来硬件越来越便宜，微机 OS 更重视方便性（图形用户界面、大量系统调用）。
- **（3）可扩充性**：为适应硬件、体系结构及应用发展，OS 必须具有很好的可扩充性。可扩充性与 OS 结构紧密相关，推动了 OS 结构发展：无结构 → 模块化结构 → 层次化结构 → 微内核结构。微内核结构能方便地增添新功能和模块，具有良好的可扩充性。
- **（4）开放性**：指系统能遵循世界标准规范，特别是遵循开放系统互连 OSI 国际标准。遵循国际标准开发的硬件和软件都能彼此兼容，方便实现互连。开放性已成为 20 世纪 90 年代以后计算机技术的核心问题。

### 4. OS 的特征 ⭐(练习3,练习4) ⚠(练习31)

操作系统的基本特征包括**并发、共享、虚拟和异步**。

- **（1）并发性**：
  - 正是系统中的程序能并发执行这一特征，才使得 OS 能有效地提高资源利用率，增加系统吞吐量。
  - **并行性**：两个或多个事件在**同一时刻**发生。
  - **并发性**：两个或多个事件在**同一时间间隔内**发生。
  - ⚠ 并发是其他三个特征的前提（练习3）。
- **（2）共享性**：
  - 在 OS 环境下，共享是指系统中的资源可供内存中多个并发执行的进程（线程）共同使用，称为资源共享或资源复用。
  - **互斥共享方式**：[原文未详述具体定义，仅列出名称]
  - **同时访问方式**：[原文未详述具体定义，仅列出名称]
- **（3）虚拟性**：
  - 通过某种技术把一个**物理实体**变为若干个**逻辑上的对应物**。物理实体是实的（实际存在的），后者是虚的（用户感觉上的东西）。
  - 实现虚拟的技术称为虚拟技术。OS 中利用两种方式实现：**时分复用技术**和**空分复用技术**。
- **（4）异步性**：
  - 多道程序环境下，系统允许多个进程并发执行。单处理机环境下，每次只允许一个进程执行，其余进程只能等待。
  - 由于资源等因素限制，进程的执行通常不可能"一气呵成"，而是以"停停走走"的方式运行。

### 5. OS 的功能 ⭐(练习5,练习8,练习10,练习16,练习24) ⚠(练习25,练习34,练习35,练习38)

#### 5.1 OS 作为计算机系统资源的管理者

- **（1）处理器管理**：在多道程序环境下，处理机的分配和运行都以进程（或线程）为基本单位，对处理机的管理可归结为对进程的管理。进程管理的主要功能包括：**进程控制、进程同步、进程通信、死锁处理、处理机调度**等。
- **（2）存储器管理**：为多道程序的运行提供良好环境，方便用户使用及提高内存利用率。主要包括：**内存分配与回收、地址映射、内存保护与共享、内存扩充**等功能。
  - ⚠ 存储器管理是对**内存**的管理，磁盘块的分配和回收不属于存储器管理（练习25）。
- **（3）文件管理**：计算机中的信息以文件形式存在，负责文件管理的部分称为文件系统。包括：**文件存储空间的管理、目录管理、文件读写管理和保护**等。
- **（4）设备管理**：完成用户的 I/O 请求，方便用户使用各种设备，提高设备利用率。主要包括：**缓冲管理、设备分配、设备处理、虚拟设备**等功能。

#### 5.2 OS 作为用户与计算机硬件系统之间的接口

- **（1）命令接口**：使用命令接口进行作业控制的主要方式有两种——联机控制方式和脱机控制方式。按作业控制方式的不同，可将命令接口分为**联机命令接口**和**脱机命令接口**。
- **（2）程序接口**：由一组**系统调用**（也称广义指令）组成。用户通过在程序中使用这些系统调用来请求 OS 为其提供服务，如使用各种外部设备、申请分配和回收内存及其他各种要求。
  - ⚠ 系统调用的目的是**请求系统服务**，不仅仅是申请系统资源（练习38）。

#### 5.3 OS 用作扩充机器

- 没有任何软件支持的计算机称为**裸机**，仅构成计算机系统的物质基础。
- OS 提供的资源管理功能和方便用户的各种服务功能，将裸机改造成功能更强、使用更方便的机器。
- 覆盖了软件的机器称为**扩充机器**或**虚拟机**。

### 6. 现代 OS 的新功能

#### 6.1 系统安全

确保系统中存储和传输数据的**保密性、完整性和系统可用性**。常见保障技术：

- **（1）认证技术**：确认被认证对象是否名副其实，确定对象真实性，防止入侵者假冒和篡改。如身份认证，通过验证被认证对象的一个或多个参数的真实性和有效性来确定。
- **（2）密码技术**：对系统中存储和传输的数据进行加密，使之成为密文。只有指定用户才能解密。
- **（3）访问控制技术**：通过两种途径保障资源安全——① 对用户权限的设置，限定用户只能访问被允许访问的资源；② 对文件属性的设置（如只读属性）。
- **（4）反病毒技术**：最好的解决方法是预防，但难以完全做到，需要有效的反病毒软件来检测和清除病毒。

#### 6.2 网络的功能和服务

网络操作系统应具备的功能：

- **（1）网络通信**：在源主机和目标主机之间实现无差错的数据传输，如建立和拆除通信链路、传输控制、差错控制和流量控制等。
- **（2）资源管理**：对网络中的共享资源（硬件和软件）实施有效管理，协调诸用户对共享资源的使用，保证数据的安全性和一致性。典型共享硬件资源有硬盘、打印机等，软件资源有文件和数据。
- **（3）应用互操作**：在互连网络中提供应用互操作功能，实现**信息的互通性**（不同网络中的用户之间能实现信息互通）和**信息的互用性**（用户可以访问不同网络中的文件系统和数据库系统中的信息）。

#### 6.3 支持多媒体

现代 OS 增加的多媒体处理功能：

- **（1）接纳控制功能**：为保证同时运行多个实时进程的截止时间，需对软实时任务（SRT，Soft Real-time）的数目、驻留内存的任务数目加以限制。包括媒体服务器的接纳控制、存储器接纳控制和进程接纳控制。
- **（2）实时调度**：多媒体系统中的任务往往是要求较严格的、周期性的软实时任务 SRT。如保证动态图像连续性，图像更新周期必须在 40ms 之内。SRT 调度不仅需要考虑进程调度策略，还要考虑进程调度的接纳度。
- **（3）多媒体文件的存储**：最重要的要求是能把硬盘上的数据快速传送到输出设备上。对传统文件系统中数据的离散存放方式以及磁盘寻道方式都要加以改进。

### 7. 推动 OS 发展的主要动力 ⭐(练习27)

- **（1）不断提高计算机资源利用率**：计算机发展初期系统特别昂贵，需千方百计提高资源利用率。由此形成了多道批处理系统。20 世纪 60 和 70 年代出现了 SPOOLing 系统（提高 I/O 设备和 CPU 利用率）和虚拟存储器技术（改善存储器利用率）。网络环境下通过网络文件系统和数据库系统进一步提高了资源利用率。
- **（2）方便用户**：资源利用率问题基本解决后，用户上机、调试程序的不方便性成为主要矛盾。20 世纪 60 年代分时系统出现，实现人-机交互。90 年代初图形用户界面出现，进一步方便用户。
- **（3）器件的不断更新换代**：微机芯片由 8 位发展到 16 位、32 位、64 位，相应 OS 也同步发展，功能和性能显著增强。外部设备也在迅速发展，OS 支持的外部设备越来越多。
- **（4）计算机体系结构的不断发展**：如单处理机系统发展为多处理机系统，相应 OS 由单处理机 OS 发展为多处理机 OS；计算机网络出现后产生了网络操作系统。
- **（5）不断提出新的应用需求**：如工业控制需求产生了实时 OS；听音乐、看电影等需求促使 OS 增添多媒体功能。

### 8. OS 的发展过程 ⭐(练习11,练习12,练习19,练习26,练习28) ⚠(练习9,练习37,练习40)

#### 8.1 未配置 OS 的计算机系统

- **（1）人工操作方式**：
  - 程序员将穿孔纸带（或卡片）装入纸带输入机（或卡片输入机），启动输入程序和数据，然后启动计算机运行。仅当程序运行完毕并取走结果，才允许下一个用户上机。
  - 缺点：① **用户独占全机**；② **CPU 等待人工操作**（装带/卸带时 CPU 及内存等资源空闲）。
  - 人机矛盾严重降低了资源利用率。CPU 速度迅速提高但 I/O 设备速度提高缓慢，矛盾更加突出。先后出现通道技术、缓冲技术，但未能很好解决，直至引入脱机 I/O 技术。

- **（2）脱机输入/输出方式**：
  - 20 世纪 50 年代末出现。事先将用户程序和数据装入纸带输入机，在**外围机**控制下把数据输入到**磁带**上。CPU 需要时再从磁带上高速调入内存。
  - 主要优点：① **减少了 CPU 的空闲时间**（装带/卸带及数据传输在脱机情况下由外围机完成，不占用主机时间）；② **提高了 I/O 速度**（CPU 直接从高速磁带输入数据到内存）。

#### 8.2 批处理阶段（OS 的出现）

- **（1）单道批处理系统**：
  - 系统对作业的处理是成批进行的，但内存中始终保持一道作业。
  - 主要特征：
    - **（a）自动性**：在顺利情况下，磁带上的一批作业能自动逐个运行，无须人工干预。
    - **（b）顺序性**：各道作业顺序进入内存，完成顺序与进入内存顺序在正常情况下完全相同（先调入先完成）。
    - **（c）单道性**：内存中仅有一道程序运行，当该程序完成或发生异常时，才换入后继程序。
  - 面临问题：每次内存中仅存放一道作业，运行期间发出 I/O 请求后，高速 CPU 处于等待低速 I/O 完成状态。为提高资源利用率和吞吐量，引入了多道程序技术。

- **（2）多道批处理系统**：
  - 多道程序设计技术允许许多个程序同时进入内存并允许它们在 CPU 中交替运行，这些程序共享系统中的各种硬/软件资源。多道程序因 I/O 请求而暂停运行时，CPU 便立即转去运行另一道程序。
  - [原文此处似乎有截断，"它不采用某些"后内容缺失]
  - ⚠ 多道批处理系统未能解决系统和用户交互的问题（练习26）。
  - ⚠ 多道程序设计技术的最主要原因是提高系统资源利用率和吞吐量，不是提高可靠性（练习40）。

### 9. 分时 OS ⭐(练习7,练习17,练习18) ⚠(练习6,练习36)

- **分时技术**：把处理器的运行时间分成很短的**时间片**，按时间片轮流把处理器分配给各联机作业使用。若某个作业在分配的时间片内不能完成计算，则暂时停止运行，把处理器让给其他作业使用，等待下一轮再继续运行。由于计算机速度很快，作业运行轮转很快，给每个用户的感觉就像自己独占一台计算机。

- **分时 OS 的特征**：
  - **（1）多路性**：系统允许将多台终端同时连接到一台主机上，并按分时原则为每个用户服务。多路性允许多个用户共享一台计算机，显著提高资源利用率，降低使用费用。
  - **（2）独立性**：每个用户在各自的终端上进行操作，彼此之间互不干扰，给用户的感觉就像他一人独占主机。
  - **（3）及时性**：用户请求能在很短时间内获得响应。这一时间间隔根据人们所能接受的等待时间确定，通常仅为 **1~3 秒钟**。
  - **（4）交互性**：用户可通过终端与系统进行广泛的人机对话。用户可以请求系统提供多方面的服务，如文件编辑和数据处理、访问文件系统和数据库系统、请求打印服务等。

### 10. 实时 OS ⭐(练习1,练习15,练习22)

实时操作系统具有以下性质：

- **（1）多路性**：信息查询系统和分时系统中的多路性都表现为系统按分时原则为多个终端用户服务；实时控制系统的多路性是指系统周期性地对多路现场信息进行采集，以及对多个对象或多个执行机构进行控制。
- **（2）独立性**：信息查询系统中每个终端用户在与系统交互时，彼此相互独立互不干扰；实时控制系统中，对信息的采集和对对象的控制也都是彼此互不干扰的。
- **（3）及时性**：信息查询系统对实时性的要求依据人所能接受的等待时间确定；多媒体系统实时性的要求是播放出来的音乐和电视能令人满意；实时控制系统的实时性以控制对象所要求的**截止时间**来确定，一般为**秒级到毫秒级**。
- **（4）交互性**：信息查询系统中人与系统的交互性仅限于访问系统中某些特定的专用服务程序，不像分时系统那样能向终端用户提供数据处理、资源共享等服务。多媒体系统的交互性也仅限于用户发送某些特定命令（如开始、停止、快进等），由系统立即响应。
- **（5）可靠性**：分时系统要求系统可靠，实时系统要求**高度可靠**，因为任何差错都可能带来无法预料的灾难性后果。实时系统往往采取**多级容错措施**来保障系统安全性及数据安全性。

### 11. OS 的分类 ⭐(练习13) ⚠(练习9,练习33,练习39)

- **（1）单用户单任务 OS**：只允许一个用户上机，且只允许用户程序作为一个任务运行。最简单的微机操作系统，主要配置在 8 位和 16 位微机上。最有代表性的是 **CP/M** 和 **MS-DOS**。
  - ⚠ MS-DOS 是单用户单任务操作系统，不是多用户多任务（练习33）。
- **（2）单用户多任务 OS**：只允许一个用户上机，但允许用户把程序分为若干个任务，使它们并发执行，从而有效改善系统性能。目前在 32 位微机上配置的操作系统大多是单用户多任务操作系统。最有代表性的是微软公司的 **Windows**。
  - 1985 年和 1987 年先后推出 Windows 1.0 和 2.0（16 位微机平台支持不好）；1990 年发布 Windows 3.0，随后 Windows 3.1（针对 386 和 486 等 32 位微机），引入友善的图形用户界面，支持多任务和扩展内存功能。
- **（3）多用户多任务 OS**：允许多个用户通过各自的终端使用同一台机器，共享主机系统中的各种资源，而每个用户程序又可进一步分为几个任务，使它们能并发执行。在大、中和小型机中配置的大多是多用户多任务操作系统，32 位微机上也有不少配置。最有代表性的是 **UNIX**。
  - ⚠ 多道≠多用户，多道仅指多个程序在系统调度下并行运行，与是否多用户无关（练习39）。

### 12. OS 的体系结构 ⭐(练习30)

- **（1）无结构 OS**：
  - 设计者只注意功能实现和获得高效率，缺乏首尾一致的设计思想。OS 为数众多的一组过程的集合，每个过程可以任意相互调用其他过程，内部既复杂又混乱，也称为**整体系统结构**。
  - 程序设计技巧只是如何编制紧凑的程序以有效利用内存。系统不断扩大时，所设计的 OS 变得既庞大又杂乱，程序错误多，调试困难，难以阅读和理解，增加维护人员负担。

- **（2）模块化 OS**：
  - 划分模块时必须充分注意模块的独立性。衡量模块独立性的两个标准：
    - **（a）内聚性**：模块内部各部分间联系的紧密程度。内聚性越高，模块独立性越强。
    - **（b）耦合度**：模块间相互联系和相互影响的程度。耦合度越低，模块独立性越好。
  - 模块-接口法开发的 OS 较之无结构 OS 的优点：① 提高 OS 设计的正确性、可理解性和可维护性；② 增强 OS 的可适应性；③ 加速 OS 的开发过程。

- **（3）分层结构 OS**：
  - 主要优点：① **易保证系统的正确性**（自下而上的设计，决定都是有序的，建立在较为可靠的基础上）；② **易扩充和易维护性**（增加、修改或替换某层模块时，只要不改变层次间接口，就不会影响其他层次）。
  - 主要缺点：**系统效率降低**。层次结构分层单向依赖，每层之间都需建立通信机制，OS 每执行一个功能通常要自上而下穿越多个层次，增加通信开销，导致效率降低。

### 13. 微内核 OS 结构 ⭐(练习29)

#### 13.1 基本概念

- 在微内核操作系统中，内核是指精心设计的、能实现现代 OS 最基本核心功能的**小型内核**。微内核并非一个完整的 OS，而只是将操作系统中最基本的部分放入微内核，通常包含与硬件处理紧密相关的部分。
- 设计基础：
  - **（1）基于客户/服务器模式**
  - **（2）应用"机制与策略分离"原理**
  - **（3）采用面向对象技术**

#### 13.2 微内核的基本功能

- **（1）进程（线程）管理**
- **（2）低级存储器管理**
- **（3）中断和陷入处理**

#### 13.3 微内核 OS 的优点

建立在模块化、层次化结构基础上，并采用客户/服务器模式和面向对象程序设计技术：

- **（1）提高了系统的可扩展性。**
- **（2）增强了系统的可靠性。**
- **（3）可移植性强。**
- **（4）提供了对分布式系统的支持。**
- **（5）融入了面向对象技术。**

#### 13.4 微内核 OS 存在的问题

- 由于采用了非常小的内核，客户/服务器模式和消息传递机制虽带来许多优点，但也存在潜在缺点：**最主要的是运行效率有所降低**（较之早期操作系统）。
- 实际情况是往往还会引起**更多的上下文切换**。
- ⚠ 微内核会增加一些开销（如上下文切换、消息传递、数据拷贝等），降低执行效率（练习29）。

### 4.2 分时 OS vs 实时 OS

| 对比维度 | 分时 OS | 实时 OS |
|----------|---------|---------|
| **多路性** | 系统按分时原则为多个终端用户服务 | 信息查询系统按分时原则为多个终端用户服务；实时控制系统周期性对多路现场信息采集及对多个对象/执行机构控制 |
| **独立性** | 每个用户在各自终端上操作，彼此互不干扰 | 每个终端用户与系统交互时彼此独立互不干扰；实时控制系统中信息采集和对象控制也彼此互不干扰 |
| **及时性** | 用户请求能在很短时间内获得响应，通常仅为 1~3 秒钟 | 信息查询系统依据人所能接受的等待时间确定；实时控制系统以控制对象要求的截止时间确定，一般为秒级到毫秒级 |
| **交互性** | 用户可通过终端与系统进行**广泛**的人机对话（文件编辑、数据处理、访问文件系统和数据库系统、打印服务等） | 仅限于访问系统中**某些特定的专用服务程序**，不像分时系统那样提供数据处理、资源共享等服务；多媒体系统仅限发送特定命令（开始、停止、快进等） |

## 五、高频考点与易错提醒

> 以下考点信号来自专项练习与答案解析，仅列考点方向，不收录题目与解答。

1. ⚠ **并发与并行的概念辨析**是高频易错点——并发=同一时间间隔内，并行=同一时刻（练习4, 练习31）。
2. ⚠ **并发是其他三个特征（共享、虚拟、异步）的前提**是高频考点（练习3）。
3. ⚠ **OS 提供给用户使用的两种方式**——命令接口和系统调用（程序接口），图形接口本质上调用了系统调用，是高频考点（练习5）。
4. ⚠ **系统调用的目的**是请求系统服务（不仅仅是申请系统资源），是易错点（练习8, 练习38）。
5. ⚠ **OS 的四大功能**——处理机管理、存储器管理、设备管理、文件管理，文档编辑不属于 OS 功能，是高频考点（练习10, 练习16）。
6. ⚠ **存储器管理是对内存的管理**，磁盘块的分配/回收不属于存储器管理，是易错点（练习25）。
7. ⚠ **OS 基本类型**为批处理、分时、实时三种，是高频考点（练习13）。
8. ⚠ **批处理系统的主要缺点是无交互能力**，是高频考点（练习12）。
9. ⚠ **多道程序设计技术的目的**是提高资源利用率和吞吐量，不是提高可靠性，是易错点（练习11, 练习40）。
10. ⚠ **实时 OS 的主要追求目标**是实时性和可靠性，资源利用率不是其主要目标，是高频考点（练习15）。
11. ⚠ **分时 OS 与实时 OS 的区别**主要在交互性、可靠性和响应时间，是高频考点（练习22）。
12. ⚠ **MS-DOS 是单用户单任务操作系统**，不是多用户多任务，是易错点（练习33）。
13. ⚠ **多道≠多用户**——多道仅指多个程序在系统调度下并行运行，与是否多用户无关，是易错点（练习9, 练习39）。
14. ⚠ **进程数越多不一定 CPU 利用率越高**——进程过多可能导致资源竞争激烈甚至死锁，是易错点（练习37）。
15. ⚠ **OS 的程序不一定全部常驻内存**——分为常驻内存程序和非常驻内存程序，是易错点（练习35）。
16. ⚠ **分时系统中用户并非真正独占文件系统**——从 OS 角度看是多个用户共享，需解决互斥使用问题，是易错点（练习36）。
17. ⚠ **资源利用率越高用户不一定越满意**——可能使用户等待时间更长，方便性、交互性受影响，是易错点（练习32）。
18. ⚠ **脱机 I/O 方式解决的问题**是人机矛盾及 CPU 和 I/O 设备间速度不匹配的矛盾，是高频考点（练习19）。
19. ⚠ **微内核 OS 的运行效率有所降低**（更多上下文切换、消息传递开销），不能提高执行效率，是易错点（练习29）。
20. ⚠ **分层结构 OS 的优点**是易保证正确性、易扩充、易维护，但系统效率降低，是高频考点（练习30）。
21. ⚠ **用户态可执行的指令类型**（特权指令 vs 非特权指令的区分）是考点方向，知识点正文未详述，需注意（练习20）。
22. ⚠ **分时系统时间片计算**——响应时间 = 时间片 × 就绪进程数，可据此反推时间片最大值（练习18，反推）。
23. ⚠ **OS 的目标**包括方便性、有效性、可扩充性、开放性，"多用户"不是 OS 的目标，是高频考点（练习23）。
24. ⚠ **OS 的作用**——作为用户与计算机硬件系统之间的接口、作为计算机系统资源的管理者、实现对计算机资源的抽象（扩充机器/虚拟机），是高频考点（练习14, 练习24）。
25. ⚠ **计算机系统资源**包括处理器、存储器、I/O 设备及信息（数据和程序），文件属于信息类资源，是易错点（练习34）。
26. ⚠ **推动 OS 发展的动力**包括计算机体系结构发展、新应用需求、对资源利用率的追求、用户对操作方便性的追求、硬件不断更新换代，与硬件价格无关，是高频考点（练习27）。

## 六、一句话记忆要点

1. **OS 定义**：OS 是配置在硬件上的第一层软件，管理设备、提高利用率与吞吐量、提供接口。
2. **OS 目标**：方便、有效、可扩充、开放——方便性和有效性最重要。
3. **OS 四大特征**：并发（前提）、共享（互斥/同时）、虚拟（时分/空分复用）、异步（停停走走）。
4. **并行 vs 并发**：并行=同一时刻，并发=同一时间间隔。
5. **OS 四大功能**：处理器管理（进程）、存储器管理（内存）、文件管理、设备管理。
6. **OS 三大角色**：资源管理者、用户与硬件的接口（命令接口+程序接口/系统调用）、扩充机器（虚拟机）。
7. **系统调用目的**：请求系统服务（不只是申请资源）。
8. **OS 发展动力**：资源利用率、方便用户、器件更新、体系结构发展、新应用需求。
9. **脱机 I/O**：外围机控制数据输入磁带，减少 CPU 空闲、提高 I/O 速度。
10. **单道批处理**：自动性、顺序性、单道性——CPU 等 I/O 是瓶颈。
11. **多道批处理**：多道程序同时驻留内存交替运行，提高利用率但无交互能力。
12. **分时 OS**：时间片轮转，特征为多路性、独立性、及时性（1~3 秒）、交互性。
13. **实时 OS**：截止时间约束，特征为多路性、独立性、及时性（秒级~毫秒级）、交互性（有限）、可靠性（高度可靠+多级容错）。
14. **OS 分类**：单用户单任务（MS-DOS）、单用户多任务（Windows）、多用户多任务（UNIX）。
15. **OS 体系结构**：无结构→模块化（内聚高/耦合低）→分层（正确性好但效率低）→微内核（可扩展/可靠/可移植但效率低）。
16. **微内核**：小型内核，含进程管理、低级存储器管理、中断和陷入处理；基于客户/服务器模式、机制与策略分离、面向对象技术。
