## 一、本章概览

本章围绕「并行处理系统」展开，解决三大问题：（1）计算机系统结构如何分类——以 Flynn 分类法为核心，区分 SISD/SIMD/MISD/MIMD 四类体系结构及其典型代表；（2）互连网络的基本概念、参数与分类，作为并行计算机内部通信的基础设施；（3）GPU 与集群技术两种典型的并行处理系统——GPU 从图形处理器发展为通用并行计算平台（CUDA/OpenCL），集群技术则通过多机互联实现高可用性与高性能。全章与前后章的关系：在掌握单处理器结构（运算器、存储器、控制器）之后，上升到多处理器/多计算机的系统级并行架构。

## 二、知识点详解

### 第一节 计算机系统结构的分类方法

### 1. 计算机系统结构的分类方法（Flynn 分类法）⭐ `(练习6, 多选2)`

- **定义**：1966年，Michael.J.Flynn 提出按**指令流**和**数据流**的多倍性对计算机系统结构进行分类。
  - **指令流**：机器执行的指令序列。
  - **数据流**：由指令流调用的数据序列，包括输入数据和中间结果。
  - **多倍性**：在系统最受限的部件上，同时处于同一执行阶段的指令或数据的最大数目。

- **分类表**：

| 指令流 | 数据流 | 名称 | 举例 |
| ---- | ---- | ---- | ---- |
| 1个 | 1个 | SISD | 传统的冯·诺依曼计算机 |
| 1个 | 多个 | SIMD | 向量计算机，阵列处理机 |
| 多个 | 1个 | MISD | ? ? ? ? ? ? |
| 多个 | 多个 | MIMD | 多处理机，多计算机 |

#### SISD 体系结构
- 单一的指令流从存储器取指令，以单一的数据流从存储器取操作数和将结果写回存储器。
- 典型机器：
  - 单功能部件处理机：IBM1401，VAX-11
  - 多功能部件处理机：IBM360/370，CDC6600
  - 流水线处理机：指标量流水线处理机

#### SIMD 体系结构
- 单一的控制部件，多个处理部件。计算机以一个控制单元从存储器取单一的指令流，一条指令同时作用到各个处理单元，控制各个处理单元对来自不同数据流的数据组进行操作。
- 典型代表：**阵列处理机**和**向量流水处理机**。
- **向量计算机**：可以在一个向量的每个元素上执行相同的操作，一般来说向量处理器是以流水线式 ALU 为核心，实现向量各个元素的并行计算采用的是**时间重叠技术**。
- **阵列计算机**：采用**资源重复方法**引入空间因素，即在系统中设置多个相同的处理单元来开发并行性。利用并行性中的**同时性**，所有处理单元必须同时进行相同操作。

#### MISD 体系结构 ⚠ `(多选2)`
- 多个处理单元，各配有相应的控制单元。各个处理单元接收不同的指令，多条指令同时在一份数据上进行操作。
- 这种计算机体系结构**已经被证明是不可能至少是不实际的**，目前为止还不存在这种类型的计算机。

#### MIMD 体系结构 ⭐ `(练习6)`
- 同时有多个处理单元，并且每个处理单元都配有相应的控制单元。各个处理单元可以接收不同的指令并对不同的数据流进行操作。
- 大多数现代的并行计算机都属于这一类，多处理机系统和多计算机系统都是 MIMD 型的计算机。
- 例如：IBM3081、IBM3084、UNIVAC-1100/80、CRAY-2

#### Flynn 分类法的局限
- 分类的对象主要是**控制驱动方式**下的串行处理和并行处理计算机。对于非控制驱动方式的计算机，就不适合采用 Flynn 分类法。
- 把两个不同等级的功能并列对待，通常，数据流受指令流控制从而造成 MISD 不存在。
- 分类太粗，对流水线处理机的划分不明确：标量流水处理机为 SISD，向量流水处理机为 SIMD。

#### 并行计算机体系结构图谱（图7-6）

```
计算机系统结构
├── SISD —— 冯·诺依曼
├── SIMD —— 向量计算机、阵列计算机
├── MISD —— ?
└── MIMD
    ├── 多处理机（共享内存）
    │   ├── UMA
    │   ├── COMA
    │   └── NUMA
    │       ├── CC-NUMA
    │       └── NC-NUMA
    │   （互连方式：总线 / 交换结构）
    └── 多计算机（消息传递）
        ├── MPP
        └── COW
        （互连方式：网格 / 超立方体）
```

#### MIMD 的两大子类
- **多处理机系统——基于共享存储器**：系统中只有唯一的地址空间，所有的处理器共享该地址空间。共享地址空间可以通过一个物理上共享的存储器来实现，也可以通过分布式存储器并在硬件和软件的支持下实现。
- **多计算机系统——基于消息传递**：每个处理器有自己的存储器，该存储器只能被该处理器访问而不能被其它处理器直接访问，这种存储器称为**局部存储器或私有存储器**。当处理器 A 需要向处理器 B 传送数据时，A 把数据以消息的形式发送给 B。

### 2. 并行计算机系统发展的原因

1. **应用的需求**永远是并行计算机系统发展的动力。随着计算机速度的提高，人们对计算机性能的要求也越来越高。例如科学计算、工程和工业设计等都需要高性能计算。芯片的速度不可能无限地提高，并行计算机可以处理越来越复杂的问题。芯片的速度要受到光速的制约，但芯片的集成度还有发展的空间。
2. 大量**商品化的处理器**的出现为设计并行计算机系统提供了可能。
3. 并行计算机系统获得快速发展和**处理器间通信技术**的发展密不可分。

### 3. 互连网络概述 ⭐ `(多选5,6,7,8,9, 判断6,7,8)`

- **定义**：互连网络是并行计算机系统内部的互连网络，由开关元件按一定拓扑结构和控制方式构成的网络以实现计算机系统内部多个处理器或多个功能部件间的相互连接。与计算机网络在工作原理、概念以及术语上有许多相同或相似之处；并且某些并行计算机系统中的互连网络就是高速以太网和 ATM 网络。

- **组成**：互连网络一般由以下五个部分组成：CPU、内存模块、接口、链路和交换结点。

- **分类**：
  - **静态网络**（Static Networks）：结点间有着固定连接通路且在程序执行期间，这种连接保持不变的网络。
  - **动态网络**（Dynamic Networks）：由开关单元构成，可按应用程序的要求动态地改变连接状态。如总线、交叉开关、多级交换网络等。

- **互连网络的参数**：

| 参数 | 定义 |
| ---- | ---- |
| 结点度 | 与结点相连接的边数，表示节点所需要的端口数。结点度 = 入度 + 出度 |
| 链路的长度 | 链路中包含的边数 |
| 距离 | 与两个结点之间相连的最少边数 |
| 网络直径 | 网络中任意两个结点间距离的最大值 |
| 网络规模 | 网络中结点数，表示该网络功能连结部件的多少 |
| 等分宽度 | 某一网络被切成相等的两半时，沿切口的最小边数 |
| 对称性 | 从任何结点看，拓扑结构都一样，这种网络实现和编程都很容易 |

- **静态互连网络有**：线性阵列、环形、全连接、树形、网格型、环网型等。
- **动态互联网络有**：总线、交叉开关、多级交换网络等。

### 第二节 GPU 简介

### 1. GPU ⭐ `(判断2)`

- **定义**：Graphics Processing Unit 的简写，是现代显卡中非常重要的一个部分，其地位与 CPU 在主板上的地位一致，主要负责的任务是加速图形处理速度。GPU 是一个**高度并行化的多线程、多核心处理器**。
- 与中央处理器（CPU）的串行设计模式不同，GPU 为图形处理设计，具有**天然的并行特性**。
- NVIDIA 在 1999 年推出了第一款 GPU 产品——GeForce 256。主要任务是进行图形渲染任务，缓解 CPU 压力。
- 市场迫切需要实时、高清晰度的 3D 图形，可编程的 GPU 已发展成为一种高度并行化、多线程、多核心的处理器，具有杰出的计算能力和极高的存储器带宽。

### 2. GPU 研究现状

- 从 GPU 诞生开始，人们希望将某些通用计算问题移植到 GPU 上来完成以提升效率，出现了所谓的 **GPGPU**（General Purpose Graphic Process Unit），但由于开发难度较大，没有被广泛接受。
- 2006年 NVIDIA 推出了第一款基于 **Tesla 架构**的 GPU（G80），GPU 开始正式向通用计算领域迈进。
- 2007年6月，NVIDIA 推出了 **CUDA**（Computer Unified Device Architecture，计算统一设备架构）。CUDA 是一种将 GPU 作为数据并行计算设备的软硬件体系。在 CUDA 的架构中，不再像过去 GPGPU 架构那样将通用计算映射到图形 API 中，开发门槛大大降低。CUDA 的编程语言基于**标准 C**，被广泛应用于石油勘测、天文计算、流体力学模拟、分子动力学仿真、生物计算、图像处理、音视频编解码等领域。

### 3. GPU 内部架构 ⭐ `(练习7)`

- **CPU vs GPU 设计差异**：
  - CPU **强控制弱计算**，更多资源用于缓存。
  - GPU **强计算弱控制**，更多资源用于数据计算。

- **传统 GPU 基于 SIMD 架构**（Single Instruction Multiple Data，单指令多数据）。

- **Tesla GT200 架构**（以 GT200 体系架构为代表对 GPU 的并行层次进行分析）：
  - Tesla GT200 由两部分组成：**可伸缩流处理器阵列**（Scalable Streaming Processor Array，SPA）和**存储器系统**，由一个片上互联网络连接。
  - SPA 由若干个**线程处理器群**（Thread Processing Cluster，TPC）构成。
  - 每个 TPC 包含 **2～3 个流多处理器**（Streaming Multiprocessor，SM）。
  - 每个流多处理器中包含 **8 个流处理器**（Streaming Processor，SP）。
  - **流处理器（SP）**：有独立的寄存器和指令指针，但**缺少取指和调度单元**。
  - **流多处理器（SM）**：拥有完整前端，包括取值、译码、发射等。每个 SM 相当于一个 **8 路 SIMD 处理器**，不同的是，GPU 实现了自动向量机化，NVIDIA 将之命名为 **SIMT**（Single Instruction Multiple Thread，单指令流多线程）。
  - 在 GPU 中，**流多处理器才能被称为真正的完整核心**，整个 SPA 可以看成是由多个 SM 组成的 **MSIMT**（Multiple SIMT）系统。
  - 线程被组织成多个**线程块**（Thread Block），分配到各个 SM 上，而每个线程块内的线程再被以 SIMT 的方式交给 SP 运行。

- **CPU 与 GPU 差异总结**：
  - CPU 的重线程与 GPU 的轻线程
  - CPU 的 MIMD 多核与 GPU 的 SIMT 众核
  - CPU 内存、缓存与 GPU 存储器
  - GPU 是以大量线程实现面向吞吐量的数据并行计算，适合于处理**计算密度高、逻辑分支简单的大规模数据并行负载**；而 CPU 则有复杂的控制逻辑和大容量的缓存减小延迟，擅长**复杂逻辑运算**。

### 4. GPU 编程

- **CUDA 基本思想**：支持大量的**线程级并行**（Thread Level Parallel），并在硬件中动态地调度和执行这些线程。
- **异构思想**：Host + Device（Coprocessor）：让 GPU 来运行一些能够被高度线程化的代码。GPU 与 CPU 协同工作，GPU 只有在计算高度数据并行任务时才发挥作用。
- **OpenCL**（Open Computing Language，开放运算语言）：第一个面向异构系统通用目的并行编程的**开放式、免费标准**，也是一个统一的编程环境，便于软件开发人员为高性能计算服务器、桌面计算系统、手持设备编写高效轻便的代码，广泛适用于多核心处理器（CPU）、图形处理器（GPU）。

### 第三节 集群技术

### 1. 集群技术概述 ⭐ `(练习9,10)`

- 对于要求有高可用性和高安全性的系统，如金融、邮电、交通、电力、证券等行业，用户对系统安全性能提出了更高的要求。集群技术应运而生。
- 集群技术是实现系统**高可用性、可靠性、稳定性、安全性**并需要快速处理能力、大容量存储及系统的可管理性的重要手段。

### 2. 定义 ⭐ `(练习3,10,11,18)`

- 一组**相互独立的服务器**在网络中表现为**单一的系统**，并以单一系统的模式加以管理。此单一系统为客户工作站提供高可靠性的服务。
- 将多台**同构或异构**的计算机连接起来协同完成特定的任务就构成了集群系统。
- 大多数模式下，集群中所有的计算机拥有一个共同的名称，集群内任一系统上运行的服务可被所有的网络客户所使用。
- **对外表现为**：一台虚拟主机。
- **功能是**：协调管理各分离的组件的错误和失败，并可透明地向 Cluster 中加入组件。实现更高的可用性、可管理性和更优异的可伸缩性。

### 3. 实现方式（原文标为「知识点2」）

- 一个 Cluster 包含两台以上拥有**共享数据存储空间**的服务器。任何一台服务器运行一个应用时，应用数据被存储在共享的数据空间内。每台服务器的操作系统和应用程序文件存储在其各自的**本地储存空间**上。
- Cluster 内各节点服务器通过一**内部局域网**相互通讯。当一台节点服务器发生故障时，这台服务器上所运行的应用程序将在另一节点服务器上被**自动接管**。当一个应用服务发生故障时，应用服务将被重新启动或被另一台服务器接管。

### 4. 分类

- **按功能侧重分**：高可用性集群、高性能集群。
- **按设备来分**：镜像服务器双机、双机与磁盘阵列柜。

### 5. 高可用性与高性能集群 ⭐ `(练习9)`

- **高可用性集群**：主要是提供**不间断的服务**。适用于必须一天二十四小时不停运转的应用程序，如所有的 web 服务器、ATM、医学与军事监测仪以及股票处理机等。对这些应用程序而言，暂时的停机都会导致数据的丢失和灾难性的后果。
- **高性能集群**：通过将多台机器连接起来同时处理复杂的计算问题。模拟星球附近的磁场、预测龙卷风的出现等情况都需要对大量的数据进行处理。传统方法是用超级计算机来完成计算工作，但价格昂贵，且可用性和可扩展性不够强。

## 三、公式 / 算法 / 定理速查表

> 本章以概念分类为主，无数学公式。以下集中列出核心定义与分类体系。

| 项目 | 内容 |
| ---- | ---- |

| 多倍性定义 | 在系统最受限的部件上，同时处于同一执行阶段的指令或数据的最大数目 |

| 距离 | 两个结点之间相连的最少边数 |

| GT200 层次结构 | SPA → TPC（2~3个）→ SM（8个SP/SM）→ SP |
| SIMT | Single Instruction Multiple Thread，单指令流多线程（NVIDIA 命名） |
| MSIMT | Multiple SIMT，整个 SPA 看成由多个 SM 组成的系统 |
| CUDA 编程模型 | Host + Device(Coprocessor)，线程级并行，基于标准 C |

## 四、重点对比与易混辨析

### 1. Flynn 四类体系结构对比

| 特征 | SISD | SIMD | MISD | MIMD |
| ---- | ---- | ---- | ---- | ---- |
| 指令流 | 1个 | 1个 | 多个 | 多个 |
| 数据流 | 1个 | 多个 | 1个 | 多个 |
| 典型代表 | 冯·诺依曼计算机 | 向量计算机、阵列处理机 | 不存在 | 多处理机、多计算机 |
| 实际机器 | IBM1401, VAX-11, IBM360/370, CDC6600 | 阵列处理机、向量流水处理机 | [原文未述实际机器] | IBM3081, IBM3084, UNIVAC-1100/80, CRAY-2 |

### 2. 多处理机系统 vs 多计算机系统

| 特征 | 多处理机系统（共享存储器） | 多计算机系统（消息传递） |
| ---- | ---- | ---- |
| 地址空间 | 唯一地址空间，所有处理器共享 | 每个处理器有私有存储器（局部存储器） |
| 通信方式 | 共享地址空间 | 消息传递 |
| 存储器实现 | 物理共享或分布式+硬软件支持 | 每个处理器自己的存储器，只能被该处理器访问 |
| 典型类型 | UMA, COMA, NUMA(CC-NUMA, NC-NUMA) | MPP, COW |

### 3. CPU vs GPU

| 特征 | CPU | GPU |
| ---- | ---- | ---- |
| 设计取向 | 强控制弱计算，更多资源用于缓存 | 强计算弱控制，更多资源用于数据计算 |
| 并行模式 | MIMD 多核 | SIMT 众核 |
| 线程 | 重线程 | 轻线程 |
| 擅长领域 | 复杂逻辑运算 | 计算密度高、逻辑分支简单的大规模数据并行负载 |
| 传统架构 | [原文未述] | 基于 SIMD 架构 |

### 4. 静态互连网络 vs 动态互连网络

| 特征 | 静态网络 | 动态网络 |
| ---- | ---- | ---- |
| 连接特性 | 固定连接通路，程序执行期间保持不变 | 可按应用程序要求动态改变连接状态 |
| 构成 | 固定连接 | 开关单元 |
| 典型类型 | 线性阵列、环形、全连接、树形、网格型、环网型 | 总线、交叉开关、多级交换网络 |

### 5. 高可用性集群 vs 高性能集群

| 特征 | 高可用性集群 | 高性能集群 |
| ---- | ---- | ---- |
| 目标 | 提供不间断的服务 | 同时处理复杂的计算问题 |
| 适用场景 | web服务器、ATM、医学与军事监测仪、股票处理机 | 模拟星球磁场、预测龙卷风等大量数据处理 |
| 替代方案 | [原文未述] | 传统方法用超级计算机，但价格昂贵且可用性和可扩展性不够强 |

### 6. SIMD 的两种典型代表对比

| 特征 | 向量计算机 | 阵列计算机 |
| ---- | ---- | ---- |

| 核心部件 | 流水线式 ALU | 多个相同的处理单元 |
| 并行性利用 | [原文未述] | 同时性（所有处理单元同时进行相同操作） |

## 五、高频考点与易错提醒

1. ⚠ **Flynn 分类法中 MISD 不存在**——已被证明是不可能至少是不实际的，目前无实际机器。`(多选2)`
2. ⭐ **多处理机系统属于 MIMD**——按指令流和数据流组织方式，多处理机=MIMD。`(练习6)`
3. ⚠ **超线程技术不是多核技术**——超线程是在一个物理核心内模拟两个逻辑核心，并非多核；超线程的并行级别高于指令级。`(练习13, 20)`
4. ⚠ **超线程没有自己独立的运算器**——硬件还是一套，涉及资源冲突仍需串行；超线程性能≠两个 CPU 的性能。`(练习1, 14)`
5. ⭐ **机群系统（COW）特征**——节点是完整计算机，拥有本地磁盘和完整操作系统，可独立运行；节点可以是同构或异构的；通过消息传递通信，不通过共享内存；机群没有共享磁盘。`(练习3, 10, 11, 18)`
6. ⚠ **机群≠局域网≠MPP**——机群物理结构上看像局域网，但区别在于机群会协同完成任务；MPP 是紧耦合结构，机群是松耦合结构。`(练习10, 17)`
7. ⭐ **并行性概念**——两个以上的事件在同一时刻发生或同一时间间隔内发生都叫并行；前者严格称为"并发"。`(练习4, 判断1)`
8. ⭐ **并行性等级**（从执行程序角度看，由低到高）：指令内部并行 < 指令级并行 < 任务级或过程级并行 < 作业或程序级并行。`(练习5, 8)`
9. ⚠ **结点距离定义**——两个结点之间跨越的图的边的**最少**条数（不是最多）。`(判断6)`
10. ⚠ **交叉开关是动态互连网络**，不是静态互连网络。`(判断8)`
11. ⭐ **互连网络参数**（线性阵列、环形、星形等具体参数值）——这些具体数值仅在练习答案中出现，知识点正文只给出了参数定义，未给出各拓扑的具体数值。`(多选6,7,8, 反推)`
12. ⭐ **NUMA 多处理器系统特点**——所有 CPU 看到单一地址空间；用 LOAD/STORE 指令访问远程内存；访问远程内存比本地慢；处理器可使用高速缓存。`(多选10, 反推)`
13. ⚠ **SMP 特征**——对称多处理共享存储并行机，多个 CPU 共享统一内存地址空间，不是以结点为单位划分。`(判断4)`
14. ⭐ **GPU 产品线分类**——GeForce（家庭/企业娱乐，面向游戏用户）、Quadro（图形工作站，面向专业级用户）、Tesla（高性能通用计算，面向研究人员）、Tegra（移动设备）。`(练习7, 反推)`
15. ⚠ **超标量流水线**——能在一个时钟周期内同时发射多条指令，可结合动态调度技术提高指令执行并行性；但不会缩短流水线功能段的处理时间。`(练习12, 反推)`
16. ⚠ **存储墙概念**——CPU 和存储系统的性能差距，处理器性能逐年提升而存储器跟不上。`(练习15, 反推)`
17. ⭐ **并行程序性能优化**——减少通信量、提高通信粒度是首先要考虑的问题；加速比和效率是最基本的评价方法。`(判断9, 10, 反推)`
18. ⚠ **总线结构 vs 交叉开关**——总线结构冲突大，不如交叉开关带宽高。`(练习19, 反推)`

## 六、一句话记忆要点

1. **Flynn 分类法**：看指令流×数据流的数量——SISD（传统单机）、SIMD（向量/阵列机）、MISD（不存在）、MIMD（多处理机/多计算机）。
2. **互连网络参数**：结点度=入度+出度；距离=最少边数；直径=最大距离；等分宽度=切半的最小边数。
3. **静态 vs 动态网络**：静态固定连接（线性阵列、环、树、网格），动态可重构（总线、交叉开关、多级交换）。
4. **GPU 架构层次**：SPA → TPC → SM（真正完整核心，8路SIMD/SIMT）→ SP（缺取指调度）。
5. **CPU vs GPU**：CPU 强控制弱计算靠缓存，GPU 强计算弱控制靠众核并行。
6. **CUDA**：Host+Device 异构编程，基于标准 C，线程级并行。
7. **集群定义**：独立服务器通过网络表现为单一系统，同构或异构，消息传递通信，对外一台虚拟主机。
8. **高可用 vs 高性能集群**：前者提供不间断服务（24小时不停），后者多机并行处理复杂计算。
9. **超线程≠多核**：超线程在一个物理核心内模拟多个逻辑核心，硬件仍是一套，并行级别高于指令级。
10. **机群（COW）vs MPP**：机群松耦合、消息传递、商品化网络；MPP 紧耦合、独立存储器。
