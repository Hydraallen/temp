## 一、本章概览

本章是数据结构的核心章节，系统讲解树与二叉树的基本概念、性质、存储结构、遍历方法及典型应用。内容可分为八大块：① 树与二叉树的定义及特殊形态（满二叉树、完全二叉树）；② 二叉树的五条重要性质；③ 二叉树与树的存储结构；④ 二叉树的四种遍历（先序、中序、后序、层次）及由遍历序列恢复二叉树；⑤ 线索二叉树（定义、存储、创建、遍历）；⑥ 最优二叉树（哈夫曼树）与哈夫曼编码；⑦ 树/森林与二叉树的相互转换；⑧ 树和森林的遍历及其与对应二叉树遍历的对应关系。本章是后续查找（二叉排序树、平衡二叉树）和排序（堆）等章节的基础。

## 二、知识点详解

### 1. 二叉树的定义 ⭐⚠ `(练习1, 多选1, 多选3, 判断1)`

- **定义**：二叉树是 $n$（$n \geq 0$）个结点的有限集，它或者为空（$n=0$），或者由一个根结点及两棵互不相交的、分别称为这个根的左子树和右子树的二叉树组成。左子树和右子树又是一棵二叉树。
- **要点**：
  - 二叉树定义具有**递归性**，使得可以采用递归的算法实现其上的操作。
  - 二叉树的定义确定了二叉树的**五种基本形态**，可作为二叉树基本操作和应用算法思路分析的出发点。
  - 二叉树是有序树，子树**有左右之分**。`(多选3)`

### 2. 树的定义 ⭐ `(练习4, 练习19, 多选3, 判断20)`

- **定义**：树是 $n$（$n \geq 0$）个结点的有限集 $T$，$T$ 为空时称为空树，否则它满足如下两个条件：
  1. 有且仅有一个特定的称为**根**的结点；
  2. 其余的结点可分为 $m$（$m \geq 0$）个互不相交的子集 $T_1, T_2, \ldots, T_m$，其中每个子集本身又是一棵树，并称为根的**子树**。
- **要点**：树中每个数据元素只可能有一个直接前驱，但可有多个直接后继。`(练习19)`

### 3. 满二叉树 `(判断4)`

- **定义**：若二叉树中每一层结点的个数都达到了最大，则称之为一棵满二叉树。

### 4. 完全二叉树 ⭐⚠ `(练习7, 练习15, 练习18, 练习21, 练习24, 多选4, 判断5, 判断11)`

- **定义**：若一棵二叉树至多只有最下面的两层上结点的度数可以小于 2，并且最下一层上的结点都集中在该层最左边的若干位置上，则此二叉树称为完全二叉树。
- **要点**：
  - 满二叉树是完全二叉树的**特例**。
  - 完全二叉树适合**顺序存储**。`(多选4)`
  - 叶子结点只可能出现在**最下层和次下层**。`(多选4, 判断5)`
  - 某结点有右子树则必有左子树。`(多选4)`
  - 若一个结点没有左孩子，则它必是叶子结点。`(判断5)`

### 5. 二叉树的性质 ⭐⚠ `(练习13, 练习14, 练习15, 练习17, 练习20, 练习21, 判断6)`

- **性质1**：二叉树的第 $i$ 层上最多有 $2^{i-1}$ 个结点。
- **性质2**：深度为 $k$ 的二叉树最多有 $2^k - 1$ 个结点。
- **性质3**：任意一棵二叉树中，若终端结点的个数为 $n_0$，度数为 2 的结点为 $n_2$，则：$n_2 = n_0 - 1$。
- **性质4**：具有 $n$ 个结点的完全二叉树的深度 $k = \lfloor \log_2 n \rfloor + 1$。
  - ⚠ 适用条件：仅适用于**完全二叉树**，非完全二叉树不适用。`(判断6)`
- **性质5**：对于具有 $n$ 个结点的完全二叉树，按层次从上至下，每层从左至右对每个结点进行编号，则编号为 $i$ 的结点有如下性质：
  1. 若 $i=1$，则 $k_i$ 为根结点，无双亲；否则 $k_i$ 的双亲编号为 $i/2$。
  2. 若 $2i \leq n$，则 $k_i$ 有左孩子，左孩子的编号为 $2i$；否则 $k_i$ 无左孩子，$k_i$ 肯定是叶子。
  3. 若 $2i+1 \leq n$，则 $k_i$ 有右孩子，右孩子的编号为 $2i+1$；否则 $k_i$ 无右孩子。
  4. 除根外，编号为奇数的结点为其双亲的右孩子，偶数结点为其双亲的左孩子。

### 6. 二叉树的存储 ⭐⚠ `(练习2, 练习6, 判断11, 判断15)`

- **顺序存储**：适于完全二叉树的存储。
- **二叉链表**：每个结点含两个指针域（lchild、rchild）。
  - ⚠ $n$ 个结点的二叉链表中有 $n+1$ 个空链域。`(练习2)`
- **三叉链表**：比二叉链表多一个指向父结点的指针。
  - 三叉链表有一个指针指向父结点，不用栈便可在不需要向分支结点深入时或无法深入时回退到父结点。`(练习6)`

### 7. 二叉树的先序遍历 ⭐ `(练习5, 练习9, 练习10, 练习11, 练习12, 判断3, 判断7, 判断10)`

- **定义**：若二叉树非空，则先访问根结点，再先根遍历左子树，最后先根遍历右子树。
- **算法实现提示**：
  - 可以用递归方式实现，也可用非递归方式实现。
  - 访问结点的时机是在**第一次遇到结点**的时候，即进入其左子树之前时访问。
  - 在用非递归方法实现时，需要用**栈**或其他别的方式记住当前进入的是哪个点的左子树，以便将来返回时可进入其右子树。

### 8. 二叉树的中序遍历 ⭐ `(练习9, 练习10, 练习11, 练习23, 判断7, 判断8)`

- **定义**：若二叉树非空，则先中序遍历左子树，再访问根结点，最后中序遍历右子树。
- **算法实现提示**：
  - 可以用递归方式实现，也可用非递归方式实现。
  - 访问结点的时机是在**第二次遇到结点**时，即进入其左子树之后并从其左子树返回时访问。
  - 在用非递归方法实现时，需要用**栈**记住当前进入的是哪个点的左子树，以便将来返回时可访问该结点并进入其右子树。

### 9. 二叉树的后序遍历 ⭐ `(练习9, 练习10, 练习11, 练习12, 判断8)`

- **定义**：若二叉树非空，则先后根遍历左子树，再后根遍历右子树，最后访问根结点。
- **算法实现提示**：
  - 可以用递归方式实现，也可用非递归方式实现。
  - 访问结点的时机是在**第三次遇到结点**的时候，即进入其左子树之后返回，又进入其右子树之后返回时才访问该结点。
  - 在用非递归方法实现时，需要用**栈**记住当前进入的是哪个点的左子树，同时还需要设置**标记**标识返回前进入的是左子树还是右子树，以便将来从右子树返回时访问该结点。

### 10. 二叉树的层次遍历 ⭐⚠ `(判断12)`

- **定义**：从二叉树的根结点开始，按自上而下、从左到右的顺序进行遍历。
- **算法实现提示**：可以利用**队列**帮助实现遍历。
- ⚠ 层次遍历使用**队列**，不是栈。`(判断12)`

### 11. 由二叉树的遍历序列恢复二叉树 ⭐⚠ `(练习10, 练习11, 判断16, 判断17)`

- **可以唯一确定二叉树的组合**：
  - 先序 + 中序
  - 中序 + 后序
  - 中序 + 层次遍历
- **不能唯一确定二叉树的组合**：
  - 先序 + 后序
  - 先序 + 层次遍历
  - 后序 + 层次遍历
- ⚠ 先序和后序不能唯一确定一棵二叉树，因为无法确定左右子树的范围。`(判断16)`

### 12. 线索二叉树的定义 `(练习3)`

- **定义**：利用二叉链表中的 $n+1$ 个空指针域存放指向结点在某种遍历次序下的前驱和后继结点的指针，为了区别指向左右孩子结点的指针，把前者称为**线索**，加了线索的二叉链表称为**线索链表**，相应的二叉树称为**线索二叉树**。
- 按中序线索建立的线索树称为**中序线索树**；同理，有先序线索树和后序线索树。

### 13. 线索二叉树的存储

- 在二叉链表中为每个结点的 lchild 域和 rchild 域分别加标记 **ltag** 和 **rtag**，取值为 1 则为线索，否则取值为 0。

### 14. 线索二叉树的创建

- 可通过对二叉树进行先序、中序和后序遍历创建相应的线索二叉树。

### 15. 线索二叉树的遍历 `(练习25)`

- 遍历不加线索的二叉树（二叉链表存储）时，通常用递归方法，不用递归方法时就要用到栈。
- 遍历线索二叉树则**不需要递归和堆栈**，其时间性能为 $O(n)$。

### 16. 最优二叉树的概念 ⭐ `(练习26, 判断13)`

- **定义**：在权为 $w_1, w_2, \ldots, w_n$ 的 $n$ 个叶子所构成的所有二叉树中，带权路径长度最小（即树的代价最小）的二叉树称为**最优二叉树**或**哈夫曼树**。
- **注意**：最优二叉树的形状不唯一，带权路径长度是唯一的。
- 哈夫曼树是自底向上生长的，权值越小的结点越远离根结点，越靠近根结点的结点权值越大。`(判断13)`

### 17. 前缀码 `(判断14)`

- **定义**：为了保证解码的唯一性，在编码时要求字符集中任一字符的编码都不是其他字符编码的前缀，这种编码称为**前缀码**。
- **注意**：采用最优二叉树方法构造的哈夫曼编码就是前缀码。

### 19. 最优前缀码 `(判断14)`

### 20. 哈夫曼编码 ⭐ `(练习27, 多选6)`

- 由哈夫曼树求得的编码是最优前缀码，称为**哈夫曼编码**。
- ⚠ 哈夫曼编码中，同层分支的 0 和 1 的编码并不是固定的，故字符对应的编码值不唯一，但每个字符编码的位数是确定的。`(多选6)`
- ⚠ 出现频率相同的字符编码长度不一定相同，因为它们可能出现在不同层上。`(判断14)`

### 21. 树的存储 `(多选2, 判断19)`

- **存储方法**：双亲表示法、孩子链表表示法、双亲孩子链表表示法、孩子兄弟表示法。
- ⚠ 双亲表示法对于求双亲操作方便，但求某结点的孩子结点需查询整个数组，很不方便。`(判断19)`

### 22. 树与二叉树的相互转换 ⭐⚠ `(练习22, 判断2)`

- **转换规则**：
  1. 原树的根为转换后二叉树的根；
  2. 原树中每个非终端结点的第一个孩子转换后成为其双亲的**左孩子**；
  3. 原树中每个结点右边的第一个兄弟孩子转换后成为它的**右孩子**。
- **树转换为二叉树后的特点**：
  1. 所转换成的二叉树其根结点的**右子树为空**；
  2. 其二叉链表可解释为原树的孩子兄弟链表。
- ⚠ 由树转换成二叉树，根结点必有左子树而没有右子树。`(判断2)`
- 二叉树转换成树是上述规则的逆过程。

### 23. 森林与二叉树的相互转换 ⭐ `(练习8, 练习22, 判断18)`

- **转换规则**：将森林中第二棵树的树根结点视为第一棵树树根的第一个兄弟，第三棵树的树根结点视为第二棵树树根的第一个兄弟，依此类推，仍按树的转换规则进行即可。
- 二叉树转换成森林是上述规则的逆过程。
- ⚠ 转换不会导致结点个数的减少，只是逻辑关系的变化。`(判断18)`

### 24. 树的遍历 `(判断3)`

- **先根遍历**：若树 $T$ 非空，则：
  1. 访问根结点 $R$；
  2. 依次先根遍历 $R$ 的各个子树 $T_1, T_2, \ldots, T_k$。
- **后根遍历**：若树 $T$ 非空，则：
  1. 依次后根遍历根 $R$ 的各个子树 $T_1, T_2, \ldots, T_k$；
  2. 访问根 $R$。

### 25. 森林的遍历

- **先序遍历**：若森林 $F$ 非空，则：
  1. 访问其第一棵树的根结点；
  2. 先序遍历第一棵树的根的子树；
  3. 先序遍历去掉第一棵树后的子森林。
- **后序遍历**：若森林 $F$ 非空，则：
  1. 后序遍历第一棵树的根结点的子树；
  2. 访问第一棵树的根结点；
  3. 中序遍历去掉第一棵树后的子森林。

### 26. 树和森林的遍历与对应二叉树遍历的关系 ⭐ `(练习8)`

- 树和森林的**先序遍历**和**后序遍历**与所转换的二叉树的**先序遍历**和**中序遍历**的结果序列对应相同。

| 树/森林遍历 | 对应二叉树遍历 |
|---|---|
| 先序遍历 | 先序遍历 |
| 后序遍历 | 中序遍历 |

## 四、重点对比与易混辨析

### 1. 满二叉树 vs 完全二叉树

| 特征 | 满二叉树 | 完全二叉树 |
|---|---|---|
| 定义 | 每一层结点个数都达到最大 | 至多最下两层结点度数可小于2，最下层结点集中在最左 |
| 关系 | 是完全二叉树的**特例** | 满二叉树是其特例 |
| 叶子位置 | 所有层 | 仅最下层和次下层 |
| 顺序存储 | 适合 | 适合 |

### 2. 四种遍历方式对比

| 遍历方式 | 访问顺序 | 访问根的时机 | 非递归辅助结构 |
|---|---|---|---|
| 先序 | 根→左→右 | 第一次遇到结点 | 栈 |
| 中序 | 左→根→右 | 第二次遇到结点 | 栈 |
| 后序 | 左→右→根 | 第三次遇到结点 | 栈 + 标记 |
| 层次 | 上→下，左→右 | — | 队列 |

### 3. 普通二叉链表 vs 线索二叉树

| 特征 | 普通二叉链表 | 线索二叉树 |
|---|---|---|
| 空指针域 | $n+1$ 个空指针域（闲置） | 利用空指针域存放前驱/后继指针（线索） |
| 标记域 | 无 | ltag / rtag（1=线索，0=孩子） |
| 遍历 | 需要递归或栈 | 不需要递归和堆栈，$O(n)$ |

### 4. 由遍历序列恢复二叉树——能否唯一确定

| 序列组合 | 能否唯一确定 |
|---|---|
| 先序 + 中序 | ✅ 能 |
| 中序 + 后序 | ✅ 能 |
| 中序 + 层次 | ✅ 能 |
| 先序 + 后序 | ❌ 不能 |
| 先序 + 层次 | ❌ 不能 |
| 后序 + 层次 | ❌ 不能 |

## 五、高频考点与易错提醒

1. ⚠ **二叉树与树的区别**是高频易错点：二叉树是有序树，子树有左右之分；二叉树可以为空树。`(练习1, 多选3, 判断1)`
2. ⚠ **完全二叉树叶子结点位置**：只在最下层和次下层出现，不能在任意层出现。`(多选4, 判断5)`
3. ⚠ **性质4（深度公式 $k = \lfloor \log_2 n \rfloor + 1$）仅适用于完全二叉树**，非完全二叉树高度有多种可能。`(判断6)`
4. ⚠ **空链域个数 $n+1$** 是高频计算考点。`(练习2)`
5. ⚠ **层次遍历用队列，不是栈**。`(判断12)`
6. ⚠ **先序+后序不能唯一确定二叉树**，先序+中序可以。`(判断16, 判断17)`
7. ⚠ **树转二叉树后根结点右子树为空**（有左子树无右子树）。`(判断2)`
8. ⚠ **森林转二叉树后，根的右子树结点数 = 其余所有树结点数之和**。`(练习8)`
9. ⚠ **哈夫曼编码不唯一**（0/1 分支可交换），但每个字符编码位数确定；频率相同的字符编码长度不一定相同。`(多选6, 判断14)`
10. ⚠ **哈夫曼树中权值较大的结点离根较近**，权值较小的结点离根较远。`(判断13)`
11. ⚠ **非完全二叉树也能用顺序存储**，只是会浪费空间。`(判断11)`
12. ⚠ **先序与后序遍历序列相反时，该二叉树只有一个叶子结点**（所有结点均无左孩子或均无右孩子）。`(练习12，反推)`
13. ⚠ **哈夫曼树结点总数 $= 2n-1$**（$n$ 为叶子结点数）。`(练习27，反推)`
14. ⚠ **叶结点在先序、中序、后序遍历中的相对次序不变**（三种遍历都是从左向右）。`(练习9)`
15. ⚠ **中序遍历中某结点的前驱是其左子树中最右的结点**。`(练习23)`
16. ⚠ **双亲表示法求孩子不方便**（需遍历整个数组）。`(判断19)`

## 六、一句话记忆要点

1. **二叉树定义**：递归定义，分左右子树，五种基本形态。
2. **树的定义**：一个根 + 互不相交的子树，递归定义。
3. **满二叉树**：每层结点数都达最大。
4. **完全二叉树**：最下两层度数可小于2，最下层靠左排；满二叉树是其特例。
5. **性质**：层最多 $2^{i-1}$，深最多 $2^k-1$，$n_2 = n_0 - 1$，完全二叉树深 $\lfloor \log_2 n \rfloor + 1$，编号 $i$ 的双亲 $i/2$、左孩子 $2i$、右孩子 $2i+1$。
6. **存储**：顺序（适完全二叉树）、二叉链表（$n+1$ 空链域）、三叉链表（多一个父指针）。
7. **先序**：根左右，第一次遇到访问，非递归用栈。
8. **中序**：左根右，第二次遇到访问，非递归用栈。
9. **后序**：左右根，第三次遇到访问，非递归用栈+标记。
10. **层次**：上到下左到右，用队列。
11. **恢复二叉树**：先+中、中+后、中+层次可唯一确定；先+后不可。
12. **线索二叉树**：利用 $n+1$ 个空指针域存前驱后继，ltag/rtag=1 为线索，遍历不需递归和栈，$O(n)$。
13. **哈夫曼树**：带权路径长度最小，形状不唯一但 WPL 唯一，权大近根。
14. **前缀码**：任一字符编码不是其他编码的前缀；哈夫曼编码即前缀码。
15. **哈夫曼编码**：最优前缀码，编码值不唯一但码长确定。
16. **树转二叉树**：第一个孩子→左孩子，第一个兄弟→右孩子，根的右子树为空。
17. **森林转二叉树**：各树根依次视为兄弟，按树转规则进行。
18. **树/森林遍历对应**：先序→二叉树先序，后序→二叉树中序。
