## 一、本章概览

本章是操作系统课程的总纲，解决"OS 是什么、为什么需要、能做什么、如何演进、怎样组织"五个根本问题。内容覆盖 OS 的定义与目标、四大基本特征、五大管理功能、发展历程（人工→批处理→分时→实时）、系统分类以及体系结构演变。后续各章（进程管理、内存管理、文件管理、设备管理）均在本章框架下展开。

## 二、操作系统的定义、目标与作用

**操作系统（OS）**：配置在计算机硬件上的第一层软件，是对硬件系统的首次扩充。主要作用是管理设备、提高利用率和系统吞吐量，并为用户和应用程序提供简单接口。*(讲义 p3)*

**计算机软件系统构成**：由系统软件和应用软件构成。OS 是最基本和最重要的系统软件；其他系统软件包括编译程序、数据库管理系统等；应用软件运行在系统软件之上。*(讲义 p3)*

### OS 的四大目标

| 目标 | 含义 | 关键词 |
|---|---|---|
| 方便性 | 配置 OS 后可用高级语言编程，通过命令操纵计算机，易学易用 | 机器语言→高级语言 |
| 有效性 | ①提高系统资源利用率；②提高系统吞吐量（合理组织工作流程） | 利用率 + 吞吐量 |
| 可扩充性 | 适应硬件/体系结构/应用发展，OS 结构从无结构→模块化→层次化→微内核 | 微内核结构 |
| 开放性 | 遵循世界标准规范（特别是 OSI 国际标准），实现互连兼容 | OSI 标准 |

*(讲义 p3-4)*

> 方便性和有效性是设计 OS 时最重要的两个目标。*(讲义 p3)*

### OS 的三大作用

**1. 作为计算机系统资源的管理者***(讲义 p4)*

- **处理机管理**：处理机的分配和运行以进程（线程）为基本单位，归结为进程管理。功能包括进程控制、进程同步、进程通信、死锁处理、处理机调度等。*(讲义 p4)*
- **存储器管理**：为多道程序提供良好环境，提高内存利用率。包括内存分配与回收、地址映射、内存保护与共享、内存扩充。*(讲义 p4)*
- **文件管理**：文件存储空间管理、目录管理、文件读写管理和保护。*(讲义 p4)*
- **设备管理**：完成用户 I/O 请求，提高设备利用率。包括缓冲管理、设备分配、设备处理、虚拟设备。*(讲义 p4)*

**2. 作为用户与计算机硬件系统之间的接口***(讲义 p4-5)*

- **命令接口**：按作业控制方式分为联机命令接口（联机控制方式）和脱机命令接口（脱机控制方式）。*(讲义 p4-5)*
- **程序接口**：由一组系统调用（也称广义指令）组成，用户在程序中使用系统调用请求 OS 服务。*(讲义 p5)*

> Caveat (题目P5-T20 / 第一章-单选-8,38): 系统调用的目的是**请求系统服务**，申请系统资源只是服务的一种，不能等同。*(题目P5-T20, 讲义 p18,19,21)*

**3. 用作扩充机器***(讲义 p5)*

- 没有任何软件支持的计算机称为**裸机**。OS 覆盖在裸机之上，将其改造成功能更强、使用更方便的机器，称为**扩充机器**或**虚拟机**。*(讲义 p5)*

## 三、操作系统的基本特征 ⭐

OS 的基本特征包括**并发、共享、虚拟和异步**。*(讲义 p3)*

### 1. 并发性 ⭐⚠

正是系统中程序能并发执行，才使得 OS 能有效提高资源利用率和吞吐量。*(讲义 p4)*

| 概念 | 定义 |
|---|---|
| 并行性 | 两个或多个事件在**同一时刻**发生 |
| 并发性 | 两个或多个事件在**同一时间间隔内**发生 |

*(讲义 p4)*

> Caveat (第一章-单选-3,4): **并发性是其他三个特征（共享、虚拟、异步）的前提**。并发性 = 同一时间间隔内；并行性 = 同一时刻。*(讲义 p18)*

### 2. 共享性

共享是指系统中的资源可供内存中多个并发执行的进程（线程）共同使用，又称资源共享或资源复用。*(讲义 p4)*

- **互斥共享方式**：一段时间内只允许一个进程访问该资源。*(讲义 p4)*
- **同时访问方式**：一段时间内允许多个进程"同时"访问该资源。*(讲义 p4)*

### 3. 虚拟性

通过某种技术把一个物理实体变为若干个逻辑上的对应物。物理实体是"实"的，逻辑对应物是"虚"的（用户感觉上的东西）。*(讲义 p4)*

- **时分复用技术***(讲义 p4)*
- **空分复用技术***(讲义 p4)*

### 4. 异步性

多道程序环境下，多个进程并发执行。单处理机环境下每次只允许一个进程执行，其余等待。由于资源等因素限制，进程执行通常不可能"一气呵成"，而是以"停停走走"的方式运行。*(讲义 p4)*

> Caveat (第一章-判断-31): 单处理机环境下程序是并发不是并行的，但**程序的执行与 I/O 操作在通道、中断、DMA 的支持下可以并行**实现。*(讲义 p21)*

## 四、现代 OS 的新功能

### 1. 系统安全 ⭐

保障系统中存储和传输数据的**保密性、完整性和系统可用性**。*(讲义 p5)*

| 技术 | 说明 |
|---|---|
| 认证技术 | 确认被认证对象是否名副其实，如身份认证 |
| 密码技术 | 对数据加密成密文，指定用户才能解密 |
| 访问控制技术 | ①设置用户权限限定访问范围；②设置文件属性（如只读）保障安全 |
| 反病毒技术 | 预防为主，反病毒软件扫描检测并清除病毒 |

*(讲义 p5)*

### 2. 网络的功能和服务

作为网络操作系统应具备的功能：*(讲义 p5-6)*

- **网络通信**：源主机和目标主机之间实现无差错数据传输（建立/拆除通信链路、传输控制、差错控制、流量控制）。*(讲义 p6)*
- **资源管理**：对网络中共享资源实施有效管理，保证数据安全性和一致性。*(讲义 p6)*
- **应用互操作**：实现信息的**互通性**（不同网络用户间信息互通）和**互用性**（用户可访问不同网络的文件系统和数据库系统）。*(讲义 p6)*

### 3. 支持多媒体

现代 OS 增加的多媒体处理功能：*(讲义 p6)*

- **接纳控制功能**：限制 SRT（软实时）任务数目和驻留内存任务数目（媒体服务器接纳控制、存储器接纳控制、进程接纳控制）。*(讲义 p6)*
- **实时调度**：SRT 任务往往为周期性软实时任务（如图像更新周期须在 40ms 内）。*(讲义 p6)*
- **多媒体文件的存储**：要求能把硬盘数据快速传送到输出设备，改进传统文件系统的离散存放方式和磁盘寻道方式。*(讲义 p6)*

## 五、OS 的发展历程与分类

### 推动OS发展的主要动力

1. 不断提高计算机资源利用率（最初推动力，形成多道批处理、SPOOLing、虚拟存储器）*(讲义 p6)*
2. 方便用户（分时系统、图形用户界面）*(讲义 p6)*
3. 器件的不断更新换代（8位→16位→32位→64位）*(讲义 p6)*
4. 计算机体系结构的不断发展（单处理机→多处理机OS；网络→网络OS）*(讲义 p7)*
5. 不断提出新的应用需求（实时控制OS、多媒体功能）*(讲义 p7)*

### OS 的发展过程

#### （1）未配置 OS 的计算机系统

**人工操作方式**（早期）：程序员将穿孔纸带/卡片装入输入机，启动运行，完毕后取走结果才允许下一用户上机。*(讲义 p7)*

缺点：*(讲义 p7)*
- 用户独占全机
- CPU 等待人工操作（人机矛盾）

**脱机输入/输出方式**（20世纪50年代末）：事先在外围机控制下将纸带数据输入到磁带上，CPU 需要时从磁带高速调入内存。*(讲义 p7)*

优点：*(讲义 p7)*
- 减少 CPU 空闲时间（装带/卸带等操作在脱机情况下由外围机完成）
- 提高 I/O 速度（CPU 直接从高速磁带读取数据）

> Caveat (第一章-单选-19): 脱机I/O 解决的是**人机矛盾及 CPU 和 I/O 设备速度不匹配**的矛盾。*(讲义 p19)*

#### （2）批处理阶段（OS 的出现）

**单道批处理系统**：系统对作业成批处理，但内存中始终保持一道作业。*(讲义 p7)*

| 特征 | 说明 |
|---|---|
| 自动性 | 磁带上一批作业能自动逐个运行，无须人工干预 |
| 顺序性 | 各道作业顺序进入内存，完成顺序与进入顺序相同（先调入先完成） |
| 单道性 | 内存中仅一道程序运行，完成或异常时才换入后继程序 |

*(讲义 p7)*

> 问题：每次内存仅一道作业，运行期间发出 I/O 请求后高速 CPU 等待低速 I/O 完成。*(讲义 p7)*

**多道批处理系统**：允许多个程序同时进入内存并交替运行，共享系统资源。某道程序因 I/O 请求暂停时，CPU 立即转去运行另一道程序。*(讲义 p7-8)*

| 优点 | 缺点 |
|---|---|
| 资源利用率高（CPU、内存、I/O 设备均保持忙碌） | 平均周转时间长（作业排队依次处理） |
| 系统吞吐量大（资源忙碌 + 切换开销小） | 无交互能力（作业提交后至完成前用户无法交互） |

*(讲义 p8)*

多道批处理系统需要解决的问题：*(讲义 p8)*
- 处理机争用问题
- 内存分配和保护问题
- I/O 设备分配问题
- 文件的组织和管理问题
- 作业管理问题
- 用户与系统的接口问题

> Caveat (第一章-判断-40): 多道程序设计技术的最主要原因是**提高资源利用率和系统吞吐量**，不是提高可靠性。*(讲义 p21)*

> Caveat (第一章-判断-39): **多道 ≠ 多用户**。多道仅指多个程序在系统调度下并行运行，与是否多用户无关。*(讲义 p21)*

> Caveat (第一章-单选-11): 提高单机资源利用率的关键技术是**多道程序设计技术**。*(讲义 p19)*

> Caveat (第一章-单选-12): 批处理系统的主要缺点是**无交互能力**。*(讲义 p19)*

### 分时 OS ⭐

把处理器的运行时间分成很短的时间片，按时间片轮流把处理器分配给各联机作业使用。若作业在分配的时间片内不能完成，则暂时停止运行，让出处理器给其他作业，等待下一轮继续运行。由于计算机速度很快，给每个用户的感觉就像独占一台计算机。*(讲义 p8)*

| 特征 | 说明 |
|---|---|
| 多路性 | 多台终端同时连接一台主机，按分时原则为每个用户服务 |
| 独立性 | 每个用户在各自终端操作，彼此互不干扰，感觉独占主机 |
| 及时性 | 用户请求能在很短时间内获得响应，通常仅 **1~3 秒** |
| 交互性 | 用户可通过终端与系统进行广泛的人机对话 |

*(讲义 p8-9)*

> Caveat (第一章-单选-17): 分时系统主要用于**交互式作业**而非批处理作业。*(讲义 p19)*

### 实时 OS ⭐

实时系统要求在严格时间限制内响应外部事件。*(讲义 p9)*

| 特征 | 分时系统 | 实时系统 |
|---|---|---|
| 多路性 | 按时分原则为多个终端用户服务 | 周期性对多路现场信息采集、对多个对象/执行机构控制 |
| 独立性 | 终端用户彼此互不干扰 | 信息采集和对象控制彼此互不干扰 |
| 及时性 | 依据人能接受的等待时间（1~3秒） | 依据控制对象要求的截止时间（秒级到毫秒级） |
| 交互性 | 广泛的人机对话（数据处理、资源共享等） | 仅限访问特定专用服务程序（信息查询）；或发送特定命令（开始/停止/快进） |
| 可靠性 | 要求系统可靠 | 要求系统**高度可靠**，采取多级容错措施 |

*(讲义 p9)*

> Caveat (第一章-单选-15): 实时 OS 的主要追求目标是**实时性和可靠性**（安全可靠、及时响应、快速处理），**资源利用率不是**其主要目标。*(讲义 p19)*

> Caveat (第一章-多选-22): 实时 OS 和分时 OS 可从**交互性、可靠性、响应时间**三方面加以区别。*(讲义 p20)*

### OS 的分类 ⭐

| 类型 | 含义 | 代表系统 |
|---|---|---|
| 单用户单任务 OS | 只允许一个用户上机，用户程序作为一个任务运行 | CP/M、MS-DOS |
| 单用户多任务 OS | 只允许一个用户上机，但允许程序分为若干任务并发执行 | Windows |
| 多用户多任务 OS | 允许多个用户通过终端使用同一台机器，共享资源，每个用户程序又可分为多个任务并发执行 | UNIX |

*(讲义 p9-10)*

> Caveat (第一章-判断-33): MS-DOS 是**单用户单任务**操作系统，不是多用户多任务。*(讲义 p21)*

> Caveat (第一章-单选-13): 操作系统的基本类型为：**批处理操作系统、分时操作系统、实时操作系统**。*(讲义 p19)*

## 六、OS 的体系结构

| 体系结构 | 核心思想 | 优点 | 缺点 |
|---|---|---|---|
| 无结构OS | 为数众多过程的集合，过程间可任意相互调用 | [原文未述] | 内部复杂混乱，错误多，难调试、难阅读理解、难维护 |
| 模块化OS | 划分模块，注重模块独立性（内聚性高、耦合度低） | 提高正确性/可理解性/可维护性；增强可适应性；加速开发过程 | [原文未述] |
| 分层结构OS | 自下而上分层设计，层间单向依赖 | 易保证正确性；易扩充和易维护 | 系统效率降低（每层间建立通信机制，穿越多层增加通信开销） |
| 微内核OS | 精心设计的小型内核，仅放入最基本核心功能 | 可扩展性、可靠性、可移植性、支持分布式、融入面向对象 | 运行效率降低（更多上下文切换） |

*(讲义 p10-11)*

**模块化OS的独立性衡量标准**：*(讲义 p10)*
- **内聚性**：模块内部各部分间联系的紧密程度。内聚性越高，模块独立性越强。
- **耦合度**：模块间相互联系和相互影响的程度。耦合度越低，模块独立性越好。

**微内核OS的设计基础**：*(讲义 p10)*
- 基于客户/服务器模式
- 应用"机制与策略分离"原理
- 采用面向对象技术

**微内核的基本功能**：*(讲义 p11)*
- 进程（线程）管理
- 低级存储器管理
- 中断和陷入处理

> Caveat (第一章-多选-29): 微内核会**降低执行效率**（增加上下文切换、消息传递、数据拷贝等开销），不能提高执行效率。*(讲义 p21)*

> Caveat (第一章-多选-30): 分层结构优点是易保证正确性、易扩充、易维护，但**执行速度会降低**。*(讲义 p21)*

## 七、高频考点与易错提醒

1. **并发 vs 并行**：并发 = 同一时间间隔内；并行 = 同一时刻。并发是共享、虚拟、异步的前提。*(讲义 p18)*
2. **系统调用的目的**：请求系统服务（不只是申请资源）。用户通过命令接口和程序接口（系统调用）使用计算机。*(讲义 p18,21)*
3. **用户态可执行指令**：修改内存单元的值（在用户自己的地址空间内）是非特权指令；屏蔽中断、设置时钟、停机、I/O 操作均为核心态特权指令。*(讲义 p20)*
4. **OS 基本类型**：批处理、分时、实时。*(讲义 p19)*
5. **实时 OS 追求目标**：实时性和可靠性，不追求资源利用率。*(讲义 p19)*
6. **多道程序设计目的**：提高资源利用率和系统吞吐量，非可靠性。*(讲义 p21)*
7. **多道 ≠ 多用户**：多道指多程序并发运行，与用户数无关。*(讲义 p21)*
8. **MS-DOS**：单用户单任务，非多用户多任务。*(讲义 p21)*
9. **批处理系统缺点**：无交互能力。*(讲义 p19)*
10. **脱机I/O 解决的问题**：人机矛盾及 CPU 与 I/O 设备速度不匹配。*(讲义 p19)*
11. **OS 功能**：处理机管理、存储器管理、设备管理、文件管理。文档编辑不是 OS 功能。*(讲义 p19)*
12. **存储器管理**：管理的是内存（分配与回收），不是磁盘块。*(讲义 p20)*
13. **进程数与 CPU 利用率**：进程多不意味着 CPU 利用率高，竞争激烈甚至死锁会降低利用率。*(讲义 p21)*
14. **分时系统文件系统**：用户并非真正独占文件系统，多用户共享需解决互斥使用。*(讲义 p21)*
15. **OS 程序不全部常驻内存**：分为常驻内存程序和非常驻内存程序。*(讲义 p21)*

## 八、一句话记忆要点

- **OS 定义**：硬件上第一层软件，管理资源、提供接口。
- **四大目标**：方便、有效、可扩充、开放。
- **四大特征**：并发（前提）、共享（互斥/同时）、虚拟（时分/空分）、异步（停停走走）。
- **四大管理**：处理机（进程）、存储器（内存）、文件、设备。
- **两大接口**：命令接口（联机/脱机）、程序接口（系统调用）。
- **发展脉络**：人工→脱机I/O→单道批→多道批→分时→实时。
- **三大基本类型**：批处理、分时、实时。
- **分时四特征**：多路、独立、及时（1~3秒）、交互。
- **实时比分时**：更及时、更可靠、交互范围窄。
- **体系结构**：无结构→模块化（内聚高/耦合低）→分层（正确但慢）→微内核（小而可靠但效率低）。

## Worked Examples

### 例 1：分时系统时间片计算 *(来自第一章-单选-18)*
Tags: 分时系统, 时间片轮转, 响应时间

**问题**：在分时系统中，为使多个进程能够及时与系统交互，关键的问题是能在短时间内，使所有就绪进程都能运行。当就绪进程数为 100 时，为保证响应时间不超过 2s，此时的时间片最大应为？*(讲义 p14)*

**解答**：
1. 分时系统中，响应时间 = 就绪进程数 × 时间片。*(讲义 p19)*
2. 要求响应时间不超过 2s，即 $100 \times \text{时间片} \leq 2\text{s}$。*(讲义 p19)*
3. 时间片最大值 $= \frac{2\text{s}}{100} = \frac{2000\text{ms}}{100} = 20\text{ms}$。*(讲义 p19)*
4. 答案选 B（20ms）。*(讲义 p19)*

### 例 2：用户态与核心态指令判断 *(来自第一章-单选-20)*
Tags: 用户态, 核心态, 特权指令

**问题**：以下哪条指令可在用户态执行？*(讲义 p15)*

**解答**：
1. 屏蔽中断、设置时钟的值、停机指令均为**特权指令**，只能在核心态执行，操作不当会损害机器。*(讲义 p20)*
2. 修改内存单元的值是**非特权指令**，可在用户态执行，但进程只能访问自己的用户空间。*(讲义 p20)*
3. 答案选 C（修改内存单元的值）。*(讲义 p20)*
